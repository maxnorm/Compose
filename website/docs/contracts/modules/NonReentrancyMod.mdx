## Overview

The NonReentrancyMod provides reusable non-reentrancy utilities to protect functions in a diamond&#x27;s facets from re-entering during execution. It ensures the integrity of execution flow, especially useful for guarding state-modifying logic in composition-heavy contracts.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

/**
 * Example Facet integrating LibNonReentrancy for reentrancy protection.
 */
library LibNonReentrancy {
    function enter() internal;
    function exit() internal;
}

contract ExampleFacet {
    /**
     * Demonstrates the non-reentrancy logic.
     * Protects against recursive calls to this function.
     */
    function safeFunction() external {
        LibNonReentrancy.enter();

        // Function logic here.

        LibNonReentrancy.exit();
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Always call &#x60;LibNonReentrancy.exit()&#x60; as the final step in protected functions.
- Ensure &#x60;LibNonReentrancy.enter()&#x60; and &#x60;exit()&#x60; pair correctly in all execution pathways.
- Test for edge cases such as uncaught reentrant execution attempts to verify facet safety.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
LibNonReentrancy interacts with diamond storage by leveraging shared execution context across facets. Ensure that functions using &#x60;enter&#x60; and &#x60;exit&#x60; respect the reentrancy lock state. Violating the requirement to call these in pairs might result in unexpected locked state, preventing legitimate execution of functions in future calls.
</Callout>
