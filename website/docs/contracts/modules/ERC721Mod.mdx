<DocSubtitle>
ERC-721 Library for Compose - Thrown when attempting to interact with a non-existent token.
</DocSubtitle>

<Callout type="info" title="Key Features">
- Provides standard ERC-721 functionality, including minting, burning, transferring, and metadata handling.
- Safely interacts with diamond proxy storage using a predefined slot with inline assembly.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

ERC721Mod provides core ERC-721 functionality as a library for Compose diamonds, supporting token minting, burning, transfer, and metadata updates. It ensures safe interaction with diamond storage, preventing issues like invalid token ownership or state corruption.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

import {ERC721Mod} from './ERC721Mod.sol';

contract ERC721ExampleFacet {
    function mintItem(address _to, uint256 _tokenId) external {
        ERC721Mod.mint(_to, _tokenId);
    }

    function burnItem(uint256 _tokenId) external {
        ERC721Mod.burn(_tokenId);
    }

    function transferItem(address _from, address _to, uint256 _tokenId) external {
        ERC721Mod.transferFrom(_from, _to, _tokenId);
    }

    function getTokenStorage() external view returns (ERC721Mod.StorageStruct memory) {
        return ERC721Mod.getStorage();
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Always check access control or authorization when minting, burning, or transferring tokens.
- Ensure storage writes do not collide or overwrite sensitive data by using the predefined storage structure.
- Use custom error handling for clarity when reverting due to invalid operations, such as nonexistent tokens.
</Callout>
