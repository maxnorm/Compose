<DocSubtitle>
ERC-173 Two-Step Contract Ownership Library - Provides two-step ownership transfer logic for facets or modular contracts.
</DocSubtitle>

<Callout type="info" title="Key Features">
- Implements a secure two-step ownership transfer process.
- Provides standard &#x60;owner()&#x60;, &#x60;pendingOwner()&#x60;, and &#x60;renounceOwnership()&#x60; functions.
- Includes &#x60;requireOwner()&#x60; for access control enforcement within facets.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

The OwnerTwoStepsMod provides a secure, two-step ownership transfer mechanism for Compose diamonds. This pattern prevents accidental ownership loss by requiring explicit acceptance from the new owner, enhancing safety and composability for critical administrative functions.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

import {IOwnerTwoStepsMod} from "../modules/OwnerTwoStepsMod.sol";

contract MyFacet {
    address internal ownerTwoStepsModAddress;

    function initialize(address _ownerTwoStepsModAddress) external {
        ownerTwoStepsModAddress = _ownerTwoStepsModAddress;
    }

    function transferDiamondOwnership(address _newOwner) external {
        IOwnerTwoStepsMod(ownerTwoStepsModAddress).transferOwnership(_newOwner);
    }

    function acceptDiamondOwnership() external {
        IOwnerTwoStepsMod(ownerTwoStepsModAddress).acceptOwnership();
    }

    function getCurrentOwner() external view returns (address) {
        return IOwnerTwoStepsMod(ownerTwoStepsModAddress).owner();
    }

    function requireCallerIsOwner() external view {
        IOwnerTwoStepsMod(ownerTwoStepsModAddress).requireOwner();
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Always use the &#x60;transferOwnership&#x60; function to initiate a transfer and ensure the new owner calls &#x60;acceptOwnership&#x60; to finalize it.
- Implement &#x60;requireOwner&#x60; checks in facets that manage critical state or administrative functions to enforce ownership constraints.
- Be aware that &#x60;renounceOwnership&#x60; permanently relinquishes control and should be used with extreme caution.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
The OwnerTwoStepsMod manages ownership state directly within its designated storage slots, accessible via inline assembly using &#x60;OWNER_STORAGE_POSITION&#x60; and &#x60;PENDING_OWNER_STORAGE_POSITION&#x60;. Facets interacting with this module should call its external functions to manage and query ownership. Changes to ownership are immediately reflected in the module&#x27;s state and are observable by all facets through its public view functions.
</Callout>
