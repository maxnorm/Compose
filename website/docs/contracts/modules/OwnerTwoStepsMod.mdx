<DocSubtitle>
ERC-173 Two-Step Contract Ownership Library - Provides two-step ownership transfer logic for facets or modular contracts.
</DocSubtitle>

<Callout type="info" title="Key Features">
- Implements ERC-173 with explicit two-step ownership transitions.
- Provides ownership checks via &#x60;requireOwner&#x60; for secure access control.
- Supports ownership renunciation, fully disabling owner-only operations.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

OwnerTwoStepsMod is a library implementing ERC-173 compatible two-step ownership transfer logic for Compose diamonds. It enables a secure and deliberate process to assign or renounce ownership, complementing the modular design with explicit access control and transition safeguards.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

contract OwnershipFacet {
    using OwnerTwoStepsMod for OwnerTwoStepsMod.OwnerStorage;

    function acceptOwnership() external {
        OwnerTwoStepsMod.acceptOwnership();
    }

    function transferOwnership(address _newOwner) external {
        OwnerTwoStepsMod.transferOwnership(_newOwner);
    }

    function owner() external view returns (address) {
        return OwnerTwoStepsMod.owner();
    }

    function pendingOwner() external view returns (address) {
        return OwnerTwoStepsMod.pendingOwner();
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Always verify caller identities in external functions using &#x60;requireOwner&#x60; when sensitive actions are involved.
- Ensure &#x60;acceptOwnership&#x60; is called by the pending owner promptly to finalize transitions.
- Prefer &#x60;transferOwnership&#x60; instead of directly modifying storage to better preserve logging and prevent unsafe transitions.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
OwnerTwoStepsMod interacts with diamond storage via dedicated storage slots specified by &#x60;OWNER_STORAGE_POSITION&#x60; and &#x60;PENDING_OWNER_STORAGE_POSITION&#x60;. These slots house &#x60;Owner&#x60; and &#x60;PendingOwner&#x60; storage structs, respectively, and are accessible across all facets using &#x60;getOwnerStorage&#x60; and &#x60;getPendingOwnerStorage&#x60;. Transition invariants include ensuring only a valid pending owner can finalize ownership transfers and that ownership functions respect proper propagation of state changes.
</Callout>
