<DocSubtitle>
ERC-173 Two-Step Contract Ownership Library - Provides two-step ownership transfer logic for facets or modular contracts.
</DocSubtitle>

<Callout type="info" title="Key Features">
- Implements ERC-173 two-step ownership transfer to prevent inadvertent changes.
- Provides utility functions for accessing and validating owner and pending owner states.
- Includes a &#x60;renounceOwnership&#x60; function to disable owner-restricted functions by setting the owner to &#x60;address(0)&#x60;.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

OwnerTwoStepsMod provides a robust two-step ownership transfer mechanism for diamonds, adhering to the ERC-173 standard. It ensures secure and deliberate transitions of ownership, reducing the risk of accidental or unauthorized changes to critical functions.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

import {OwnerTwoStepsMod} from './OwnerTwoStepsMod.sol';

contract OwnershipFacet {
    function initiateOwnershipTransfer(address _newOwner) external {
        OwnerTwoStepsMod.transferOwnership(_newOwner);
    }

    function finalizeOwnershipTransfer() external {
        OwnerTwoStepsMod.acceptOwnership();
    }

    function renounceContractOwnership() external {
        OwnerTwoStepsMod.renounceOwnership();
    }

    function getCurrentOwner() external view returns (address) {
        return OwnerTwoStepsMod.owner();
    }

    function getPendingOwner() external view returns (address) {
        return OwnerTwoStepsMod.pendingOwner();
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Call &#x60;transferOwnership&#x60; with the intended new owner&#x27;s address to initiate the transfer.
- Ensure the pending owner explicitly calls &#x60;acceptOwnership&#x60; to finalize the transfer.
- Use &#x60;requireOwner&#x60; to enforce access control for functions restricted to the current owner.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
OwnerTwoStepsMod uses inline assembly to directly access diamond storage for ownership and pending ownership states, ensuring compatibility with other facets. Storage layout must remain consistent across upgrades. Facets relying on ownership checks should use &#x60;requireOwner&#x60; to enforce access control, and state changes to ownership are immediately visible to all facets once finalized.
</Callout>
