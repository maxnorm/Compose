<DocSubtitle>
ERC-721 Enumerable Library for Compose - Thrown when attempting to interact with a non-existent token.
</DocSubtitle>

<Callout type="info" title="Key Features">
- Manages the addition and removal of tokens from internal enumeration lists during mint, transfer, and burn operations.
- Provides functions to mint new tokens and transfer existing ones while preserving enumerable state integrity.
- Includes a &#x60;getStorage&#x60; function to retrieve the ERC-721 enumerable storage struct, allowing for direct inspection of enumeration data if necessary.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

The ERC721EnumerableMod provides essential functions for managing ERC-721 tokens within a Compose diamond, specifically handling enumeration of owned tokens. It ensures that minted, transferred, and burned tokens are correctly reflected in the token tracking lists, maintaining the integrity of the enumerable state.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

import {IERC721EnumerableMod} from "@compose/modules/erc721/interfaces/IERC721EnumerableMod.sol";
import {IERC721Storage} from "@compose/modules/erc721/storage/IERC721Storage.sol";

contract MyNFTSalesFacet {
    IERC721EnumerableMod private immutable _erc721EnumerableMod;
    IERC721Storage private immutable _erc721Storage;

    constructor(address _diamondAddress) {
        // Assuming the module is registered and accessible via the diamond proxy
        _erc721EnumerableMod = IERC721EnumerableMod(_diamondAddress);
        _erc721Storage = IERC721Storage(_diamondAddress);
    }

    /**
     * @notice Mints a new token and adds it to the enumerable lists.
     * @param _to The address to mint the token to.
     * @param _tokenId The ID of the token to mint.
     */
    function mintNewToken(address _to, uint256 _tokenId) external {
        // Call the mint function provided by the ERC721EnumerableMod
        _erc721EnumerableMod.mint(_to, _tokenId);
    }

    /**
     * @notice Transfers a token, updating enumerable lists.
     * @param _from The address to transfer from.
     * @param _to The address to transfer to.
     * @param _tokenId The ID of the token to transfer.
     */
    function transferToken(address _from, address _to, uint256 _tokenId) external {
        // Call the transferFrom function provided by the ERC721EnumerableMod
        _erc721EnumerableMod.transferFrom(_from, _to, _tokenId);
    }

    /**
     * @notice Burns a token, removing it from enumerable lists.
     * @param _tokenId The ID of the token to burn.
     */
    function burnToken(uint256 _tokenId) external {
        // Call the burn function provided by the ERC721EnumerableMod
        _erc721EnumerableMod.burn(_tokenId);
    }
}
`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure the &#x60;_erc721EnumerableMod&#x60; is correctly initialized with the diamond proxy address. Access control for minting, transferring, and burning should be handled by the calling facet, not within the module itself, promoting composability.
- Always check for the existence of tokens before attempting to burn or transfer them. The module handles these checks internally, but facets should be aware of potential reverts.
- Be mindful of gas costs associated with enumerable operations. Maintaining large token lists can increase transaction complexity.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
The ERC721EnumerableMod interacts directly with the ERC-721 storage slot defined by the diamond&#x27;s storage pattern. Functions like &#x60;mint&#x60;, &#x60;transferFrom&#x60;, and &#x60;burn&#x60; modify this storage to update the internal tracking of tokens. Facets calling these module functions do not need to directly manipulate the storage; the module handles all state changes related to token enumeration. The order of storage within the ERC-721 struct is critical for the module&#x27;s operation; never reorder or remove fields from the beginning or middle of the struct.
</Callout>
