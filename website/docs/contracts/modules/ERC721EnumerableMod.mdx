<DocSubtitle>
ERC-721 Enumerable Library for Compose - Thrown when attempting to interact with a non-existent token.
</DocSubtitle>

<Callout type="info" title="Key Features">
- Provides storage for token enumeration, enabling efficient enumeration and owner token tracking.
- Includes comprehensive validation (authorization, existence of tokens, non-zero addresses) for token management.
- Integrates seamlessly with the diamond storage pattern for modular and upgradable development.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

The ERC721EnumerableMod module extends Compose diamonds with ERC-721 enumeration capabilities. It provides efficient token tracking, enabling enumeration of token-owned and total supply data while ensuring secure, composable storage interactions.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

interface IERC721EnumerableMod {
    function burn(uint256 _tokenId) external;
    function mint(address _to, uint256 _tokenId) external;
    function transferFrom(address _from, address _to, uint256 _tokenId) external;
    function getStorage() external view returns (bytes32 storageStruct);
}

contract ExampleFacet {
    IERC721EnumerableMod private erc721Module;

    constructor(address _moduleAddress) {
        erc721Module = IERC721EnumerableMod(_moduleAddress);
    }

    function mintToken(address to, uint256 tokenId) external {
        erc721Module.mint(to, tokenId);
    }

    function burnToken(uint256 tokenId) external {
        erc721Module.burn(tokenId);
    }

    function transferToken(address from, address to, uint256 tokenId) external {
        erc721Module.transferFrom(from, to, tokenId);
    }

    function retrieveStorage() external view returns (bytes32) {
        return erc721Module.getStorage();
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Validate access control for mint, transfer, and burn actions to prevent unauthorized operations.
- Ensure proper management of token enumeration state across facets; avoid conflicts in diamond storage updates.
- Validate input parameters (e.g., non-zero addresses) before passing to this module&#x27;s functions for robust error handling.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
ERC721EnumerableMod relies on a predefined diamond storage slot for ERC-721 enumeration data. This ensures that all facets interacting with ERC721 storage remain consistent. Developers must not rearrange the storage layout or skip variables within the module to preserve correct functionality. Proper sequencing of operations, such as minting before transferring, is essential to maintain enumeration integrity.
</Callout>
