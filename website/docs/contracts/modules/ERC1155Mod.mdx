ERC-1155 Token Receiver Interface - Handles the receipt of a single ERC-1155 token type.
</DocSubtitle>

<Callout type="info" title="Key Features">
- Provides mint and burn functionality for single and batched ERC-1155 tokens.
- Implements safe transfer methods compliant with EIP-1155 standards, including contract validation.
- Supports metadata management through base URI and token-specific URI updates.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

The ERC1155Mod module implements the ERC-1155 token receiver interface and provides core functionality for minting, burning, transferring, and managing the metadata of ERC-1155 token types. It ensures compliance with EIP-1155 standards while leveraging diamond storage for efficient state management and composability across facets.

## Storage

### ERC1155Storage

ERC-8042 compliant storage struct for ERC-1155 token data. **Note:** storage-location: erc8042:compose.erc1155

<ExpandableCode language="solidity" maxLines={15} title="Definition">
{`struct ERC1155Storage {
mapping(uint256 id => mapping(address account => uint256 balance)) balanceOf;
mapping(address account => mapping(address operator => bool)) isApprovedForAll;
string uri;
string baseURI;
mapping(uint256 tokenId => string) tokenURIs;
}`}
</ExpandableCode>

Storage position: &#x60;STORAGE_POSITION&#x60; - Used for diamond storage pattern.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

import {ERC1155Mod} from "./ERC1155Mod.sol";

contract ERC1155Facet {
    /**
     * Mint a single ERC-1155 token to an address.
     */
    function mintToken(address _to, uint256 _id, uint256 _amount) external {
        ERC1155Mod.mint(_to, _id, _amount, "");
    }

    /**
     * Burn a specified amount of an ERC-1155 token from an address.
     */
    function burnToken(address _from, uint256 _id, uint256 _amount) external {
        ERC1155Mod.burn(_from, _id, _amount);
    }

    /**
     * Update the base URI for token metadata.
     */
    function updateBaseURI(string calldata _newBaseURI) external {
        ERC1155Mod.setBaseURI(_newBaseURI);
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure proper access control for minting, burning, and URI updates to prevent unauthorized actions.
- Use &#x60;getStorage&#x60; to retrieve and interact with the ERC-1155 storage struct, ensuring changes remain consistent across facets.
- Always validate recipient addresses and perform safe transfer checks when transferring tokens to contracts.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
The ERC1155Mod module uses a predefined diamond storage slot to manage state, ensuring that all facets accessing this module interact with the same storage struct. Changes made via mint, burn, or transfer functions are immediately reflected in this shared state. Safe transfer operations enforce ERC1155Receiver compliance, and metadata updates are propagated using emitted events. Maintain proper storage struct ordering and avoid breaking changes to ensure compatibility during upgrades.
</Callout>
