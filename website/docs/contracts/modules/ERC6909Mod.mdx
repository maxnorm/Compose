<Callout type="info" title="Key Features">

</Callout>

## Overview

ERC6909Mod provides core functionality for managing ERC-6909 tokens within a Compose diamond. It handles token minting, burning, transferring, and approval while interacting with diamond storage for efficient state management. This module ensures composability and storage reusability across facets.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

contract TokenFacet {
    function mintToken(address _to, uint256 _id, uint256 _amount) external {
        ERC6909Mod.mint(_to, _id, _amount);
    }

    function transferToken(address _from, address _to, uint256 _id, uint256 _amount) external {
        ERC6909Mod.transfer(_from, _to, _id, _amount);
    }

    function approveSpender(address _spender, uint256 _id, uint256 _amount) external {
        ERC6909Mod.approve(_spender, _id, _amount);
    }

    function setTokenOperator(address _spender, bool _isOperator) external {
        ERC6909Mod.setOperator(_spender, _isOperator);
    }

    function burnToken(address _from, uint256 _id, uint256 _amount) external {
        ERC6909Mod.burn(_from, _id, _amount);
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure storage visibility consistency by utilizing getStorage when accessing diamond storage.
- Validate sender permissions (operatorship or allowance) in facets before invoking transfer or burn.
- Revert on invalid parameters like mismatched &#x60;_from&#x60; or &#x60;_amount&#x60; to ensure safety.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
ERC6909Mod interacts directly with diamond storage using the &#x60;getStorage&#x60; function, which accesses the predefined storage slot via inline assembly. Facets invoking ERC-6909 functions must ensure proper initialization of diamond state. Consistent storage layout across upgrades is critical, as it avoids breaking invariants or corrupting state.
</Callout>
