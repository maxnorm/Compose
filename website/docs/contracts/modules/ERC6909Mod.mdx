LibERC6909 — ERC-6909 Library - Thrown when the sender has insufficient balance.
</DocSubtitle>

<Callout type="info" title="Key Features">
- Supports standard ERC-6909 token operations including mint, burn, transfer, approve, and operator management.
- Adheres to the diamond storage pattern, allowing for seamless integration and upgradeability.
- Provides explicit functions for managing token balances and approvals, reducing ambiguity.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

The ERC6909Mod provides a robust implementation for the ERC-6909 standard, enabling efficient token management within a Compose diamond. This module is crucial for diamonds requiring fungible token functionalities, offering features like minting, burning, transferring, and managing allowances and operators, all while adhering to the diamond&#x27;s storage pattern for composability and upgradeability.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

import {IERC6909} from "@compose-protocol/diamond-contracts/contracts/interfaces/IERC6909.sol";
import {ERC6909Mod} from "@compose-protocol/diamond-contracts/contracts/modules/ERC6909Mod.sol";

contract MyDiamondFacet is IERC6909 {
    address immutable DIAMOND_ADDRESS;

    constructor(address _diamondAddress) {
        DIAMOND_ADDRESS = _diamondAddress;
    }

    function mintToken(address _to, uint256 _id, uint256 _amount) external {
        ERC6909Mod(DIAMOND_ADDRESS).mint(_to, _id, _amount);
    }

    function transferToken(address _from, address _to, uint256 _id, uint256 _amount) external {
        ERC6909Mod(DIAMOND_ADDRESS).transfer(_from, _to, _id, _amount);
    }

    // Implement other IERC6909 functions as needed

    function approve(address _spender, uint256 _id, uint256 _amount) external override {
        ERC6909Mod(DIAMOND_ADDRESS).approve(_spender, _id, _amount);
    }

    function burn(address _from, uint256 _id, uint256 _amount) external override {
        ERC6909Mod(DIAMOND_ADDRESS).burn(_from, _id, _amount);
    }

    function setOperator(address _operator, bool _approved) external override {
        ERC6909Mod(DIAMOND_ADDRESS).setOperator(_operator, _approved);
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure proper access control is implemented at the facet level for sensitive operations like minting and burning.
- Be mindful of the &#x60;type(uint256).max&#x60; allowance behavior and operator status when implementing transfer logic in facets.
- When upgrading the diamond, ensure the storage layout of &#x60;ERC6909Mod&#x60; remains compatible to prevent data corruption.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
The ERC6909Mod interacts with the diamond&#x27;s shared storage. Facets can access and modify token balances, allowances, and operator statuses through the functions provided by this module. The &#x60;getStorage&#x60; function allows direct access to the ERC6909 storage struct via inline assembly, enabling facets to read state efficiently. Any changes made via the module&#x27;s functions are immediately visible across all facets interacting with the diamond&#x27;s storage.
</Callout>
