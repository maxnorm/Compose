LibERC6909 — ERC-6909 Library - Thrown when the sender has insufficient balance.
</DocSubtitle>

<Callout type="info" title="Key Features">
- Provides a complete implementation of ERC-6909 with composability for diamond proxies.
- Handles approvals, operators, and transfers seamlessly with optimized storage access.
- Inline assembly ensures efficient access to designated storage slots.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

ERC6909Mod provides a robust implementation of the ERC-6909 standard, enabling tokenized balances with enhanced composability and flexibility for diamond architectures. The module is designed for safe interaction with diamond storage, ensuring proper state management and reusability across multiple facets while adhering to storage layout principles.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

interface IERC6909Facet {
    function mint(address _to, uint256 _id, uint256 _amount) external;
    function transfer(address _from, address _to, uint256 _id, uint256 _amount) external;
    function burn(address _from, uint256 _id, uint256 _amount) external;
}

contract ExampleFacet {
    // Assume ERC6909Mod functionalities are integrated via diamond storage

    function exampleMint(address to, uint256 id, uint256 amount) external {
        IERC6909Facet(address(this)).mint(to, id, amount);
    }

    function exampleTransfer(address from, address to, uint256 id, uint256 amount) external {
        IERC6909Facet(address(this)).transfer(from, to, id, amount);
    }

    function exampleBurn(address from, uint256 id, uint256 amount) external {
        IERC6909Facet(address(this)).burn(from, id, amount);
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure &#x60;getStorage&#x60; is used consistently across all facets to avoid storage corruption.
- Verify proper access controls (where applicable) for critical functions like &#x60;mint&#x60; and &#x60;burn&#x60;.
- Maintain compatibility with the ERC-6909 standard when extending or upgrading facets.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
ERC6909Mod interacts with diamond storage via a designated storage slot defined by &#x60;STORAGE_POSITION&#x60;. Inline assembly is used to provide a pointer to the dedicated storage struct, making it accessible to all facets using &#x60;getStorage&#x60;. Ensure all facets interacting with this module respect the defined storage layout to prevent state conflicts.
</Callout>
