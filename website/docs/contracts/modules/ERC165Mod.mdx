LibERC165 — ERC-165 Standard Interface Detection Library - Provides internal functions and storage layout for ERC-165 interface detection.
</DocSubtitle>

<Callout type="info" title="Key Features">
- Provides a fixed storage layout for ERC-165 interface detection.
- Enables registration of multiple supported interfaces during initialization.
- Optimized for modularity and safe composition across diamond facets.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

ERC165Mod provides internal functions and storage for implementing ERC-165 interface detection in Compose diamonds. It ensures that facets can consistently register and query supported interfaces, enabling modular, standard-compliant composition across diamond architectures.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

/**
 * Example facet using LibERC165 for interface registration.
 */
import {LibERC165} from "./ERC165Mod.sol";

contract MyFacet {
    /**
     * Initialize the facet and register ERC-721 interface.
     */
    function initialize() external {
        LibERC165.registerInterface(type(IERC721).interfaceId);
    }

    /**
     * Example function to get a reference to ERC165 storage.
     */
    function getStorageReference() external view returns (bytes4[] memory) {
        return LibERC165.getStorage().supportedInterfaces;
    }
}

interface IERC721 {
    function balanceOf(address owner) external view returns (uint256);
    function ownerOf(uint256 tokenId) external view returns (address);
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Always call &#x60;registerInterface&#x60; during the initialization phase of a facet to ensure proper detection by external contracts.
- Prevent accidental overwrites of &#x60;supportedInterfaces&#x60; by adhering strictly to the storage layout defined by LibERC165.
- Avoid invoking &#x60;getStorage&#x60; in non-constant functions unless absolutely necessary to prevent unintended state mutations.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
LibERC165 uses a fixed storage slot to manage supported interfaces, ensuring consistent behavior across facets. Changes made using &#x60;registerInterface&#x60; are immediately visible to other facets in the same diamond via &#x60;getStorage&#x60;. The immutable nature of the storage slot enforces stability, but developers must ensure compatibility when adding or modifying facets that use ERC165 storage.
</Callout>
