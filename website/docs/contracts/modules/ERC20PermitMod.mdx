LibERC20Permit — Library for ERC-2612 Permit Logic - Thrown when a permit signature is invalid or expired.
</DocSubtitle>

<Callout type="info" title="Key Features">
- Provides robust implementation for ERC-2612 permit logic to enable gasless approvals.
- Guarantees mitigation against signature replay attacks through strict domain separator enforcement.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

The ERC20PermitMod library implements critical components for ERC-2612 permit functionality within the Compose diamond framework. It ensures off-chain signature verification for gasless approvals using the domain separator and allowance logic, enabling trustless approval workflows while protecting against signature replay attacks.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

interface IERC20Permit {
    function permit(address owner, address spender, uint256 value, uint256 deadline, uint8 v, bytes32 r, bytes32 s) external;
    function DOMAIN_SEPARATOR() external view returns (bytes32);
}

contract ExampleFacet {
    function usePermit(address token, address owner, address spender, uint256 amount, uint256 deadline, uint8 v, bytes32 r, bytes32 s) external {
        IERC20Permit(token).permit(owner, spender, amount, deadline, v, r, s);
        // Proceed to perform transferFrom or other logic
    }

    function getDomainSeparator(address token) external view returns (bytes32) {
        return IERC20Permit(token).DOMAIN_SEPARATOR();
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Validate that DOMAIN_SEPARATOR remains consistent across upgrades to prevent replay vulnerabilities.
- Ensure the &#x60;permit&#x60; function is never executed without first verifying the signature and expiration checks.
- Emit the Approval event correctly after calling &#x60;permit&#x60;. This must be done by the invoking facet/contract.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
ERC20PermitMod leverages distinct storage slots for ERC20 and permit state, including maintaining the domain separator for signature security. Each diamond facet invoking this module must ensure correct initialization of these storage structures. Permit signatures depend on chain ID and deployment-specific details; always update and verify domain separator post-upgrade. Comply with EIP-2612 by ensuring Approval events are emitted as required.
</Callout>
