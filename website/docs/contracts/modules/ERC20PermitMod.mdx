LibERC20Permit — Library for ERC-2612 Permit Logic - Thrown when a permit signature is invalid or expired.
</DocSubtitle>

<Callout type="info" title="Key Features">
- Implements ERC-2612 permit logic for gasless token approvals.
- Provides &#x60;DOMAIN_SEPARATOR&#x60; to ensure signature replay protection across chains.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

ERC20PermitMod provides core support for the ERC-2612 permit functionality, allowing token approvals to be granted via signed messages instead of on-chain transactions. This module is crucial for enabling gasless approvals and ensuring robust signature validation within a Compose diamond architecture.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

contract MyTokenFacet {
    function usePermit(address _owner, address _spender, uint256 _value, uint256 _deadline, uint8 _v, bytes32 _r, bytes32 _s) external {
        ERC20PermitMod.permit(_owner, _spender, _value, _deadline, _v, _r, _s);
        // Emit approval event as required by ERC20PermitMod
        emit Approval(_owner, _spender, _value);
    }

    function getDomainSeparator() external view returns (bytes32) {
        return ERC20PermitMod.DOMAIN_SEPARATOR();
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure the calling facet emits the &#x60;Approval&#x60; event after invoking &#x60;permit&#x60; to comply with ERC-2612.
- Validate inputs thoroughly and handle potential reverts from invalid or expired permits.
- Avoid modifying the domain separator after deployment to maintain signature validity.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
ERC20PermitMod interacts with diamond storage for both ERC20 and permit-specific state. Any facet calling &#x60;permit&#x60; must respect the domain separator&#x27;s immutability to ensure signature integrity. Changes made by &#x60;permit&#x60; (e.g., updating allowances) should be immediately visible to all facets sharing the same storage layout.
</Callout>
