LibERC20Permit — Library for ERC-2612 Permit Logic - Thrown when a permit signature is invalid or expired.
</DocSubtitle>

<Callout type="info" title="Key Features">
- Provides ERC-2612 permit implementation allowing gasless approvals.
- Includes a DOMAIN_SEPARATOR function to prevent signature replay attacks.
- Ensures compatibility with diamond storage patterns.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

The ERC20PermitMod module provides reusable library functions to implement ERC-2612 permit functionality in diamonds. It includes utilities for signature verification, allowance setting, and access to domain-specific storage, enabling secure token transfers without relying on traditional approval transactions. This promotes gas savings and enhances user experience within composable diamond systems.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

import {LibERC20Permit} from "./path/to/LibERC20Permit.sol";

contract MyTokenFacet {
    /**
     * Facet function implementation requiring permit logic.
     */
    function usePermit(address _owner, address _spender, uint256 _value, uint256 _deadline, uint8 _v, bytes32 _r, bytes32 _s) external {
        LibERC20Permit.permit(_owner, _spender, _value, _deadline, _v, _r, _s);

        // Optionally emit the Approval event from this facet, as required.
        emit Approval(_owner, _spender, _value);
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure the calling facet emits the Approval event after invoking &#x60;permit&#x60;.
- Maintain storage layout alignment with &#x60;getERC20Storage&#x60; and &#x60;getPermitStorage&#x60; for diamond upgrade safety.
- Validate the integrity of the DOMAIN_SEPARATOR across all deployments to prevent replay attacks.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
ERC20PermitMod interacts with diamond storage through getERC20Storage and getPermitStorage functions. Facets calling &#x60;permit&#x60; must manually emit the Approval event to match the interface specifications. Since DOMAIN_SEPARATOR is chain and contract-specific, facets relying on it must ensure its consistent computation during upgrades. Any misuse of storage alignment or domain separator integrity can lead to undefined behavior or security vulnerabilities.
</Callout>
