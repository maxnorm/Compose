<Callout type="info" title="Key Features">
- Supports dynamic addition, replacement, and removal of facet functions.
- Compliant with EIP-2535 and integrates seamlessly with diamond proxy architectures.
- Provides optional delegatecall initialization to set up state for new facets.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

The DiamondCutMod module provides essential functions for managing the composition of a diamond proxy. It allows adding, removing, and replacing facet functions dynamically, enabling flexible upgrades and maintenance. This ensures efficient storage reuse while maintaining compliance with EIP-2535 (Diamond Standard).

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

contract ExampleFacet {
    /**
     * Adds new functions to the diamond.
     */
    function addFacetFunctions(address _facet, bytes4[] memory _selectors) external {
        DiamondCutMod.diamondCut()({
            facets: [DiamondCutMod.FacetCut({facetAddress: _facet, action: DiamondCutMod.FacetCutAction.Add, functionSelectors: _selectors})],
            init: address(0),
            calldata: ''
        });
    }

    /**
     * Replaces an existing function in the diamond.
     */
    function replaceFacetFunction(address _newFacet, bytes4 _selector) external {
        DiamondCutMod.diamondCut()({
            facets: [DiamondCutMod.FacetCut({facetAddress: _newFacet, action: DiamondCutMod.FacetCutAction.Replace, functionSelectors: [_selector]})],
            init: address(0),
            calldata: ''
        });
    }

    /**
     * Removes functions from the diamond.
     */
    function removeFacetFunctions(bytes4[] memory _selectors) external {
        DiamondCutMod.diamondCut()({
            facets: [DiamondCutMod.FacetCut({facetAddress: address(0), action: DiamondCutMod.FacetCutAction.Remove, functionSelectors: _selectors})],
            init: address(0),
            calldata: ''
        });
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Always validate caller permissions when modifying the diamond (e.g., restrict to an admin or owner role).
- Ensure functions added or replaced maintain compatibility with existing storage layouts.
- Test edge cases for diamond upgrades to prevent unintended delegation logic changes.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
The module directly interacts with diamond storage to manage function selectors and associated facet addresses. Changes made via &#x60;diamondCut&#x60; are immediately visible across all facets in the diamond. When replacing or removing functions, ensure selectors are not in use or referenced elsewhere to avoid breaking dependencies. Initializations triggered via delegatecall must align with the existing storage layout to maintain state integrity.
</Callout>
