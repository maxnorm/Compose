Documentation for DiamondCutMod
</DocSubtitle>

<Callout type="info" title="Key Features">
- Provides atomic addition, replacement, or removal of functions within a diamond proxy.
- Optionally executes a delegatecall to initialize or configure the diamond after a cut operation.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

DiamondCutMod enables dynamic updates to a Compose diamond&#x27;s functionality by allowing atomic addition, replacement, or removal of facet functions. This module ensures modularity and flexibility while adhering to the diamond standard&#x27;s storage and upgrade patterns, making it critical for maintaining and evolving diamond proxies safely.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

/**
 * ExampleFacet demonstrates how to use DiamondCutMod to modify the diamond's functions.
 */
contract ExampleFacet {

    address private immutable diamondCutMod;

    /**
     * Initialize with the address of the DiamondCutMod facet.
     */
    constructor(address _diamondCutMod) {
        diamondCutMod = _diamondCutMod;
    }

    /**
     * Example function to add a facet to the diamond.
     */
    function addNewFacet(IDiamondCut.FacetCut[] calldata _facetCuts) external {
        DiamondCutMod(diamondCutMod).diamondCut(
            _facetCuts,
            address(0),
            ""
        );
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure proper access control on any function invoking &#x60;diamondCut&#x60; to prevent unauthorized modifications.
- Test all &#x60;diamondCut&#x60; transitions off-chain to verify correctness and ensure no storage or function collisions occur.
- Maintain a clear fallback mechanism to handle unexpected upgrades or revert to a valid state if the execution of &#x60;diamondCut&#x60; fails.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
DiamondCutMod operates on the diamond storage structure, directly modifying the function selectors and updating the mapping between selectors and their corresponding facets. Facets invoking &#x60;diamondCut&#x60; must account for any state changes or side effects triggered by their updates. Invariants such as the uniqueness of function selectors must be preserved, and developers should order operations to avoid unintentional removal or shadowing of existing functions.
</Callout>
