LibRoyalty - ERC-2981 Royalty Standard Library - Thrown when default royalty fee exceeds 100% (10000 basis points).
</DocSubtitle>

<Callout type="info" title="Key Features">
- Implements the ERC-2981 royalty logic for seamless token sale royalty calculation.
- Allows both default royalty settings and token-specific overrides for flexible configuration.
- Safeguards against setting royalty fees above 100% to maintain compliance with ERC standards.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

The RoyaltyMod module provides reusable library functions for managing royalties compliant with the ERC-2981 Royalty Standard in a diamond proxy architecture. It enables facets to set up default royalty settings and token-specific overrides, allowing for dynamic royalty calculation and storage reuse through the diamond storage pattern.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

contract ExampleFacet {
    using RoyaltyMod for RoyaltyMod.RoyaltyStorage;

    function setDefaultRoyalty(address receiver, uint96 feeNumerator) external {
        RoyaltyMod.setDefaultRoyalty(receiver, feeNumerator);
    }

    function resetSpecificTokenRoyalty(uint256 tokenId) external {
        RoyaltyMod.resetTokenRoyalty(tokenId);
    }

    function configureTokenRoyalty(uint256 tokenId, address receiver, uint96 feeNumerator) external {
        RoyaltyMod.setTokenRoyalty(tokenId, receiver, feeNumerator);
    }

    function getRoyaltyDetails(uint256 tokenId, uint256 salePrice) external view returns (address, uint256) {
        return RoyaltyMod.royaltyInfo(tokenId, salePrice);
    }

    function clearAllDefaultRoyalties() external {
        RoyaltyMod.deleteDefaultRoyalty();
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Use &#x60;setDefaultRoyalty&#x60; with a valid &#x60;receiver&#x60; address and ensure that &#x60;feeNumerator&#x60; does not exceed 10000 basis points (100%).
- Call &#x60;resetTokenRoyalty&#x60; or &#x60;deleteDefaultRoyalty&#x60; only when you intend to revert to the default royalty configuration or clear royalties altogether.
- Validate access control in facets to prevent unauthorized updates to royalty settings.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
The module utilizes a diamond storage slot dedicated to royalty information, accessed via &#x60;getStorage&#x60; using inline assembly. Any updates to default or token-specific royalties are immediately reflected in subsequent calls to &#x60;royaltyInfo&#x60;. Maintain the storage structure as defined and avoid altering the ordering or names of variables to ensure compatibility.
</Callout>
