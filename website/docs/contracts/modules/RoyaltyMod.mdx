## Overview

The RoyaltyMod provides functionality to manage and query royalties for NFTs, adhering to the ERC-2981 royalty standard. It allows for setting both default royalties and token-specific royalties, with mechanisms for managing storage safety in a diamond context. This ensures consistent royalty behavior across tokens in compliance with established standards while optimizing storage usage in diamond proxies.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

contract RoyaltyFacet {
    function setDefaultRoyaltyForAll(address receiver, uint96 feeNumerator) external {
        // Sets default royalty for all tokens
        RoyaltyMod.setDefaultRoyalty(receiver, feeNumerator);
    }

    function setRoyaltyForToken(uint256 tokenId, address receiver, uint96 feeNumerator) external {
        // Overrides default royalty for a specific token
        RoyaltyMod.setTokenRoyalty(tokenId, receiver, feeNumerator);
    }

    function removeDefaultRoyalty() external {
        // Removes the default royalty configuration
        RoyaltyMod.deleteDefaultRoyalty();
    }

    function fetchRoyalty(uint256 tokenId, uint256 salePrice) external view returns (address receiver, uint256 royaltyAmount) {
        // Queries royalty info for a token
        return RoyaltyMod.royaltyInfo(tokenId, salePrice);
    }

    function resetTokenToDefaultRoyalty(uint256 tokenId) external {
        // Resets a tokenâ€™s royalty to the default setting
        RoyaltyMod.resetTokenRoyalty(tokenId);
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Always validate receiver addresses and feeNumerator values to ensure correct royalty configuration.
- Use deleteDefaultRoyalty carefully; deleting default royalties might leave tokens without proper royalty settings.
- Avoid directly manipulating storage; rely on getStorage for managed access to diamond storage to ensure data consistency.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
The RoyaltyMod operates on composite diamond storage, leveraging a predefined slot for compatibility and composability. Ensure storage layouts align with the existing implementation to prevent overwrite. Reverting royalty configuration changes is possible by targeting correct token or default settings. Changes made to storage via one facet are visible across all facets employing this module. Ensure proper validation and error handling to maintain invariants.
</Callout>
