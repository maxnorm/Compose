<Callout type="info" title="Key Features">
- Implements the ERC-2981 standard for seamless integration with marketplaces and tools.
- Supports both default royalties and token-specific configurations, enabling flexible usage.
- Durable interaction with Compose diamond storage for efficient data management.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

The RoyaltyMod library implements the ERC-2981 royalty standard for managing and querying royalty information related to NFTs in a Compose diamond. It provides functions to set and retrieve default and token-specific royalty settings, ensuring proper revenue distribution for creators and rights holders while leveraging the diamond&#x27;s shared storage layout for optimal resource use.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

import {RoyaltyMod} from 'path-to-library/RoyaltyMod.sol';

contract RoyaltyFacet {
    function configureDefaultRoyalty(address _receiver, uint96 _feeNumerator) external {
        RoyaltyMod.setDefaultRoyalty(_receiver, _feeNumerator);
    }

    function clearDefaultRoyalty() external {
        RoyaltyMod.deleteDefaultRoyalty();
    }

    function configureTokenRoyalty(uint256 _tokenId, address _receiver, uint96 _feeNumerator) external {
        RoyaltyMod.setTokenRoyalty(_tokenId, _receiver, _feeNumerator);
    }

    function queryRoyalty(uint256 _tokenId, uint256 _salePrice) external view returns (address, uint256) {
        return RoyaltyMod.royaltyInfo(_tokenId, _salePrice);
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Verify that royalty fees (basis points) do not exceed 10000 (100%).
- Ensure that the receiver address is not zero when setting default or token-specific royalties.
- Regularly verify compliance with ERC-2981 by testing royaltyInfo against expected results for edge cases.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
The RoyaltyMod library directly modifies shared diamond storage using the defined slot for royalty data. When using this module, facets must verify correct configuration and integrity of the storage structure. Avoid making direct modifications to the royalty storage struct outside this library. Royalty configurations set via facets are immediately visible to all interactions querying royaltyInfo or related methods.
</Callout>
