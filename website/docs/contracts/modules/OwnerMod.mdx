## Overview

The OwnerMod module implements ERC-173 contract ownership functionality for diamonds, enabling safe and consistent owner management. It provides internal methods for accessing owner data, enforcing ownership checks, and transferring ownership, leveraging the diamond storage pattern for modularity and state isolation.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

contract MyFacet {
    using OwnerMod for OwnerMod.Storage;

    function getOwner() external view returns (address) {
        return OwnerMod.owner();
    }

    function transferContractOwnership(address _newOwner) external {
        OwnerMod.requireOwner(msg.sender);
        OwnerMod.transferOwnership(_newOwner);
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Always use the requireOwner function to enforce access control for critical operations.
- Ensure accurate and consistent use of the diamond storage slot defined by OwnerMod to avoid storage collisions.
- Use transferOwnership to renounce ownership by setting the new owner to address(0), ensuring any necessary fallback mechanisms are in place.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
The OwnerMod module interacts with diamond storage by using a fixed storage slot defined as STORAGE_POSITION. This ensures that the owner data is isolated from other facets while being accessible to all facets that use the same storage pattern. Modifying the storage layout or slot requires careful consideration to avoid breaking compatibility with existing facets. Ownership changes through transferOwnership are immediately reflected across all facets accessing the same storage.
</Callout>
