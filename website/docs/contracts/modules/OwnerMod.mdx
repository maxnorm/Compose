<Callout type="info" title="Key Features">
- Implements ERC-173 standard contract ownership for diamond patterns.
- Provides clear access control via &#x60;requireOwner&#x60;.
- Includes facilities for ownership renouncement by setting the owner to &#x60;address(0)&#x60;.

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

OwnerMod is a module implementing ERC-173 contract ownership using the diamond storage pattern. It provides internal functions for managing contract ownership, ensuring clear and safe access control within a diamond proxy&#x27;s facets. This module enables secure delegation of ownership, including renouncement, while maintaining storage integrity.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

import { OwnerMod } from './OwnerMod.sol';

contract ExampleFacet {
    function safeAction() external {
        // Ensure only the owner can access this function
        OwnerMod.requireOwner();

        // Perform sensitive operations
    }

    function changeOwnership(address _newOwner) external {
        // Transfer ownership using OwnerMod
        OwnerMod.transferOwnership(_newOwner);
    }

    function getOwner() external view returns (address) {
        // Retrieve the current owner
        return OwnerMod.owner();
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Use &#x60;requireOwner&#x60; to ensure only the current owner can access sensitive or administrative functions.
- Clearly document any functionality affected by owner renouncement (setting new owner to &#x60;address(0)&#x60;).
- Always initialize ownership during deployment or when linking the facet to avoid unintentional access risks.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
The module operates on a specific storage slot defined by its internal constant &#x60;STORAGE_POSITION&#x60;. Facets interacting with ownership data must use this module&#x27;s functions to ensure consistency and prevent direct access to diamond storage. Ownership changes through &#x60;transferOwnership&#x60; are immediately reflected for all facets, as they share the same diamond storage.
</Callout>
