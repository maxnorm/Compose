<Callout type="info" title="Key Features">
- Supports granting roles with expiration timestamps to enforce time-limited access.

<Callout type="info" title="Module Usage">

## Overview

AccessControlTemporalMod provides role-based access control with time-limited permissions for Compose diamonds. It allows granting roles with expiry timestamps, ensuring precise control over access duration. This module leverages diamond storage for safe and efficient state management.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

contract ExampleFacet {
    function assignTemporalRole(address target, bytes32 role, uint256 expiryTimestamp) external {
        // Grant the specified role with an expiry timestamp
        AccessControlTemporalMod.grantRoleWithExpiry(target, role, expiryTimestamp);
    }

    function checkRoleValidity(address target, bytes32 role) external view returns (bool) {
        // Check if the specified role is still valid
        try AccessControlTemporalMod.requireValidRole(target, role) {
            return true;
        } catch {
            return false;
        }
    }

    function revokeExpiredRole(address target, bytes32 role) external {
        // Revoke the role if it is expired
        if (AccessControlTemporalMod.isRoleExpired(target, role)) {
            AccessControlTemporalMod.revokeTemporalRole(target, role);
        }
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Always check the role expiry status using &#x60;isRoleExpired&#x60; before relying on role-based access logic.
- Use &#x60;requireValidRole&#x60; for critical access control paths to enforce both existence and non-expiry.
- Review and update expiry timestamps carefully to maintain synchronization with business requirements.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
This module uses diamond storage to manage role and expiry data, ensuring compatibility and safety when extending or upgrading diamonds. Facets interacting with this module must follow the diamond storage layout to avoid overwriting data. Consider using the &#x60;getAccessControlStorage&#x60; and &#x60;getStorage&#x60; functions to interact with the respective storage segments. Roles must be granted and validated consistently with the provided expiration mechanics to prevent unauthorized access.
</Callout>
