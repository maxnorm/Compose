Event emitted when a role is granted with an expiry timestamp.
</DocSubtitle>

<Callout type="info" title="Key Features">
- Allows assigning roles with specified expiration timestamps via &#x60;grantRoleWithExpiry&#x60;.
- Validates role assignments dynamically with &#x60;requireValidRole&#x60;, ensuring time-bound access control through diamond proxy composition.
- Supports efficient storage of expiration data with diamond storage pattern, improving composability and reducing contract size.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

AccessControlTemporalMod provides functionality for managing role-based access control with expiration timestamps in a Compose diamond. It enables fine-grained permission management by allowing roles to be granted and revoked dynamically while storing expiration data in diamond storage for efficiency and reusability.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

interface IAccessControlTemporalMod {
    function grantRoleWithExpiry(bytes32 _role, address _account, uint256 _expiry) external;
    function isRoleExpired(bytes32 _role, address _account) external view returns (bool);
}

contract MyFacet {
    IAccessControlTemporalMod private immutable accessControlModule;

    constructor(address _accessControlAddress) {
        accessControlModule = IAccessControlTemporalMod(_accessControlAddress);
    }

    function assignRole(bytes32 _role, address _account, uint256 _expiry) external {
        accessControlModule.grantRoleWithExpiry(_role, _account, _expiry);
    }

    function checkIfRoleExpired(bytes32 _role, address _account) external view returns (bool) {
        return accessControlModule.isRoleExpired(_role, _account);
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure role expiration timestamps are set to a future date when invoking &#x60;grantRoleWithExpiry&#x60; to avoid unintended immediate expirations.
- Use &#x60;requireValidRole&#x60; judiciously to enforce validity checks and handle its custom errors properly (&#x60;AccessControlUnauthorizedAccount&#x60;, &#x60;AccessControlRoleExpired&#x60;).
- Regularly monitor and prune expired roles by revoking them using &#x60;revokeTemporalRole&#x60; to maintain storage hygiene.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
Functions like &#x60;grantRoleWithExpiry&#x60; and &#x60;revokeTemporalRole&#x60; interact with diamond storage to manage temporal role assignments. Each role&#x27;s expiration data is stored in diamond storage, making it accessible across facets. All modifications should preserve the integrity of roles and expiry timestamps to ensure consistent system behavior. There are no specific ordering constraints, but facets relying on role validity must account for the role&#x27;s expiration status dynamically.
</Callout>
