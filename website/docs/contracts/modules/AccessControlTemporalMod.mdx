Event emitted when a role is granted with an expiry timestamp.
</DocSubtitle>

<Callout type="info" title="Key Features">
- Enables granting roles with an expiry timestamp for time-based security policies.
- Validates role expiration with &#x60;isRoleExpired&#x60; and &#x60;requireValidRole&#x60; functions, enhancing safety.
- Allows revocation of roles at any time using &#x60;revokeTemporalRole&#x60; for precise control.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

The AccessControlTemporalMod provides functionality for time-bound role management within a Compose diamond. It enables granting roles that automatically expire after a specified timestamp, ensuring time-based access control. This improves flexibility, enhances security, and minimizes reliance on manual revocation of roles.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

contract MyAccessControlFacet {
    AccessControlTemporalMod internal accessControlModule;

    constructor(address _accessControlModule) {
        accessControlModule = AccessControlTemporalMod(_accessControlModule);
    }

    function assignTemporalRole(address _account, bytes32 _role, uint256 _expiry) external {
        accessControlModule.grantRoleWithExpiry(_account, _role, _expiry);
    }

    function verifyRole(address _account, bytes32 _role) external view {
        accessControlModule.requireValidRole(_account, _role);
    }

    function removeTemporalRole(address _account, bytes32 _role) external {
        accessControlModule.revokeTemporalRole(_account, _role);
    }

    function isRoleStillValid(address _account, bytes32 _role) external view returns (bool) {
        return accessControlModule.isRoleExpired(_account, _role);
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure expiry timestamps are set according to system requirements to avoid unintended role expiration.
- Use &#x60;requireValidRole&#x60; for role verification before executing restricted functions to enforce access control.
- Validate inputs for functions like &#x60;grantRoleWithExpiry&#x60; and &#x60;revokeTemporalRole&#x60; to prevent invalid or unintended role assignments.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
AccessControlTemporalMod integrates with diamond storage using its own storage struct, which can be retrieved with &#x60;getAccessControlStorage&#x60; or &#x60;getStorage&#x60;. Roles and their expiry timestamps are stored within this dedicated structure and are shared across interacting facets. Ensure there is no conflict when using other access control modules, and maintain a strict ordering of storage initialization and assignment before accessing or modifying roles.
</Callout>
