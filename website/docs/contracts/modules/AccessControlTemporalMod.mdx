<DocSubtitle>
Event emitted when a role is granted with an expiry timestamp.
</DocSubtitle>

<Callout type="info" title="Key Features">
- Supports assigning roles with explicit expiry timestamps.
- Provides built-in validation for expired roles to enhance security.
- Enables revocation of roles before their natural expiry for dynamic access control.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

The AccessControlTemporalMod module manages role-based access control with temporal restrictions for Compose diamonds. It allows granting roles with expiry timestamps, validating active roles, and revoking them when necessary, ensuring fine-grained and time-bound permission handling.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

contract AccessControlFacet {
    AccessControlTemporalMod private _accessControl;

    constructor(address accessControlModule) {
        _accessControl = AccessControlTemporalMod(accessControlModule);
    }

    function assignRole(address account, bytes32 role, uint256 expiry) external {
        _accessControl.grantRoleWithExpiry(role, account, expiry);
    }

    function checkRole(address account, bytes32 role) external view returns (bool) {
        return !_accessControl.isRoleExpired(role, account);
    }

    function validateRole(address account, bytes32 role) external view {
        _accessControl.requireValidRole(role, account);
    }

    function removeRole(address account, bytes32 role) external {
        _accessControl.revokeTemporalRole(role, account);
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Always validate role expiry using &#x60;isRoleExpired&#x60; or &#x60;requireValidRole&#x60; before relying on access decisions.
- Maintain strict access control when calling &#x60;grantRoleWithExpiry&#x60; and &#x60;revokeTemporalRole&#x60; to prevent unauthorized modifications.
- Ensure that expiry timestamps are well-coordinated to avoid unintended access revocations or extensions.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
This module utilizes diamond storage for AccessControl and AccessControlTemporal data, ensuring efficient state management. Facets must interact with the module&#x27;s functions to grant, validate, and revoke roles. Expiry timestamps are checked at runtime, so it is critical to call &#x60;isRoleExpired&#x60; or &#x60;requireValidRole&#x60; when enforcing permissions. Ensure consistent ordering in updates to avoid data integrity issues.
</Callout>
