Diamond Library - Internal functions and storage for diamond proxy functionality.
</DocSubtitle>

<Callout type="info" title="Key Features">
- Adds new functionality to diamonds by mapping selectors to facet implementations.
- Dynamically routes calls to their correct facet using &#x60;diamondFallback&#x60;.
- Centralized storage access via &#x60;getStorage&#x60; helps maintain diamond state integrity.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

The DiamondMod library provides internal functionality and storage utilities for managing the facets and selector mapping of a diamond proxy. It enables adding new facets and routing external calls to their appropriate implementations, ensuring that diamonds maintain modularity and scalability.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

interface IDiamond {
    function addFacets(address[] calldata _facets) external;
    fallback() external payable;
}

contract ExampleFacet {
    IDiamond private diamondProxy;

    constructor(address _diamondProxy) {
        diamondProxy = IDiamond(_diamondProxy);
    }

    function addNewFacets(address[] calldata newFacets) external {
        diamondProxy.addFacets(newFacets);
    }

    fallback() external payable {
        (bool success, bytes memory data) = address(diamondProxy).call(msg.data);
        require(success, "Diamond call failed");
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Invoke &#x60;addFacets&#x60; only during diamond deployment to uphold compatibility with existing selectors.
- Ensure proper testing of added facets to verify their integration with the diamond.
- Avoid calling &#x60;diamondFallback&#x60; directly; ensure actual diamond fallback behavior adheres to EIP-2535 standards.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
DiamondMod functions interact directly with diamond storage, maintaining the selector-to-facet mapping and enabling extensibility. Changes through &#x60;addFacets&#x60; are immediately visible to facets that invoke routed calls. Upgrades must follow an order preserving existing selector integrity to ensure seamless behavior.
</Callout>
