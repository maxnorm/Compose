LibERC20 — ERC-20 Library - Thrown when a sender attempts to transfer or burn more tokens than their balance.
</DocSubtitle>

<Callout type="info" title="Key Features">
- Provides foundational ERC-20 functions (approve, mint, burn, transfer, transferFrom) in a diamond-compatible library.
- Leverages the diamond storage pattern for predictable and safe state management.
- Implements balance and allowance updates securely and atomically.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

ERC20Mod provides essential functionality for implementing ERC-20 token standards within a diamond proxy architecture. It ensures stateful operations like balance updates, allowance management, and supply adjustments are performed securely and efficiently. The module follows the diamond storage pattern, enabling seamless composability and upgradeability across facets.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

contract ERC20Facet {
    function mintTokens(address _to, uint256 _amount) external {
        ERC20Mod.mint(_to, _amount);
    }

    function burnTokens(address _from, uint256 _amount) external {
        ERC20Mod.burn(_from, _amount);
    }

    function approveSpender(address _spender, uint256 _amount) external {
        ERC20Mod.approve(_spender, _amount);
    }

    function transferTokens(address _to, uint256 _amount) external {
        ERC20Mod.transfer(msg.sender, _to, _amount);
    }

    function transferFromTokens(address _from, address _to, uint256 _amount) external {
        ERC20Mod.transferFrom(_from, _to, _amount);
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure all balance-altering operations (mint, burn, transfer) use access-controlled facets or adhere to tokenomics constraints.
- Always verify storage alignment when integrating or upgrading facets using &#x60;ERC20Mod&#x60;.
- Use external validation for user inputs like &#x60;amount&#x60; to prevent overflows or malicious transactions.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
ERC20Mod integrates with diamond storage via a dedicated storage slot for the ERC-20 struct, accessed through &#x60;getStorage()&#x60;. Facets using this module must ensure proper configuration of diamond storage to prevent data overlap. Any changes to the ERC-20 state, such as balances or allowances, are immediately reflected across all facets accessing the same storage slot. Maintain ordering and alignment of storage structs during upgrades to avoid breaking invariants.
</Callout>
