LibERC20 — ERC-20 Library - Thrown when a sender attempts to transfer or burn more tokens than their balance.
</DocSubtitle>

<Callout type="info" title="Key Features">
- Supports core ERC-20 functionality, including minting, burning, transfers, and approvals.
- Efficiently reuses diamond storage for tokens, reducing state conflicts and memory footprint.
- Accessed via stateless library functions for optimal gas efficiency in facet calls.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

The ERC20Mod module provides a lightweight implementation of core ERC-20 token mechanics, including minting, burning, transferring, and allowance management. It enables seamless storage reuse and efficient token operations within the Compose diamond framework while ensuring compliance with the ERC-20 standard.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

// Example Facet Utilizing ERC20Mod

import {ERC20Mod} from "./ERC20Mod.sol";

contract TokenFacet {
    function mintTokens(address _to, uint256 _amount) external {
        ERC20Mod.mint(_to, _amount);
    }

    function burnTokens(address _from, uint256 _amount) external {
        ERC20Mod.burn(_from, _amount);
    }

    function transferTokens(address _to, uint256 _amount) external {
        ERC20Mod.transfer(_to, _amount);
    }

    function approveSpender(address _spender, uint256 _amount) external {
        ERC20Mod.approve(_spender, _amount);
    }

    function transferFromTokens(address _from, address _to, uint256 _amount) external {
        ERC20Mod.transferFrom(_from, _to, _amount);
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Always use &#x60;getStorage()&#x60; to access the shared ERC-20 storage struct. Avoid direct storage slot manipulation.
- Ensure that minting and burning logic incorporates access control to prevent unauthorized token supply changes.
- When spreading functionality across facets, ensure consistency in interpreting storage operations to avoid unintended state changes.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
ERC20Mod relies on the diamond storage pattern to manage token data and operations. The &#x60;getStorage()&#x60; function ensures safe and consistent access to the shared token storage struct. Always call &#x60;getStorage()&#x60; before performing operations to maintain storage integrity. Any upgrades or facet additions must preserve the original storage layout, ensuring no reordering or removal of existing fields to prevent breaking interactions.
</Callout>
