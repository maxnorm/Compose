LibERC20 — ERC-20 Library - Thrown when a sender attempts to transfer or burn more tokens than their balance.
</DocSubtitle>

<Callout type="info" title="Key Features">
- Provides standard ERC-20 token behaviors including minting, burning, and transferring.
- Storage is managed using the diamond storage pattern, ensuring composability and upgrade safety.
- Allows efficient state management by accessing a fixed storage pointer via &#x60;getStorage&#x60;.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

ERC20Mod is a core library implementing the standard ERC-20 token functionality for Compose diamonds. It provides essential token features like transfers, approvals, minting, and burning, directly interfacing with the diamond&#x27;s storage. This enables facets to integrate token behavior while maintaining composable and upgradeable design patterns.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

import {ERC20Mod} from './ERC20Mod.sol';

contract ERC20Facet {
    /**
     * Transfers \`amount\` tokens from the caller to the \`recipient\`.
     * Updates balances directly without using allowances.
     */
    function transfer(address recipient, uint256 amount) external {
        ERC20Mod.transfer(msg.sender, recipient, amount);
    }

    /**
     * Mints \`amount\` tokens to the \`recipient\`.
     * Increases total supply and the recipient's balance.
     */
    function mint(address recipient, uint256 amount) external {
        // Implement access control here if required
        ERC20Mod.mint(recipient, amount);
    }

    /**
     * Approves \`spender\` to spend \`amount\` of tokens on behalf of the caller.
     */
    function approve(address spender, uint256 amount) external {
        ERC20Mod.approve(msg.sender, spender, amount);
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Validate input parameters in calling facet functions to ensure security and data integrity.
- Implement robust access control for administrative actions like &#x60;mint&#x60; and &#x60;burn&#x60;.
- Test thoroughly before upgrading a diamond that relies on this library to avoid breaking storage compatibility.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
ERC20Mod binds its operations to a shared diamond storage slot using the diamond storage pattern, ensuring all facets access a consistent and unified state. Facets should not modify or overwrite the ERC20Mod storage layout to maintain functional integrity. Any field modifications or extensions in storage should follow the storage reuse guidelines by extending the struct in a non-breaking manner.
</Callout>
