## Overview

The AccessControlMod module provides a flexible and efficient mechanism to manage role-based access control within diamond contracts. It enables seamless role assignments, checks, and administrative role management, adhering to Compose's diamond storage pattern for consistent cross-facet role visibility.


[View Source](https://github.com/maxnorm/Compose/blob/120348af74dbcdccde5d091d5c92f84ffed4753d/src/access/AccessControl/AccessControlMod.sol)



<Callout type="info" title="Key Features">
- Enables dynamic role assignments across facets.\n- Role verification via "hasRole" and "requireRole" simplifies access checks.\n- Supports hierarchical role management using "setRoleAdmin".
</Callout>



<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Usage Example

```solidity
contract AccessControlExample {\n    AccessControlMod private accessControl;\n\n    constructor(address _accessControlFacetAddress) {\n        accessControl = AccessControlMod(_accessControlFacetAddress);\n    }\n\n    function assignRole(bytes32 _role, address _account) external {\n        // Grant a role to an account.\n        accessControl.grantRole(_role, _account);\n    }\n\n    function checkRole(bytes32 _role, address _account) external view returns (bool) {\n        // Check if an account has a specific role.\n        return accessControl.hasRole(_role, _account);\n    }\n\n    function enforceRole(bytes32 _role, address _account) external view {\n        // Revert if the account does not have the required role.\n        accessControl.requireRole(_role, _account);\n    }\n}
```



## Best Practices

<Callout type="tip" title="Best Practice">
- Always use "requireRole" in functions where permission checks are essential to prevent unauthorized actions.\n- Organize roles logically and reduce unnecessary role creation to minimize complexity.\n- Use "setRoleAdmin" to establish clear administrative hierarchies for efficient role management.
</Callout>





## Integration Notes

<Callout type="success" title="Shared Storage">
AccessControlMod leverages the diamond storage pattern, ensuring that role states and administrative hierarchies set by this module are directly accessible across all facets. Changes to roles or admin assignments propagate consistently throughout the diamond's shared storage.
</Callout>
