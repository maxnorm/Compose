<Callout type="info" title="Key Features">
- Granular role-based access control to secure diamond-based operations.
- Functions for assigning, revoking, and validating roles to enforce permissions.
- Error handling with &#x60;AccessControlUnauthorizedAccount&#x60; for clear feedback on permission violations.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

The AccessControlMod module provides granular access management capabilities for Compose diamonds. It enables role-based permissions through functions to grant, revoke, and verify roles, ensuring controlled access to the diamond&#x27;s functionality. This module facilitates secure storage reuse and makes permissioned operations across facets straightforward and consistent.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

contract AccessControlFacet {
    function checkAndGrantRole(bytes32 role, address account) external {
        // Check if the caller has the necessary role
        AccessControlMod.requireRole(role, msg.sender);

        // Grant the role to the specified account
        AccessControlMod.grantRole(role, account);
    }

    function verifyRole(bytes32 role, address account) external view returns (bool) {
        // Check if the given account has the specified role
        return AccessControlMod.hasRole(role, account);
    }

    function removeRole(bytes32 role, address account) external {
        // Revoke the role from the specified account
        AccessControlMod.revokeRole(role, account);
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Always use &#x60;requireRole&#x60; early to enforce access control at the start of your functions.
- Keep roles dynamic and flexible by updating admins using &#x60;setRoleAdmin&#x60; whenever role management responsibilities change.
- Regularly audit role management functions to ensure roles are granted/revoked correctly and no unintended users hold unnecessary roles.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
AccessControlMod must leverage diamond storage for consistent state management across facets. Ensure &#x60;getStorage&#x60; is used to access the same storage slot as expected by all facets interacting with roles. Function ordering and proper access control checks are crucial to maintaining integrity and preventing unauthorized actions. Any changes, such as setting a new role admin, propagate to all facets relying on this module.
</Callout>
