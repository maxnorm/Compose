<DocSubtitle>
Emitted when the admin role for a role is changed.
</DocSubtitle>

<Callout type="info" title="Key Features">
- Supports role granularity, allowing multiple roles per account.
- Implements centralized role management via a shared diamond storage pattern.
- Emits errors for unauthorized role access using the &#x60;AccessControlUnauthorizedAccount&#x60; error.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

The AccessControlMod module manages role-based access control for a Compose diamond. It provides mechanisms to grant, revoke, and validate roles, ensuring secure and role-restricted operations. Using this module facilitates reusable and consistent access control across diamond facets, leveraging shared storage for role management.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

contract ExampleFacet {
    function grantAdminRole(address _account) external {
        AccessControlMod.grantRole(keccak256('ADMIN_ROLE'), _account);
    }

    function isAdmin(address _account) external view returns (bool) {
        return AccessControlMod.hasRole(keccak256('ADMIN_ROLE'), _account);
    }

    function requireAdminRole(address _account) external view {
        AccessControlMod.requireRole(keccak256('ADMIN_ROLE'), _account);
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Always validate input parameters (e.g., role hashes) before delegating to these functions.
- Use &#x60;requireRole&#x60; for critical operations to enforce necessary permissions at runtime.
- Avoid hardcoding roles with sensitive privileges; use consistently hashed constants to define roles.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
AccessControlMod integrates with diamond storage, storing role membership and admin relationships in a well-defined storage structure. All facets accessing roles or permissions must ensure they use the same role hash definitions. Modifying &#x60;setRoleAdmin&#x60; affects all dependent role functionality and should be performed with caution to avoid disrupting enforcement logic.
</Callout>
