Emitted when the admin role for a role is changed.
</DocSubtitle>

<Callout type="info" title="Key Features">
- Supports role-based access control for diamonds with minimal gas overhead.
- Provides storage reuse with a single source of truth for roles across facets.
- Role-based permissions can be dynamically updated with &#x60;grantRole&#x60; and &#x60;revokeRole&#x60; commands.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

The AccessControlMod provides role-based access control functionality for Compose diamonds. It facilitates permission management by allowing facets to grant, revoke, and check roles efficiently while leveraging diamond storage for consistent state access. This ensures composability, safety, and role hierarchy configuration.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

contract MyFacet {
    function checkAndGrantRole(bytes32 _role, address _account) external {
        // Ensure the caller has the role admin permissions
        AccessControlMod.requireRole(_role, msg.sender);

        // Grant the specified role to the provided account
        AccessControlMod.grantRole(_role, _account);
    }

    function isAccountAuthorized(bytes32 _role, address _account) external view returns (bool) {
        // Check if the account possesses the specified role
        return AccessControlMod.hasRole(_role, _account);
    }

    function updateRoleAdmin(bytes32 _role, bytes32 _newAdmin) external {
        // Ensure the caller is the current admin of the role
        AccessControlMod.requireRole(_role, msg.sender);

        // Set a new admin for the role
        AccessControlMod.setRoleAdmin(_role, _newAdmin);
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Use &#x60;requireRole&#x60; to enforce access control in facets before performing sensitive operations.
- Regularly audit role assignments and hierarchy via &#x60;getStorage&#x60; outputs to ensure accurate setup.
- Carefully update role admins only when necessary to prevent accidental access escalation.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
AccessControlMod integrates with diamond storage to maintain a shared state accessible by all facets. Changes to roles or admins instantly propagate to other facets using the same storage slot. When updating role admins, ensure all dependent facets are designed to respect the new hierarchy. Improper usage or manipulation of roles may inadvertently break system-wide permissions.
</Callout>
