<DocSubtitle>
Emitted when the admin role for a role is changed.
</DocSubtitle>

<Callout type="info" title="Key Features">
- Provides fine-grained role-based access control for Compose diamonds.
- Supports dynamic assignment, revocation, and hierarchical management of roles.
- Enforces access control through the &#x60;requireRole&#x60; function with a custom error for unauthorized access.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

The AccessControlMod module provides role-based access control mechanisms for Compose diamonds, allowing facets to manage permissions robustly. It enables dynamic role assignments, role checks, and admin role configuration using diamond storage, ensuring safe and modular access control management.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

interface IAccessControlMod {
    function grantRole(bytes32 role, address account) external;
    function hasRole(bytes32 role, address account) external view returns (bool);
    function requireRole(bytes32 role, address account) external view;
    function revokeRole(bytes32 role, address account) external;
    function setRoleAdmin(bytes32 role, bytes32 adminRole) external;
}

contract SomeFacet {
    IAccessControlMod constant accessControl = IAccessControlMod(0xYourAccessControlFacetAddress);

    bytes32 public constant ADMIN_ROLE = keccak256("ADMIN_ROLE");
    bytes32 public constant USER_ROLE = keccak256("USER_ROLE");

    function assignUserRole(address account) external {
        accessControl.requireRole(ADMIN_ROLE, msg.sender);
        accessControl.grantRole(USER_ROLE, account);
    }

    function revokeUserRole(address account) external {
        accessControl.requireRole(ADMIN_ROLE, msg.sender);
        accessControl.revokeRole(USER_ROLE, account);
    }

    function checkUserRole(address account) external view returns (bool) {
        return accessControl.hasRole(USER_ROLE, account);
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Always validate that the caller has the required role using &#x60;requireRole&#x60; before performing sensitive actions.
- Use &#x60;setRoleAdmin&#x60; to establish clear role hierarchies and maintain proper delegation of responsibilities.
- Regularly audit role assignments and ensure no orphaned or unintended roles exist after upgrades.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
AccessControlMod integrates with diamond storage to maintain role assignments and admin role mappings. Facets can access and modify this storage through the exposed functions, ensuring consistent state across the diamond. It is crucial to avoid direct storage modifications outside the module to maintain integrity. Changes in role assignments are immediately effective and visible to other facets using the AccessControlMod.
</Callout>
