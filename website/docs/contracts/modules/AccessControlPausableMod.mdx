<DocSubtitle>
Event emitted when a role is paused.
</DocSubtitle>

<Callout type="info" title="Key Features">
- Allows pausing and unpausing of specific roles for enhanced control and safety during critical operations.
- Emits defined errors ensuring clear debugging, such as AccessControlUnauthorizedAccount and AccessControlRolePaused.
- Permissionless facets can enforce role-based access while accounting for paused states.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

The AccessControlPausableMod module extends the diamond&#x27;s access control capabilities by providing the ability to pause and unpause specific roles. This is crucial for managing sensitive actions within a diamond proxy, ensuring controlled execution and safety during upgrades or emergency scenarios.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

interface IAccessControlPausable {
    function pauseRole(bytes32 role) external;
    function unpauseRole(bytes32 role) external;
    function isRolePaused(bytes32 role) external view returns (bool);
    function requireRoleNotPaused(bytes32 role) external view;
}

contract ExampleFacet {
    IAccessControlPausable constant accessControl = IAccessControlPausable(address(0xYourDiamondAddress));

    function performAdminAction(bytes32 adminRole) external {
        accessControl.requireRoleNotPaused(adminRole);
        // Add your logic here
    }

    function pauseAdminRole(bytes32 adminRole) external {
        accessControl.pauseRole(adminRole);
    }

    function unpauseAdminRole(bytes32 adminRole) external {
        accessControl.unpauseRole(adminRole);
    }

    function checkAdminRolePaused(bytes32 adminRole) external view returns (bool) {
        return accessControl.isRolePaused(adminRole);
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Always validate access control before pausing or unpausing roles to prevent privilege misuse.
- Avoid pausing critical roles without backup plans to maintain contract functionality.
- Ensure facets interacting with roles account for paused states to prevent unintended behavior.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
The module relies on diamond storage patterns where AccessControl and AccessControlPausable storage structs are accessed via getAccessControlStorage and getStorage. Pausing a role updates the diamond&#x27;s shared state, affecting all facets relying on the role&#x27;s permissions. Ensure role state changes are well-communicated to any dependent functionality within the diamond.
</Callout>
