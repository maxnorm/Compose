<DocSubtitle>
Event emitted when a role is paused.
</DocSubtitle>

<Callout type="info" title="Key Features">
- Enables role-based pausing, avoiding global contract shutdowns.
- Error-driven flow: emits clear errors such as AccessControlUnauthorizedAccount and AccessControlRolePaused.
- Provides view function isRolePaused to check the state of any role.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

AccessControlPausableMod provides granular role-based pause functionality within a diamond. It enhances safety by enabling temporary suspension of specific roles without affecting the entire contract, allowing for fine-grained control in response to incidents or maintenance needs.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

interface IAccessControlPausable {
    function pauseRole(bytes32 _role) external;
    function unpauseRole(bytes32 _role) external;
    function isRolePaused(bytes32 _role) external view returns (bool);
}

contract ExampleFacet {
    IAccessControlPausable internal accessControl;

    constructor(address _accessControlAddress) {
        accessControl = IAccessControlPausable(_accessControlAddress);
    }

    function pauseAdminRole() external {
        bytes32 adminRole = keccak256("ADMIN_ROLE");
        accessControl.pauseRole(adminRole);
    }

    function unpauseAdminRole() external {
        bytes32 adminRole = keccak256("ADMIN_ROLE");
        accessControl.unpauseRole(adminRole);
    }

    function isAdminRolePaused() external view returns (bool) {
        bytes32 adminRole = keccak256("ADMIN_ROLE");
        return accessControl.isRolePaused(adminRole);
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure only authorized entities can pause or unpause roles by implementing strict access control.
- Avoid leaving critical roles in a permanently paused state unless explicitly planned as part of a deactivation strategy.
- Handle errors AccessControlUnauthorizedAccount and AccessControlRolePaused consistently, especially in cross-contract integrations.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
AccessControlPausableMod uses diamond storage to maintain state across facets. The storage is accessed via getAccessControlStorage and getStorage functions, ensuring consistency and reusability across facets. Any changes made to pause state (via pauseRole or unpauseRole) are immediately visible to all facets using the shared storage. During upgrades, maintain storage integrity by preserving existing slot layout, and avoid reordering fields to uphold contract invariants.
</Callout>
