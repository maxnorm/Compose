<DocSubtitle>
Event emitted when a role is paused.
</DocSubtitle>

<Callout type="info" title="Key Features">
- Allows pausing and unpausing of specific roles dynamically.
- Supports robust role-based access control with pause-aware constraints.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

AccessControlPausableMod provides functionality to manage user roles with pausing capabilities in Compose diamonds. It enables role-based access control while allowing roles to be paused or unpaused dynamically, enhancing modularity and security in facet operations.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

interface IAccessControlPausableMod {
    function pauseRole(bytes32 _role) external;
    function isRolePaused(bytes32 _role) external view returns (bool);
    function unpauseRole(bytes32 _role) external;
    function requireRoleNotPaused(bytes32 _role, address _account) external view;
}

contract ExampleFacet {
    address private accessControlPausableAddress;

    constructor(address _accessControlPausableAddress) {
        accessControlPausableAddress = _accessControlPausableAddress;
    }

    function exampleFunction(bytes32 role, address account) external {
        // Ensure the role is not paused
        IAccessControlPausableMod(accessControlPausableAddress).requireRoleNotPaused(role, account);

        // Perform operations knowing role is active
    }

    function pauseRoleExample(bytes32 role) external {
        IAccessControlPausableMod(accessControlPausableAddress).pauseRole(role);
    }

    function unpauseRoleExample(bytes32 role) external {
        IAccessControlPausableMod(accessControlPausableAddress).unpauseRole(role);
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Use &#x60;requireRoleNotPaused&#x60; to enforce active role checks before critical logic.
- Manage role states (pause/unpause) judiciously to avoid unintended downtime for authorized actions.
- Ensure trusted access control management for pause/unpause functions to prevent misuse.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
AccessControlPausableMod interacts with diamond storage to manage role states. Any modifications to role pause status in one facet are immediately visible across all facets using the shared storage. Ensure consistent storage layout and avoid unauthorized alterations to preserve operational integrity. Use &#x60;requireRoleNotPaused&#x60; in all critical sections that depend on active roles.
</Callout>
