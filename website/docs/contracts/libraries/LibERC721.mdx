## Overview

LibERC721 is a dedicated Solidity library designed to provide core ERC-721 functionality for diamond contracts. It enables facets to efficiently implement token transfers, minting, burning, and storage management using the diamond storage pattern while ensuring clarity, composability, and auditability.


[View Source](https://github.com/maxnorm/Compose/blob/38545161423dd99e31b48b0eff41d260712fe731/src/token/ERC721/ERC721/LibERC721.sol)


<Callout type="info" title="Library Usage">
This library provides internal functions for use in your custom facets. Import it to access shared storage.
</Callout>

## Functions


### getStorage


Returns the ERC-721 storage struct from its predefined slot. Uses inline assembly to access diamond storage location.



```solidity
function getStorage() internal pure returns (ERC721Storage storage s);
```

## Errors


### ERC721NonexistentToken


Thrown when attempting to interact with a non-existent token.



```solidity
error ERC721NonexistentToken(uint256 _tokenId);
```



### ERC721IncorrectOwner


Thrown when the sender is not the owner of the token.



```solidity
error ERC721IncorrectOwner(address _sender, uint256 _tokenId, address _owner);
```



### ERC721InvalidSender


Thrown when the sender address is invalid (e.g., zero address).



```solidity
error ERC721InvalidSender(address _sender);
```



### ERC721InvalidReceiver


Thrown when the receiver address is invalid (e.g., zero address).



```solidity
error ERC721InvalidReceiver(address _receiver);
```



### ERC721InsufficientApproval


Thrown when an operator lacks sufficient approval to manage a token.



```solidity
error ERC721InsufficientApproval(address _operator, uint256 _tokenId);
```

## Usage Example

```solidity
pragma solidity ^0.8.0;

import { LibERC721 } from &quot;./LibERC721.sol&quot;;

contract ExampleFacet {
    function transferToken(address _from, address _to, uint256 _tokenId) external {
        LibERC721.transferFrom(_from, _to, _tokenId);
    }

    function mintToken(address _to, uint256 _tokenId) external {
        LibERC721.mint(_to, _tokenId);
    }

    function burnToken(uint256 _tokenId) external {
        LibERC721.burn(_tokenId);
    }

    function getERC721Storage() external view returns (bytes32 storageSlot) {
        return LibERC721.getStorage();
    }
}
```



## Best Practices

<Callout type="tip" title="Best Practice">
- Use the getStorage function to consistently access the correct slot without redundancy.
- Ensure validation (ownership, approval, address checks) is done explicitly using provided library utilities.
- Avoid externalizing storage manipulations outside of the library to maintain consistency and prevent errors.
</Callout>



## Integration Notes

<Callout type="success" title="Shared Storage">
LibERC721 uses a diamond storage pattern to manage ERC-721 data, ensuring that updates and state changes through the library are immediately visible to all facets relying on the same storage slot. This guarantees coherent and synchronized state management across the diamond.
</Callout>
