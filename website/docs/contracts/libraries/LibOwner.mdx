## Overview

LibOwner is a lightweight library designed to manage ownership logic in diamond proxy contracts following the ERC-173 standard. It simplifies the handling of ownership-related operations like getting the owner, transferring ownership, and enforcing ownership requirements while integrating seamlessly with the diamond storage pattern for state management.


[View Source](https://github.com/maxnorm/Compose/blob/38545161423dd99e31b48b0eff41d260712fe731/src/access/Owner/LibOwner.sol)


<Callout type="info" title="Library Usage">
This library provides internal functions for use in your custom facets. Import it to access shared storage.
</Callout>

## Usage Example

```solidity
pragma solidity ^0.8.0;
import &quot;./LibOwner.sol&quot;;

contract OwnershipFacet {
    function owner() external view returns (address) {
        return LibOwner.owner();
    }

    function transferOwnership(address _newOwner) external {
        LibOwner.requireOwner();
        LibOwner.transferOwnership(_newOwner);
    }

    function renounceOwnership() external {
        LibOwner.requireOwner();
        LibOwner.renounceOwnership();
    }
}
```



## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure the storage slot for ERC-173 is consistent across all facets and libraries to avoid corruption.
- Use clear, permissionless flows where suitable; minimize unnecessary owner checks to enhance composability.
- Always execute role-critical functions (like transferOwnership) through the requireOwner check to uphold security.
</Callout>



## Integration Notes

<Callout type="success" title="Shared Storage">
LibOwner leverages the diamond storage pattern to manage and update contract state directly through a predefined storage slot (STORAGE_POSITION). Any changes made to ownership logic through LibOwner are immediately reflected and accessible across all facets interacting with the same diamond storage.
</Callout>
