## Overview

LibOwner is a library for managing ownership functionality in diamond contracts, aligning with ERC-173 standards. It provides efficient functions for ownership checks, transfers, and renunciation, facilitating secure, permissioned controls while remaining composable within the diamond storage pattern.


[View Source](https://github.com/maxnorm/Compose/blob/7206f045d711880c211b967e80d72745b52f5cfc/src/access/Owner/LibOwner.sol)


<Callout type="info" title="Library Usage">
This library provides internal functions for use in your custom facets. Import it to access shared storage.
</Callout>

## Usage Example

```solidity
pragma solidity ^0.8.0;

import { LibOwner } from &quot;LibOwner.sol&quot;;

contract UseLibOwner {
    using LibOwner for LibOwner.Storage;

    function checkOwner() external view {
        address _owner = LibOwner.owner();
        require(msg.sender == _owner, &quot;Caller is not the owner&quot;);
    }
    
    function transferContractOwnership(address _newOwner) external {
        LibOwner.requireOwner();
        LibOwner.transferOwnership(_newOwner);
    }
}
```



## Best Practices

<Callout type="tip" title="Best Practice">
- Call &#39;requireOwner&#39; to enforce ownership checks before executing privileged actions.
- Utilize &#39;transferOwnership&#39; for clear, explicit transitions of contract control.
- Use &#39;renounceOwnership&#39; sparingly and ensure its implications are fully understood in your application.
</Callout>



## Integration Notes

<Callout type="success" title="Shared Storage">
LibOwner relies on the diamond storage pattern to manage ownership data, ensuring consistent access and updates across facets. Changes such as ownership transfers are visible to all facets referencing this library.
</Callout>
