## Overview

LibDiamond is a utility library designed for managing function additions, replacements, and removals in diamond contracts. It facilitates the modular assembly of facets, ensuring seamless upgradeability and composability while adhering to the diamond storage pattern.


[View Source](https://github.com/maxnorm/Compose/blob/7206f045d711880c211b967e80d72745b52f5cfc/src/diamond/LibDiamond.sol)


<Callout type="info" title="Library Usage">
This library provides internal functions for use in your custom facets. Import it to access shared storage.
</Callout>

## Usage Example

```solidity
import &quot;./LibDiamond.sol&quot;;

contract MyDiamond {
    function performDiamondCut() external {
        bytes4[] memory selectorsToAdd = new bytes4[](2);
        selectorsToAdd[0] = bytes4(keccak256(&quot;functionOne()&quot;));
        selectorsToAdd[1] = bytes4(keccak256(&quot;functionTwo()&quot;));

        address facetAddress = 0x1234567890123456789012345678901234567890;

        LibDiamond.diamondCut(
            [
                LibDiamond.FacetCut({
                    facetAddress: facetAddress,
                    action: LibDiamond.Add,
                    functionSelectors: selectorsToAdd
                })
            ],
            address(0),
            new bytes(0)
        );
    }
}
```



## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure functionSelectors are correctly computed using compatible hash algorithms.
- Use modular facet additions to minimize coupling and retain composability.
- Consistently verify diamond storage integrity post-diamondCut operations.
</Callout>



## Integration Notes

<Callout type="success" title="Shared Storage">
LibDiamond leverages the diamond storage pattern, ensuring all state changes through diamondCut are immediately visible to facets. Facets interact directly with shared storage, enabling composable and upgradeable contract designs.
</Callout>
