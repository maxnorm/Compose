## Overview

LibDiamond is a core library designed for use with the diamond standard (EIP-2535), enabling the dynamic addition, replacement, and removal of contract functions. It provides a robust and efficient way to manage functionality changes, ensuring that smart contracts remain flexible and modular over time.


[View Source](https://github.com/maxnorm/Compose/blob/38545161423dd99e31b48b0eff41d260712fe731/src/diamond/LibDiamond.sol)


<Callout type="info" title="Library Usage">
This library provides internal functions for use in your custom facets. Import it to access shared storage.
</Callout>

## Usage Example

```solidity
pragma solidity ^0.8.0;

import {LibDiamond} from &#39;./LibDiamond.sol&#39;;

contract ExampleDiamond {
    function diamondCutExample() external {
        LibDiamond.FacetCut[] memory diamondCuts = new LibDiamond.FacetCut[](2);

        diamondCuts[0] = LibDiamond.FacetCut({
            facetAddress: address(0x1234...5678),
            action: LibDiamond.Add,
            functionSelectors: new bytes4[](1)
        });
        diamondCuts[0].functionSelectors[0] = bytes4(keccak256(&quot;newFunction()&quot;));

        diamondCuts[1] = LibDiamond.FacetCut({
            facetAddress: address(0x9abc...def0),
            action: LibDiamond.Replace,
            functionSelectors: new bytes4[](1)
        });
        diamondCuts[1].functionSelectors[0] = bytes4(keccak256(&quot;existingFunction()&quot;));

        LibDiamond.diamondCut(
            diamondCuts,
            address(0),
            &quot;&quot;
        );
    }
}
```



## Best Practices

<Callout type="tip" title="Best Practice">
- Always ensure to test the diamondCut function in a controlled environment before deploying to production.
- Avoid unauthorized modifications by implementing strong access control mechanisms when integrating this library in contracts.
- Use custom errors appropriately for efficient error reporting and gas optimization.
</Callout>



## Integration Notes

<Callout type="success" title="Shared Storage">
LibDiamond operates seamlessly with the diamond storage pattern by directly modifying storage structures. Any changes made via LibDiamond functions are immediately visible to facets, ensuring consistency across the diamond contract. It is essential to follow EIP-2535 guidelines when using this library to maintain storage integrity and avoid conflicts.
</Callout>
