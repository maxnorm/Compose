## Overview

LibAccessControlPausable is a library designed to extend access control mechanisms in diamond contracts with the ability to pause specific roles. It simplifies role-based management for developers by providing utility functions to pause, unpause, and check the status of roles while seamlessly integrating with the diamond storage pattern.


[View Source](https://github.com/maxnorm/Compose/blob/38545161423dd99e31b48b0eff41d260712fe731/src/access/AccessControlPausable/LibAccessControlPausable.sol)


<Callout type="info" title="Library Usage">
This library provides internal functions for use in your custom facets. Import it to access shared storage.
</Callout>

## Usage Example

```solidity
library LibAccessControlPausable {
    struct AccessControlPausableStorage {
        mapping(bytes32 =&gt; bool) pausedRoles;
    }

    function getStorage() internal pure returns (AccessControlPausableStorage storage strg) {
        bytes32 slot = keccak256(&quot;diamond.standard.accesscontrolpausable.storage&quot;);
        assembly {
            strg.slot := slot
        }
    }

    function isRolePaused(bytes32 _role) internal view returns (bool) {
        return getStorage().pausedRoles[_role];
    }

    function pauseRole(bytes32 _role) internal {
        getStorage().pausedRoles[_role] = true;
    }

    function unpauseRole(bytes32 _role) internal {
        delete getStorage().pausedRoles[_role];
    }
}
```



## Best Practices

<Callout type="tip" title="Best Practice">
- Always check if a role is paused using isRolePaused before executing role-specific actions.
- Avoid removing roles or modifying AccessControlPausableStorage structure in ways that change storage slot layout.
- Use requireRoleNotPaused to ensure role consistency and prevent unexpected permission handling.
</Callout>



## Integration Notes

<Callout type="success" title="Shared Storage">
LibAccessControlPausable uses the diamond storage pattern to manage paused roles. Changes made to the storage by this library are immediately visible to all facets within the diamond, ensuring seamless composability and consistency across all access control related operations.
</Callout>
