## Overview

LibAccessControlPausable provides functionality for managing roles with pause capabilities in diamond contracts. It simplifies role-based access control and enhances security by allowing roles to be explicitly paused, preventing unintended access. This library is optimized for clarity and seamless integration with the diamond storage pattern.


[View Source](https://github.com/maxnorm/Compose/blob/e3e377cf3d77e94dc30e10812da2bdd1907ca159/src/access/AccessControlPausable/LibAccessControlPausable.sol)


<Callout type="info" title="Library Usage">
This library provides internal functions for use in your custom facets. Import it to access shared storage.
</Callout>

## Usage Example

```solidity

import {LibAccessControlPausable} from &quot;./LibAccessControlPausable.sol&quot;;

contract ExampleFacet {
    function pauseAdminRole() external {
        LibAccessControlPausable.pauseRole(keccak256(&quot;ADMIN_ROLE&quot;));
    }

    function isAdminRolePaused() external view returns (bool) {
        return LibAccessControlPausable.isRolePaused(keccak256(&quot;ADMIN_ROLE&quot;));
    }

    function performAction() external {
        LibAccessControlPausable.requireRoleNotPaused(msg.sender, keccak256(&quot;USER_ROLE&quot;));
        // Action logic here
    }
}
```



## Best Practices

<Callout type="tip" title="Best Practice">
- Always validate role status using `requireRoleNotPaused` before performing critical actions.
- Use role pausing selectively to avoid unnecessary interruptions in contract operations.
- Avoid removing or reordering storage variables within diamond contracts to maintain compatibility.
</Callout>



## Integration Notes

<Callout type="success" title="Shared Storage">
LibAccessControlPausable uses the diamond storage pattern to manage role states, ensuring all changes are interoperable and visible across facets in the same diamond contract. This design minimizes coupling and maximizes composability for customizable access control solutions.
</Callout>
