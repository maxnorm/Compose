## Overview

The LibAccessControlPausable library facilitates role-based access control in diamond contracts with the added functionality of pausing roles. It ensures fine-grained control over user permissions while maintaining compatibility with the diamond storage pattern, enabling modular, reusable management of roles and their states.


[View Source](https://github.com/maxnorm/Compose/blob/2ba2081cb9214075b097344cc10dc74bec360da3/src/access/AccessControlPausable/LibAccessControlPausable.sol)


<Callout type="info" title="Library Usage">
This library provides internal functions for use in your custom facets. Import it to access shared storage.
</Callout>

## Usage Example

```solidity
library LibAccessControlPausable {
    struct AccessControlPausableStorage {
        mapping(bytes32 =&gt; bool) pausedRoles;
    }

    function getStorage() internal pure returns (AccessControlPausableStorage storage ds) {
        bytes32 position = keccak256(&quot;diamond.library.LibAccessControlPausable&quot;);
        assembly {
            ds.slot := position
        }
    }

    function pauseRole(bytes32 _role) internal {
        getStorage().pausedRoles[_role] = true;
    }
}

contract ExampleFacet {
    function pauseAdminRole() external {
        LibAccessControlPausable.pauseRole(keccak256(&quot;ADMIN_ROLE&quot;));
    }
}
```



## Best Practices

<Callout type="tip" title="Best Practice">
- Always verify the role and pause state using `isRolePaused` before performing sensitive operations.
- Use descriptive role identifiers (e.g., `keccak256(&quot;ADMIN_ROLE&quot;)`) to avoid miscommunication and errors.
- Regularly audit your access control and role pause logic to ensure no unwanted overrides occur.
</Callout>



## Integration Notes

<Callout type="success" title="Shared Storage">
The LibAccessControlPausable library leverages the diamond storage pattern to ensure reliable storage management across facets. Any role state changes made through this library are immediately available and visible to other facets sharing the same diamond contract storage.
</Callout>
