## Overview

LibAccessControlTemporal provides functionality for managing access control with temporal restrictions in diamond contracts. It allows assigning roles with expiry timestamps, ensuring fine-grained time-based permissions within a diamond&#39;s facets.


[View Source](https://github.com/maxnorm/Compose/blob/7206f045d711880c211b967e80d72745b52f5cfc/src/access/AccessControlTemporal/LibAccessControlTemporal.sol)


<Callout type="info" title="Library Usage">
This library provides internal functions for use in your custom facets. Import it to access shared storage.
</Callout>

## Usage Example

```solidity
pragma solidity ^0.8.0;

import {LibAccessControlTemporal} from &quot;./LibAccessControlTemporal.sol&quot;;

contract ExampleFacet {
    function assignRole(address _account, bytes32 _role, uint256 _expiry) external {
        // Grant a role with an expiry timestamp
        LibAccessControlTemporal.grantRoleWithExpiry(_account, _role, _expiry);
    }

    function checkRole(address _account, bytes32 _role) external view returns (bool) {
        // Check if the role is valid and not expired
        return LibAccessControlTemporal.requireValidRole(_account, _role);
    }

    function revokeRole(address _account, bytes32 _role) external {
        // Revoke a temporal role
        LibAccessControlTemporal.revokeTemporalRole(_account, _role);
    }
}
```



## Best Practices

<Callout type="tip" title="Best Practice">
- Always specify accurate expiry timestamps when using grantRoleWithExpiry.
- Regularly clean up expired roles by revoking them to maintain clarity in AccessControl storage.
- Use requireValidRole to enforce permissions instead of duplicating expiration checks.
</Callout>



## Integration Notes

<Callout type="success" title="Shared Storage">
LibAccessControlTemporal operates seamlessly with the diamond storage pattern, ensuring that all changes to AccessControl or AccessControlTemporal storage are immediately visible to all facets using the same storage design.
</Callout>
