## Overview

LibAccessControlTemporal provides role-based access control functionalities with temporal (time-based) constraints. It enables role assignments with expiry timestamps, ensuring fine-grained and self-expiring permissions in diamond contracts.


[View Source](https://github.com/maxnorm/Compose/blob/2ba2081cb9214075b097344cc10dc74bec360da3/src/access/AccessControlTemporal/LibAccessControlTemporal.sol)


<Callout type="info" title="Library Usage">
This library provides internal functions for use in your custom facets. Import it to access shared storage.
</Callout>

## Usage Example

```solidity
pragma solidity ^0.8.0;

import &quot;LibAccessControlTemporal.sol&quot;;

contract ExampleDiamond {
    function assignTemporalRole(address _account, bytes32 _role, uint256 _expiry) external {
        LibAccessControlTemporal.grantRoleWithExpiry(_account, _role, _expiry);
    }

    function validateRole(address _account, bytes32 _role) external view returns (bool) {
        return !LibAccessControlTemporal.isRoleExpired(_account, _role);
    }

    function ensureAccountHasValidRole(address _account, bytes32 _role) external view {
        LibAccessControlTemporal.requireValidRole(_account, _role);
    }
}
```



## Best Practices

<Callout type="tip" title="Best Practice">
- Use `grantRoleWithExpiry` to assign roles with precise time bounds, minimizing unnecessary authorizations.
- Always check role validity using `isRoleExpired` or `requireValidRole` before performing sensitive actions to ensure access compliance.
- Revoke expired roles proactively using `revokeTemporalRole` to keep storage clean and access policies up-to-date.
</Callout>



## Integration Notes

<Callout type="success" title="Shared Storage">
This library adheres to the diamond storage pattern, directly modifying storage structures shared across facets. Changes to role assignments or expirations are immediately visible to all compatible facets within the diamond.
</Callout>
