## Overview

LibERC20 is a library designed to manage ERC-20 token operations within a diamond proxy architecture. It provides core functionality like minting, burning, transferring, and approving tokens, while adhering to the diamond storage pattern for seamless state management across facets.


[View Source](https://github.com/maxnorm/Compose/blob/7206f045d711880c211b967e80d72745b52f5cfc/src/token/ERC20/ERC20/LibERC20.sol)


<Callout type="info" title="Library Usage">
This library provides internal functions for use in your custom facets. Import it to access shared storage.
</Callout>

## Usage Example

```solidity
pragma solidity ^0.8.0;

library LibERC20 {
    struct ERC20Storage {
        mapping(address =&gt; uint256) balances;
        mapping(address =&gt; mapping(address =&gt; uint256)) allowances;
        uint256 totalSupply;
    }

    function getStorage() internal pure returns (ERC20Storage storage erc20Storage) {
        bytes32 position = keccak256(&quot;erc20.storage&quot;);
        assembly {
            erc20Storage.slot := position
        }
    }
}

contract TokenFacet {
    function mint(address _to, uint256 _amount) external {
        LibERC20.ERC20Storage storage erc20Storage = LibERC20.getStorage();
        erc20Storage.balances[_to] += _amount;
        erc20Storage.totalSupply += _amount;
    }
}
```



## Best Practices

<Callout type="tip" title="Best Practice">
- Always verify the storage slot used by getStorage aligns with your diamond configuration.
- Use the library&#39;s functions consistently across facets to ensure predictable interaction with ERC-20 state.
- Avoid modifying the library&#39;s base struct directly; extend functionality via additional methods or new facets.
</Callout>



## Integration Notes

<Callout type="success" title="Shared Storage">
LibERC20 utilizes the diamond storage pattern to ensure a single shared state across all facets using the library. Changes made to state via LibERC20 affect the global ERC-20 storage, ensuring facets operate seamlessly with accurate token balances and allowances.
</Callout>
