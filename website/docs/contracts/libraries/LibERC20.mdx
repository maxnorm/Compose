## Overview

LibERC20 is a utility library designed to handle common ERC-20 token operations like minting, burning, transferring, and approvals while conforming to the Compose diamond storage pattern. It enables efficient, composable implementation of token functionality within a diamond contract framework.


[View Source](https://github.com/maxnorm/Compose/blob/38545161423dd99e31b48b0eff41d260712fe731/src/token/ERC20/ERC20/LibERC20.sol)


<Callout type="info" title="Library Usage">
This library provides internal functions for use in your custom facets. Import it to access shared storage.
</Callout>

### mint


### burn

## Errors


### ERC20InsufficientBalance


Thrown when a sender attempts to transfer or burn more tokens than their balance.



```solidity
error ERC20InsufficientBalance(address _sender, uint256 _balance, uint256 _needed);
```



### ERC20InvalidSender


Thrown when the sender address is invalid (e.g., zero address).



```solidity
error ERC20InvalidSender(address _sender);
```



### ERC20InvalidReceiver


Thrown when the receiver address is invalid (e.g., zero address).



```solidity
error ERC20InvalidReceiver(address _receiver);
```



### ERC20InsufficientAllowance


Thrown when a spender tries to spend more than their allowance.



```solidity
error ERC20InsufficientAllowance(address _spender, uint256 _allowance, uint256 _needed);
```



### ERC20InvalidSpender


Thrown when the spender address is invalid (e.g., zero address).



```solidity
error ERC20InvalidSpender(address _spender);
```

## Usage Example

```solidity
pragma solidity ^0.8.17;
import &quot;LibERC20.sol&quot;;

contract ERC20Facet {
    function mintTokens(address _to, uint256 _amount) external {
        LibERC20.mint(_to, _amount);
    }

    function burnTokens(address _from, uint256 _amount) external {
        LibERC20.burn(_from, _amount);
    }

    function transferTokens(address _to, uint256 _amount) external {
        LibERC20.transfer(msg.sender, _to, _amount);
    }
}
```



## Best Practices

<Callout type="tip" title="Best Practice">
- Always validate inputs (e.g., non-zero recipient address) before calling LibERC20 functions.
- Use explicit permission checks when minting or burning tokens if required by your application logic.
- Ensure the storage layout is compatible with the diamond storage pattern to avoid state conflicts.
</Callout>



## Integration Notes

<Callout type="success" title="Shared Storage">
LibERC20 leverages the diamond storage pattern to manage ERC-20 token state consistently across facets. Changes made using this library are immediately visible to other facets relying on the same shared storage.
</Callout>
