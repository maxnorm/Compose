## Overview

LibERC20 is a foundational library for implementing ERC-20 token logic within diamond contracts, streamlining token operations like minting, burning, transferring, and approvals. It adheres to the diamond storage pattern and ensures consistent, gas-efficient integration with facets.


[View Source](https://github.com/maxnorm/Compose/blob/e3e377cf3d77e94dc30e10812da2bdd1907ca159/src/token/ERC20/ERC20/LibERC20.sol)


<Callout type="info" title="Library Usage">
This library provides internal functions for use in your custom facets. Import it to access shared storage.
</Callout>

## Usage Example

```solidity
pragma solidity ^0.8.0;

library LibERC20 {
    struct ERC20Storage {
        mapping(address =&gt; uint256) balances;
        mapping(address =&gt; mapping(address =&gt; uint256)) allowances;
        uint256 totalSupply;
    }

    bytes32 private constant STORAGE_POSITION = keccak256(&quot;compose.erc20.storage&quot;);

    function getStorage() internal pure returns (ERC20Storage storage ds) {
        assembly {
            ds.slot := STORAGE_POSITION
        }
    }

    function mint(address _to, uint256 _amount) internal {
        ERC20Storage storage ds = getStorage();
        ds.balances[_to] += _amount;
        ds.totalSupply += _amount;
    }

    // Example facet using LibERC20
    contract ERC20Facet {
        function mintTokens(address to, uint256 amount) external {
            LibERC20.mint(to, amount);
        }
    }
```



## Best Practices

<Callout type="tip" title="Best Practice">
- Always ensure the library is consistently integrated with the diamond storage pattern.
- Use explicit initialization processes for library-dependent data to avoid storage collisions.
- Prefer permissionless token functions unless admin control is essential for your use case.
</Callout>



## Integration Notes

<Callout type="success" title="Shared Storage">
LibERC20 adheres to the diamond storage pattern, ensuring changes to storage through this library are seamless and visible across other facets in the diamond. Properly designed slot usage avoids conflicts while enabling interoperable token logic across the contract.
</Callout>
