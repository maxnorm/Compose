## Overview

LibAccessControl is a library for managing role-based access control in diamond contracts. It simplifies assigning, revoking, and verifying roles across facets while maintaining security and composability. This ensures a cohesive authorization system compatible with the diamond storage pattern.


[View Source](https://github.com/maxnorm/Compose/blob/7206f045d711880c211b967e80d72745b52f5cfc/src/access/AccessControl/LibAccessControl.sol)


<Callout type="info" title="Library Usage">
This library provides internal functions for use in your custom facets. Import it to access shared storage.
</Callout>

## Usage Example

```solidity
pragma solidity ^0.8.0;

import &quot;LibAccessControl.sol&quot;;

contract ExampleDiamondFacet {
    function secureAction() external {
        // Ensure caller has the required role before proceeding
        LibAccessControl.requireRole(msg.sender, keccak256(&quot;ROLE_ADMIN&quot;));
        // Business logic
    }

    function assignRole(address _account) external {
        // Grant ROLE_USER to the specified account
        LibAccessControl.grantRole(_account, keccak256(&quot;ROLE_USER&quot;));
    }

    function revokeRole(address _account) external {
        // Revoke ROLE_USER from the specified account
        LibAccessControl.revokeRole(_account, keccak256(&quot;ROLE_USER&quot;));
    }
}
```



## Best Practices

<Callout type="tip" title="Best Practice">
- Always define clear and distinct roles to avoid confusion.
- Use LibAccessControl.requireRole for critical function access verifications.
- Organize role assignments and revocations in admin-managed flows.
</Callout>



## Integration Notes

<Callout type="success" title="Shared Storage">
LibAccessControl operates within the diamond storage pattern, ensuring roles and permissions defined in storage are accessible and consistent across all facets. Changes made to roles are immediately visible to other facets interacting with the same diamond storage.
</Callout>
