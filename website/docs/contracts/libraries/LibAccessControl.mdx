## Overview

LibAccessControl is an internal library designed for diamond proxy contracts to efficiently manage role-based access control. It ensures decentralized and permissioned interactions with precise role management, seamlessly integrating with the diamond storage pattern.


[View Source](https://github.com/maxnorm/Compose/blob/38545161423dd99e31b48b0eff41d260712fe731/src/access/AccessControl/LibAccessControl.sol)


<Callout type="info" title="Library Usage">
This library provides internal functions for use in your custom facets. Import it to access shared storage.
</Callout>

## Usage Example

```solidity
pragma solidity ^0.8.0;

import {LibAccessControl} from &quot;./LibAccessControl.sol&quot;;

contract ExampleDiamondFacet {
    function grantManagerRole(address account) external {
        LibAccessControl.grantRole(keccak256(&quot;MANAGER_ROLE&quot;), account);
    }

    function checkRole(address account) external view {
        LibAccessControl.requireRole(keccak256(&quot;MANAGER_ROLE&quot;), account);
    }

    function removeManagerRole(address account) external {
        LibAccessControl.revokeRole(keccak256(&quot;MANAGER_ROLE&quot;), account);
    }
}
```



## Best Practices

<Callout type="tip" title="Best Practice">
- Use readable `keccak256`-hashed string constants for roles to reduce ambiguity.
- Grant or revoke roles only through explicitly defined functions for clarity and security.
- Ensure role-specific admin roles are appropriately set for hierarchical control.
</Callout>



## Integration Notes

<Callout type="success" title="Shared Storage">
LibAccessControl uses the diamond storage pattern to store and manage roles, ensuring changes are immediately reflected across all associated facets. This enables cohesive, centralized role management within the diamond proxy structure.
</Callout>
