## Overview

LibERC20Bridgeable enables cross-chain minting and burning of ERC20 tokens in diamond contract architectures. It ensures secure functionality by leveraging role-based access control and the Compose storage and diamond proxy patterns, providing a reusable, scalable solution.


[View Source](https://github.com/maxnorm/Compose/blob/38545161423dd99e31b48b0eff41d260712fe731/src/token/ERC20/ERC20Bridgeable/LibERC20Bridgeable.sol)


<Callout type="info" title="Library Usage">
This library provides internal functions for use in your custom facets. Import it to access shared storage.
</Callout>

## Usage Example

```solidity
pragma solidity ^0.8.20;

import &quot;./LibERC20Bridgeable.sol&quot;;

contract MyTokenFacet {
    function mintCrosschain(address _to, uint256 _amount) external {
        require(LibERC20Bridgeable.checkTokenBridge(msg.sender), &quot;Not a trusted bridge&quot;);
        LibERC20Bridgeable.crosschainMint(_to, _amount);
    }

    function burnCrosschain(address _from, uint256 _amount) external {
        require(LibERC20Bridgeable.checkTokenBridge(msg.sender), &quot;Not a trusted bridge&quot;);
        LibERC20Bridgeable.crosschainBurn(_from, _amount);
    }
}
```



## Best Practices

<Callout type="tip" title="Best Practice">
- Carefully define and assign the `trusted-bridge` role when calling crosschainMint or crosschainBurn.
- Validate and audit bridge callers to ensure permissions are correctly enforced.
- Avoid directly modifying any diamond storage outside of the library to maintain consistency.
</Callout>



## Integration Notes

<Callout type="success" title="Shared Storage">
LibERC20Bridgeable adheres to the diamond storage pattern, ensuring compatibility with other facets. Changes made using this library (such as minting or burning) are immediately visible to all facets that utilize the same diamond storage.
</Callout>
