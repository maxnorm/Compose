## Overview

LibERC721Enumerable is a Solidity library designed for implementing ERC-721 enumerable token functionality within the diamond standard. It provides efficient and clear operations for token ownership enumeration, including methods for transferring, minting, and burning tokens while maintaining enumeration integrity.


[View Source](https://github.com/maxnorm/Compose/blob/38545161423dd99e31b48b0eff41d260712fe731/src/token/ERC721/ERC721Enumerable/LibERC721Enumerable.sol)


<Callout type="info" title="Library Usage">
This library provides internal functions for use in your custom facets. Import it to access shared storage.
</Callout>

## Usage Example

```solidity
pragma solidity ^0.8.0;

import &quot;./LibERC721Enumerable.sol&quot;;

contract TestDiamondFacet {
    function mintToken(address _to, uint256 _tokenId) external {
        LibERC721Enumerable.mint(_to, _tokenId);
    }

    function transfer(address _from, address _to, uint256 _tokenId) external {
        LibERC721Enumerable.transferFrom(_from, _to, _tokenId);
    }

    function burnToken(uint256 _tokenId) external {
        LibERC721Enumerable.burn(_tokenId);
    }

    function getStorage() external view returns (LibERC721Enumerable.ERC721EnumerableStorage storage) {
        return LibERC721Enumerable.getStorage();
    }
}
```



## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure the diamond storage slot for enumerables is correctly and consistently set when integrating this library.
- Always validate input addresses and token IDs to prevent unintended behavior or state corruption.
- Maintain appropriate access controls to restrict sensitive operations like minting or burning tokens where required.
</Callout>



## Integration Notes

<Callout type="success" title="Shared Storage">
LibERC721Enumerable leverages the diamond storage pattern by accessing a predefined storage slot via inline assembly. Any changes made using the library&#39;s methods directly update this storage and are seamlessly visible to all facets of the diamond using the same shared storage.
</Callout>
