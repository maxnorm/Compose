## Overview

LibERC6909 is a utility library designed for use with the diamond storage pattern in ERC-6909 tokens. It provides optimized and direct methods for token minting, burning, transferring, and authorization, making it ideal for scalable, reusable, and composable multi-facet contracts.


[View Source](https://github.com/maxnorm/Compose/blob/7206f045d711880c211b967e80d72745b52f5cfc/src/token/ERC6909/ERC6909/LibERC6909.sol)


<Callout type="info" title="Library Usage">
This library provides internal functions for use in your custom facets. Import it to access shared storage.
</Callout>

## Usage Example

```solidity
pragma solidity ^0.8.0;

import &quot;./LibERC6909.sol&quot;;

contract TokenFacet {
    function mintToken(uint256 _id, address _to, uint256 _amount) external {
        LibERC6909.mint(_id, _to, _amount);
    }

    function transferToken(uint256 _id, address _from, address _to, uint256 _amount) external {
        LibERC6909.transfer(_id, _from, _to, _amount);
    }

    function approveAndSetOperator(uint256 _id, address _spender, uint256 _amount, bool _isOperator) external {
        require(_amount &gt; 0, &quot;Amount must be positive&quot;);
        LibERC6909.approve(_id, _spender, _amount);
        LibERC6909.setOperator(msg.sender, _spender, _isOperator);
    }
}
```



## Best Practices

<Callout type="tip" title="Best Practice">
- Call `getStorage` to securely access and manipulate the ERC-6909 Diamond storage slot.
- Always validate inputs like `_amount` or addresses `_to`/`_from` before using library functions to ensure correctness.
- Implement clear and permissionless usage patterns ensuring this library aligns well with your diamond contract architecture.
</Callout>



## Integration Notes

<Callout type="success" title="Shared Storage">
LibERC6909 integrates seamlessly with the diamond storage pattern by providing access to a uniquely defined slot for ERC-6909 storage. Any updates to the storage through it are immediately accessible across all facets, preserving consistency and composability within the diamond architecture.
</Callout>
