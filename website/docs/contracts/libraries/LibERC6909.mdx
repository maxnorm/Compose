## Overview

LibERC6909 is a library providing core functionalities for managing ERC-6909 token operations in the diamond standard architecture. It includes methods for minting, burning, transferring, approving tokens, and managing operators, all while ensuring alignment with diamond storage patterns.


[View Source](https://github.com/maxnorm/Compose/blob/38545161423dd99e31b48b0eff41d260712fe731/src/token/ERC6909/ERC6909/LibERC6909.sol)


<Callout type="info" title="Library Usage">
This library provides internal functions for use in your custom facets. Import it to access shared storage.
</Callout>

## Usage Example

```solidity
pragma solidity ^0.8.0;

library LibERC6909 {
    struct ERC6909Storage {
        mapping(address =&gt; mapping(uint256 =&gt; uint256)) balances;
        mapping(address =&gt; mapping(address =&gt; bool)) operators;
    }

    bytes32 constant STORAGE_POSITION = keccak256(&quot;diamond.standard.ERC6909.storage&quot;);

    function getStorage() internal pure returns (ERC6909Storage storage es) {
        bytes32 position = STORAGE_POSITION;
        assembly {
            es.slot := position
        }
    }

    function mint(address _to, uint256 _id, uint256 _amount) internal {
        ERC6909Storage storage es = getStorage();
        es.balances[_to][_id] += _amount;
    }

    function transfer(address _from, address _to, uint256 _id, uint256 _amount, address _by) internal {
        ERC6909Storage storage es = getStorage();
        require(es.balances[_from][_id] &gt;= _amount, &quot;Insufficient balance&quot;);
        es.balances[_from][_id] -= _amount;
        es.balances[_to][_id] += _amount;
    }
}

// Usage in a Facet
contract ERC6909Facet {
    function mintToken(address to, uint256 id, uint256 amount) external {
        LibERC6909.mint(to, id, amount);
    }

    function transferToken(address from, address to, uint256 id, uint256 amount) external {
        LibERC6909.transfer(from, to, id, amount, msg.sender);
    }
}
```



## Best Practices

<Callout type="tip" title="Best Practice">
- Always use the `getStorage` function when accessing ERC-6909 storage to ensure compliance with diamond storage patterns.
- Avoid making external-facing calls directly from the library; instead, call functions within the appropriate facet.
- Regularly audit diamond storage patterns to prevent conflicts and maintain consistent memory layout.
</Callout>



## Integration Notes

<Callout type="success" title="Shared Storage">
This library operates strictly within the diamond storage pattern. By accessing storage using the `getStorage` function, it ensures consistent data visibility across all facets of the diamond that interact with ERC-6909 tokens. Changes made via library functions immediately reflect in other facets sharing the same storage slot, maintaining integrity and synchronization.
</Callout>
