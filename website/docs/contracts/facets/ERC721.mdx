## Overview

The ERC721 facet facilitates the integration and management of ERC-721 non-fungible tokens (NFTs) within a diamond proxy. It provides essential functions such as minting, burning, transferring tokens, and managing metadata while conforming to the ERC-721 standard.


[View Source](https://github.com/maxnorm/Compose/blob/f773c1f2b6ca1fc289a7b9ea4f149315edd34199/src/token/ERC721/ERC721/ERC721.sol)



<Callout type="info" title="Key Features">
- Provides minting and burning functions compliant with the ERC-721 standard
- Facilitates secure ownership transfer and approval management
- Enables diamond proxy users to access storage slots specifically for ERC-721 implementation
</Callout>

## Errors


### ERC721IncorrectOwner


Thrown when the sender is not the owner of the token. **Parameters**



```solidity
error ERC721IncorrectOwner(address _sender, uint256 _tokenId, address _owner);
```



### ERC721InsufficientApproval


Thrown when an operator lacks sufficient approval to manage a token. **Parameters**



```solidity
error ERC721InsufficientApproval(address _operator, uint256 _tokenId);
```



### ERC721InvalidReceiver


Thrown when the receiver address is invalid (e.g., zero address). **Parameters**



```solidity

```



### ERC721InvalidSender


Thrown when the sender address is invalid (e.g., zero address). **Parameters**



```solidity

```



### ERC721NonexistentToken

## Usage Example

```solidity
pragma solidity ^0.8.0;

import {Diamond} from &quot;./Diamond.sol&quot;; // Simplified example
import {ERC721Facet} from &quot;./facets/ERC721Facet.sol&quot;;

contract MyDiamond is Diamond {
    function exampleUsage() external {
        // Add ERC721 facet to the diamond
        bytes memory initFunction = abi.encodeWithSelector(ERC721Facet.mint.selector, _msgSender(), 1);
        Diamond.addFacet(address(new ERC721Facet()), initFunction);

        // Example: Mint an ERC-721 token
        ERC721Facet(address(this)).mint(0xAbc123456789abcdef, 1234);

        // Example: Transfer a token
        ERC721Facet(address(this)).transferFrom(0xAbc123456789abcdef, 0xdef987654321fedcba, 1234);
    }
}
```



## Best Practices

<Callout type="tip" title="Best Practice">
- Always verify proper initialization of the ERC-721 facet before invoking any function to avoid unexpected errors.
- Use meaningful and unique token IDs to prevent overlapping among tokens managed by the diamond proxy.
- Ensure sufficient gas limits are set for minting/burning functions, especially if metadata or event emission is involved.
</Callout>



## Security Considerations

<Callout type="warning" title="Security">
Ensure all external functions enforce proper access control and validation mechanisms. Avoid calling mint or burn functions within the same transaction that interacts with external contracts (to prevent reentrancy attacks). Confirm that receiver addresses in transferFrom are valid to mitigate undesirable token states or loss.
</Callout>
