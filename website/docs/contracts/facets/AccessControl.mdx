## Overview

The AccessControl facet enables flexible and modular role-based access control within a Compose diamond. It allows dynamically assigning, revoking, and verifying roles, as well as defining administrative privileges for role management. This is essential in diamond contracts to enforce permissioned actions without embedding redundant logic within every facet.


[View Source](https://github.com/maxnorm/Compose/blob/f773c1f2b6ca1fc289a7b9ea4f149315edd34199/src/access/AccessControl/AccessControl.sol)



<Callout type="info" title="Key Features">
- Secure role assignment and revocation with detailed logging
- Allows hierarchical role-based structures by setting role administrators
- Enables flexible and composable access control for diamonds
</Callout>

## Usage Example

```solidity
pragma solidity ^0.8.17;

import { IDiamondCut } from &quot;diamond/interfaces/IDiamondCut.sol&quot;;
import { AccessControl } from &quot;diamond/facets/AccessControl.sol&quot;;

contract MyDiamond {
    function deployAccessControlFacet() external {
        IDiamondCut diamondCut = IDiamondCut(diamondAddress);

        // Add AccessControl facet to the diamond
        diamondCut.diamondCut(
            [
                IDiamondCut.FacetCut({
                    facetAddress: address(new AccessControl()),
                    action: IDiamondCut.FacetCutAction.Add,
                    functionSelectors: AccessControl.getSelectors()
                })
            ],
            address(0),
            &quot;&quot;
        );

        // Grant an initial role to an account
        AccessControl(facetAddress).grantRole(keccak256(&quot;ADMIN&quot;), msg.sender);
    }
}
```



## Best Practices

<Callout type="tip" title="Best Practice">
- Use meaningful and descriptive role identifiers, e.g., keccak256 string hashes to avoid accidental conflicts.
- Revoke roles immediately when no longer needed or after an account is compromised.
- Call `hasRole` or `requireRole` before executing sensitive functions to preemptively check access permissions.
</Callout>



## Security Considerations

<Callout type="warning" title="Security">
Always validate access permissions using `requireRole` or `hasRole` to enforce restrictions. Carefully manage `_admin` accounts to avoid privilege escalation. Take care to prevent reentrancy issues if AccessControl roles are used in conjunction with sensitive external calls.
</Callout>
