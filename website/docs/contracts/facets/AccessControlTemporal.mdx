## Overview

The AccessControlTemporal facet enables role-based access control with temporal constraints in diamond contracts. It provides functionality to assign roles with expiry timestamps, validate their validity, and revoke them, enhancing flexibility and security in access management.


[View Source](https://github.com/maxnorm/Compose/blob/f773c1f2b6ca1fc289a7b9ea4f149315edd34199/src/access/AccessControlTemporal/AccessControlTemporal.sol)



<Callout type="info" title="Key Features">
- Assign roles with defined expiry timestamps.
- Validate if a role is expired before granting access.
- Revoke temporal roles explicitly.
</Callout>

## Usage Example

```solidity
pragma solidity ^0.8.0;

interface IDiamond {
    function facetAddress(bytes4 _functionSelector) external view returns (address);
}

contract Example {
    IDiamond public diamond;
    
    constructor(address _diamondAddress) {
        diamond = IDiamond(_diamondAddress);
    }

    function grantTemporaryRole(address account, bytes32 role, uint256 expiry) external {
        // Call the grantRoleWithExpiry function from the AccessControlTemporal facet
        (bool success, bytes memory data) = diamond.facetAddress(bytes4(keccak256(&quot;grantRoleWithExpiry&quot;))).delegatecall(
            abi.encodeWithSignature(&quot;grantRoleWithExpiry(address,bytes32,uint256)&quot;, account, role, expiry)
        );
        require(success, &quot;Role assignment failed&quot;);
    }
}
```



## Best Practices

<Callout type="tip" title="Best Practice">
- Always set appropriate expiry timestamps to minimize long-term risk.
- Use requireValidRole strategically to enforce access constraints where necessary.
- Regularly check for and revoke expired roles to maintain a secure state.
</Callout>



## Security Considerations

<Callout type="warning" title="Security">
Ensure roles with administrative or critical permissions have tighter expiry constraints. Validate user input to prevent assigning roles with excessively long durations. Test for reentrancy vulnerabilities in interactions with external contracts.
</Callout>
