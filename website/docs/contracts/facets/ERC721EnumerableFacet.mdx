<Callout type="info" title="Key Features">
- Supports standard ERC721 token operations: name, symbol, totalSupply, balanceOf, ownerOf, tokenURI.
- Provides enumerable functionality: tokenOfOwnerByIndex, allowing iteration over a specific owner&#x27;s tokens.
- Manages token approvals and operator authorizations (approve, setApprovalForAll, getApproved, isApprovedForAll).
- Includes internal and external transfer functions with safety checks (internalTransferFrom, transferFrom, safeTransferFrom).
</Callout>

## Overview

The ERC721EnumerableFacet provides standard ERC721 functionality with added support for enumeration of tokens. It exposes functions to retrieve token metadata, ownership details, and allows for approvals and transfers, while enabling tracking of total supply and token ownership by index.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

import { IERC721Enumerable } from "@openzeppelin/contracts/token/ERC721/extensions/IERC721Enumerable.sol";
import { IDiamondCut } from "@compose/diamond-contracts/contracts/interfaces/IDiamondCut.sol";

contract ERC721EnumerableDeployment {
    address constant ERC721_FACET_ADDRESS = address(0x...); // Address where ERC721EnumerableFacet is deployed

    function deploy() public {
        // Assume diamondCut is an instance of IDiamondCut on the diamond proxy
        address diamondCut = address(0x...);

        IDiamondCut(diamondCut).diamondCut(
            IDiamondCut.FacetCut[] memory cuts,
            address(0), // init address
            bytes("") // init calldata
        );
    }

    function interactWithERC721(address _diamondAddress) public view {
        IERC721Enumerable erc721 = IERC721Enumerable(_diamondAddress);

        // Get token name and symbol
        string memory name = erc721.name();
        string memory symbol = erc721.symbol();

        // Get total supply
        uint256 supply = erc721.totalSupply();

        // Get owner of a specific token
        address owner = erc721.ownerOf(1);

        // Get token at index for a specific owner
        uint256 tokenId = erc721.tokenOfOwnerByIndex(owner, 0);

        // Get token URI
        string memory uri = erc721.tokenURI(tokenId);
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Initialize the facet with essential ERC721 metadata (name, symbol) during diamond deployment.
- Ensure proper access control is implemented at the diamond proxy level for sensitive functions like transfers and approvals.
- When upgrading, ensure the storage layout remains compatible to prevent data loss or corruption.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
Implement robust access control at the diamond proxy level for functions that modify state (approve, setApprovalForAll, transfers) to prevent unauthorized actions. Ensure input validation for token IDs and addresses to prevent unexpected behavior. The &#x60;internalTransferFrom&#x60; function should only be called by authorized internal logic within the diamond to maintain state integrity.
</Callout>
