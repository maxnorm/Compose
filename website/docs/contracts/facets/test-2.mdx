## Overview

The AccessControlFacet is a self-contained module for managing role-based access control (RBAC) within a diamond proxy. It enables flexible role assignment, revocation, and validation, supporting both single and batch operations. This facet ensures fine-grained control of permissions, making it essential for managing access in composable smart contract systems.


[View Source](https://github.com/maxnorm/Compose/blob/38545161423dd99e31b48b0eff41d260712fe731/src/access/AccessControl/AccessControlFacet.sol)



<Callout type="info" title="Key Features">
- Fine-grained role-based access control
- Batch operations for efficient role management
- Emits events for all role changes to ensure visibility
</Callout>

## Usage Example

```solidity
pragma solidity ^0.8.0;
import { IDiamondCut } from &quot;./interfaces/IDiamondCut.sol&quot;;

diamondCut = [
    IDiamondCut.FacetCut({
        facetAddress: address(accessControlFacet),
        action: IDiamondCut.FacetCutAction.Add,
        functionSelectors: [
            accessControlFacet.hasRole.selector,
            accessControlFacet.grantRole.selector,
            accessControlFacet.revokeRole.selector,
            accessControlFacet.requireRole.selector
        ]
    })
];
// Add the AccessControlFacet to the diamond.
diamond.diamondCut(facets);

// Example usage: Check role and grant role.
AccessControlFacet(diamondAddress).grantRole(keccak256(&quot;ADMIN_ROLE&quot;), adminAddress);
bool hasAdminRole = AccessControlFacet(diamondAddress).hasRole(keccak256(&quot;ADMIN_ROLE&quot;), adminAddress);
```



## Best Practices

<Callout type="tip" title="Best Practice">
- Assign roles responsibly, especially admin roles, to prevent unauthorized access.
- Regularly validate role settings to avoid privilege escalation and unintended access.
- Use `requireRole` for critical functions to enforce strict access control.
</Callout>



## Security Considerations

<Callout type="warning" title="Security">
Ensure that only trusted accounts are granted roles with high privileges, especially admin roles, as they have the authority to modify other roles. Functions that modify roles (e.g., `grantRole`, `setRoleAdmin`) should be carefully controlled to prevent unauthorized changes. Reentrancy is not directly a concern here, but external function calls from callbacks should be audited for reentrancy vulnerabilities.
</Callout>
