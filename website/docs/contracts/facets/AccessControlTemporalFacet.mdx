Contract documentation for AccessControlTemporalFacet
</DocSubtitle>

<Callout type="info" title="Key Features">
- Grants roles with time-based restrictions via the &#x60;grantRoleWithExpiry&#x60; function.
- Validates the status of a role using &#x60;isRoleExpired&#x60; or &#x60;requireValidRole&#x60;.
- Emits clear events for granted and revoked roles to support auditability.
</Callout>

## Overview

The AccessControlTemporalFacet manages role-based access control with temporal restrictions within a diamond. It provides functionality to assign roles with expiry timestamps, validate role validity, and revoke expired or temporal roles, supporting finer-grained access control in modular contracts.

## Best Practices

<Callout type="tip" title="Best Practice">
- Utilize &#x60;grantRoleWithExpiry&#x60; for roles that need time-limited authorization. Ensure the role expiry aligns with business needs.
- Always check the validity of a role using &#x60;requireValidRole&#x60; before granting permissions to ensure expired roles are not exploited.
- Implement a robust framework for handling access control upgrades to maintain role integrity when extending or modifying the diamond.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
Restrict &#x60;grantRoleWithExpiry&#x60; and &#x60;revokeTemporalRole&#x60; to role administrators to prevent unauthorized modifications. Validate all role expirations using &#x60;requireValidRole&#x60; to enforce temporal restrictions. Protect against reentrancy where such risks may arise by ensuring external calls are minimized and state changes occur before any external interaction.
</Callout>
