Contract documentation for AccessControlTemporalFacet
</DocSubtitle>

<Callout type="info" title="Key Features">
- Supports granting roles with specific expiry timestamps for enhanced access control.
- Provides runtime validation of role expiry for secure and dynamic authorization checks.
- Emits events (&#x60;RoleGrantedWithExpiry&#x60;, &#x60;TemporalRoleRevoked&#x60;) for transparent role changes.
</Callout>

## Overview

The AccessControlTemporalFacet provides time-bound role management for diamonds, enabling dynamic access control with expiration timestamps. It extends standard role-based access control by introducing temporal constraints, ensuring roles are automatically invalidated after a specified duration.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

import { IDiamondCut } from "./IDiamondCut.sol";
import { AccessControlTemporalFacet } from "./AccessControlTemporalFacet.sol";

contract DiamondExample {
    function useAccessControlTemporalFacet(address diamondAddress) external {
        AccessControlTemporalFacet facet = AccessControlTemporalFacet(diamondAddress);

        // Grant a role with an expiry timestamp
        bytes32 role = keccak256("ADMIN");
        address account = 0x1234567890123456789012345678901234567890;
        uint256 expiry = block.timestamp + 7 days;
        facet.grantRoleWithExpiry(role, account, expiry);

        // Check if a role is valid and not expired
        require(facet.requireValidRole(role, account), "Role is invalid or expired");

        // Revoke a temporal role
        facet.revokeTemporalRole(role, account);
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Initialize the facet by assigning the admin roles during diamond deployment to prevent unauthorized access.
- Use &#x60;requireValidRole&#x60; to validate role assignments and ensure expired roles are not mistakenly accepted.
- Follow diamond upgrade guidelines to preserve the integrity of the AccessControl storage layout when adding or updating facets.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
Ensure only authorized accounts can call &#x60;grantRoleWithExpiry&#x60; and &#x60;revokeTemporalRole&#x60; by enforcing strict admin checks. Validate all input parameters to prevent improperly set expiry timestamps or invalid role identifiers. Use &#x60;requireValidRole&#x60; in all sensitive logic to guard against expired or unauthorized roles. Avoid reentrancy issues by not performing external calls while roles are being granted, revoked, or validated.
</Callout>
