Contract documentation for AccessControlTemporalFacet
</DocSubtitle>

<Callout type="info" title="Key Features">
- Time-bound role grants: Assign roles that automatically expire after a specified timestamp.
- Temporal role revocation: Explicitly revoke time-bound roles before their expiry.
- Expiry validation: Functions to check if a role assignment is still valid or has expired.
</Callout>

## Overview

The AccessControlTemporalFacet extends standard role-based access control by introducing time-bound role assignments. It allows for granting roles with specific expiry timestamps and checking their validity, enhancing dynamic permission management within a Compose diamond.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

import {IAccessControlTemporal} from "@compose/contracts/src/facets/AccessControl/IAccessControlTemporal.sol";
import {IDiamondCut} from "@compose/contracts/src/interfaces/IDiamondCut.sol";

contract AccessControlTemporalDeployment {
    function deploy() public {
        // Assume diamondCutFacet is already deployed and accessible
        address diamondCutFacet = address(0x123);
        address accessControlTemporalFacet = address(0x456); // Deployed facet address

        // Add the AccessControlTemporalFacet to the diamond
        IDiamondCut(diamondCutFacet).diamondCut(new IDiamondCut.FacetCut[](0), address(0), bytes(''));
        IDiamondCut(diamondCutFacet).diamondCut(
            new IDiamondCut.FacetCut[](1) = {
                IDiamondCut.FacetCut(0,
                    accessControlTemporalFacet,
                    IDiamondCut.FacetCutAction.Add
                )
            },
            address(0),
            ''
        );

        // Grant a role with expiry
        // Assume 'DIAMOND_ADMIN_ROLE' is defined and accessible
        bytes32 role = keccak256("DIAMOND_ADMIN_ROLE");
        uint256 expiryTimestamp = block.timestamp + 3600; // Role expires in 1 hour
        address user = address(0x789);

        IAccessControlTemporal(address(this)).grantRoleWithExpiry(role, user, expiryTimestamp);

        // Check if role is valid
        bool isValid = IAccessControlTemporal(address(this)).requireValidRole(role, user);
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Initialize roles and grant initial administrative privileges before deploying this facet.
- Ensure the &#x60;diamondCut&#x60; function is called with appropriate &#x60;FacetCutAction.Add&#x60; when integrating this facet into a diamond.
- Use &#x60;grantRoleWithExpiry&#x60; judiciously, ensuring the granting account possesses the necessary administrative privileges for the target role.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
Access to &#x60;grantRoleWithExpiry&#x60; and &#x60;revokeTemporalRole&#x60; is restricted to the admin of the role, preventing unauthorized role modifications. The &#x60;requireValidRole&#x60; function explicitly checks for role expiry, preventing access by accounts with expired roles. Ensure that role administration is correctly configured to prevent privilege escalation.
</Callout>
