## Overview

The AccessControlTemporalFacet enables time-based role management in a diamond contract, allowing roles to be granted with specific expiration times. It ensures improved flexibility and fine-grained access control by verifying role validity based on time, while also adhering to the diamond storage and modular design principles.


[View Source](https://github.com/maxnorm/Compose/blob/7050baa250694d5e2445aa4c6b96c62c326cb1cb/src/access/AccessControlTemporal/AccessControlTemporalFacet.sol)



<Callout type="info" title="Key Features">
- Allows time-bound roles by granting them with expiration timestamps.\n- Includes validation for expired roles, improving security.\n- Emits events on role changes for improved traceability.
</Callout>

## Usage Example

```solidity
pragma solidity ^0.8.19;\n\nimport {DiamondCutFacet} from &#39;./facets/DiamondCutFacet.sol&#39;;\nimport {AccessControlTemporalFacet} from &#39;./facets/AccessControlTemporalFacet.sol&#39;;\n\ncontract Diamond {\n    event RoleGrantedWithExpiry(bytes32 role, address account, uint256 expiry);\n    event TemporalRoleRevoked(bytes32 role, address account);\n\n    constructor(address diamondCutFacet, address accessControlTemporalFacet) {\n        // Add facets to the diamond\n        DiamondCutFacet(diamondCutFacet).diamondCut([\n            DiamondCutFacet.FacetCut({facetAddress: accessControlTemporalFacet, action: DiamondCutFacet.FacetCutAction.Add, functionSelectors: new bytes4[](0)})\n        ], address(0), &#39;&#39;);\n    }\n\n    function assignRole(address account, bytes32 role, uint256 expiry) external {\n        AccessControlTemporalFacet(address(this)).grantRoleWithExpiry(role, account, expiry);\n    }\n\n    function checkRole(address account, bytes32 role) external view returns (bool) {\n        return !AccessControlTemporalFacet(address(this)).isRoleExpired(role, account);\n    }\n}
```



## Best Practices

<Callout type="tip" title="Best Practice">
- Define a clear role hierarchy and ensure only authorized accounts can grant roles.\n- Regularly check and revoke expired roles to maintain clarity in access.\n- Use `requireValidRole` wherever role-dependent functionality is implemented to enforce role validity.
</Callout>




## Security Considerations

<Callout type="warning" title="Security">
Ensure that the role admin only grants/revokes roles where necessary to prevent improper access. Always apply a reentrancy guard or checks-effects-interactions pattern if integrating this facet's role-reliant methods with external calls. Validate inputs, such as account and timestamp, to mitigate misuse or unintended behavior.
</Callout>
