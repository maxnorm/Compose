## Overview

The AccessControlTemporalFacet provides role-based access control with time-based expiry for roles in a diamond contract. It enhances traditional access control by introducing temporal constraints, ensuring fine-grained and time-sensitive permission management.


[View Source](https://github.com/maxnorm/Compose/blob/2ba2081cb9214075b097344cc10dc74bec360da3/src/access/AccessControlTemporal/AccessControlTemporalFacet.sol)



<Callout type="info" title="Key Features">
- Grant roles with specific expiry timestamps.
- Revoke time-based roles securely.
- Verify role validity with expiration awareness.
</Callout>

## Usage Example

```solidity
pragma solidity ^0.8.0;

import { Diamond } from &quot;.../Diamond.sol&quot;;
import { AccessControlTemporalFacet } from &quot;.../AccessControlTemporalFacet.sol&quot;;

contract UseAccessControlTemporal {
    function setupRoleWithExpiry(address diamond, bytes32 role, address account, uint256 expiry) external {
        AccessControlTemporalFacet(diamond).grantRoleWithExpiry(role, account, expiry);
    }

    function checkRole(address diamond, bytes32 role, address account) external view {
        AccessControlTemporalFacet(diamond).requireValidRole(role, account);
    }

    function revokeExpiredRole(address diamond, bytes32 role, address account) external {
        AccessControlTemporalFacet(diamond).revokeTemporalRole(role, account);
    }
}
```



## Best Practices

<Callout type="tip" title="Best Practice">
- Always set clear and reasonable expiry timestamps to prevent long-standing permissions.
- Regularly audit role expiry statuses using isRoleExpired to identify outdated accounts.
- Use requireValidRole checks before performing sensitive operations.
</Callout>



## Security Considerations

<Callout type="warning" title="Security">
- Ensure only authorized accounts (e.g., role admins) perform grant and revoke actions.
- Avoid granting unnecessarily long expiry durations to minimize risk of privilege abuse.
- Address reentrancy by using the Checks-Effects-Interactions pattern within role management logic.
</Callout>
