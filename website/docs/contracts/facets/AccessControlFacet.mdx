## Overview

The AccessControlFacet provides role-based access control mechanisms tailored for diamond proxy architecture. It enables fine-grained permission management with functionality to grant, revoke, or check roles for accounts, ensuring secure and modular authorization within the diamond&#39;s components.


[View Source](https://github.com/maxnorm/Compose/blob/7206f045d711880c211b967e80d72745b52f5cfc/src/access/AccessControl/AccessControlFacet.sol)



<Callout type="info" title="Key Features">
- Granular role-based access control.
- Batch grant and revoke functionality for efficiency.
- Admin assignability for dynamic role management.
</Callout>

import { AccessControlFacet } from &quot;./facets/AccessControlFacet.sol&quot;;

    function deployDiamondWithAccessControl(address _diamondOwner) external {
        // Deploy the diamond and add the AccessControlFacet
        IDiamondCut.FacetCut[] memory diamondCut = new IDiamondCut.FacetCut[](1);

        diamondCut[0] = IDiamondCut.FacetCut({
            facetAddress: address(new AccessControlFacet()),
            action: IDiamondCut.FacetCutAction.Add,
            functionSelectors: generateSelectors(&quot;AccessControlFacet&quot;)
        });

        // Add the diamond facets (example assumes DiamondInit exists)
        IDiamondCut(diamondCutDiamondAddress).diamondCut(
            diamondCut,
            address(0),
            &quot;&quot;
        );
    }

    function grantAdminRole(address accessControlFacetAddress, bytes32 role, address account) external {
        AccessControlFacet(accessControlFacetAddress).grantRole(role, account);
    }
}

## Best Practices

<Callout type="tip" title="Best Practice">
- Properly document role assignments to ensure clarity for developers and auditors.
- Use `requireRole` for sensitive operations where a specific role is mandatory.
- Avoid assigning overly broad or ambiguous admin roles to maintain clear boundaries.
</Callout>



## Security Considerations

<Callout type="warning" title="Security">
- Ensure only intended administrators can call `grantRole`, `revokeRole`, and similar functions.
- Prevent reentrancy attacks by avoiding external contract calls within role management functions.
- Use a clear and immutable hierarchy for roles to avoid unintended privilege escalation.
</Callout>
