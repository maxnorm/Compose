<Callout type="info" title="Key Features">
- Fine-grained, role-based access control for diamonds.
- Support for batch role assignment and revocation for efficient updates.
- Admin management functionality to define hierarchical roles and access levels.
</Callout>

## Overview

The AccessControlFacet provides granular role-based access control functionality for a Compose diamond. It enables flexible role management, including granting, revoking, and renouncing roles, as well as defining role hierarchies via admin roles. This facet is essential for managing permissions and ensuring secure access within a diamond architecture.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

// Example usage of AccessControlFacet in a diamond project
import { IDiamondCut } from "@compose/diamond/interfaces/IDiamondCut.sol";
import { AccessControlFacet } from "path/to/AccessControlFacet.sol";

contract DiamondDeployer {
    function deployDiamond() external {
        // Deploy facet contract
        AccessControlFacet accessControlFacet = new AccessControlFacet();

        // Define facet cuts
        IDiamondCut.FacetCut[] memory diamondCut = new IDiamondCut.FacetCut[](1);
        diamondCut[0] = IDiamondCut.FacetCut({
            facetAddress: address(accessControlFacet),
            action: IDiamondCut.FacetCutAction.Add,
            functionSelectors: generateSelectors("AccessControlFacet")
        });

        // Perform diamond cut
        IDiamondCut(diamond).diamondCut(diamondCut, address(0), "");
    }

    function generateSelectors(string memory _facetName) internal pure returns (bytes4[] memory) {
        if (keccak256(abi.encodePacked(_facetName)) == keccak256("AccessControlFacet")) {
            bytes4[] memory selectors = new bytes4[](10);
            selectors[0] = AccessControlFacet.getStorage.selector;
            selectors[1] = AccessControlFacet.hasRole.selector;
            selectors[2] = AccessControlFacet.requireRole.selector;
            selectors[3] = AccessControlFacet.getRoleAdmin.selector;
            selectors[4] = AccessControlFacet.setRoleAdmin.selector;
            selectors[5] = AccessControlFacet.grantRole.selector;
            selectors[6] = AccessControlFacet.revokeRole.selector;
            selectors[7] = AccessControlFacet.grantRoleBatch.selector;
            selectors[8] = AccessControlFacet.revokeRoleBatch.selector;
            selectors[9] = AccessControlFacet.renounceRole.selector;
            return selectors;
        }
        revert("Invalid facet name");
    }

    function useAccessControl(address diamond, address account, bytes32 role) external view returns (bool) {
        return AccessControlFacet(diamond).hasRole(role, account);
    }
}

// Example external call
// AccessControlFacet(diamondAddress).grantRole(role, account);`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Initialize roles and admin assignments during deployment to prevent unauthorized access.
- Use facets and modular permissions carefully to avoid accidental configuration errors.
- Manage diamond cuts securely to prevent unintended re-ordering of function selectors.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
Ensure proper access control in diamondCut to prevent unauthorized facet replacement. Validate role inputs to prevent misuse. Avoid reentrancy attacks by carefully designing callable functions. Use role revoking and renouncing features responsibly to prevent privilege escalation or accidental loss of access.
</Callout>
