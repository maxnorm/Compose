## Overview

The AccessControlFacet provides role-based access management for diamond contracts, enabling developers to assign and enforce permissions through roles. Its functionalities ensure secure, scalable, and organized authorization handling while allowing dynamic updates to roles and their associated permissions.


[View Source](https://github.com/maxnorm/Compose/blob/2ba2081cb9214075b097344cc10dc74bec360da3/src/access/AccessControl/AccessControlFacet.sol)



<Callout type="info" title="Key Features">
- Enables dynamic role-based access control with minimal storage overhead.
- Supports batch operations for granting and revoking roles in a single transaction.
- Includes robust authorization checks for admin operations.
</Callout>

## Usage Example

```solidity
pragma solidity ^0.8.0;

interface IDiamond {
    function facetAddress(bytes4 _selector) external view returns (address);
}

interface IAccessControlFacet {
    function grantRole(bytes32 _role, address _account) external;
    function hasRole(bytes32 _role, address _account) external view returns (bool);
    function revokeRole(bytes32 _role, address _account) external;
}

contract Example {
    IDiamond diamond;

    bytes32 constant ADMIN_ROLE = keccak256(&quot;ADMIN_ROLE&quot;);

    constructor(address _diamond) {
        diamond = IDiamond(_diamond);
    }

    function manageAccess(address _user) external {
        address accessFacet = diamond.facetAddress(IAccessControlFacet.grantRole.selector);
        require(accessFacet != address(0), &quot;AccessControlFacet not available&quot;);
        IAccessControlFacet(accessFacet).grantRole(ADMIN_ROLE, _user);
    }

    function checkAccess(address _user) external view returns (bool) {
        address accessFacet = diamond.facetAddress(IAccessControlFacet.hasRole.selector);
        require(accessFacet != address(0), &quot;AccessControlFacet not available&quot;);
        return IAccessControlFacet(accessFacet).hasRole(ADMIN_ROLE, _user);
    }
}
```



## Best Practices

<Callout type="tip" title="Best Practice">
- Assign admin roles to trusted accounts only and regularly audit their permissions.
- Use batch grant/revoke functions where possible to save gas and ensure operational consistency.
- Avoid hardcoding role values; use keccak256() for defining roles to reduce potential collisions.
</Callout>



## Security Considerations

<Callout type="warning" title="Security">
Perform stringent access control audits to ensure only trusted accounts manage critical roles. Avoid granting unnecessary permissions to mitigate the risk of abuse. Verify that the caller has the appropriate admin role before performing sensitive operations such as granting or revoking roles. Utilize the requireRole function to enforce access restrictions explicitly within dependent functionalities.
</Callout>
