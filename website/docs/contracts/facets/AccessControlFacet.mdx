## Overview

The AccessControlFacet provides role-based access control functionality for diamond contracts, allowing fine-grained permission management of accounts. It includes scalable methods for granting, revoking, and managing roles, making it essential for modular and secure contract design.


[View Source](https://github.com/maxnorm/Compose/blob/eabc77b81e461d751a5a8e003c3a9c2e818b9d3c/src/access/AccessControl/AccessControlFacet.sol)



<Callout type="info" title="Key Features">
- Scalable role-based access control\n- Batch operations for efficient role management\n- Customizable admin hierarchy for roles
</Callout>

## Usage Example

```solidity
pragma solidity ^0.8.0;\n\ninterface IAccessControlFacet {\n    function hasRole(bytes32 _role, address _account) external view returns (bool);\n    function grantRole(bytes32 _role, address _account) external;\n    function revokeRole(bytes32 _role, address _account) external;\n}\n\ncontract UseAccessControlFacet {\n    address public diamondAddress;\n    bytes32 public constant ADMIN_ROLE = keccak256(&quot;ADMIN_ROLE&quot;);\n\n    constructor(address _diamondAddress) {\n        diamondAddress = _diamondAddress;\n    }\n\n    function grantAdminRole(address _account) external {\n        IAccessControlFacet(diamondAddress).grantRole(ADMIN_ROLE, _account);\n    }\n\n    function checkRole(address _account) external view returns (bool) {\n        return IAccessControlFacet(diamondAddress).hasRole(ADMIN_ROLE, _account);\n    }\n}
```



## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure role assignments and admin role changes go through proper authorization checks.\n- Use batch functions (grantRoleBatch/revokeRoleBatch) to optimize gas costs when managing multiple accounts.\n- Always verify emitted events for auditability and detection of role adjustments.
</Callout>




## Security Considerations

<Callout type="warning" title="Security">
Implement strict access controls to protect the granter/revoker functions from unauthorized users.\nValidate addresses and roles when using batch functions to avoid granting/revoking unintended roles.\nRegularly audit and monitor emitted RoleGranted and RoleRevoked events to detect unauthorized changes.
</Callout>
