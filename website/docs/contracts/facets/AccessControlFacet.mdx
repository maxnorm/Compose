## Overview

The AccessControlFacet provides a robust, role-based access control mechanism within the Compose diamond framework. It allows administrators to manage permissions dynamically by assigning or revoking roles to addresses, making it an essential tool for maintaining modular and secure diamond contracts.


[View Source](https://github.com/maxnorm/Compose/blob/120348af74dbcdccde5d091d5c92f84ffed4753d/src/access/AccessControl/AccessControlFacet.sol)



<Callout type="info" title="Key Features">
- Dynamic role assignment and revocation.\n- Batch operations for efficient role management.\n- Strict role verification with custom errors for unauthorized access.
</Callout>

## Usage Example

```solidity
pragma solidity ^0.8.0;\n\nimport {Diamond} from &#39;diamond-foundation/Diamond.sol&#39;;\n\ncontract ExampleDiamond {\n    Diamond private diamond;\n\n    constructor() {\n        diamond = new Diamond();\n        diamond.addFacet(address(new AccessControlFacet()));\n    }\n\n    function assignAdminRole(address _admin) external {\n        AccessControlFacet(address(diamond)).grantRole(keccak256(&#39;ADMIN_ROLE&#39;), _admin);\n    }\n\n    function performAdminTask() external {\n        AccessControlFacet(address(diamond)).requireRole(keccak256(&#39;ADMIN_ROLE&#39;), msg.sender);\n        // Perform admin-specific logic here\n    }\n}
```



## Best Practices

<Callout type="tip" title="Best Practice">
- Use keccak256 hashing for role identifiers to prevent collisions.\n- Limit the usage of the setRoleAdmin function to trusted parties to avoid privilege escalation.\n- Regularly revoke roles from inactive or unnecessary accounts to minimize attack vectors.
</Callout>




## Security Considerations

<Callout type="warning" title="Security">
- Ensure that only authorized accounts can call grantRole, revokeRole, and setRoleAdmin to maintain permission integrity.\n- Prevent reentrancy attacks by avoiding external calls within role-altering functions to maintain atomicity.\n- Use requireRole to enforce proper access control before executing sensitive business logic.
</Callout>
