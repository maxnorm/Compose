## Overview

The ERC6909Facet facilitates token operations using the ERC-6909 standard within a diamond contract. It provides functions for managing balances, allowances, operators, and token transfers while leveraging the modularity and efficiency of the diamond proxy pattern.


[View Source](https://github.com/maxnorm/Compose/blob/7050baa250694d5e2445aa4c6b96c62c326cb1cb/src/token/ERC6909/ERC6909/ERC6909Facet.sol)



<Callout type="info" title="Key Features">
- Provides standard functions for ERC-6909 balance and allowance management.\n- Supports direct token transfers and operator-based approvals.
</Callout>

## Usage Example

```solidity
pragma solidity ^0.8.0; \n\nimport { Diamond } from &#39;Compose/Diamond.sol&#39;; \nimport { ERC6909Facet } from &#39;./ERC6909Facet.sol&#39;; \n\ncontract MyDiamond { \n    Diamond private diamond; \n    \n    constructor() { \n        diamond = new Diamond(); \n        diamond.addFacet(address(new ERC6909Facet())); \n    } \n\n    function performTransfer(address _to, uint256 _id, uint256 _amount) external { \n        // Call the ERC6909Facet&#39;s transfer function from the diamond\n        diamond.call(abi.encodeWithSignature(&quot;transfer(address,uint256,uint256)&quot;, _to, _id, _amount)); \n    } \n}
```



## Best Practices

<Callout type="tip" title="Best Practice">
- Always validate input parameters (e.g., ensure sufficient balances for transfers).\n- Ensure `approve` and `setOperator` actions are integrated with the intended authorization logic.\n- Avoid over-reliance on a single facet; combine functionalities wisely for enhanced modularity.
</Callout>




## Security Considerations

<Callout type="warning" title="Security">
Ensure proper access controls for `setOperator` and `approve` functions to avoid unauthorized operator assignments. Prevent reentrancy in `transfer` and `transferFrom` by following best practices such as using checks-effects-interactions pattern.
</Callout>
