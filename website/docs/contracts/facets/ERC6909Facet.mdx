## Overview

The ERC6909Facet adds token management capabilities to a Compose diamond. It implements core token operations like balance tracking, transfer, approval, and operator management, following the ERC-6909 custom standard. This facet enables flexible token logic within a diamond’s modular architecture.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

import { IDiamondCut } from "@compose/diamond/interfaces/IDiamondCut.sol";
import { ERC6909Facet } from "./facets/ERC6909Facet.sol";

contract DeployDiamond {
    function deployAndAddFacet(address diamondAddress) external {
        IDiamondCut.FacetCut[] memory facetCut = new IDiamondCut.FacetCut[](1);

        // Deploy ERC6909Facet
        ERC6909Facet erc6909Facet = new ERC6909Facet();

        // Add ERC6909Facet functions to the diamond
        facetCut[0] = IDiamondCut.FacetCut({
            facetAddress: address(erc6909Facet),
            action: IDiamondCut.FacetCutAction.Add,
            functionSelectors: abi.encodePacked(
                ERC6909Facet.getStorage.selector,
                ERC6909Facet.balanceOf.selector,
                ERC6909Facet.allowance.selector,
                ERC6909Facet.isOperator.selector,
                ERC6909Facet.transfer.selector,
                ERC6909Facet.transferFrom.selector,
                ERC6909Facet.approve.selector,
                ERC6909Facet.setOperator.selector
            )
        });

        // Execute diamond cut to add the facet
        IDiamondCut(diamondAddress).diamondCut(facetCut, address(0), "");

        // Example usage
        ERC6909Facet(diamondAddress).approve(msg.sender, 0x1234, 100);
        ERC6909Facet(diamondAddress).transferFrom(msg.sender, 0x4567, 0x1234, 50);
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure STORAGE_POSITION is unique across facets to prevent storage collision.
- Properly initialize operators and allowances before transferring tokens.
- When upgrading, verify new facets don’t alter token balances or allow unauthorized use.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
Validate all function inputs to prevent storage corruption or out-of-bounds errors. Enforce access checks for sensitive operations like setOperator. Prevent reentrancy in transfer and approve interactions by following the checks-effects-interactions pattern.
</Callout>
