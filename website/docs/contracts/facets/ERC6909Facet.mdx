<Callout type="info" title="Key Features">
- Implements ERC-6909 token standards at facet-level granularity for diamonds.
- Efficiently manages storage and modifications using inline assembly.
- Includes support for both operator-based approvals and traditional allowances.
</Callout>

## Overview

The ERC6909Facet provides essential ERC-6909 functionality for token management within a Compose diamond. It includes methods for handling token balances, allowances, operator settings, and token transfers, enabling efficient and modular ERC-6909 integration within a diamond proxy.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

import {ERC6909Facet} from './ERC6909Facet.sol';

contract SampleDiamond {
    function example() external {
        // Example usage of ERC6909Facet functions
        ERC6909Facet facet = ERC6909Facet(address(this));

        address owner = 0x123...;
        address receiver = 0x456...;
        uint256 id = 1;
        uint256 amount = 100;

        // Check balance
        uint256 balance = facet.balanceOf(owner, id);

        // Approve spender
        address spender = 0x789...;
        facet.approve(spender, id, amount);

        // Transfer tokens
        facet.transfer(receiver, id, amount);

        // Verify operator status
        bool isOp = facet.isOperator(owner, spender);

        // Set a spender as operator
        facet.setOperator(spender, true);
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Always initialize ERC-6909 storage properly when deploying the diamond.
- Enforce rigorous access control on &#x60;transferFrom&#x60; and &#x60;setOperator&#x60; to prevent unauthorized token transfers.
- Use the diamond implementation guidelines to handle state slot configuration consistently and avoid collisions.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
Ensure proper input validation for token IDs, amounts, and addresses to prevent invalid operations. Avoid reentrancy vulnerabilities in &#x60;transfer&#x60; and &#x60;transferFrom&#x60;; consider utilizing &#x60;checks-effects-interactions&#x60; or &#x60;ReentrancyGuard&#x60;. Manage operator settings cautiously to avoid unintended permissions.
</Callout>
