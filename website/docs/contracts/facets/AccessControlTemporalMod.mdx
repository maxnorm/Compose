## Overview

The AccessControlTemporalMod facet extends the diamond's access control capabilities by allowing role assignments with optional expiration timestamps. This ensures temporary permissions can be granted and automatically expire, enhancing security and reducing the risk of forgotten access rights in diamond contracts.


[View Source](https://github.com/maxnorm/Compose/blob/7050baa250694d5e2445aa4c6b96c62c326cb1cb/src/access/AccessControlTemporal/AccessControlTemporalMod.sol)



<Callout type="info" title="Key Features">
- Supports time-bound role assignments with expiration timestamps.\n- Provides explicit methods to verify and revoke expired roles.
</Callout>

## Usage Example

```solidity
contract Diamond {\n    // Assuming &#39;AccessControlTemporalMod&#39; facet is included in the diamond\n\n    function assignTemporaryRole(address user, bytes32 role, uint256 expiryTimestamp) external {\n        // Grant a role with an expiry timestamp\n        AccessControlTemporalMod(address(this)).grantRoleWithExpiry(role, user, expiryTimestamp);\n    }\n\n    function checkRoleStatus(address user, bytes32 role) external view returns (bool) {\n        // Validate if the user has an unexpired role\n        return !AccessControlTemporalMod(address(this)).isRoleExpired(role, user);\n    }\n\n    function revokeTemporaryRole(address user, bytes32 role) external {\n        // Revoke a role regardless of expiry\n        AccessControlTemporalMod(address(this)).revokeTemporalRole(role, user);\n    }\n}
```



## Best Practices

<Callout type="tip" title="Best Practice">
- Use meaningful timestamps for role expirations to avoid unnecessary privilege extension.\n- Implement access control checks (e.g., `msg.sender` authorization) when calling `grantRoleWithExpiry` or `revokeTemporalRole`.\n- Regularly audit and monitor temporary role assignments to ensure security requirements are met.
</Callout>




## Security Considerations

<Callout type="warning" title="Security">
Ensure only authorized entities can grant or revoke roles by implementing robust access checks. Always validate input timestamps to prevent bypass or extended invalid access. Prevent or mitigate reentrancy issues if granting or revoking roles interacts with external contracts.
</Callout>
