Contract documentation for AccessControlPausableFacet
</DocSubtitle>

<Callout type="info" title="Key Features">
- Enables pausing and unpausing of roles for temporary access control.
- Provides checks to ensure paused roles cannot be used until explicitly unpaused.
</Callout>

## Overview

The AccessControlPausableFacet provides role-based access control with the ability to pause specific roles. This facet enables fine-grained permission management and temporary disablement of roles, enhancing operational control within a diamond proxy.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

import { IDiamondCut } from "./IDiamondCut.sol";
import { AccessControlPausableFacet } from "./AccessControlPausableFacet.sol";

contract Example {
    function setupAccessControl(address diamond) external {
        // Assuming AccessControlPausableFacet is already added to the diamond
        AccessControlPausableFacet facet = AccessControlPausableFacet(diamond);

        // Check if a role is paused
        bool isPaused = facet.isRolePaused(keccak256("ROLE_ADMIN"));

        // Pause a role
        facet.pauseRole(keccak256("ROLE_ADMIN"));

        // Unpause a role
        facet.unpauseRole(keccak256("ROLE_ADMIN"));

        // Ensure a role is not paused before execution
        facet.requireRoleNotPaused(msg.sender, keccak256("ROLE_ADMIN"));
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Always verify that only authorized accounts (role admins) invoke &#x60;pauseRole&#x60; and &#x60;unpauseRole&#x60; to prevent mismanagement.
- Use &#x60;requireRoleNotPaused&#x60; wherever role-based authorization is critical to prevent inadvertent use of paused roles.
- Ensure all role and storage keys are deterministic and consistent across facets when accessing shared state.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
Ensure role admin accounts are securely managed to prevent unauthorized use of &#x60;pauseRole&#x60; and &#x60;unpauseRole&#x60;. Validate role identifiers to prevent unintended behavior. Use &#x60;requireRoleNotPaused&#x60; to avoid executing operations with paused roles. Verify that pausing and unpausing do not introduce reentrancy vulnerabilities.
</Callout>
