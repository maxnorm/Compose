## Overview

The AccessControlPausableFacet adds fine-grained control to role-based functionalities by allowing roles to be temporarily paused and unpaused by their respective administrators. This ensures greater flexibility and safety when managing role-permissions in diamond contracts, enabling dynamic adjustments during runtime.


[View Source](https://github.com/maxnorm/Compose/blob/7050baa250694d5e2445aa4c6b96c62c326cb1cb/src/access/AccessControlPausable/AccessControlPausableFacet.sol)



<Callout type="info" title="Key Features">
- Allows pausing of roles to temporarily disable functionality.\n- Prevents misuse by restricting pause/unpause actions to role administrators.\n- Emits clear events for role state changes for improved observability.
</Callout>

### pauseRole


Temporarily disables a role, preventing all accounts from using it. Only the admin of the role can pause it. Emits a RolePaused event. error: AccessControlUnauthorizedAccount If the caller is not the admin of the role.

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure role-specific admins carefully manage the pause and unpause functionality to prevent disruptions.\n- Always check if a role is paused using `isRolePaused` before using it in business logic.\n- Document roles and their pause/unpause criteria clearly to support maintenance and audits.
</Callout>




## Security Considerations

<Callout type="warning" title="Security">
Ensure only authorized admin accounts can pause or unpause roles. This prevents malicious or accidental disruption of functionality. Use `requireRoleNotPaused` in critical operations to enforce role state checks and prevent inconsistencies. Protect against reentrancy attacks by ensuring no external calls are made after performing state updates.
</Callout>
