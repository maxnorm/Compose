## Overview

The AccessControlPausableFacet provides a mechanism to manage access control roles with an additional feature to pause and unpause individual roles. This is particularly useful in diamond contracts to temporarily disable specific roles during upgrades or emergencies while maintaining fine-grained access control.


[View Source](https://github.com/maxnorm/Compose/blob/38545161423dd99e31b48b0eff41d260712fe731/src/access/AccessControlPausable/AccessControlPausableFacet.sol)



<Callout type="info" title="Key Features">
- Temporarily disable roles using the pauseRole function.
- Re-enable paused roles using the unpauseRole function.
- Efficiently check whether a role is paused.
</Callout>

    }

        AccessControlPausableFacet(address(this)).requireRoleNotPaused(msg.sender, role);

## Best Practices

<Callout type="tip" title="Best Practice">
- Only allow trusted admins to manage role pausing/unpausing.
- Use explicit checks (e.g., requireRoleNotPaused) in critical functions to enforce paused state restrictions.
- Document the role hierarchy thoroughly to avoid misuse or misconfiguration of roles.
</Callout>



## Security Considerations

<Callout type="warning" title="Security">
Ensure only authorized administrators can invoke pauseRole and unpauseRole to prevent unauthorized changes to access control. Include requireRoleNotPaused checks in all critical functions to enforce role-based restrictions during paused states. Regularly review and audit role hierarchies for potential privilege escalation or abuse.
</Callout>
