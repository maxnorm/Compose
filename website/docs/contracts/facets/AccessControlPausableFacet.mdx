## Overview

The AccessControlPausableFacet provides mechanisms to manage and temporarily disable (pause) specific roles in a diamond contract. This allows developers to enforce granular access control while having the flexibility to pause roles during maintenance or in response to critical situations, improving contract security and operability.


[View Source](https://github.com/maxnorm/Compose/blob/2ba2081cb9214075b097344cc10dc74bec360da3/src/access/AccessControlPausable/AccessControlPausableFacet.sol)



<Callout type="info" title="Key Features">
- Temporarily disable roles using `pauseRole`.
- Admin-restricted role management for controlled access.
- Ensures role-based operations are only performed when roles are active.
</Callout>

Temporarily disables a role, preventing all accounts from using it. Only the admin of the role can pause it. Emits a RolePaused event. error: AccessControlUnauthorizedAccount If the caller is not the admin of the role.

## Usage Example

```solidity
contract MyDiamond {
    function examplePauseRole(address diamond, bytes32 role) external {
        // Pause a specific role using the facet&#39;s function
        AccessControlPausableFacet(diamond).pauseRole(role);
    }

    function exampleCheckRoleNotPaused(address diamond, address account, bytes32 role) external view {
```

## Best Practices

<Callout type="tip" title="Best Practice">
- Assign role admins cautiously; only trusted accounts should have the ability to pause or unpause roles.
- Use `requireRoleNotPaused` before performing critical functions to ensure the role is active.
- Regularly audit role usage and states to prevent inactivity of vital roles.
</Callout>



## Security Considerations

<Callout type="warning" title="Security">
Only trusted admins should have the ability to pause or unpause roles, as improper use can disrupt functionalities. Ensure role pausing checks are implemented to prevent unauthorized access to critical functions. Beware of potential reentrancy issues when implementing roles-based checks within external calls.
</Callout>
