Contract documentation for AccessControlPausableFacet
</DocSubtitle>

<Callout type="info" title="Key Features">
- Role-specific pausing: Allows granular control over which functionalities are temporarily disabled.
- Admin-controlled pausing: Ensures that only authorized administrators can manage the paused state of roles.
- Emergency stop capability: Provides a mechanism to halt critical operations in case of unforeseen issues.
</Callout>

## Overview

This facet extends the AccessControl functionality by introducing role-based pausing. It allows specific roles within the diamond to be temporarily disabled, preventing any account from executing functions associated with that role. This provides a mechanism for controlled downtime or emergency stops for critical operations.

## Best Practices

<Callout type="tip" title="Best Practice">
- Initialize roles and their respective admin roles before attempting to pause or unpause them.
- Grant the &#x27;ROLE_ADMIN&#x27; role judiciously, as this role has the authority to pause and unpause other roles.
- Integrate &#x60;requireRoleNotPaused&#x60; checks within your facet functions to enforce the paused state of roles.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
Access to &#x60;pauseRole&#x60; and &#x60;unpauseRole&#x60; is restricted to the admin of the specific role being managed, enforced by the &#x60;AccessControlUnauthorizedAccount&#x60; error. The &#x60;requireRoleNotPaused&#x60; function prevents execution if a role is paused, raising an &#x60;AccessControlRolePaused&#x60; error. Ensure that the role administration logic is correctly implemented and secured.
</Callout>
