## Overview

The AccessControlPausableFacet provides functionality to manage role-based access control with added support for pausing specific roles. This is crucial for implementing advanced permission hierarchies or temporarily disabling certain operations in a diamond contract without affecting others.


[View Source](https://github.com/maxnorm/Compose/blob/e3e377cf3d77e94dc30e10812da2bdd1907ca159/src/access/AccessControlPausable/AccessControlPausableFacet.sol)



<Callout type="info" title="Key Features">
- Role-based access management
- Capability to pause roles temporarily
- Admin-restricted pausing and unpausing operations
</Callout>

### pauseRole


Temporarily disables a role, preventing all accounts from using it. Only the admin of the role can pause it. Emits a RolePaused event. error: AccessControlUnauthorizedAccount If the caller is not the admin of the role.

## Usage Example

```solidity
pragma solidity ^0.8.0;

import {Diamond} from &quot;./Diamond.sol&quot;;

contract MyContract {
    Diamond diamond;

    constructor(address _diamond) {
        diamond = Diamond(_diamond);
    }

    function pauseUserRole(bytes32 role) external {
        diamond.pauseRole(role);
    }

    function unpauseUserRole(bytes32 role) external {
        diamond.unpauseRole(role);
    }

    function isRoleCurrentlyPaused(bytes32 role) external view returns (bool) {
        return diamond.isRolePaused(role);
    }
}
```



## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure that only authorized accounts, typically admins, can call pause and unpause functions by verifying role-based access in the diamond proxy.
- Use `requireRoleNotPaused` in operations that must not proceed when a role is paused to enforce security restrictions.
- Clearly document roles and their administrative hierarchy for clarity and audit purposes.
</Callout>



## Security Considerations

<Callout type="warning" title="Security">
- Ensure robust access control to prevent unauthorized accounts from pausing or unpausing roles.
- Guard against potential reentrancy issues in pause or unpause logic by minimizing external calls within these functions.
- Use `requireRoleNotPaused` consistently to ensure reliability in critical access logic.
</Callout>
