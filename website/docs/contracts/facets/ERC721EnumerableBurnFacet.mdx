<Callout type="info" title="Key Features">
- Provides a &#x60;burn&#x60; function that removes tokens both from existence and enumeration tracking, fully aligning with ERC-721 Enumerable standards.
- Offers &#x60;getStorage&#x60; for exposing and managing dedicated facet storage.
</Callout>

## Overview

ERC721EnumerableBurnFacet enables token burning within a diamond and ensures proper removal from enumeration tracking, complying with the ERC-721 Enumerable standard. By separating this functionality into a distinct facet, it aligns with Compose&#x27;s modular, composable diamond architecture.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

import {LibDiamond} from "./libraries/LibDiamond.sol";

contract Diamond {
    fallback() external payable {
        address facet = LibDiamond.findFacet(msg.sig);
        require(facet != address(0), "Function does not exist.");
        assembly {
            calldatacopy(0, 0, calldatasize())
            let result := delegatecall(gas(), facet, 0, calldatasize(), 0, 0)
            returndatacopy(0, 0, returndatasize())
            if eq(result, 0) { revert(0, returndatasize()) }
            return(0, returndatasize())
        }
    }
}

// Example usage of ERC721EnumerableBurnFacet
contract ExampleConsumer {
    Diamond diamond;
    constructor(address _diamond) {
        diamond = Diamond(_diamond);
    }
    function executeBurn(uint256 _tokenId) external {
        (bool success, ) = address(diamond).call(abi.encodeWithSignature("burn(uint256)", _tokenId));
        require(success, "Token burn failed");
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Initialize the diamond storage correctly and gate the burn function if proper permissions are required.
- Ensure that &#x60;getStorage&#x60; is consistent with other facets to prevent unintended collisions or overwrites.
- Carefully consider selector collision risks when integrating this facet with other ERC-721 operations.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
Ensure the &#x60;burn&#x60; function checks for ownership of the token being burned, including permissioning if required for burning operations. Validate token ID inputs to prevent unintended behavior. If chaining other facet calls using a diamond, watch for potential reentrancy vulnerabilities.
</Callout>
