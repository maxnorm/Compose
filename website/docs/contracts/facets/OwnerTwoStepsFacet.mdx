- Designed for compatibility with Compose diamond storage patterns, allowing seamless integration.

## Overview

The OwnerTwoStepsFacet manages ownership transitions within a Compose diamond. It provides functionality to safely transfer ownership using a two-step process, ensuring both current and pending owners explicitly acknowledge the change. This aids in maintaining control and accountability within the diamondâ€™s governance structure.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

import {IDiamondCut} from "diamond/interfaces/IDiamondCut.sol";

contract Diamond {
    address public ownerFacetAddress;

    constructor(address _ownerFacetAddress) {
        ownerFacetAddress = _ownerFacetAddress;
        (bool success,) = _ownerFacetAddress.delegatecall(
            abi.encodeWithSignature("transferOwnership(address)", msg.sender)
        );
        require(success, "Failed to set initial ownership");
    }

    function transferOwnership(address newOwner) external {
        (bool success,) = ownerFacetAddress.delegatecall(
            abi.encodeWithSignature("transferOwnership(address)", newOwner)
        );
        require(success, "Ownership transfer call failed");
    }

    function acceptOwnership() external {
        (bool success,) = ownerFacetAddress.delegatecall(
            abi.encodeWithSignature("acceptOwnership()")
        );
        require(success, "Ownership acceptance call failed");
    }

    function renounceOwnership() external {
        (bool success,) = ownerFacetAddress.delegatecall(
            abi.encodeWithSignature("renounceOwnership()")
        );
        require(success, "Renounce ownership call failed");
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure correct initialization of the facet during diamond deployment, especially setting the initial owner.
- Use &#x60;transferOwnership&#x60; followed by &#x60;acceptOwnership&#x60; instead of directly modifying ownership in storage to preserve two-step confirmation.
- Avoid unnecessary updates to OWNER_STORAGE_POSITION or PENDING_OWNER_STORAGE_POSITION slots; maintain structured access.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
Ownership transfer requires two explicit steps, mitigating risks of accidental or unauthorized takeovers. Ensure &#x60;transferOwnership&#x60; and &#x60;acceptOwnership&#x60; are properly permissioned to restrict calls to the intended users. Validate pending owner addresses for correctness to prevent assignment to non-receivable accounts. Avoid reentrancy vulnerabilities by ensuring no external calls within the storage modification sequence.
</Callout>
