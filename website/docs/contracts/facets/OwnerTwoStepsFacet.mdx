## Overview

The OwnerTwoStepsFacet provides secure, two-step ownership transfer functionality within a diamond contract architecture. It ensures that ownership changes are intentional and verified, reducing the risk of accidental or malicious misconfiguration. This facet is essential for managing administrative control in a transparent and secure manner.


[View Source](https://github.com/maxnorm/Compose/blob/7050baa250694d5e2445aa4c6b96c62c326cb1cb/src/access/OwnerTwoSteps/OwnerTwoStepsFacet.sol)



<Callout type="info" title="Key Features">
- Implements two-step ownership transfer with explicit acceptance.\n- Includes functionality to view current and pending owners.\n- Allows renouncing ownership for decentralized control.
</Callout>

## Usage Example

```solidity
pragma solidity ^0.8.0;

import { Diamond } from &quot;./Diamond.sol&quot;;
import { OwnerTwoStepsFacet } from &quot;./OwnerTwoStepsFacet.sol&quot;;

contract Example {
    Diamond public diamond;

    constructor(address _diamondAddress) {
        diamond = Diamond(_diamondAddress);
    }

    function setNewOwner(address _newOwner) external {
        // Call transferOwnership to initiate two-step ownership transfer
        OwnerTwoStepsFacet(diamond).transferOwnership(_newOwner);
    }

    function acceptNewOwnership() external {
        // New owner calls acceptOwnership to complete ownership transfer
        OwnerTwoStepsFacet(diamond).acceptOwnership();
    }

    function getCurrentOwner() external view returns (address) {
        // Retrieve the current owner of the contract
        return OwnerTwoStepsFacet(diamond).owner();
    }
}
```



## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure the new owner explicitly accepts ownership to complete the transfer.\n- Call renounceOwnership only when administrative control is no longer needed or appropriate.\n- Use separate addresses for ownership and operational roles to minimize risks.
</Callout>




## Security Considerations

<Callout type="warning" title="Security">
- Verify the authenticity of the new owner's address before initiating a transfer.\n- Double-check that only authorized parties can call transferOwnership, acceptOwnership, or renounceOwnership functions.\n- Address any reentrancy risks by structuring the contract to follow checks-effects-interactions where applicable.
</Callout>
