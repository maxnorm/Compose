<Callout type="info" title="Key Features">
- Supports two-step ownership transfers for secure transitions.
- Provides helper functions to fetch owner and pending owner information.
- Direct storage access via defined slots with inline assembly for optimized gas usage.
</Callout>

## Overview

The OwnerTwoStepsFacet provides a secure mechanism for managing ownership transitions in a diamond proxy. It includes functions for retrieving current and pending owner details, initiating ownership transfers, and finalizing them, ensuring controlled and deliberate changes to contract ownership.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

import {IDiamondCut} from "./interfaces/IDiamondCut.sol";

contract DeployDiamond {
    address public diamond;

    function deploy() external {
        // Deploy the Diamond
        diamond = address(new Diamond(address(this)));

        // Configure facets
        address ownerTwoStepsFacet = address(new OwnerTwoStepsFacet());
        IDiamondCut(diamond).facetCut(
            ownerTwoStepsFacet,
            [\"getOwnerStorage()\", \"getPendingOwnerStorage()\", \"owner()\", \"pendingOwner()\", \"transferOwnership(address)\", \"acceptOwnership()\", \"renounceOwnership()\"]
        );

        // Initialize Ownership
        IOwnerTwoStepsFacet(diamond).transferOwnership(msg.sender);
    }

    function invokeExample() external {
        IOwnerTwoStepsFacet(diamond).transferOwnership(0x123);
        IOwnerTwoStepsFacet(diamond).acceptOwnership();
    }
}

interface IOwnerTwoStepsFacet {
    function getOwnerStorage() external view returns (address);
    function getPendingOwnerStorage() external view returns (address);
    function owner() external view returns (address);
    function pendingOwner() external view returns (address);
    function transferOwnership(address _newOwner) external;
    function acceptOwnership() external;
    function renounceOwnership() external;
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Use &#x60;transferOwnership&#x60; to delegate a new owner and provide them time to review before accepting.
- Always ensure the &#x60;PendingOwner&#x60; has consented before calling &#x60;acceptOwnership&#x60; to avoid accidental transition.
- On upgrades, confirm storage slots &#x60;OWNER_STORAGE_POSITION&#x60; and &#x60;PENDING_OWNER_STORAGE_POSITION&#x60; are secure and unchanged.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
Ensure only the current owner can execute &#x60;transferOwnership&#x60; and &#x60;renounceOwnership&#x60;. Use input validation to prevent unexpected results, such as setting the owner address to a zero address. As the facet involves inline assembly for accessing storage, audit the smart contract to confirm slot correctness and prevent unintended overwrites.
</Callout>
