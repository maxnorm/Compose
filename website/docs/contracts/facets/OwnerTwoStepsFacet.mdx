## Overview

The OwnerTwoStepsFacet provides two-step ownership transfer for diamond contracts, aligning with the ERC-173 standard. It enables improved security and transparency through pending owner confirmation before finalizing the ownership change.


[View Source](https://github.com/maxnorm/Compose/blob/2ba2081cb9214075b097344cc10dc74bec360da3/src/access/OwnerTwoSteps/OwnerTwoSteps.sol)



<Callout type="info" title="Key Features">
- Implements two-step ownership transfer for added security
- Fully conforms to ERC-173
- Provides functions for initiating and accepting ownership transfer
</Callout>

## Usage Example

```solidity
pragma solidity ^0.8.0;

import {DiamondCutFacet} from &quot;./DiamondCutFacet.sol&quot;;
import {OwnerTwoStepsFacet} from &quot;./OwnerTwoStepsFacet.sol&quot;;

contract ExampleDiamond {
    constructor(address _contractOwner) {
        DiamondCutFacet diamondCutFacet = new DiamondCutFacet();
        OwnerTwoStepsFacet ownerFacet = new OwnerTwoStepsFacet();

        // Example: Set initial owner using OwnerTwoStepsFacet
        IDiamondCut.FacetCut[] memory cut = new IDiamondCut.FacetCut[](2);
        cut[0] = IDiamondCut.FacetCut({facetAddress: address(diamondCutFacet), action: IDiamondCut.FacetCutAction.Add, functionSelectors: diamondCutFacet.getSelectors()});
        cut[1] = IDiamondCut.FacetCut({facetAddress: address(ownerFacet), action: IDiamondCut.FacetCutAction.Add, functionSelectors: ownerFacet.getSelectors()});

        diamondCutFacet.diamondCut(cut, address(0), &quot;&quot;);
        ownerFacet.transferOwnership(_contractOwner);
    }
}
```



## Best Practices

<Callout type="tip" title="Best Practice">
- Use two-step transfer to avoid accidental ownership changes or malicious takeovers.
- Always validate pending ownership actions before accepting them.
- Ensure `renounceOwnership` is only used when intentionally relinquishing control.
</Callout>



## Security Considerations

<Callout type="warning" title="Security">
Ensure the pending owner address is verified before accepting ownership. Reentrancy protections are not required as state changes are minimal, but conditions should be explicitly verified to avoid misuse.
</Callout>
