## Overview

OwnerTwoStepsFacet enables secure and controlled ownership transfers in diamond proxy contracts by implementing a two-step process ensuring deliberate change in ownership. This facet simplifies verification of ownership status and transition logic within ERC-173 compliant systems.


[View Source](https://github.com/maxnorm/Compose/blob/7206f045d711880c211b967e80d72745b52f5cfc/src/access/OwnerTwoSteps/OwnerTwoSteps.sol)



<Callout type="info" title="Key Features">
- Implements ERC-173 compliant ownership functions.
- Secure, deliberate two-step ownership transfer.
- Streamlined integration for diamond storage slots.
</Callout>

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure ownership changes are intentional by using the two-step acceptance process.
- Use `getStorage` to verify storage configuration and ownership details.
- Regularly check `pendingOwner` to track ownership transfer progress.
</Callout>



## Security Considerations

<Callout type="warning" title="Security">
Verify the new owner address before `transferOwnership`.
Always validate the caller of `acceptOwnership` to match the `pendingOwner`.
Avoid using `renounceOwnership` in cases requiring written owner permissions for critical operations.
</Callout>
