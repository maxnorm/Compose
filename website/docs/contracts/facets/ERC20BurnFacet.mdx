Contract documentation for ERC20BurnFacet
</DocSubtitle>

<Callout type="info" title="Key Features">
- Supports direct self-burn via &#x60;burn&#x60;.
- Allows delegated burns using allowances via &#x60;burnFrom&#x60;.
- Emits standard Transfer events to indicate supply changes.
</Callout>

## Overview

The ERC20BurnFacet provides token burn functionality for a diamond by implementing &#x60;burn&#x60; and &#x60;burnFrom&#x60;. It reduces token supply through balance destruction and allowance deduction while ensuring event emissions for transparency.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

import {DiamondCutFacet} from "./DiamondCutFacet.sol";
import {ERC20BurnFacet} from "./ERC20BurnFacet.sol";

contract DeployDiamond {
    function deploy() external {
        DiamondCutFacet diamondCut = new DiamondCutFacet();
        ERC20BurnFacet erc20BurnFacet = new ERC20BurnFacet();

        // Example diamond cut operation
        diamondCut.diamondCut(
            [
                { facetAddress: address(erc20BurnFacet), action: FacetCutAction.Add, functionSelectors: [
                    ERC20BurnFacet.burn.selector,
                    ERC20BurnFacet.burnFrom.selector,
                    ERC20BurnFacet.getStorage.selector
                ] }
            ],
            address(0),
            new bytes(0)
        );

        // Example burn call
        ERC20BurnFacet(address(diamond)).burn(100 * 10**18);

        // Example burnFrom call
        ERC20BurnFacet(address(diamond)).burnFrom(\"0x1234...abcd\", 50 * 10**18);
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Always correctly initialize the ERC20 storage before adding this facet to a diamond.
- Ensure selectors for the facet are properly aligned in diamond&#x27;s function mapping during deployment.
- Avoid partial upgrades that exclude critical functions like &#x60;getStorage&#x60;.

</Callout>

## Security Considerations

<Callout type="warning" title="Security">
Use proper access control to protect &#x60;burnFrom&#x60; functionality from misuse. Validate caller allowance and ensure correct deduction to mitigate unauthorized token destruction. Reentrancy is non-existent since no external calls are made.
</Callout>
