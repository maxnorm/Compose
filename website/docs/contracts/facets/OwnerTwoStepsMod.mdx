## Overview

The OwnerTwoStepsMod facet implements ERC-173 compliant two-step ownership transfer functionality, allowing for secure and deliberate ownership transitions in diamond proxy contracts. It ensures clarity and control over ownership changes, which are critical for the secure management of modular and composable systems.


[View Source](https://github.com/maxnorm/Compose/blob/7050baa250694d5e2445aa4c6b96c62c326cb1cb/src/access/OwnerTwoSteps/OwnerTwoStepsMod.sol)



<Callout type="info" title="Key Features">
- Implements two-step ownership transfer for secure transitions.\n- Supports ownership renouncement, enabling fully permissionless functionality if desired.
</Callout>

## Usage Example

```solidity
pragma solidity ^0.8.0;\n\ninterface IDiamondCut {\n    function diamondCut(bytes[] calldata _cut, address _init, bytes calldata _calldata) external;\n}\n\nimport &quot;./OwnerTwoStepsMod.sol&quot;;\n\ncontract DiamondExample {\n    address public diamondOwner;\n\n    constructor() {\n        diamondOwner = msg.sender;\n        // Initial facet setup\n        IDiamondCut(address(this)).diamondCut(new bytes[](0), address(this), abi.encodeWithSignature(&quot;transferOwnership(address)&quot;, msg.sender));\n    }\n\n    function initiateOwnershipTransfer(address _newOwner) external {\n        OwnerTwoStepsMod.transferOwnership(_newOwner);\n    }\n\n    function completeOwnershipTransfer() external {\n        OwnerTwoStepsMod.acceptOwnership();\n    }\n\n    function getCurrentOwner() external view returns (address) {\n        return OwnerTwoStepsMod.owner();\n    }\n}
```



## Best Practices

<Callout type="tip" title="Best Practice">
- Only call `transferOwnership` with a carefully intended new owner address.\n- Always complete the ownership transfer process promptly using `acceptOwnership`.\n- Periodically audit ownership-related functions to ensure no undesired transfers have occurred.
</Callout>




## Security Considerations

<Callout type="warning" title="Security">
Ensure access to restricted functions is safeguarded using `requireOwner`. Use the two-step process carefully to avoid accidental loss of ownership. Be vigilant while renouncing ownership, as it cannot be undone and disables owner-restricted functions permanently.
</Callout>
