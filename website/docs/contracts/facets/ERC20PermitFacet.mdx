## Overview

The ERC20PermitFacet allows ERC-20 token allowances to be set via cryptographic EIP-2612 signatures instead of on-chain transactions. This functionality facilitates gasless approvals and improves user experience in diamond proxy contracts while adhering to EIP standards.


[View Source](https://github.com/maxnorm/Compose/blob/38545161423dd99e31b48b0eff41d260712fe731/src/token/ERC20/ERC20/ERC20PermitFacet.sol)



<Callout type="info" title="Key Features">
- Implements EIP-2612 standard for gasless token approvals.
- Provides nonce tracking to prevent signature replay attacks.
</Callout>

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure DOMAIN_SEPARATOR is correctly calculated and unique per deployment to prevent replay attacks.
- Use secure handling for function parameters and signature verification logic.
- Regularly test the permit function with various scenarios such as edge cases for nonce expiration and signature correctness.
</Callout>



## Security Considerations

<Callout type="warning" title="Security">
Verify that DOMAIN_SEPARATOR remains consistent across transactions and chains, as inconsistency can compromise replay attack prevention. Validate signatures thoroughly inside the `permit` function to ensure they are legitimate and match the owner&#39;s intent. Test for reentrancy attacks in surrounding external calls when using `permit` in multi-step workflows.
</Callout>
