Contract documentation for ERC20PermitFacet
</DocSubtitle>

<Callout type="info" title="Key Features">
- Supports EIP-2612 permit standard for gasless approvals.
- Provides DOMAIN_SEPARATOR for signature integrity across chains.
- Tracks nonces per owner to prevent signature replay attacks.
</Callout>

## Overview

The ERC20PermitFacet provides EIP-2612 permit functionality, enabling gasless approvals via signed messages. It also exposes utility methods like nonces and DOMAIN_SEPARATOR for signature management within the diamond proxy structure.

        address diamondAddress = 0xYourDiamondAddress;

        // Example: Calling DOMAIN_SEPARATOR
        bytes32 domainSeparator = ERC20PermitFacet(diamondAddress).DOMAIN_SEPARATOR();

        // Example: Fetching a user's nonce
        uint256 userNonce = ERC20PermitFacet(diamondAddress).nonces(msg.sender);

        // Example: Using permit to set an allowance
        ERC20PermitFacet(diamondAddress).permit(
            msg.sender, // owner
            0xSpenderAddress, // spender
            1000, // value
            block.timestamp + 3600, // deadline
            v, r, s // signature components
        );
    }
}`}

## Best Practices

<Callout type="tip" title="Best Practice">
- Initialize DOMAIN_SEPARATOR properly during diamond deployment to bind it to the correct chain ID.
- Verify the permit functionality is integrated with the ERC20 facet for seamless allowance management.
- Regularly audit nonce handling to ensure proper signature replay protection.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
Ensure the DOMAIN_SEPARATOR is correctly initialized to prevent replay attacks across chains. Validate all signature inputs in the permit function to mitigate misuse. Prevent reentrancy issues by avoiding external calls before state updates in the permit implementation. Test nonce integrity to ensure unique nonce values across transactions.
</Callout>
