Contract documentation for ERC20PermitFacet
</DocSubtitle>

<Callout type="info" title="Key Features">
- Implements EIP-2612 permit functionality to enable gasless approvals.
- Includes methods for securely verifying nonces and preventing signature replay.
- Provides domain isolation based on chain ID to ensure signature validity.
</Callout>

## Overview

The ERC20PermitFacet provides EIP-2612 permit functionality within a diamond. It allows token holders to approve allowances via off-chain signatures instead of on-chain transactions, reducing gas costs and improving user experience.

## Best Practices

<Callout type="tip" title="Best Practice">
- Initialize the DOMAIN_SEPARATOR when deploying or upgrading the diamond to ensure signature security.
- Use the &#x60;nonces&#x60; function to manage and verify signature uniqueness.
- When adding this facet to a diamond, check for compatibility with other facets that modify allowance or transfer logic.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
Ensure proper validation of signature inputs, including &#x60;v&#x60;, &#x60;r&#x60;, &#x60;s&#x60; values. Secure &#x60;permit&#x60; against replay attacks by using the &#x60;nonces&#x60; function combined with the &#x60;DOMAIN_SEPARATOR&#x60;. Avoid reentrancy risks in downstream calls when integrating &#x27;transferFrom&#x27; logic within diamonds.
</Callout>
