## Overview

The ERC721Enumerable facet provides enumeration functionality for ERC-721 tokens, enabling optimized token management within the Compose diamond framework. It supports minting, burning, transferring tokens, and efficient retrieval of storage data following the EIP-2535 diamond storage pattern for modular reuse.


[View Source](https://github.com/maxnorm/Compose/blob/f773c1f2b6ca1fc289a7b9ea4f149315edd34199/src/token/ERC721/ERC721Enumerable/ERC721Enumerable.sol)



<Callout type="info" title="Key Features">
- Supports ERC-721 token enumeration as per EIP-721 standard.
- Modular design, compatible with diamond proxy contracts.
- Handles mint, burn, and transfer operations with automatic enumeration updates.
</Callout>

## Usage Example

```solidity
pragma solidity ^0.8.0;

contract Diamond {

    // Example usage of the ERC721Enumerable facet
    function mintToken(address _to, uint256 _tokenId) external {
        ERC721EnumerableFacet(address(this)).mint(_to, _tokenId);
    }

    function transferToken(address _from, address _to, uint256 _tokenId) external {
        ERC721EnumerableFacet(address(this)).transferFrom(_from, _to, _tokenId);
    }

    function burnToken(uint256 _tokenId) external {
        ERC721EnumerableFacet(address(this)).burn(_tokenId);
    }

    function getTokenStorage() external view returns (ERC721EnumerableStorageLib.StorageStruct memory) {
        return ERC721EnumerableFacet(address(this)).getStorage();
    }
}
```



## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure proper access controls for functions like minting and burning to prevent unauthorized actions.
- Utilize `getStorage` function for direct interaction with ERC-721 enumerable storage instead of duplicating enumeration logic.
- Verify all input parameters (e.g., non-zero addresses) before invoking minting, burning, or transfer functions.
</Callout>



## Security Considerations

<Callout type="warning" title="Security">
Ensure that mint and burn operations are restricted to authorized addresses to prevent unauthorized minting or destruction of tokens. Validate ownership and approvals during transfers to avoid unauthorized token transfers. As with any enumerable functionality, be cautious about the potential for overflow or unexpected states from reentrancy. Utilize appropriate safeguards wherever external calls are made, if applicable.
</Callout>
