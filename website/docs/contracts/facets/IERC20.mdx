## Overview

**Title:**

ERC-20 Token Standard Interface


[View Source](https://github.com/maxnorm/Compose/blob/486d859471e9877e9fab1d6c2ba1751795e49c90/src/interfaces/IERC20.sol)




## Functions


### name


Returns the name of the token.






### symbol


Returns the symbol of the token.






### decimals


Returns the number of decimals used for token precision.






### totalSupply


Returns the total supply of tokens.






### balanceOf


Returns the balance of a specific account.






### allowance


Returns the remaining number of tokens that a spender is allowed to spend on behalf of an owner.






### approve


Approves a spender to transfer up to a certain amount of tokens on behalf of the caller. Emits an Approval event.






### transfer


Transfers tokens to another address. Emits a Transfer event.






### transferFrom


Transfers tokens on behalf of another account, provided sufficient allowance exists. Emits a Transfer event and decreases the spender's allowance.






### burn


Burns (destroys) a specific amount of tokens from the caller's balance. Emits a Transfer event to the zero address.






### burnFrom


Burns tokens from another account, deducting from the caller's allowance. Emits a Transfer event to the zero address.






### nonces


Returns the current nonce for an owner. This value changes each time a permit is used.






### DOMAIN_SEPARATOR


Returns the domain separator used in the encoding of the signature for permit. This value is unique to a contract and chain ID combination to prevent replay attacks.






### permit


Sets the allowance for a spender via a signature. This function implements EIP-2612 permit functionality.








## Events


### Transfer


Emitted when tokens are transferred between two addresses.





### Approval


Emitted when an approval is made for a spender by an owner.







## Errors


### ERC20InsufficientBalance


Thrown when an account has insufficient balance for a transfer or burn.




### ERC20InvalidSender


Thrown when the sender address is invalid (e.g., zero address).




### ERC20InvalidReceiver


Thrown when the receiver address is invalid (e.g., zero address).




### ERC20InsufficientAllowance


Thrown when a spender tries to use more than the approved allowance.




### ERC20InvalidSpender


Thrown when the spender address is invalid (e.g., zero address).




### ERC2612InvalidSignature


Thrown when a permit signature is invalid or expired.
