## Overview

The Royalty facet implements the ERC-2981 royalty standard, enabling streamlined management of royalty information for NFTs in a diamond contract architecture. It supports both default and token-specific royalties, making it flexible enough for various royalty distribution scenarios while ensuring compliance with industry standards.


[View Source](https://github.com/maxnorm/Compose/blob/f773c1f2b6ca1fc289a7b9ea4f149315edd34199/src/token/Royalty/Royalty.sol)



<Callout type="info" title="Key Features">
- Compliant with ERC-2981 Royalty Standard
- Supports default and token-specific royalty configuration
- Allows efficient management and querying of royalty information
</Callout>

## Errors


### ERC2981InvalidDefaultRoyalty


Thrown when default royalty fee exceeds 100% (10000 basis points). **Parameters**



```solidity
error ERC2981InvalidDefaultRoyalty(uint256 _numerator, uint256 _denominator);
```



### ERC2981InvalidDefaultRoyaltyReceiver


Thrown when default royalty receiver is the zero address. **Parameters**



```solidity
error ERC2981InvalidDefaultRoyaltyReceiver(address _receiver);
```



### ERC2981InvalidTokenRoyalty


Thrown when token-specific royalty fee exceeds 100% (10000 basis points). **Parameters**



```solidity
error ERC2981InvalidTokenRoyalty(uint256 _tokenId, uint256 _numerator, uint256 _denominator);
```



### ERC2981InvalidTokenRoyaltyReceiver

## Usage Example

```solidity
pragma solidity ^0.8.20;

import {Diamond} from &quot;./Diamond.sol&quot;;

contract RoyaltyDemo {
    Diamond private diamond;

    constructor(address _diamondAddress) {
        diamond = Diamond(_diamondAddress);
    }

    function configureRoyalty() external {
        // Set default royalty to 5% (500 basis points)
        diamond.setDefaultRoyalty(address(this), 500);
        
        // Override royalty for token ID 1 to 10%
        diamond.setTokenRoyalty(1, address(this), 1000);
        
        // Query royalty information
        (address receiver, uint256 royaltyAmount) = diamond.royaltyInfo(1, 100 ether);

        // Clear override royalty so it defaults to the setDefaultRoyalty 
        diamond.resetTokenRoyalty(1);
    }
}
```



## Best Practices

<Callout type="tip" title="Best Practice">
- Always validate royalty receiver address and ensure it is operational to avoid disruptions in royalty payments.
- Set reasonable default and token-specific royalty fees to align with industry standards and prevent issues with excessive royalties.
- Reset token-specific royalties to prevent outdated or unwanted configurations when they're no longer needed.
</Callout>



## Security Considerations

<Callout type="warning" title="Security">
Ensure access control mechanisms prevent unauthorized or malicious updates to royalty configurations. Beware of reentrancy vulnerabilities in royalty payment flows when integrating royalty receiver logic with external contract calls.
</Callout>
