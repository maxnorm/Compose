<Callout type="info" title="Key Features">
- Implements ERC-2981 royalty functions for token sales.
- Supports fallback from token-specific royalties to a globally configured default.
</Callout>

## Overview

The RoyaltyFacet facilitates ERC-2981 compliant royalty calculations for tokens within a Compose diamond. It provides modularized storage access and royalty computation either on a token-specific or default basis.

### royaltyInfo

Returns royalty information for a given token and sale price. Returns token-specific royalty if set, otherwise falls back to default royalty. Royalty amount is calculated as a percentage of the sale price using basis points. Implements the ERC-2981 royaltyInfo function.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

interface IRoyaltyFacet {
    function getStorage() external pure returns (bytes32 slot);
    function royaltyInfo(uint256 _tokenId, uint256 _salePrice)
        external
        view
        returns (address receiver, uint256 royaltyAmount);
}

contract RoyaltyExample {
    IRoyaltyFacet private immutable royaltyFacet;

    constructor(address _diamond) {
        royaltyFacet = IRoyaltyFacet(_diamond);
    }

    function viewRoyalty(uint256 tokenId, uint256 salePrice) external view returns (address, uint256) {
        return royaltyFacet.royaltyInfo(tokenId, salePrice);
    }

    function storageSlot() external pure returns (bytes32) {
        return royaltyFacet.getStorage();
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Initialize the facet storage explicitly if adding new state fields.
- Handle royalty percentages and precision carefully to avoid unexpected values.
- Secure upgrades by maintaining consistent storage layout, especially for new royalty-related fields.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
Ensure royalty percentages and calculations do not exceed 100%. Validate all inputs to prevent unintended reentrancy risks. The royalty fallback mechanism may require auditing during upgrades to ensure consistent behavior.
</Callout>
