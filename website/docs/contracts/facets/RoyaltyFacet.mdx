## Overview

The RoyaltyFacet provides functionality to manage and query royalty information in conformance with the ERC-2981 standard within a diamond proxy contract. It allows for efficient retrieval of royalty data, supporting both token-specific and default royalty settings, and calculates royalty amounts based on sale price and basis points.


[View Source](https://github.com/maxnorm/Compose/blob/7206f045d711880c211b967e80d72745b52f5cfc/src/token/Royalty/RoyaltyFacet.sol)



<Callout type="info" title="Key Features">
- Supports ERC-2981-compliant royalties.
- Token-specific and default royalty fallback.
- Simple storage retrieval for extension or debugging.
</Callout>

### royaltyInfo


Returns royalty information for a given token and sale price. Returns token-specific royalty if set, otherwise falls back to default royalty. Royalty amount is calculated as a percentage of the sale price using basis points. Implements the ERC-2981 royaltyInfo function.



```solidity
function royaltyInfo(uint256 _tokenId, uint256 _salePrice)
    external
    view
    returns (address receiver, uint256 royaltyAmount);
```

## Usage Example

```solidity
pragma solidity ^0.8.0;

import {IDiamondLoupe} from &quot;../interfaces/IDiamondLoupe.sol&quot;;

contract UseRoyaltyFacet {
    address public diamond;

    constructor(address _diamond) {
        diamond = _diamond;
    }

    function getRoyaltyInfo(uint256 _tokenId, uint256 _salePrice) external view returns (address receiver, uint256 royaltyAmount) {
        (bool success, bytes memory data) = diamond.staticcall(abi.encodeWithSignature(&quot;royaltyInfo(uint256,uint256)&quot;, _tokenId, _salePrice));
        require(success, &quot;Royalty query failed&quot;);
        return abi.decode(data, (address, uint256));
    }
}
```



## Best Practices

<Callout type="tip" title="Best Practice">
- Always set default and token-specific royalties explicitly to avoid inconsistent outputs.
- Conform royalty calculations to the ERC-2981 standard to ensure compatibility.
- Use getStorage internally for debugging or extension purposes, but avoid exposing it unless necessary.
</Callout>



## Security Considerations

<Callout type="warning" title="Security">
Ensure royaltyInfo is protected against manipulation by unauthorized entities. Validate all external inputs to avoid miscalculations in royalty amounts. If integrating with marketplaces, confirm they respect ERC-2981 for royalty enforcement.
</Callout>
