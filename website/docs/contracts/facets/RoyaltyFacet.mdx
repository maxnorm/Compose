## Overview

The RoyaltyFacet enables ERC-2981 compliant royalty management within a diamond proxy. It supports querying royalty information both per-token and globally via fallback, providing a standardized interface for marketplace integrations.

### royaltyInfo

Returns royalty information for a given token and sale price. Returns token-specific royalty if set, otherwise falls back to default royalty. Royalty amount is calculated as a percentage of the sale price using basis points. Implements the ERC-2981 royaltyInfo function.

<ExpandableCode language="solidity" maxLines={8}>
{`function royaltyInfo(uint256 _tokenId, uint256 _salePrice)
    external
    view

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

/** Imports necessary for diamond initialization */
import { RoyaltyFacet } from './RoyaltyFacet.sol';

contract DiamondDeployer {
    function deployDiamond() external {
        // Initialize diamond with RoyaltyFacet
        Diamond diamond = new Diamond();

        // Add RoyaltyFacet to the diamond
        diamond.addFacet(address(new RoyaltyFacet()), \'getStorage()\', \'royaltyInfo(uint256,uint256)\');

        // Example usage of royaltyInfo via the diamond
        (address receiver, uint256 royaltyAmount) = RoyaltyFacet(address(diamond)).royaltyInfo(123, 10000);
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Always initialize default royalty values where applicable to ensure consistent behavior across tokens.
- Store royalty percentages in basis points to avoid floating-point inaccuracies.
- Thoroughly validate new royalty rules during diamond upgrades to ensure backward compatibility.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
Carefully validate input values in royaltyInfo to ensure basis points do not exceed 10,000 (100%). Protect getStorage with internal visibility to avoid misuse of storage pointers. Ensure that any external marketplace code invoking royaltyInfo appropriately handles the returned values.
</Callout>
