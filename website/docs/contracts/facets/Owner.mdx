## Overview

The Owner facet implements ERC-173 contract ownership management, allowing a contract to track and control an owner address. It provides essential functionality such as verifying ownership and transferring or renouncing ownership, which is critical in managing access control within diamond contracts.


[View Source](https://github.com/maxnorm/Compose/blob/f773c1f2b6ca1fc289a7b9ea4f149315edd34199/src/access/Owner/Owner.sol)



<Callout type="info" title="Key Features">
- Implements ERC-173 ownership standard
- Includes ownership assertion and transfer capabilities
- Prevents unauthorized access with enforced ownership checks
</Callout>

## Usage Example

```solidity
pragma solidity ^0.8.0;

import { Diamond } from &quot;./Diamond.sol&quot;;
import { Owner } from &quot;./facets/Owner.sol&quot;;

contract Example {
    Diamond private diamond;

    constructor(address diamondAddress) {
        diamond = Diamond(diamondAddress);
    }

    function transferDiamondOwnership(address _newOwner) external {
        bytes memory callData = abi.encodeWithSelector(Owner.transferOwnership.selector, _newOwner);
        (bool success, ) = address(diamond).call(callData);
        require(success, &quot;Ownership transfer failed&quot;);
    }

    function getCurrentOwner() external view returns (address) {
        bytes memory callData = abi.encodeWithSelector(Owner.owner.selector);
        (bool success, bytes memory result) = address(diamond).staticcall(callData);
        require(success, &quot;Failed to fetch owner&quot;);
        return abi.decode(result, (address));
    }
}
```



## Best Practices

<Callout type="tip" title="Best Practice">
- Use `requireOwner` to guard any function requiring exclusive owner access.
- Always check the return values when transferring or renouncing ownership.
- Document owner-related changes clearly in project governance or on-chain logs.
</Callout>



## Security Considerations

<Callout type="warning" title="Security">
Ensure `transferOwnership` is properly protected with `requireOwner` to prevent unauthorized access. Be cautious when using `address(0)` in `transferOwnership` as it renounces ownership permanently. Verify ownership changes are reflected correctly to avoid governance issues.
</Callout>
