## Overview

The ERC20Facet provides core ERC20 token functionality within the diamond proxy framework, enabling token management with minimal duplication and maximum reusability. It implements essential token operations such as transfers, approvals, and balance queries directly within diamond contracts.


[View Source](https://github.com/maxnorm/Compose/blob/eabc77b81e461d751a5a8e003c3a9c2e818b9d3c/src/token/ERC20/ERC20/ERC20Facet.sol)



<Callout type="info" title="Key Features">
- Full ERC20 token support\n- Seamless composability within diamonds\n- Internal balance and allowance management using diamond storage
</Callout>

## Errors


### ERC20InsufficientBalance


Thrown when an account has insufficient balance for a transfer or burn.



```solidity

```



### ERC20InvalidSender


Thrown when the sender address is invalid (e.g., zero address).



```solidity

```



### ERC20InvalidReceiver


Thrown when the receiver address is invalid (e.g., zero address).



```solidity

```



### ERC20InsufficientAllowance


Thrown when a spender tries to use more than the approved allowance.



```solidity
error ERC20InsufficientAllowance(address _spender, uint256 _allowance, uint256 _needed);
```



### ERC20InvalidSpender


Thrown when the spender address is invalid (e.g., zero address).




## Usage Example

```solidity
contract BasicDiamond {\n    // Include diamond core logic setup\n    Diamond diamond;\n\n    constructor() {\n        // Deploying ERC20Facet as part of the diamond\n        ERC20Facet facet = new ERC20Facet();\n        // Add the ERC20Facet to the diamond with its selectors\n        bytes4[] memory selectors = new bytes4[](8);\n        selectors[0] = ERC20Facet.name.selector;\n        selectors[1] = ERC20Facet.symbol.selector;\n        selectors[2] = ERC20Facet.decimals.selector;\n        selectors[3] = ERC20Facet.totalSupply.selector;\n        selectors[4] = ERC20Facet.balanceOf.selector;\n        selectors[5] = ERC20Facet.allowance.selector;\n        selectors[6] = ERC20Facet.approve.selector;\n        selectors[7] = ERC20Facet.transfer.selector;\n\n        diamond.addFacet(address(facet), selectors);\n    }\n}
```



## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure consistent storage slot usage when integrating with other facets.\n- Implement ERC20 event emissions (Approval, Transfer) to maintain standard compliance.\n- Use clear and accurate naming for custom implementations to distinguish token behaviors.
</Callout>




## Security Considerations

<Callout type="warning" title="Security">
- Validate input parameters such as addresses and amounts to prevent manipulation.\n- Implement access controls for functions like totalSupply adjustments if modifiable.\n- Protect against reentrancy attacks in transfer or transferFrom by following the checks-effects-interactions pattern.
</Callout>
