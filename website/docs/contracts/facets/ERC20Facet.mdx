Contract documentation for ERC20Facet
</DocSubtitle>

<Callout type="info" title="Key Features">
- Implements all essential ERC-20 transferable and balanceable functions.
- Leverages diamond storage for seamless integration with existing facets.
- Emits standard Approval and Transfer events for interoperability with ERC-20 wallets and dApps.
</Callout>

## Overview

The ERC20Facet provides a standard implementation of ERC-20 token functionality for a Compose diamond. It enables basic token operations such as transferring tokens, approving allowances, and querying balances while leveraging the diamond storage pattern for state management.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

import {DiamondBase} from 'ComposeDiamond/DiamondBase.sol';
import {ERC20Facet} from './ERC20Facet.sol';

contract Diamond is DiamondBase {
    constructor(address[] memory _facetAddresses) {
        for (uint256 i = 0; i < _facetAddresses.length; i++) {
            addFacet(_facetAddresses[i]);
        }
    }
}

contract ERC20Example {
    function useERC20Functions(address diamondAddress) external {
        ERC20Facet erc20 = ERC20Facet(diamondAddress);

        // Approve spender to transfer 100 tokens from the caller
        bool success = erc20.approve(address(0xSpender), 100);
        require(success, 'Approval failed');

        // Query for balance
        uint256 balance = erc20.balanceOf(msg.sender);

        // Transfer tokens
        erc20.transfer(address(0xRecipient), 50);

        // Transfer tokens on behalf of another
        erc20.transferFrom(address(0xOwner), address(0xRecipient), 25);
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure all contract facets explicitly define permissions for modifying the ERC20 storage.
- Verify allowance and balance checks within external integration tests.
- Maintain compatibility with other facets when upgrading or modifying the structure.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
Implement thorough input validation for transfer and allowance limits to prevent overflows. Use reentrancy guards to secure token transfers. Audit interactions with other facets to avoid unexpected state mutations.
</Callout>
