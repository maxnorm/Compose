## Overview

ERC20Facet provides essential ERC-20 token functionality for diamond contracts, such as managing balances, allowances, and transfers. It offers a complete implementation of the ERC-20 standard within a self-contained facet to simplify integration while ensuring consistency and modularity.


[View Source](https://github.com/maxnorm/Compose/blob/7206f045d711880c211b967e80d72745b52f5cfc/src/token/ERC20/ERC20/ERC20Facet.sol)



<Callout type="info" title="Key Features">
- Full ERC-20 functionality for diamonds
- Self-contained facet requiring no external imports
- Emit standard Approval and Transfer events
</Callout>

## Usage Example

```solidity
pragma solidity ^0.8.17;

import { IDiamondCut } from &quot;diamond/interfaces/IDiamondCut.sol&quot;;

contract MyTokenDiamond {
    function addERC20Facet(address _erc20Facet) external {
        IDiamondCut.FacetCut memory facetCut = IDiamondCut.FacetCut({
            facetAddress: _erc20Facet,
            action: IDiamondCut.FacetCutAction.Add,
            functionSelectors: new bytes4[](0)
        });

        IDiamondCut(address(this)).diamondCut(
            [facetCut],
            address(0),
            &quot;&quot;
        );
    }

    function transferMyTokens(address _to, uint256 _amount) external {
        ERC20Facet(address(this)).transfer(_to, _amount);
    }
}
```

## Security Considerations

<Callout type="warning" title="Security">
Validate input parameters to prevent overflow or improper approvals. Implement access control when integrating the facet into a diamond to prevent unauthorized `facetCut` actions. Ensure no reentrancy vulnerabilities exist during state changes in functions like `transfer` or `transferFrom`.
</Callout>
