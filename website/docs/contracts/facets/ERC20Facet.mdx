Contract documentation for ERC20Facet

<Callout type="info" title="Key Features">
- Implements full ERC-20 compliance, including metadata, transfers, and approvals.
- Uses the diamond storage pattern for safe and reusable token state management.
- Emits standard ERC-20 events (Approval, Transfer) for compatibility with external tools and contracts.
</Callout>

## Overview

The ERC20Facet provides standard ERC-20 token functionality within a Compose diamond. It manages token state, including balances, allowances, and supply, while exposing essential token operations such as transfer, approval, and balance queries. This facet enables seamless integration of ERC-20 compliance into a diamond architecture.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

import { IDiamondCut } from "@compose/interfaces/IDiamondCut.sol";
import { ERC20Facet } from "path/to/ERC20Facet.sol";

contract DeployDiamond {
    function deploy() external {
        // Deploy the diamond
        Diamond diamond = new Diamond(/* params */);

        // Prepare ERC20Facet for addition
        ERC20Facet erc20Facet = new ERC20Facet();
        IDiamondCut.FacetCut[] memory facetCuts = new IDiamondCut.FacetCut[](1);
        facetCuts[0] = IDiamondCut.FacetCut({
            facetAddress: address(erc20Facet),
            action: IDiamondCut.FacetCutAction.Add,
            functionSelectors: getERC20Selectors()
        });

        // Add ERC20Facet to the diamond
        IDiamondCut(address(diamond)).diamondCut(facetCuts, address(0), "");

        // Example usage
        ERC20Facet(address(diamond)).approve(address(0x1234), 1000);
        ERC20Facet(address(diamond)).transfer(address(0x5678), 500);
    }

    function getERC20Selectors() internal pure returns (bytes4[] memory selectors) {
        selectors = new bytes4[](9);
        selectors[0] = ERC20Facet.name.selector;
        selectors[1] = ERC20Facet.symbol.selector;
        selectors[2] = ERC20Facet.decimals.selector;
        selectors[3] = ERC20Facet.totalSupply.selector;
        selectors[4] = ERC20Facet.balanceOf.selector;
        selectors[5] = ERC20Facet.allowance.selector;
        selectors[6] = ERC20Facet.approve.selector;
        selectors[7] = ERC20Facet.transfer.selector;
        selectors[8] = ERC20Facet.transferFrom.selector;
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure the facet is initialized with proper token metadata (name, symbol, decimals, etc.) before use.
- Use the diamond storage pattern in all state-related functions to prevent storage collision.
- During upgrades, preserve the storage layout and ensure selectors are updated consistently.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
Validate all input parameters, such as recipient addresses and amounts, to prevent misuse. Ensure reentrancy protections are in place for state-modifying functions like transfer and transferFrom. Be cautious with allowance updates to avoid the known double-spend issue in ERC-20 (e.g., use safe approve patterns). Verify that the diamond owner controls the addition or replacement of this facet to prevent unauthorized changes.
</Callout>
