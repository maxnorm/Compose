## Overview

The OwnerFacet provides essential ownership management functions within a diamond contract. It offers a secure and straightforward way to manage the ownership of a contract, including transferring ownership or renouncing it entirely, while adhering to the diamond storage pattern for state management.


[View Source](https://github.com/maxnorm/Compose/blob/38545161423dd99e31b48b0eff41d260712fe731/src/access/Owner/OwnerFacet.sol)



<Callout type="info" title="Key Features">
- Simple and secure ownership management via diamond storage.
- Functions to get, set, and renounce ownership explicitly.
- Designed for easy integration into diamond contracts.
</Callout>

## Usage Example

```solidity
contract ExampleDiamond {
    OwnerFacet private ownerFacet;

    constructor(address _ownerFacetAddress) {
        ownerFacet = OwnerFacet(_ownerFacetAddress);
    }

    function getOwner() external view returns (address) {
        return ownerFacet.owner();
    }

    function changeOwner(address _newOwner) external {
        ownerFacet.transferOwnership(_newOwner);
    }

    function renounceOwnership() external {
        ownerFacet.renounceOwnership();
    }
}
```



## Best Practices

<Callout type="tip" title="Best Practice">
- Always validate that only the current owner can call ownership-modifying functions (e.g., transferOwnership or renounceOwnership).
- Use the getStorage function only in internal contexts where ownership-related state must be accessed securely and explicitly.
- When transferring ownership, ensure that the new owner address is valid and capable of managing the facet for secure contract governance.
</Callout>



## Security Considerations

<Callout type="warning" title="Security">
Ensure proper access control is implemented for ownership-changing functions to prevent unauthorized access. Avoid improperly setting the owner address to avoid permissions mismanagement. The renounceOwnership function should be used with caution as it makes the contract permanently permissionless and could lead to loss of administrative capability.
</Callout>
