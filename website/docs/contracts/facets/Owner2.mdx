## Overview

Owner2 provides core functionality for managing and verifying ownership of a diamond contract in compliance with the ERC-173 standard. It enables controlled access to critical operations by enforcing permission checks for the owner, ensuring robust and secure contract management.


[View Source](https://github.com/maxnorm/Compose/blob/f773c1f2b6ca1fc289a7b9ea4f149315edd34199/src/access/Owner/Owner2.sol)



<Callout type="info" title="Key Features">
- Provides ERC-173 compliant contract ownership management
- Supports ownership transfer and renunciation
- Includes permission checks to ensure only the owner can perform specific operations
</Callout>

## Usage Example

```solidity
pragma solidity ^0.8.0;

import { LibDiamond } from &quot;./libraries/LibDiamond.sol&quot;;
import { Owner2 } from &quot;./facets/Owner2.sol&quot;;

contract DiamondExample {
    function transferOwnership(address _newOwner) external {
        Owner2.transferOwnership(_newOwner);
    }

    function getCurrentOwner() external view returns (address) {
        return Owner2.owner();
    }

    function ensureCallerIsOwner() external view {
        Owner2.requireOwner();
    }
}
```



## Best Practices

<Callout type="tip" title="Best Practice">
- Always validate ownership using `requireOwner` before executing sensitive operations.
- Use `transferOwnership` responsibly and ensure `_newOwner` is a legitimate and trusted address.
- Regularly audit the contract to ensure no unintentional bypass of ownership checks.
</Callout>



## Security Considerations

<Callout type="warning" title="Security">
Ensure the `transferOwnership` function is not called unintentionally or without verifying the legitimacy of `_newOwner`. Be cautious of renouncing ownership by setting the new owner to `address(0)` as it makes the contract fully permissionless, which is irreversible. Use the `requireOwner` function to safeguard critical operations and prevent unauthorized access.
</Callout>
