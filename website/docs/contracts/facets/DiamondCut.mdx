## Overview

The DiamondCut facet is a core component of the Compose diamond proxy library, enabling dynamic upgrades of diamond contract functions. It provides mechanisms to add, remove, or replace functions in a diamond while ensuring flexibility and consistency in on-chain applications.


[View Source](https://github.com/maxnorm/Compose/blob/f773c1f2b6ca1fc289a7b9ea4f149315edd34199/src/diamond/DiamondCut.sol)



<Callout type="info" title="Key Features">
- Add, remove, or replace diamond functions dynamically
- Supports optional delegatecall execution after updates
- Enables modular and flexible smart contract systems
</Callout>

## Usage Example

```solidity
pragma solidity ^0.8.0;

import { DiamondCut } from &quot;./DiamondCut.sol&quot;;

contract MyDiamond {
    DiamondCut private diamondCut;

    constructor(address _diamondCutAddress) {
        diamondCut = DiamondCut(_diamondCutAddress);
    }

    function upgradeDiamond(address[] calldata _newFacets) external {
        bytes[] memory emptyData = new bytes[](_newFacets.length);
        diamondCut.diamondCut(
            _newFacets,
            emptyData,
            address(0),
            &quot;&quot;
        );
    }
}
```



## Best Practices

<Callout type="tip" title="Best Practice">
- Always validate new function selectors to avoid overwriting unintended functions.
- Test all changes in a controlled environment before applying them to the diamond.
- Ensure facets being added follow the Compose guidelines for readability and transparency.
</Callout>



## Security Considerations

<Callout type="warning" title="Security">
Ensure only authorized users can invoke the diamondCut function to prevent unauthorized modifications. Validate facet addresses and selectors carefully to avoid potential injection or corruption. Test updates comprehensively to mitigate the risk of unexpected behavior in production.
</Callout>
