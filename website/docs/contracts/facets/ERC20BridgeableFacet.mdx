## Overview

The ERC20BridgeableFacet enables seamless cross-chain minting and burning of ERC-20 tokens, leveraging diamond storage and precise access control. It ensures security by restricting privileged operations to addresses with the &#39;trusted-bridge&#39; role, making it ideal for cross-chain token interoperability in diamond proxy architectures.


[View Source](https://github.com/maxnorm/Compose/blob/e3e377cf3d77e94dc30e10812da2bdd1907ca159/src/token/ERC20/ERC20Bridgeable/ERC20BridgeableFacet.sol)



<Callout type="info" title="Key Features">
- Supports cross-chain mint and burn functionalities.
- Built-in access control with role-based permissions.
- Efficient use of diamond storage for ERC-20 token data.
</Callout>

## Usage Example

```solidity
pragma solidity ^0.8.0;

import { ERC20BridgeableFacet } from &quot;./facets/ERC20BridgeableFacet.sol&quot;;

contract DiamondExample {
    function mintTokensAcrossChains(address _recipient, uint256 _amount) external {
        // Calls crosschainMint from the ERC20BridgeableFacet within the diamond
        ERC20BridgeableFacet.crosschainMint(_recipient, _amount);
    }

    function burnTokensAcrossChains(address _tokenOwner, uint256 _amount) external {
        // Calls crosschainBurn from the ERC20BridgeableFacet within the diamond
        ERC20BridgeableFacet.crosschainBurn(_tokenOwner, _amount);
    }
}
```



## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure the &#39;trusted-bridge&#39; role is properly configured to prevent unauthorized minting or burning.
- Regularly audit storage layout and access control settings to maintain diamond consistency.
- Implement comprehensive testing for cross-chain token operations to safeguard user balances.
</Callout>



## Security Considerations

<Callout type="warning" title="Security">
Restrict access to the &#39;trusted-bridge&#39; role for privileged operations to prevent unauthorized token manipulation. Ensure all operations validate input parameters thoroughly to prevent misuse or unexpected errors. Monitor access control storage integrity and implement reentrancy protection if applicable.
</Callout>
