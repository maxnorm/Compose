## Overview

The ERC20BridgeableFacet enables cross-chain interoperability for ERC20 tokens within a diamond architecture by providing minting and burning functions specifically for trusted bridge roles. This facet ensures seamless token movement across chains securely and efficiently while adhering to access control policies.


[View Source](https://github.com/maxnorm/Compose/blob/7206f045d711880c211b967e80d72745b52f5cfc/src/token/ERC20/ERC20Bridgeable/ERC20BridgeableFacet.sol)



<Callout type="info" title="Key Features">
- Cross-chain minting support for trusted bridge addresses
- Controlled token burning by verified bridge operators
- Seamless storage integration using diamond pattern
</Callout>

## Usage Example

```solidity
interface IERC20BridgeableFacet {
    function crosschainMint(address _recipient, uint256 _amount) external;
    function crosschainBurn(address _sender, uint256 _amount) external;
}

contract TestDiamondIntegration {
    IERC20BridgeableFacet constant bridgeableFacet = IERC20BridgeableFacet(address(0xYourDiamondAddress));

    function executeCrosschainMint(address recipient, uint256 amount) external {
        // Ensure the caller prepares the necessary conditions for minting
        bridgeableFacet.crosschainMint(recipient, amount);
    }

    function executeCrosschainBurn(address sender, uint256 amount) external {
        // Perform cross-chain burn, callable only by the trusted bridge
        bridgeableFacet.crosschainBurn(sender, amount);
    }
}
```



## Best Practices

<Callout type="tip" title="Best Practice">
- Assign the `trusted-bridge` role only to known, verified bridge operators to maintain integrity.
- Use the `checkTokenBridge` internally before any minting or burning to validate the caller&#39;s role.
- Ensure that the cross-chain token supply is correctly synchronized to prevent inconsistencies.
</Callout>



## Security Considerations

<Callout type="warning" title="Security">
- Ensure `crosschainMint` and `crosschainBurn` are only callable by addresses holding the `trusted-bridge` role.
- Validate caller identities and roles rigorously before token transfer operations.
- Be cautious of potential gas limit issues during complex cross-chain operations, ensuring all interactions are efficient.
</Callout>
