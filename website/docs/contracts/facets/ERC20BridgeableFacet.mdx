<Callout type="info" title="Key Features">
- Facilitates cross-chain minting and burning with strict access control using role-based permissions.
- Leverages dedicated diamond storage patterns to safely manage ERC20 and AccessControl data.
</Callout>

## Overview

The ERC20BridgeableFacet enables cross-chain token operations within a diamond proxy infrastructure. It provides mechanisms for minting and burning tokens across chains, while enforcing access control based on a trusted-bridge role.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

import {ERC20BridgeableFacet} from "./ERC20BridgeableFacet.sol";

contract ExampleDiamond {
    function executeCrossChainMint(address _recipient, uint256 _amount) external {
        // Assumes ERC20BridgeableFacet is added to the diamond
        ERC20BridgeableFacet(address(this)).crosschainMint(_recipient, _amount);
    }

    function executeCrossChainBurn(address _account, uint256 _amount) external {
        ERC20BridgeableFacet(address(this)).crosschainBurn(_account, _amount);
    }
}

// Deployment script example

ERC20BridgeableFacet facet = new ERC20BridgeableFacet();
IDiamondCut.FacetCut[] memory cuts = new IDiamondCut.FacetCut[](1);
cuts[0] = IDiamondCut.FacetCut({
    facetAddress: address(facet),
    action: IDiamondCut.FacetCutAction.Add,
    functionSelectors: new bytes4[](5)
});
cuts[0].functionSelectors[0] = ERC20BridgeableFacet.getERC20Storage.selector;
cuts[0].functionSelectors[1] = ERC20BridgeableFacet.getAccessControlStorage.selector;
cuts[0].functionSelectors[2] = ERC20BridgeableFacet.crosschainMint.selector;
cuts[0].functionSelectors[3] = ERC20BridgeableFacet.crosschainBurn.selector;
cuts[0].functionSelectors[4] = ERC20BridgeableFacet.checkTokenBridge.selector;

// Perform the diamond cut to attach the facet
diamondCut.diamondCut(cuts, address(0), "");`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure trusted-bridge roles are assigned explicitly within AccessControl storage before invoking cross-chain operations.
- Verify the ERC20 and AccessControl storage layouts align with the specification to avoid unexpected inconsistencies.
- Use internal checks like &#x60;checkTokenBridge&#x60; to enforce access permissions and maintain security guarantees during upgrades.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
Restrict the crosschainMint and crosschainBurn functions to trusted addresses; verify and enforce their role membership. Use the &#x60;checkTokenBridge&#x60; function for caller validation in critical paths. Ensure the AccessControl storage is properly initialized to define trusted roles. Validate caller input and handle edge cases to prevent misbehavior.
</Callout>
