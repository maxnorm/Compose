**Title:**
</DocSubtitle>

<Callout type="info" title="Key Features">
- Enables secure cross-chain token minting and burning operations.
- Enforces &#x60;trusted-bridge&#x60; role for all cross-chain token movements.
- Provides internal checks for bridge trust via &#x60;checkTokenBridge&#x60;.
</Callout>

## Overview

The ERC20BridgeableFacet facilitates cross-chain operations for ERC20 tokens within a Compose diamond. It provides functions to mint and burn tokens based on trusted bridge interactions, ensuring secure and controlled cross-chain token transfers.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

import {ERC20BridgeableFacet} from "@compose/contracts/src/facets/ERC20BridgeableFacet.sol";
import {IDiamondCut} from "@compose/contracts/src/interfaces/IDiamondCut.sol";

contract DeployERC20BridgeableFacet {
    address diamond = 0x...; // Address of your Compose diamond

    function deploy() external {
        ERC20BridgeableFacet erc20BridgeableFacet = new ERC20BridgeableFacet();

        IDiamondCut.FacetCut[] memory cuts = new IDiamondCut.FacetCut[](1);
        cuts[0] = IDiamondCut.FacetCut({
            facetAddress: address(erc20BridgeableFacet),
            action: IDiamondCut.Action.ADD,
            isPure: false,
            selectors: IDiamondCut.getSelectors(erc20BridgeableFacet)
        });

        bytes32 diamondCutStorageSlot = 0x...; // Storage slot for diamond cut
        address owner = 0x...; // Owner of the diamond

        // Assuming a DiamondLoupeFacet or similar is used for upgrades
        // This is a simplified example; actual upgrade logic may vary.
        // owner.call(abi.encodeWithSignature("diamondCut(bytes32,IDiamondCut.FacetCut[],address)",
        //    diamondCutStorageSlot,
        //    cuts,
        //    address(0) // target address for upgrade, 0 for self
        // ));
    }

    function mintCrosschain(address _token, uint256 _amount, address _to) external {
        ERC20BridgeableFacet(diamond).crosschainMint(_token, _amount, _to);
    }

    function burnCrosschain(address _token, uint256 _amount, address _from) external {
        ERC20BridgeableFacet(diamond).crosschainBurn(_token, _amount, _from);
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Initialize trusted bridge roles using the AccessControl facet before calling crosschain mint/burn functions.
- Ensure the &#x60;ERC20BridgeableFacet&#x60; is added to the diamond with appropriate selectors.
- Use &#x60;getERC20Storage&#x60; and &#x60;getAccessControlStorage&#x60; to access facet-specific storage if direct interaction is needed, though direct calls to facet functions are preferred.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
Access to &#x60;crosschainMint&#x60; and &#x60;crosschainBurn&#x60; is restricted to addresses with the &#x60;trusted-bridge&#x60; role, managed by the AccessControl facet. The &#x60;checkTokenBridge&#x60; function prevents calls from unauthorized addresses or the zero address. Ensure proper role management within the AccessControl facet to maintain security.
</Callout>
