## Overview

The ERC721Facet provides a comprehensive implementation of the ERC-721 functionality within a diamond proxy system. This facet allows developers to integrate and manage non-fungible tokens (NFTs) seamlessly, offering key methods like ownership tracking, token transfers, and metadata retrieval, making it a fundamental building block for ERC-721 diamond deployments.


[View Source](https://github.com/maxnorm/Compose/blob/7206f045d711880c211b967e80d72745b52f5cfc/src/token/ERC721/ERC721/ERC721Facet.sol)



<Callout type="info" title="Key Features">
- Full implementation of ERC-721 standard.
- Provides secure, modular NFT functionality for diamond patterns.
</Callout>

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure `getStorage` is used internally only, and never exposed directly to external calls.
- Always perform proper checks (e.g., ownership, approvals) before executing transfer functions.
- Use `safeTransferFrom` when handling transactions with potential unknown or unsafe recipients.
</Callout>



## Security Considerations

<Callout type="warning" title="Security">
Ensure proper access control, particularly when setting approvals or managing token ownership. Avoid direct external calls to low-level functions like `getStorage`, as they may expose sensitive contract state. Always implement reentrancy mitigation strategies such as using &#39;checks-effects-interactions&#39; pattern or reentrancy guards for critical operations like `transferFrom`.
</Callout>
