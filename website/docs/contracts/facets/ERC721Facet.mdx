<Callout type="info" title="Key Features">
- Fully implements core ERC-721 functions and metadata handling.
- Compatible with diamond storage pattern for extended flexibility.
- Provides safe transfer methods to handle ERC-721 receiver contracts.
</Callout>

## Overview

The ERC721Facet provides reusable ERC-721 functionality within the Compose diamond framework. It facilitates token interactions such as transfers, approvals, and metadata retrieval while leveraging the diamond storage pattern to maintain compliance with ERC-721 standards.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

import {DiamondCutFacet} from './DiamondCutFacet.sol';
import {ERC721Facet} from './ERC721Facet.sol';

contract DeployDiamond {
    function deployDiamond() external {
        DiamondCutFacet diamondCutFacet = new DiamondCutFacet();
        ERC721Facet erc721Facet = new ERC721Facet();

        // Create diamond with initial facets
        bytes4[] memory selectors = new bytes4[](2);
        selectors[0] = ERC721Facet.name.selector;
        selectors[1] = ERC721Facet.balanceOf.selector;

        IDiamondCut.FacetCut[] memory diamondCut = new IDiamondCut.FacetCut[](1);
        diamondCut[0] = IDiamondCut.FacetCut({
            facetAddress: address(erc721Facet),
            action: IDiamondCut.FacetCutAction.Add,
            functionSelectors: selectors
        });

        diamondCutFacet.diamondCut(diamondCut, address(0), '');

        // Interact with ERC-721 facet
        ERC721Facet diamondERC721 = ERC721Facet(address(diamondCutFacet));
        require(keccak256(abi.encodePacked(diamondERC721.name())) == keccak256('My Collection'), 'Unexpected name');
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Initialize ERC-721 specific state using a separate initialization function post-deployment.
- Ensure Solidity version compatibility and test with the same version used in the diamond framework.
- Avoid removing ERC-721 storage layout fields unless they are the last fields in the struct to preserve state integrity.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
Enforce proper access control for administrative functions like adding/removing facets. Validate inputs in methods like approve, setApprovalForAll, and transferFrom to prevent unintended use. Use internalTransferFrom only within the facet to ensure secure internal logic compliance. Avoid potential reentrancy attacks in safeTransferFrom by transferring tokens only after external calls.
</Callout>
