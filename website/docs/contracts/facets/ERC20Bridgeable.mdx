## Overview

ERC20Bridgeable is a facet designed to support cross-chain token operations in diamond contracts. It enables trusted bridges to perform mint and burn operations while providing secure access control mechanisms tied to a storage slot. This library ensures safe and efficient token bridging across chains by verifying roles and maintaining integrity.


[View Source](https://github.com/maxnorm/Compose/blob/f773c1f2b6ca1fc289a7b9ea4f149315edd34199/src/token/ERC20/ERC20Bridgeable/ERC20Bridgeable.sol)



<Callout type="info" title="Key Features">
- Secure cross-chain minting and burning of tokens.
- Access control tied to trusted bridge roles via diamond storage.
- Internal helper functions to streamline access and storage retrieval.
</Callout>

## Usage Example

```solidity
pragma solidity ^0.8.0;

import { LibDiamond } from &quot;path/to/LibDiamond.sol&quot;;
import { ERC20Bridgeable } from &quot;path/to/ERC20Bridgeable.sol&quot;;

contract MyDiamond {
    function initializeBridge(address _trustedBridge) external {
        ERC20Bridgeable.getAccessControlStorage().trustedBridge = _trustedBridge;
    }

    function bridgeTokens(uint256 amount, address receiver) external {
        ERC20Bridgeable.crosschainBurn(msg.sender, amount);
        // Call external bridge logic here...
    }

    function receiveCrossChainTokens(uint256 amount, address recipient) external {
        require(msg.sender == ERC20Bridgeable.getAccessControlStorage().trustedBridge, &quot;Unauthorized&quot;);
        ERC20Bridgeable.crosschainMint(recipient, amount);
    }
}
```



## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure the trusted bridge address is set during initialization and is only modifiable through strict administrative access.
- Use the provided access control storage check functions instead of redundant external checks for clarity and consistency.
- Implement reentrancy guards or checks in calling functions, especially when handling external bridge operations.
</Callout>



## Security Considerations

<Callout type="warning" title="Security">
Access control is critical: only trusted bridges should be allowed to perform mint or burn operations. Ensure reentrancy is managed in the contract logic calling cross-chain functions. Verify that receiver addresses and parameters are valid before execution to prevent unintended token loss or disruptions.
</Callout>
