## Overview

The DiamondCutMod facet provides the core functionality for managing facets within a diamond using the diamondCut function. It enables the addition, replacement, and removal of facet functions, facilitating on-chain module upgrades and extending contract functionality seamlessly. This facet is critical for maintaining flexible and upgradable diamond-based architectures.


[View Source](https://github.com/maxnorm/Compose/blob/7050baa250694d5e2445aa4c6b96c62c326cb1cb/src/diamond/DiamondCutMod.sol)



<Callout type="info" title="Key Features">
- Modify diamond functionality dynamically by adding, replacing, or removing facet functions.\n- Supports executing additional actions during the diamondCut operation.\n- Provides direct access to the storage layout through getStorage.
</Callout>

## Usage Example

```solidity
contract ExampleDiamond {\n    address constant DIAMOND_CUT_MOD_FACET = 0x...; // Address of DiamondCutMod facet\n\n    function performDiamondCut() external {\n        IDiamondCutMod diamondCut = IDiamondCutMod(DIAMOND_CUT_MOD_FACET);\n\n        // Define function selectors and new implementation address\n        bytes4[] memory functionSelectors = new bytes4[](1);\n        functionSelectors[0] = bytes4(keccak256(&quot;newFunctionName()&quot;));\n        address implementation = address(0x123...);\n\n        // Add new functionality\n        IDiamondCutMod.FacetCut[] memory cuts = new IDiamondCutMod.FacetCut[](1);\n        cuts[0] = IDiamondCutMod.FacetCut({\n            facetAddress: implementation,\n            action: IDiamondCutMod.FacetCutAction.Add,\n            functionSelectors: functionSelectors\n        });\n        \n        diamondCut.diamondCut(cuts, address(0), new bytes(0));\n    }\n}
```



## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure the diamondCut action aligns with the intended operation (add, replace, or remove).\n- Thoroughly test new or modified facets off-chain before performing an upgrade on the diamond.\n- Always maintain a backup of the current diamond state before performing disruptive upgrades.
</Callout>




## Security Considerations

<Callout type="warning" title="Security">
Access to diamondCut and other management functions must be restricted to authorized accounts, typically the diamond owner or an admin role. Improper use can lead to unintended loss of functionality, state corruption, or security vulnerabilities. Ensure to validate any code being added or replaced and account for potential reentrancy concerns within the facets, especially when other facets call externally or interact with user-provided input.
</Callout>
