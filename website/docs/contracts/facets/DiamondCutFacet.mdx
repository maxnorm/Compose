<Callout type="info" title="Key Features">
- Enables function addition, replacement, and removal in a single atomic operation.
- Supports optional initialization via delegatecall after applying the diamond cut.
</Callout>

## Overview

The DiamondCutFacet manages the addition, replacement, and removal of functions within a diamond, acting as the core mechanism for modular upgrades. It uses the diamond storage pattern to maintain configuration and ensures precise function routing to intended facets.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

import {DiamondCutFacet} from "path-to-your-facets/DiamondCutFacet.sol";

contract Example {
    function executeDiamondCut(address diamondCutFacet, bytes memory cutData, address initFacet, bytes memory initData) external {
        // Assume diamondCutFacet is deployed and integrated into a diamond
        (bool success, ) = diamondCutFacet.delegatecall(
            abi.encodeWithSelector(DiamondCutFacet.diamondCut.selector, cutData, initFacet, initData)
        );
        require(success, "Diamond cut failed");
    }
}

// Example integration
// cutData should be ABI-encoded as tuples representing the cut operations (Action=0/1/2, FunctionSelectors, FacetAddress).
// initFacet and initData allow delegatecall initialization after the cut.
// Front-end tools or scripts can prepare the \`cutData\` dynamically based on developer requirements.`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure &#x60;diamondCut&#x60; is only callable via a secure and authenticated access control mechanism.
- Maintain proper ordering within &#x60;cutData&#x60;, ensuring selectors align correctly with facet addresses.
- Test upgrades in a controlled environment to prevent unintended overwrites or removals of critical functionalities.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
Restrict access to &#x60;diamondCut&#x60; to prevent unauthorized changes. Validate &#x60;cutData&#x60; thoroughly to ensure selectors match specified actions and facet addresses. Be cautious with delegatecall during initialization to prevent reentrancy or state corruption.
</Callout>
