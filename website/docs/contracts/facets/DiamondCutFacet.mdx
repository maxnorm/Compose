## Overview

The DiamondCutFacet provides the ability to add, replace, or remove functions within a diamond contract. It leverages the diamond standard (EIP-2535) to enable modular and upgradable contracts by facilitating dynamic function management. This facet ensures that diamonds can be composed flexibly while maintaining clarity and simplicity.


[View Source](https://github.com/maxnorm/Compose/blob/38545161423dd99e31b48b0eff41d260712fe731/src/diamond/DiamondCutFacet.sol)



<Callout type="info" title="Key Features">
- Supports dynamic addition, replacement, and removal of functions
- Fully compliant with the diamond standard (EIP-2535)
- Provides optional delegatecall execution post function update
</Callout>

## Usage Example

```solidity
pragma solidity ^0.8.0;

import {DiamondCutFacet} from &quot;./DiamondCutFacet.sol&quot;;

contract DiamondExample {
    DiamondCutFacet private diamondCutFacet;

    constructor(address _diamondCutFacet) {
        diamondCutFacet = DiamondCutFacet(_diamondCutFacet);
    }

    function updateDiamond() external {
        // Example: Adding new functionality to the diamond
        bytes4[] memory selectors = new bytes4[](1);
        selectors[0] = bytes4(keccak256(&quot;newFunction()&quot;));

        // Add new functions using diamondCut
        diamondCutFacet.diamondCut([
            DiamondCutFacet.FacetCut({
                facetAddress: address(newFacet),
                action: 0, // Add
                functionSelectors: selectors
            })], address(0), new bytes(0));
    }
}
```



## Best Practices

<Callout type="tip" title="Best Practice">
- Always validate and verify selectors and facet addresses before executing `diamondCut` to avoid unintended behavior
- Ensure `diamondCut` calls maintain backward compatibility if required
- Use clear documentation to track added/replaced/removed selectors
</Callout>



## Security Considerations

<Callout type="warning" title="Security">
Ensure all `diamondCut` calls are properly authorized to avoid unauthorized modifications. Verify facet implementations thoroughly to prevent malicious or unexpected behavior. Be cautious of delegatecall execution and its implications for state and reentrancy risks.
</Callout>
