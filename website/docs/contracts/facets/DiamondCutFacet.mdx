<Callout type="info" title="Key Features">
- Supports dynamic modular upgrades by adding, replacing, or removing facets and their selectors.
- Allows optional initialization via delegatecall during diamondCut to setup newly added facets.
- Adheres to the diamond storage schema for robust and conflict-free state management.
</Callout>

## Overview

The DiamondCutFacet provides fundamental functionality for dynamically managing the function dispatch table of a diamond proxy contract. It allows adding, replacing, and removing functions or facets at runtime using the diamondCut function, enabling modular upgrades and extensibility.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

interface IDiamondCut {
    enum FacetCutAction { Add, Replace, Remove }

    struct FacetCut {
        address facet;
        FacetCutAction action;
        bytes4[] functionSelectors;
    }

    function diamondCut(
        FacetCut[] memory _cut,
        address _init,
        bytes memory _calldata
    ) external;
}

contract ExampleDiamond {
    function performUpgrade(address diamondCutFacet, address newFacet) external {
        IDiamondCut.FacetCut[] memory cut = new IDiamondCut.FacetCut[](1);
        cut[0] = IDiamondCut.FacetCut({
            facet: newFacet,
            action: IDiamondCut.FacetCutAction.Add,
            functionSelectors: new bytes4[](1)
        });
        cut[0].functionSelectors[0] = bytes4(keccak256("newFunctionName()"));

        IDiamondCut(diamondCutFacet).diamondCut(cut, address(0), "");
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Always verify the correctness of function selectors and the target facet addresses before applying changes using the diamondCut function.
- Ensure proper authorization checks are implemented so only trusted parties can access diamondCut.
- Thoroughly test facet upgrades in a controlled environment to maintain diamond proxy integrity and compatibility.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
Only authorized accounts should invoke the diamondCut function, as improper usage could compromise the diamond. Validate all inputs to ensure they conform to expected formats and lengths. When using delegatecall for initialization during upgrades, ensure the provided calldata and initializer facet are trusted and secure. Avoid state overwrites by carefully managing storage slots across facets.
</Callout>
