## Overview

The OwnerTwoSteps library implements a secure two-step ownership transfer process for modular contracts using the diamond standard. It ensures proper access control by introducing pending ownership states, thus permitting the new owner to actively accept ownership before transferring control, improving both security and clarity in ownership transitions.


[View Source](https://github.com/maxnorm/Compose/blob/f773c1f2b6ca1fc289a7b9ea4f149315edd34199/src/access/OwnerTwoSteps/OwnerTwoSteps.sol)



<Callout type="info" title="Key Features">
- Implements ERC-173 standard ownership patterns.
- Enhances ownership security with a pending state.
- Easy integration with diamond storage layouts.
</Callout>

## Usage Example

```solidity
pragma solidity ^0.8.0;

import &quot;./OwnerTwoSteps.sol&quot;;

contract ExampleDiamond {
    using OwnerTwoSteps for bytes32;

    /**
     * Storage Position Constants
     */
    bytes32 internal constant OWNER_SLOT = keccak256(&quot;example.owner.slot&quot;);
    bytes32 internal constant PENDING_OWNER_SLOT = keccak256(&quot;example.pending.owner.slot&quot;);

    function transferOwnership(address _newOwner) external {
        OWNER_SLOT.transferOwnership(_newOwner);
    }

    function acceptOwnership() external {
        OWNER_SLOT.acceptOwnership();
    }

    function owner() external view returns (address) {
        return OWNER_SLOT.owner();
    }
}
```



## Best Practices

<Callout type="tip" title="Best Practice">
- Always verify that a new owner actively accepts the transfer by calling `acceptOwnership`.
- Use `requireOwner` to restrict critical functions to the current owner.
- Rename or isolate owner/pending-owner storage slots to avoid conflicts in multi-facet setups.
</Callout>



## Security Considerations

<Callout type="warning" title="Security">
Ensure all critical functions utilize `requireOwner` for access control. Validate that the pending owner is a trusted and verified account before initiating the transfer via `transferOwnership`. Avoid transferring ownership to contract addresses unless they are explicitly authorized.
</Callout>
