## Overview

The ERC20 facet provides core ERC-20 functionality such as transferring, minting, burning, and approving tokens. It enables modular token logic in a diamond contract for seamless contract upgrades and efficient token management.


[View Source](https://github.com/maxnorm/Compose/blob/f773c1f2b6ca1fc289a7b9ea4f149315edd34199/src/token/ERC20/ERC20/ERC20.sol)



<Callout type="info" title="Key Features">
- Basic ERC-20 functionality including `transfer`, `approve`, `mint`, and `burn`.
- Efficient pointer access to shared ERC-20 storage using inline assembly.
- Designed for modular inclusion in diamond proxy contracts.
</Callout>

```solidity
struct ERC20Storage {
mapping(address owner =&gt; uint256 balance) balanceOf;
uint256 totalSupply;
mapping(address owner =&gt; mapping(address spender =&gt; uint256 allowance)) allowance;
uint8 decimals;

string symbol;
}
```





## Usage Example

```solidity
contract MyToken is ModifiableDiamond {
    function initializeERC20() external {
        LibERC20Storage.ERC20Storage storage ds = LibERC20Storage.getStorage();
        ds.totalSupply = 1000000 * 10 ** 18;
        ds.name = &quot;MyToken&quot;;
        ds.symbol = &quot;MTK&quot;;
        ds.decimals = 18;
    }

    function mintTokens(address _to, uint256 _amount) external onlyOwner {
        LibERC20.mint(_to, _amount);
    }

    function transferTokens(address _to, uint256 _amount) external {
        LibERC20.transfer(_to, _amount);
    }

    function burnTokens(uint256 _amount) external {
        LibERC20.burn(msg.sender, _amount);
    }
}
```



## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure all access control checks are implemented to prevent unauthorized minting or burning.
- Design permissionless functions like `transfer` to follow the standard ERC-20 behavior for usability and interoperability.
- Avoid modifying or removing storage variables in the struct to maintain storage consistency across upgrades.
</Callout>



## Security Considerations

<Callout type="warning" title="Security">
Access control mechanisms need to be enforced strictly for privileged operations like `mint` and `burn`. Proper validation of user inputs is essential to avoid overflows or unintended token operations. Reentrancy concerns in token transfers must be addressed, especially if integrating with other external contracts.
</Callout>
