## Overview

The ERC165 facet provides functionality to manage and detect supported interfaces as per the ERC-165 standard. It simplifies interface registration and detection within diamond proxy contracts, ensuring composability and modularity in contract design.


[View Source](https://github.com/maxnorm/Compose/blob/f773c1f2b6ca1fc289a7b9ea4f149315edd34199/src/interfaceDetection/ERC165/ERC165.sol)



<Callout type="info" title="Key Features">
- Provides compliant ERC-165 interface standard detection.
- Enables modular registration of interfaces during initialization.
- Simplifies integration with other facets through interface detection.
</Callout>

## Usage Example

```solidity
pragma solidity ^0.8.0;

import &quot;./LibERC165.sol&quot;;

default DiamondInit {
    function initialize() external {
        // Register the IERC721 interface ID during initialization
        LibERC165.registerInterface(type(IERC721).interfaceId);
    }
    
    function supportsInterface(bytes4 interfaceId) external view returns (bool) {
        // Retrieve ERC165 storage and check if an interface is supported
        LibERC165.Storage storage storageERC165 = LibERC165.getStorage();
        return storageERC165.supportedInterfaces[interfaceId];
    }
}
```



## Best Practices

<Callout type="tip" title="Best Practice">
- Always register interfaces during the initialization phase of your diamond or facet.
- Use meaningful and standardized interface IDs to avoid collisions.
- Ensure updates to storage follow immutable struct patterns to protect existing definitions.
</Callout>



## Security Considerations

<Callout type="warning" title="Security">
Ensure that only legitimate interfaces are registered by limiting interface registration calls to protected initialization functions. Avoid reentrancy vulnerabilities by keeping interface registration logic simple and straightforward.
</Callout>
