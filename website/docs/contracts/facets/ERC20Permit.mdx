## Overview

The ERC20Permit library provides functionality required for implementing ERC-2612 permit signatures, enabling users to set allowances using off-chain signatures instead of on-chain transactions. This enhances usability by reducing gas costs and enabling meta-transactions in diamond-based proxy architectures.


[View Source](https://github.com/maxnorm/Compose/blob/f773c1f2b6ca1fc289a7b9ea4f149315edd34199/src/token/ERC20/ERC20Permit/ERC20Permit.sol)



<Callout type="info" title="Key Features">
- Implements ERC-2612 permit functionality for setting allowances via signatures.
- Reduces gas costs by enabling off-chain approval.
- Ensures compatibility with diamond storage patterns.
</Callout>

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure the Approval event is emitted in the calling facet or contract to comply with ERC-2612.
- Confirm the DOMAIN_SEPARATOR uses the correct chain ID and contract address to avoid replay attacks.
- Validate input parameters like `_deadline` to prevent misuse.
</Callout>



## Security Considerations

<Callout type="warning" title="Security">
Ensure proper access control during facet assembly to prevent unauthorized calls to permit-related functions. Validate all input parameters closely, especially `_deadline` and `_spender` to avoid abuse. Ensure that domain separation is correctly calculated to prevent cross-contract replay attacks.
</Callout>
