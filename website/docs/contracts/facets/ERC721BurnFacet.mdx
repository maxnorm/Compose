<Callout type="info" title="Key Features">
- Provides inline storage access to the ERC-721 struct for efficient updates.
- Implements token burning by removing it from enumeration tracking, ensuring compliance with ERC-721 behavior.
</Callout>

## Overview

The ERC721BurnFacet provides functions to burn tokens within an ERC-721 diamond implementation. It handles removal from token enumeration and maintains adherence to the ERC-721 specification by modifying diamond storage directly.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

import {LibDiamond} from "./libraries/LibDiamond.sol";
import {ERC721BurnFacet} from "./facets/ERC721BurnFacet.sol";

contract Diamond {
    ERC721BurnFacet private burnFacet;

    constructor() {
        // Initialize the ERC721BurnFacet during diamond deployment
        burnFacet = new ERC721BurnFacet();
    }

    function burnToken(uint256 _tokenId) external {
        // Delegate burn operation to the facet
        burnFacet.burn(_tokenId);
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Always initialize the storage slot using the getStorage function provided by the facet.
- Limit the access to the &#x60;burn&#x60; function to authorized parties (e.g., owner or approved operators).
- Integrate with a diamond upgrade process that ensures the storage layout adheres to this facet&#x27;s expectations.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
Ensure the &#x60;burn&#x60; function is protected by access control to avoid unauthorized token destruction. Validate input token IDs to confirm ownership or operator approval before performing the burn. Reentrancy risks are minimal but should be mitigated by appropriate checks if external calls are introduced in future adjustments.
</Callout>
