## Overview

The ERC721BurnFacet provides burning functionality for ERC-721 tokens within a diamond proxy. It enhances composability by enabling token destruction while maintaining enumeration and updating associated storage accurately.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

interface IDiamond {
    function facetAddress(bytes4 _functionSelector) external view returns (address facet);
    function execute(bytes calldata data) external;
}

contract UseERC721BurnFacet {
    IDiamond private immutable diamond;

    constructor(address _diamond) {
        diamond = IDiamond(_diamond);
    }

    function burnToken(uint256 _tokenId) external {
        // Function selector for burn(uint256)
        bytes4 selector = bytes4(keccak256("burn(uint256)"));
        address facet = diamond.facetAddress(selector);
        require(facet != address(0), "Facet not found");

        // Execute burn through the diamond proxy
        diamond.execute(abi.encodeWithSelector(selector, _tokenId));
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Initialize diamond with ERC721 storage and ensure storage slot uniqueness.
- Scope burning logic to prevent unauthorized token destruction.
- Carefully follow upgrade procedures to maintain consistent storage layout.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
Ensure the burn function is restricted to authorized users or conditions to prevent abuse. Validate token ownership and existence to avoid unintended storage overwrites. Consider the reentrancy risk when destroying tokens and handling other callbacks.
</Callout>
