## Overview

The ERC721BurnFacet provides support for burning ERC-721 tokens in a diamond contract. It includes functionality to securely destroy tokens and update enumeration tracking, ensuring accurate state management in compliance with ERC-721 standards.


[View Source](https://github.com/maxnorm/Compose/blob/7206f045d711880c211b967e80d72745b52f5cfc/src/token/ERC721/ERC721/ERC721BurnFacet.sol)



<Callout type="info" title="Key Features">
- Provides secure token burning functionality.
- Fully compliant with ERC-721 enumeration tracking.
- Supports diamond contract design for modularity.
</Callout>

## Usage Example

```solidity
pragma solidity ^0.8.0;

import {ERC721BurnFacet} from &quot;./ERC721BurnFacet.sol&quot;;

interface Diamond {
    function execute(address facet, bytes calldata data) external;
}

contract Example {
    Diamond public diamond;

    constructor(address _diamond) {
        diamond = Diamond(_diamond);
    }

    function burnToken(uint256 _tokenId) external {
        ERC721BurnFacet burnFacet = ERC721BurnFacet(address(diamond));
        diamond.execute(address(burnFacet), abi.encodeWithSelector(burnFacet.burn.selector, _tokenId));
    }
}
```



## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure correct integration with ERC-721 storage using proper storage slot identifiers.
- Verify state changes (e.g., token enumeration updates) after calls to `burn`.
- Use precise access control to avoid unauthorized token burns.
</Callout>



## Security Considerations

<Callout type="warning" title="Security">
Implement strict access control to prevent unauthorized token burns. Carefully test storage slot usage to avoid corruption. Protect against potential reentrancy attacks by ensuring no external calls are made mid-operation in the `burn` function.
</Callout>
