<Callout type="info" title="Key Features">
- Enables the permanent destruction of ERC721 tokens.
- Updates internal enumeration tracking to reflect token removal.
</Callout>

## Overview

The ERC721BurnFacet provides the functionality to destroy ERC721 tokens within a Compose diamond. It allows for the removal of tokens from the system, updating internal tracking mechanisms to reflect their destruction.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

import {IERC721BurnFacet} from "./interfaces/IERC721BurnFacet.sol";

contract ERC721BurnFacetExample {
    IERC721BurnFacet public immutable erc721BurnFacet;

    constructor(address _diamondAddress) {
        erc721BurnFacet = IERC721BurnFacet(_diamondAddress);
    }

    function burnToken(uint256 _tokenId) public {
        // Assuming caller is the owner of the token or has approval
        erc721BurnFacet.burn(_tokenId);
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure the &#x60;ERC721BurnFacet&#x60; is correctly initialized and its storage slot is properly configured within the diamond&#x27;s storage layout.
- The &#x60;burn&#x60; function should be protected by appropriate access control mechanisms, typically requiring the caller to be the owner of the token or have explicit approval.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
The &#x60;burn&#x60; function&#x27;s access control is critical. Ensure that only authorized addresses (token owner or approved addresses) can call this function to prevent unauthorized token destruction. Reentrancy is not a concern as the function does not make external calls.
</Callout>
