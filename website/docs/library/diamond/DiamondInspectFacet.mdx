<DocSubtitle>
Inspect diamond storage and facet mappings
</DocSubtitle>

<Callout type="info" title="Key Features">
- Directly accesses diamond storage via inline assembly.
- Provides a comprehensive mapping of function selectors to their respective facet addresses.
- Read-only access, ensuring state integrity during inspection.
</Callout>

## Overview

The DiamondInspectFacet provides essential read-only capabilities for understanding a Compose diamond's state and function distribution. It allows developers to query raw storage data and map function selectors to their implementing facets, crucial for debugging and runtime analysis.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

import {IDiamondInspect} from "@compose/diamond-contracts/src/facets/DiamondInspectFacet.sol";

contract Consumer {
    address immutable diamondAddress;

    constructor(address _diamondAddress) {
        diamondAddress = _diamondAddress;
    }

    function inspectStorage() external view returns (bytes memory) {
        IDiamondInspect diamondInspect = IDiamondInspect(diamondAddress);
        return diamondInspect.getStorage();
    }

    function inspectFunctionFacets() external view returns (struct IDiamondInspect.FunctionFacetPair[] memory) {
        IDiamondInspect diamondInspect = IDiamondInspect(diamondAddress);
        return diamondInspect.functionFacetPairs();
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Integrate `DiamondInspectFacet` into your diamond to enable runtime introspection and debugging.
- Use `getStorage` cautiously, as direct storage manipulation is not recommended; it's primarily for inspection.
- Leverage `functionFacetPairs` to verify function routing and understand facet responsibilities.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
The `getStorage` function retrieves raw storage and should be used for inspection purposes only. Direct manipulation of retrieved storage is not supported and could lead to state corruption. The `functionFacetPairs` function is inherently safe as it only reads mappings.
</Callout>
