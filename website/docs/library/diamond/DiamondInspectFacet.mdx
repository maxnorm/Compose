<DocSubtitle>
Inspect diamond storage and facet mappings
</DocSubtitle>

<Callout type="info" title="Key Features">
- Exposes external view function `functionFacetPairs()` for diamond introspection.
- Provides internal access to diamond storage via `getStorage()`.
- Facilitates understanding of diamond facet distribution.
- Compliant with ERC-2535 diamond standard.
</Callout>

## Overview

This facet provides external view functions to inspect the diamond's storage layout and its function-to-facet mappings. It routes calls through the diamond proxy, enabling developers to query deployed facets and their associated functions without direct storage access. This facilitates understanding and debugging of diamond implementations.

## Functions

### functionFacetPairs

Returns an array of all function selectors and their corresponding facet addresses. Iterates through the diamond's stored selectors and pairs each with its facet.

## Best Practices

<Callout type="tip" title="Best Practice">
- Use the `functionFacetPairs()` function to map function selectors to their respective facet addresses within the diamond.
- Understand that `getStorage()` is an internal function and not directly callable through the diamond proxy; use it within other facets or for internal diamond logic.
- Integrate this facet to provide an introspection layer for deployed diamond functions.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
All exposed functions are view functions and do not modify state, thus posing no reentrancy risk. Input validation is handled internally by the diamond proxy mechanism for function routing. Follow standard Solidity security practices for interacting with diamond contracts.
</Callout>
