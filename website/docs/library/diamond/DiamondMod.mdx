## Overview

This module provides internal functions for managing diamond facets and routing calls. Facets can import this module to add new functions or find the correct facet for an incoming call via `diamondFallback`. It utilizes the diamond storage pattern to maintain a registry of functions and their associated facets.

## Best Practices

<Callout type="tip" title="Best Practice">
- Use `addFacets` only during initial diamond deployment to prevent conflicts.
- Verify that the correct facet address and function selectors are passed to `addFacets`.
- Rely on the diamond proxy contract for the `diamondFallback` mechanism, rather than replicating it within individual facets.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
This module interacts with diamond storage at `DIAMOND_STORAGE_POSITION`, identified by `keccak256(\"compose.diamond\")`. It uses the `DiamondStorage` struct to map function selectors to facet addresses. Changes made via `addFacets` are immediately reflected in the diamond's routing logic.
</Callout>
