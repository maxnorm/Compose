Core diamond proxy functionality and storage management
</DocSubtitle>

<Callout type="info" title="Key Features">
- Manages facet registration and lifecycle within the diamond pattern.
- Provides internal functions for diamond storage interaction.
- Includes fallback mechanism for unassigned function selectors.
- Supports diamond upgradeability through facet replacement.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

This module provides core diamond proxy logic, including facet management and fallback handling. It exposes internal functions for interacting with the diamond storage pattern, enabling facets to be added, removed, or replaced. Changes to facets and their selectors are managed through this module, ensuring consistent state across all facets.

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure facet addition and removal operations are properly authorized.
- Verify that function selectors are correctly encoded and mapped to facets.
- Handle potential errors during facet operations, such as `FunctionNotFound` or `CannotAddFunctionToDiamondThatAlreadyExists`.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
This module utilizes diamond storage at the `DIAMOND_STORAGE_POSITION` (keccak256("erc8153.diamond")). It manages facet information within the `DiamondStorage` struct, specifically using the `facetList` field. All functions are internal, designed for direct use by diamond proxy logic or custom facets interacting with the diamond's core. Changes to `facetList` are immediately reflected across all facets that access this storage.
</Callout>
