<DocSubtitle>
Manage diamond facets and functions
</DocSubtitle>

<Callout type="info" title="Key Features">
- Programmatic addition, replacement, and removal of facets.
- Supports adding/replacing/removing multiple functions in a single transaction.
- Enables arbitrary `delegatecall` execution for facet initialization or complex upgrade logic.
</Callout>

## Overview

The DiamondCutFacet allows for the programmatic management of facets and functions within a Compose diamond. It enables adding, replacing, and removing functions, as well as executing arbitrary `delegatecall` operations during upgrades. This facet is crucial for diamond upgrades and dynamic facet composition.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

import {DiamondCutFacet} from "@compose-protocol/diamond-contracts/contracts/facets/DiamondCutFacet.sol";
import {IDiamondCut} from "@compose-protocol/diamond-contracts/contracts/interfaces/IDiamondCut.sol";

contract Deployer {
    address immutable diamondAbi;

    constructor(address _diamondAbi) {
        diamondAbi = _diamondAbi;
    }

    function upgradeDiamond(address _newFacetAddress, bytes4[] memory _selectors) public {
        DiamondCutFacet diamondCutFacet = DiamondCutFacet(diamondAbi);

        // Example: Add a new facet and its functions
        IDiamondCut.FacetCut[] memory cut = new IDiamondCut.FacetCut[](1);
        cut[0] = IDiamondCut.FacetCut({
            facetAddress: _newFacetAddress,
            action: IDiamondCut.Action.Add,
            functionSelectors: _selectors
        });

        // You can also include replace or remove actions in the same cut array

        diamondCutFacet.diamondCut(cut, address(0), ""); // address(0) and "" for no init call
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure the `diamondCut` function is called by the diamond's owner or an authorized address.
- Carefully manage function selectors to avoid conflicts or unintended overwrites.
- Use the `diamondCut` function with an initialization call for deploying new facets that require setup.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
Access to `diamondCut` must be strictly controlled, typically to the diamond's owner, to prevent unauthorized modifications. Incorrectly managing function selectors can lead to broken functionality or security vulnerabilities. Ensure that any initialization function called via `diamondCut` is thoroughly audited, as it executes with the diamond's context.
</Callout>
