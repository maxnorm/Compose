<DocSubtitle>
Manage diamond facets and upgradeability
</DocSubtitle>

<Callout type="info" title="Key Features">
- Enables adding, replacing, and removing facets on a diamond.
- Supports optional initialization calls after a cut.
- Includes specific errors for upgrade-related failures.
- Integrates with the Compose diamond storage pattern.
</Callout>

## Overview

This facet provides core functionality for managing diamond facets and enabling diamond upgrades. It exposes external functions to add, replace, or remove facets, and optionally execute an initialization function via delegatecall. Developers integrate this facet to control the diamond's functionality surface and maintain upgradeability.

## Best Practices

<Callout type="tip" title="Best Practice">
- Initialize the diamond with the DiamondCutFacet and an owner.
- Enforce access control on the `diamondCut` function to prevent unauthorized upgrades.
- Carefully validate `_diamondCut` parameters to prevent unintended state changes or function removals.
- Use `InitializationFunctionReverted` error to catch initialization failures during `diamondCut`.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
The `diamondCut` function is highly sensitive and must be protected by strict access control, typically restricted to the diamond's owner. Ensure all facet addresses are verified and selectors are correctly provided to prevent unintended facet additions or replacements. Initialization calls should be thoroughly tested to prevent reentrancy issues or state corruption. All facet functions added or replaced should be audited for security.
</Callout>
