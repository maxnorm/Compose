<DocSubtitle>
Manage diamond facets and initialization
</DocSubtitle>

<Callout type="info" title="Key Features">
- External `diamondCut` function for adding, replacing, and removing facets.
- Supports optional initialization calls via delegatecall.
- Exposes internal functions to access diamond and owner storage.
- Includes custom errors for various cut operation failures.
</Callout>

## Overview

This facet provides functions to add, replace, and remove facets from a diamond, along with an initialization mechanism. It allows for diamond upgrades and configuration through the `diamondCut` function, ensuring controlled modifications to the diamond's functionality. Developers use this facet to orchestrate facet deployments and updates.

## Best Practices

<Callout type="tip" title="Best Practice">
- Initialize the diamond with facets and owner using `diamondCut` during deployment.
- Verify that facet addresses and function selectors are correct before executing `diamondCut`.
- Ensure the `init` address and `calldata` are valid for initialization calls.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
The `diamondCut` function is highly sensitive and should only be callable by authorized accounts, typically the diamond owner. Ensure input validation for `_diamondCut`, `_init`, and `_calldata` to prevent unexpected behavior or reentrancy. Access control for calling `diamondCut` is critical to prevent unauthorized modifications. Initialization function reverts will cause the `diamondCut` transaction to revert.
</Callout>
