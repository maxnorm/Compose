<DocSubtitle>
Manage diamond facet upgrades and metadata
</DocSubtitle>

<Callout type="info" title="Key Features">
- Atomically adds, replaces, and removes facets from a diamond.
- Supports delegate calls for post-upgrade initialization or state changes.
- Emits events for `FacetAdded`, `FacetReplaced`, and `FacetRemoved`.
- Provides `exportSelectors` for mechanism for discovering facet selectors.
</Callout>

## Overview

This facet provides core functionality for upgrading diamond contracts by adding, replacing, and removing facets. It exposes external functions for managing the diamond's facet set and includes event emissions for tracking these changes. Additionally, it supports delegate calls for initialization or state modification post-upgrade and allows for exporting selectors.

## Functions

### upgradeDiamond

Upgrade the diamond by adding, replacing, or removing facets. Facets are added first, then replaced, then removed. These events are emitted to record changes to facets: - `FacetAdded(address indexed _facet)` - `FacetReplaced(address indexed _oldFacet, address indexed _newFacet)` - `FacetRemoved(address indexed _facet)` If `_delegate` is non-zero, the diamond performs a `delegatecall` to `_delegate` using `_delegateCalldata`. The `DiamondDelegateCall` event is emitted. The `delegatecall` is done to alter a diamond's state or to initialize, modify, or remove state after an upgrade. However, if `_delegate` is zero, no `delegatecall` is made and no `DiamondDelegateCall` event is emitted. If _tag is non-zero or if _metadata.length &gt; 0 then the `DiamondMetadata` event is emitted.

<ExpandableCode language="solidity" maxLines={8}>

## Best Practices

<Callout type="tip" title="Best Practice">
- Initialize the diamond with the DiamondUpgradeFacet during deployment.
- Use `upgradeDiamond` to manage facet additions, replacements, and removals atomically.
- Utilize `DiamondMetadata` event for tracking upgrade tags and associated data.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
Input validation is crucial for `upgradeDiamond` parameters (`_addFacets`, `_replaceFacets`, `_removeFacets`). Ensure `_delegate` calls are carefully reviewed for reentrancy and state manipulation risks. Use `OwnerUnauthorizedAccount` to protect sensitive upgrade operations. `DelegateCallReverted` error indicates issues during delegate calls.
</Callout>
