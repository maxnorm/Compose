<DocSubtitle>
Upgrade diamond by adding, replacing, or removing facets
</DocSubtitle>

<Callout type="info" title="Key Features">
- Supports adding, replacing, and removing facets atomically.
- Emits `FacetAdded`, `FacetReplaced`, and `FacetRemoved` events for upgrade tracking.
- Allows optional `delegatecall` for initialization or state modification post-upgrade.
- Integrates with diamond storage via `DIAMOND_STORAGE_POSITION`.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

This module provides the core logic for upgrading a diamond by managing its facets. It exposes internal functions to add, replace, and remove facets, ensuring the diamond's functionality can be updated. Changes made through this module are immediately reflected across all facets interacting with the diamond's storage.

### upgradeDiamond

Upgrade the diamond by adding, replacing, or removing facets. Facets are added first, then replaced, then removed. These events are emitted to record changes to facets: - `FacetAdded(address indexed _facet)` - `FacetReplaced(address indexed _oldFacet, address indexed _newFacet)` - `FacetRemoved(address indexed _facet)` If `_delegate` is non-zero, the diamond performs a `delegatecall` to `_delegate` using `_delegateCalldata`. The `DiamondDelegateCall` event is emitted. The `delegatecall` is done to alter a diamond's state or to initialize, modify, or remove state after an upgrade. However, if `_delegate` is zero, no `delegatecall` is made and no `DiamondDelegateCall` event is emitted. If _tag is non-zero or if _metadata.length &gt; 0 then the `DiamondMetadata` event is emitted.

<ExpandableCode language="solidity" maxLines={8}>

bytes32 _tag,

## Best Practices

<Callout type="tip" title="Best Practice">
- Call `upgradeDiamond` only from authorized contexts to prevent unauthorized upgrades.
- Ensure facet addresses provided to `upgradeDiamond` are valid and contain the intended bytecode.
- Handle errors such as `CannotAddFunctionToDiamondThatAlreadyExists` and `FacetToReplaceDoesNotExist` when performing upgrades.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
This module interacts with diamond storage located at `DIAMOND_STORAGE_POSITION`. It manages the `facetList` within the `DiamondStorage` struct. Facet additions, replacements, and removals modify this list, making the changes immediately visible to all facets that read from the diamond's storage.
</Callout>
