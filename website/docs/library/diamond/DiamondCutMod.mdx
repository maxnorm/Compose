<DocSubtitle>
Manages diamond facet additions, replacements, and removals.
</DocSubtitle>

<Callout type="info" title="Key Features">
- Atomically adds, replaces, or removes multiple functions in a single transaction.
- Supports optional delegatecall to an initialization function after cut operations.
- Includes robust error handling for various invalid cut scenarios.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

The DiamondCutMod provides essential functions for managing the diamond's facets. It allows for atomic updates to the diamond's function mappings, enabling dynamic extension and modification of its capabilities. This module is crucial for implementing upgradeability patterns and ensuring the diamond's state remains consistent during facet operations.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

import {IDiamondCut} from "@compose/diamond-core/contracts/interfaces/IDiamondCut.sol";
import {DiamondCutMod} from "@compose/diamond-core/contracts/modules/DiamondCutMod.sol";

contract MyFacet {
    IDiamondCut internal diamondCutFacet = IDiamondCut(address(this)); // Example: Assuming facet is deployed at the diamond address

    function upgradeDiamond() external {
        // Example facet addresses and selectors
        address newFacetAddress = address(0x123);
        bytes4[] memory newSelectors = new bytes4[](1);
        newSelectors[0] = bytes4(keccak256("newFunction()"));

        address oldFacetAddress = address(0x456);
        bytes4[] memory oldSelectors = new bytes4[](1);
        oldSelectors[0] = bytes4(keccak256("oldFunction()"));

        // Replace old function with new one
        diamondCutFacet.diamondCut(
            IDiamondCut.FacetCut[] memory _facetCuts,
            address _init,
            bytes memory _calldata
        );
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Always use custom errors for revert conditions to improve clarity and gas efficiency.
- Ensure that any initialization function called during `diamondCut` is properly tested and reverts only on critical failures.
- Be mindful of potential reentrancy risks if the initialization function interacts with external contracts.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
The DiamondCutMod modifies the diamond's function selector-to-facet address mappings. These changes are immediately visible to all facets interacting with the diamond proxy. Facets should be designed to be stateless with respect to function ownership and rely on the diamond's registry for correct dispatch. Ensure that the order of operations in `diamondCut` is carefully considered, especially when replacing or removing functions that might be called during an initialization process.
</Callout>
