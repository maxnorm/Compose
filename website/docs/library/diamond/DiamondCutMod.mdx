<DocSubtitle>
Manage diamond facets and function registrations.
</DocSubtitle>

<Callout type="info" title="Key Features">
- Dynamically add, replace, or remove functions from the diamond proxy.
- Supports batch operations for multiple function and facet modifications in a single transaction.
- Includes an optional `delegatecall` execution for immediate initialization or setup after a cut operation.
- Employs custom errors for precise error reporting, enhancing clarity and gas efficiency.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

The DiamondCut module provides essential functionality for managing the diamond proxy's facets. It enables adding, replacing, and removing functions, thereby allowing for upgrades and extensions of the diamond's capabilities. This module is fundamental for dynamic and composable smart contract architectures.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

import {IDiamondCut} from "@compose-protocol/diamond/contracts/modules/diamondCut/IDiamondCut.sol";
import {Selectors} from "@compose-protocol/diamond/contracts/utils/Selectors.sol";

contract MyFacet {
    function myNewFunction() external pure {
        // implementation
    }
}

contract DiamondUpgrade {
    IDiamondCut internal diamondCutModule;

    function upgradeDiamond(address _diamondCutModuleAddress, address _newFacetAddress) external {
        diamondCutModule = IDiamondCut(_diamondCutModuleAddress);

        // Example: Add a new function
        Selectors.Storage memory selectors = Selectors.Storage(new bytes[](1));
        selectors.selectors[0] = Selectors.getSelector("myNewFunction()", "MyFacet.sol");

        // Note: In a real upgrade, you would typically use a deployment script
        // to deploy the facet and then pass its address here.
        // The facet address for adding is usually the address of the deployed facet.
        address[] memory facetAddresses = new address[](1);
        facetAddresses[0] = _newFacetAddress; // Address of the deployed MyFacet

        diamondCutModule.diamondCut(facetAddresses, _newFacetAddress, selectors, selectors, selectors, new bytes[](0));
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Use the `diamondCut` function with explicit selector arrays to manage facet additions, replacements, and removals. Ensure all actions are clearly defined to prevent unintended state changes.
- Always verify that the facet addresses provided for additions and replacements point to valid, deployed facet contracts. Use `NoBytecodeAtAddress` error as a safeguard.
- Be mindful of immutable functions. The `CannotRemoveImmutableFunction` and `CannotReplaceImmutableFunction` errors prevent accidental removal or replacement of critical, unchanging logic.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
The DiamondCut module directly interacts with the diamond proxy's function selector-to-facet address mapping. When functions are added, replaced, or removed, the module updates this mapping. Facets that call functions managed by this module will resolve to the correct implementation address based on the current mapping. Changes made by `diamondCut` are immediately visible to all subsequent calls to the diamond proxy. The module's storage is implicitly managed by the diamond's storage layout.
</Callout>
