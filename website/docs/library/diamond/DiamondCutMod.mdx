<DocSubtitle>
Add, replace, and remove diamond facets and functions
</DocSubtitle>

<Callout type="info" title="Key Features">
- Supports adding, replacing, and removing functions from diamond facets.
- Allows optional initialization execution via `delegatecall`.
- Utilizes diamond storage at `DIAMOND_STORAGE_POSITION` for facet management.
- Functions are `external` for direct interaction with the diamond proxy.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

This module provides core functionality for managing diamond facets and their associated functions. Facets import this module to perform diamond cut operations, enabling dynamic upgrades and composition of diamond logic. Changes are applied atomically and are immediately visible to all interacting facets.

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure the `diamondCut` function is called with valid `FacetCut` actions and selectors.
- Verify that the `_init` address and `_calldata` are correctly formatted for initialization functions.
- Handle errors such as `CannotAddFunctionToDiamondThatAlreadyExists` and `CannotRemoveImmutableFunction`.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
This module integrates with the diamond storage pattern by managing facet definitions and function selectors. It reads and writes to the `DiamondStorage` struct located at `DIAMOND_STORAGE_POSITION` (keccak256("compose.diamond")). All modifications made through `diamondCut` are atomic and immediately reflected in the diamond's operational state.
</Callout>
