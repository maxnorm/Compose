<DocSubtitle>
Add, replace, and remove diamond facets and logic
</DocSubtitle>

<Callout type="info" title="Key Features">
- Manages addition, replacement, and removal of facet functions.
- Supports an initialization function call during diamond cuts.
- Utilizes EIP-8042 diamond storage pattern.
- Functions are `external` to allow calls from the diamond proxy.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

This module provides core functionality for managing diamond facets, including adding, replacing, and removing functions. It enables dynamic upgrades and composition of logic within a diamond proxy. Changes made through this module are immediately visible to all facets interacting with the diamond storage.

## Best Practices

<Callout type="tip" title="Best Practice">
- Call `diamondCut` only from authorized addresses.
- Ensure new facets are correctly implemented and tested before adding.
- Verify that selectors are unique and actions are appropriate for the desired state change.
- Handle errors like `InitializationFunctionReverted` and `NoSelectorsProvidedForFacet`.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
This module interacts with diamond storage at the `DIAMOND_STORAGE_POSITION` using the `DiamondStorage` struct. All facet and function management operations directly modify this shared storage, making changes immediately visible to all facets that adhere to the diamond storage pattern. The `diamondCut` function is the primary interface for altering the diamond's facet registry.
</Callout>
