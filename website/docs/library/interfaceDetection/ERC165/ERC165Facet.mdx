Implements ERC-165 interface detection for diamonds
</DocSubtitle>

<Callout type="info" title="Key Features">
- Implements the ERC-165 standard for interface detection.
- Exposes the `supportsInterface` external view function.
- Uses internal assembly for direct storage access.
- Self-contained with no external dependencies or inheritance.
</Callout>

## Overview

This facet provides ERC-165 interface detection capabilities for a diamond proxy. It exposes the `supportsInterface` function, allowing external contracts to query if the diamond implements specific interfaces. The facet uses internal assembly to access its dedicated storage slot, ensuring efficient and isolated state management.

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure the `ERC165Facet` is added to the diamond during initialization.
- Calls to `supportsInterface` should be routed through the diamond proxy.
- Verify the `STORAGE_POSITION` for `ERC165Storage` does not conflict with other facets.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
The `supportsInterface` function is a view function and does not modify state, reducing reentrancy risks. Input validation is inherent to the `bytes4` type. Ensure that the diamond's access control mechanisms are properly configured if this facet were to be extended with state-changing functions in the future. Follow standard Solidity security practices.
</Callout>
