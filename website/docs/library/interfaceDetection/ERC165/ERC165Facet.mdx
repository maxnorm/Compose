ERC-165 interface detection for diamonds
</DocSubtitle>

<Callout type="info" title="Key Features">
- Implements ERC-165 standard for interface detection.
- Exposes `supportsInterface` function for external queries.
- Utilizes diamond storage pattern for upgradeability.
- Self-contained with no external dependencies.
</Callout>

## Overview

This facet implements ERC-165 interface detection within a Compose diamond. It exposes an external `supportsInterface` function, allowing external contracts to query the diamond's capabilities. This facet routes calls through the diamond proxy and accesses shared storage, ensuring upgradeability and composability.

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure this facet is added to the diamond during its initialization.
- Do not modify the storage position for ERC-165 storage.
- Verify that the diamond correctly routes `supportsInterface` calls to this facet.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
The `supportsInterface` function is view-only and does not modify state, posing minimal security risks. Input validation for `_interfaceId` is handled by the Solidity compiler. Follow standard Solidity security practices for diamond proxy interactions.
</Callout>
