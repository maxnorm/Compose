<Callout type="info" title="Key Features">
- Implements the ERC-165 standard for interface detection.
- Allows querying supported interfaces via the diamond proxy.
</Callout>

## Overview

The ERC165Facet provides standard interface detection capabilities for Compose diamonds. It allows external contracts to query which ERC-165 supported interfaces a diamond implements, enhancing composability and interoperability.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

import {DiamondProxy} from "@compose/diamond-proxy/DiamondProxy.sol";
import {IERC165Facet} from "@compose/erc165/interfaces/IERC165Facet.sol";

contract MyDiamond is DiamondProxy {
    // Facet addresses would be configured here during deployment
    address internal constant ERC165_FACET_ADDRESS = address(0x...);

    function supportsInterface(bytes4 _interfaceId) external view returns (bool) {
        // Delegate to the ERC165Facet
        return IERC165Facet(ERC165_FACET_ADDRESS).supportsInterface(_interfaceId);
    }

    // Other diamond functions...
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure the ERC165Facet is deployed and its address correctly registered within the diamond's facet registry.
- Call `supportsInterface` on the diamond proxy itself, which will route the call to this facet.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
This facet is read-only and does not modify state, making it inherently secure against reentrancy and state manipulation. Ensure the facet's address is correctly set to prevent routing to unintended logic.
</Callout>
