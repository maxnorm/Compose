## Overview

This module provides internal functions for registering and detecting supported ERC-165 interfaces within a diamond. Facets can import this module to manage interface support using shared diamond storage. Changes made through this module are immediately visible to all facets accessing the same storage.

## Best Practices

<Callout type="tip" title="Best Practice">
- Call `registerInterface` during diamond initialization to declare supported interfaces.
- Ensure `ERC165Mod.getStorage()` is correctly bound to its storage slot.
- Integrate interface detection logic into your diamond's core ERC-165 implementation.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
This module utilizes diamond storage at `STORAGE_POSITION` (derived from `keccak256(\"compose.erc165\")`) to store interface support. The `ERC165Storage` struct is bound to this position using inline assembly via `ERC165Mod.getStorage()`. Any facet that imports and correctly initializes this module will contribute to the diamond's overall interface detection capabilities.
</Callout>
