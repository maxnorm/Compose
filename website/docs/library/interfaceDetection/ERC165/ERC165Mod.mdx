<Callout type="info" title="Key Features">
- Provides internal functions for ERC-165 interface registration.
- Uses a dedicated diamond storage slot for interface support.
- Functions are `internal`, promoting composability within facets.
- No external dependencies, ensuring minimal on-chain footprint.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

This module provides internal functions and storage for ERC-165 interface detection. Facets can import this module to register supported interfaces during initialization. All registered interfaces are stored in a shared diamond storage slot, ensuring consistency across the diamond.

## Best Practices

<Callout type="tip" title="Best Practice">
- Call `ERC165Mod.registerInterface` during facet initialization to declare supported interfaces.
- Ensure the `ERC165Storage` struct is correctly positioned in the diamond's storage layout.
- Do not call `ERC165Mod.registerInterface` outside of an initialization context to maintain storage integrity.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
This module utilizes the diamond storage pattern, mapping the `ERC165Storage` struct to a specific storage slot identified by `keccak256("erc165")`. All facets that implement ERC-165 should import and use `ERC165Mod` to register their supported interfaces within this shared storage. Changes to registered interfaces are immediately visible to all facets interacting with the diamond's storage.
</Callout>
