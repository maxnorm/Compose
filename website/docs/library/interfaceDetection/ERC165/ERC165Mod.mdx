<DocSubtitle>
Detects supported ERC-165 interfaces.
</DocSubtitle>

<Callout type="info" title="Key Features">
- Implements ERC-165 standard for interface detection.
- Allows dynamic registration of supported interfaces.
- Provides a centralized mechanism for `supportsInterface` calls.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

This module provides the necessary storage and functions to implement the ERC-165 standard for interface detection. By registering supported interfaces during initialization, your diamond can reliably report which standards it adheres to, enhancing interoperability with other smart contracts and tools.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

import {LibERC165} from "@compose/contracts/src/modules/erc165/LibERC165.sol";
import {IERC165} from "@openzeppelin/contracts/utils/introspection/IERC165.sol";

contract MyFacet is IERC165 {
    /**
     * @notice Initializes the ERC165 module by registering supported interfaces.
     */
    function init() external {
        // Register ERC165 and any other interfaces this facet supports.
        LibERC165.registerInterface(type(IERC165).interfaceId);
        // Example: LibERC165.registerInterface(type(IERC721).interfaceId);
    }

    /**
     * @notice Checks if the contract implements the interface with \`interfaceId\`.
     * @param interfaceId The interface ID to check.
     * @return \`true\` if the contract implements the interface, \`false\` otherwise.
     */
    function supportsInterface(bytes4 interfaceId) external view override returns (bool) {
        return LibERC165.supportsInterface(interfaceId);
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Register all supported interfaces during facet initialization. Ensure the `supportsInterface` function correctly delegates to the ERC165 module.
- Avoid modifying the ERC165 storage layout to maintain compatibility.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
The `LibERC165` module utilizes a dedicated storage slot to maintain a mapping of supported interface IDs. Facets that implement `IERC165` should delegate their `supportsInterface` function to `LibERC165.supportsInterface(interfaceId)`. The `registerInterface` function must be called during the facet's initialization to populate the supported interfaces.
</Callout>
