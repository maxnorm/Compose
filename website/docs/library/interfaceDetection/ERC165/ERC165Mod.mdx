<DocSubtitle>
ERC-165 interface detection and registration.
</DocSubtitle>

<Callout type="info" title="Key Features">
- Registers ERC-165 supported interfaces at a fixed storage slot.
- Provides a public getter for interface support checks.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

This module provides the core logic for ERC-165 interface detection within a Compose diamond. It allows facets to register supported interfaces and enables other contracts to query them via the diamond proxy.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

import {LibERC165} from "@compose/modules/erc165/LibERC165.sol";
import {IERC721} from "@openzeppelin/contracts/token/ERC721/IERC721.sol";

contract MyERC721Facet {
    function initialize(address _diamondProxy) external {
        // Register ERC721 interface support
        LibERC165.registerInterface(type(IERC721).interfaceId);
    }

    // ... other ERC721 functions ...
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Call `registerInterface` during facet initialization to declare supported ERC-165 interfaces.
- Ensure the ERC165 storage slot is correctly managed by the diamond proxy.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
ERC165Mod uses a fixed storage slot for its `ERC165Storage` struct. Facets should call `LibERC165.registerInterface` during their initialization to add their supported interfaces to this storage. The diamond proxy will route calls to `supportsInterface` to this facet, which will internally use the registered interfaces.
</Callout>
