Manages ERC-20 token metadata within a diamond
</DocSubtitle>

<Callout type="info" title="Key Features">
- Provides internal functions for setting and retrieving ERC-20 metadata.
- Uses the diamond storage pattern at a dedicated slot (`keccak256(\"erc20.metadata\")`).
- Stores metadata in a shared `ERC20MetadataStorage` struct.
- Facet functions are `external` or `internal`, adhering to Compose standards.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

This module provides functions to set and retrieve ERC-20 token metadata, such as name, symbol, and decimals. It utilizes the diamond storage pattern to store this information in a predefined slot, ensuring consistency across all facets that interact with the ERC-20 token. This approach allows for composable metadata management within a single diamond proxy.

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure metadata is set only once, typically during diamond initialization.
- Access metadata using `getStorage()` and then accessing struct fields for clarity.
- Verify the `ERC20MetadataMod` instance is correctly initialized before use.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
This module stores ERC-20 metadata in a dedicated diamond storage slot, identified by `keccak256(\"erc20.metadata\")`. The `ERC20MetadataStorage` struct, containing `name`, `symbol`, and `decimals`, is directly accessible via the `getStorage()` function. Any facet with access to this module and the correct storage slot can read or modify this metadata. Changes made through `setMetadata` are immediately reflected for all interacting facets.
</Callout>
