Provides ERC-20 token name, symbol, and decimals
</DocSubtitle>

<Callout type="info" title="Key Features">
- Exposes external view functions for token metadata retrieval.
- Accesses metadata directly from diamond storage.
- Self-contained, adhering to facet composition principles.
- Provides standard ERC-20 metadata interface.
</Callout>

## Overview

This facet implements ERC-20 metadata functions, exposing the token's name, symbol, and decimal precision through the diamond proxy. It accesses this information directly from the diamond's shared storage. Developers integrate this facet to provide standard ERC-20 metadata for their tokens within a Compose diamond.

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure the `ERC20MetadataFacet` is added to the diamond during deployment.
- Verify that the `name`, `symbol`, and `decimals` are correctly set in the diamond's storage slot.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
This facet only exposes view functions that read from storage. It does not perform any state changes. Follow standard Solidity security practices for diamond proxy interactions.
</Callout>
