<DocSubtitle>
Approves token spending on behalf of a token owner
</DocSubtitle>

<Callout type="info" title="Key Features">
- Exposes the ERC-20 `approve` function for external calls.
- Utilizes diamond storage for managing token approval state.
- Exports its selectors for diamond registration.
- Emits an `Approval` event upon successful approval.
</Callout>

## Overview

This facet implements the ERC-20 approve functionality within a diamond. It provides an external interface for users to grant permission to other addresses (spenders) to withdraw tokens from their account. Calls are routed through the diamond proxy and interact with shared token storage.

## Best Practices

<Callout type="tip" title="Best Practice">
- Initialize the `ERC20ApproveFacet` with the correct storage slot during diamond deployment.
- Ensure that access control is enforced on functions that modify token approvals if required by your diamond's architecture.
- Verify that the `approve` function is correctly dispatched by the diamond proxy to this facet.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
The `approve` function allows a token owner to grant spending permission to a spender. It is crucial to ensure that the `spender` address is validated before granting approval to prevent unexpected token transfers. The `ERC20InvalidSpender` error is emitted if the spender address is invalid. Follow standard Solidity security practices for input validation and access control.
</Callout>
