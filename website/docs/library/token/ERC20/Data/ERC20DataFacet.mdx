<DocSubtitle>
ERC-20 token data within a diamond
</DocSubtitle>

<Callout type="info" title="Key Features">
- Exposes external view functions for ERC-20 data queries.
- Accesses token data via diamond storage.
- Provides `exportSelectors` for diamond discovery.
- Self-contained with no external dependencies.
</Callout>

## Overview

This facet provides read-only access to ERC-20 token data, such as total supply, balances, and allowances, through a diamond proxy. It leverages the diamond storage pattern to access shared state, enabling composability and upgradeability for token functionalities within a diamond.

## Best Practices

<Callout type="tip" title="Best Practice">
- Initialize the diamond with this facet to expose ERC-20 data functions.
- Access token data by calling functions through the diamond proxy address.
- Ensure this facet is compatible with other token-related facets in the diamond.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
This facet only exposes view functions and does not modify state. Follow standard Solidity security practices for contract interactions. Ensure proper input validation when interacting with the diamond proxy.
</Callout>
