<DocSubtitle>
Internal functions for burning ERC-20 tokens
</DocSubtitle>

<Callout type="info" title="Key Features">
- Provides internal functions (`burn`) for token burning operations.
- Utilizes diamond storage (EIP-8042) for shared token state.
- Emits `Transfer` events upon successful burns.
- Reverts with `ERC20InsufficientBalance` or `ERC20InvalidSender` errors for specific failure conditions.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

This module provides internal functions for burning ERC-20 tokens, reducing both the total supply and an account's balance. Facets can import this module to integrate burning functionality while leveraging shared diamond storage for token state. Changes made via this module are immediately reflected across all facets interacting with the same storage.

## Functions

### burn

Burns tokens from a specified address. Decreases both total supply and the sender's balance. This module does not perform allowance checks. Ensure proper allowance or authorization validation before calling this function.

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure caller authorization and sufficient balance checks are performed within the facet before calling `burn()`.
- Handle the `ERC20InsufficientBalance` and `ERC20InvalidSender` errors that may be emitted by the module.
- Verify that the `ERC20Storage` struct layout remains compatible when upgrading facets that use this module.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
This module interacts with diamond storage at the `STORAGE_POSITION` defined by `keccak256(\"erc20\")`. The `getStorage()` function provides inline assembly access to the `ERC20Storage` struct. Any modifications to `totalSupply` made through this module are immediately visible to all facets that access the same storage slot.
</Callout>
