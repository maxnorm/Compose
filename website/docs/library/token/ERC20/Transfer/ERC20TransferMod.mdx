<DocSubtitle>
Internal ERC-20 token transfer functions
</DocSubtitle>

<Callout type="info" title="Key Features">
- All functions are `internal` for use within custom facets.
- Directly manipulates balances in diamond storage.
- Implements core ERC-20 transfer logic without allowance checks in `transfer`.
- Utilizes diamond storage pattern for shared state.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

This module exposes internal functions for performing ERC-20 token transfers and transfers from other accounts. Facets import this module to manage token balances and execute transfers using shared diamond storage. The functions directly update balances, bypassing the allowance mechanism for direct transfers.

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure proper initialization of the module within the diamond deployment flow.
- Call `transfer` for direct sender-to-receiver transfers and `transferFrom` when an allowance is involved.
- Handle custom errors like `ERC20InsufficientBalance` and `ERC20InvalidReceiver` returned by the module functions.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
This module uses the diamond storage pattern, accessing state at the `STORAGE_POSITION` defined by `keccak256("erc20")`. The `ERC20Storage` struct, containing fields like `totalSupply`, is managed here. Functions operate directly on this shared storage, making any changes immediately visible to all facets that access the same storage slot.
</Callout>
