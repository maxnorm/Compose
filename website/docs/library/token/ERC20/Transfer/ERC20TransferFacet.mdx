<DocSubtitle>
ERC-20 token transfers within a diamond
</DocSubtitle>

<Callout type="info" title="Key Features">
- Exposes external functions for diamond routing.
- Implements ERC-20 transfer and transferFrom logic.
- Emits Transfer events on successful transfers.
- Exports its own selectors for diamond discovery.
</Callout>

## Overview

This facet implements ERC-20 token transfers as external functions in a diamond. It routes calls through the diamond proxy and accesses shared storage. Developers add this facet to expose token functionality while maintaining upgradeability.

## Best Practices

<Callout type="tip" title="Best Practice">
- Initialize ERC20Storage during diamond setup.
- Enforce access control on functions that modify token balances or allowances.
- Verify storage compatibility before upgrading to new facet versions.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
Functions `transfer` and `transferFrom` validate sender, receiver, and balance/allowance before state changes. Reentrancy is mitigated by the checks-effects-interactions pattern. Input validation for addresses and values is performed. Follow standard Solidity security practices.
</Callout>
