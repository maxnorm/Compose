Internal functions for minting ERC-20 tokens

<Callout type="info" title="Key Features">
- All functions are `internal`, designed for composition within facets.
- Uses the diamond storage pattern for shared state management.
- Emits a `Transfer` event upon successful minting.
- No external dependencies or `using` directives.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

This module provides internal functions for minting new ERC-20 tokens. Facets can import and use these functions to increase token supply and recipient balances, leveraging the shared diamond storage pattern. Changes to token supply and balances are immediately visible across all facets interacting with the same storage.

## Best Practices

<Callout type="tip" title="Best Practice">
- Call `mint` only after verifying caller permissions within your facet.
- Ensure the `_account` address is valid and not a zero address before calling `mint`.
- Handle potential `ERC20InvalidReceiver` errors if the module implementation evolves to include receiver validation.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
This module interacts with diamond storage at the `STORAGE_POSITION` (keccak256("erc20")) to manage ERC-20 token state. The `mint` function directly modifies the `totalSupply` and the recipient's balance within this shared storage. The `getStorage()` function provides access to the `ERC20Storage` struct, which is currently defined with no fields. Any facet that accesses this storage position will see changes made by this module immediately.
</Callout>
