<DocSubtitle>
Cross-chain ERC-20 token minting and burning
</DocSubtitle>

<Callout type="info" title="Key Features">
- Enables cross-chain minting and burning of ERC-20 tokens.
- Enforces access control via the `trusted-bridge` role.
- Integrates seamlessly with the diamond storage pattern.
- Functions are external, accessible via the diamond proxy.
</Callout>

## Overview

This facet enables cross-chain ERC-20 token minting and burning operations within a diamond. It exposes functions to mint tokens to specific accounts and burn tokens from specified addresses, requiring the caller to possess the `trusted-bridge` role. This facet routes calls through the diamond proxy, interacting with shared diamond storage for token management.

## Best Practices

<Callout type="tip" title="Best Practice">
- Initialize the `trusted-bridge` role for authorized bridge contracts during diamond setup.
- Ensure the `ERC20BridgeableFacet` is correctly added to the diamond's facet registry.
- Verify that the diamond storage slot for ERC20 data is correctly configured.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
Cross-chain operations require strict validation of the `trusted-bridge` role. The `crosschainMint` and `crosschainBurn` functions revert if the caller does not possess this role. Input validation is performed on recipient and sender addresses. Follow standard Solidity security practices for input validation and reentrancy.
</Callout>
