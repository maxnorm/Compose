<DocSubtitle>
Manage cross-chain ERC20 token transfers and burns.
</DocSubtitle>

<Callout type="info" title="Key Features">
- Enables secure cross-chain token minting and burning operations.
- Leverages AccessControl for role-based authorization of bridge operations.
- Provides internal checks for trusted bridge addresses.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

This module provides functionality for cross-chain token operations, enabling secure minting and burning of ERC20 tokens across different chains. It relies on an AccessControl facet to manage trusted bridge addresses, ensuring only authorized entities can initiate cross-chain transfers.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

import {IERC20BridgeableMod} from "@compose/modules/erc20-bridgeable/IERC20BridgeableMod.sol";
import {AccessControlStorage} from "@compose/modules/access-control/AccessControlStorage.sol";

contract ERC20BridgeableFacet {
    IERC20BridgeableMod private immutable erc20BridgeableMod;
    AccessControlStorage private immutable accessControlStorage;

    constructor(address _diamondAddress) {
        erc20BridgeableMod = IERC20BridgeableMod(_diamondAddress);
        accessControlStorage = AccessControlStorage(_diamondAddress);
    }

    /**
     * @notice Mints tokens on a remote chain.
     * @param _to The recipient address on the remote chain.
     * @param _amount The amount of tokens to mint.
     */
    function crosschainMint(address _to, uint256 _amount) external {
        // Ensure the caller has the 'trusted-bridge' role.
        accessControlStorage.checkRole(AccessControlStorage.Roles.TRUSTED_BRIDGE);
        erc20BridgeableMod.crosschainMint(_to, _amount);
    }

    /**
     * @notice Burns tokens on the current chain to be minted on a remote chain.
     * @param _to The recipient address on the remote chain.
     * @param _amount The amount of tokens to burn.
     */
    function crosschainBurn(address _to, uint256 _amount) external {
        // Ensure the caller has the 'trusted-bridge' role.
        accessControlStorage.checkRole(AccessControlStorage.Roles.TRUSTED_BRIDGE);
        erc20BridgeableMod.crosschainBurn(_to, _amount);
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure the `trusted-bridge` role is correctly managed by the AccessControl facet to restrict cross-chain operations to authorized entities.
- Handle `ERC20InvalidCallerAddress`, `ERC20InvalidReciever`, and `ERC20InvalidSender` errors to gracefully manage invalid addresses during cross-chain operations.
- Be aware of potential gas implications for cross-chain transactions and ensure sufficient gas is provided by the calling context.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
This module interacts with the diamond's storage to manage ERC20 token balances and access control roles. The `getERC20Storage` and `getAccessControlStorage` functions provide direct access to these storage areas. Ensure that the AccessControl facet is deployed and configured with the `trusted-bridge` role before utilizing the cross-chain functions. The `checkTokenBridge` internal function relies on the `trusted-bridge` role being set in the AccessControl storage.
</Callout>
