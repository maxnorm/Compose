<Callout type="info" title="Key Features">
- Provides `internal` functions for cross-chain minting and burning.
- Enforces `trusted-bridge` role access control for sensitive operations.
- Interacts with shared diamond storage for ERC20 state.
- No external dependencies, ensuring composability.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

This module provides internal functions for cross-chain ERC20 token transfers. Facets can import this module to perform minting or burning operations on behalf of trusted bridge accounts. It enforces access control to ensure only authorized bridge addresses can execute these operations, interacting with shared diamond storage for state management.

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure the caller has the `trusted-bridge` role before executing `crosschainMint` or `crosschainBurn`.
- Verify the `_account` parameter in `crosschainMint` and `_from` parameter in `crosschainBurn` are valid addresses.
- Handle potential reverts from `crosschainMint` and `crosschainBurn` due to invalid bridge accounts or insufficient balance.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
This module utilizes the diamond storage pattern, reading and writing to the `ERC20_STORAGE_POSITION` slot, identified by `keccak256("compose.erc20")`. Functions like `crosschainMint` and `crosschainBurn` modify the ERC20 state directly in this shared storage, making changes immediately visible to all facets accessing the same storage slot.
</Callout>
