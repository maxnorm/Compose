<DocSubtitle>
Internal functions for cross-chain token bridging
</DocSubtitle>

<Callout type="info" title="Key Features">
- Provides `internal` functions for cross-chain minting and burning.
- Enforces access control using the `trusted-bridge` role.
- Utilizes diamond storage for `ERC20Storage` and `AccessControlStorage`.
- Compatible with ERC-2535 diamonds.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

This module provides internal functions for cross-chain token bridging, enabling secure minting and burning operations across different chains. Facets using this module can interact with trusted bridge addresses, leveraging diamond storage for shared state management. This ensures consistent and auditable cross-chain operations within the diamond.

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure the `trusted-bridge` role is correctly managed via AccessControl.
- Call `checkTokenBridge` before executing `crosschainMint` or `crosschainBurn` if external authorization is required.
- Verify that the `ERC20_STORAGE_POSITION` is correctly configured in the diamond registry.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
This module reads and writes to diamond storage at the `ERC20_STORAGE_POSITION`, identified by `keccak256(\"erc20\")`. It uses the `ERC20Storage` struct for token-related state and `AccessControlStorage` for role management. Functions like `crosschainMint` and `crosschainBurn` are internal and rely on the caller possessing the `trusted-bridge` role, managed via the access control system.
</Callout>
