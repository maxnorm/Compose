<DocSubtitle>
Cross-chain token minting and burning for diamonds
</DocSubtitle>

<Callout type="info" title="Key Features">
- Enables cross-chain minting and burning operations.
- Enforces `trusted-bridge` role for authorized bridge addresses.
- Exports selectors for diamond discovery.
- Follows EIP-2535 diamond standard for composability.
</Callout>

## Overview

This facet enables cross-chain token minting and burning operations within a diamond proxy. It enforces access control for trusted bridge addresses and exposes external functions for these operations. Integrate this facet to manage token supply across different chains via a secure, upgradeable diamond.

## Best Practices

<Callout type="tip" title="Best Practice">
- Initialize the `trusted-bridge` role with the appropriate address during diamond setup.
- Ensure that only trusted bridge addresses can call `crosschainMint` and `crosschainBurn` functions.
- Verify that the `checkTokenBridge` function is correctly implemented and accessible for bridge verification.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
Cross-chain operations are sensitive. The `crosschainMint` and `crosschainBurn` functions are restricted to addresses holding the `trusted-bridge` role, enforced via access control. The `checkTokenBridge` function validates the caller's role, reverting with `ERC20InvalidBridgeAccount` or `AccessControlUnauthorizedAccount` if unauthorized. Input validation for `_account` and `_from` addresses is crucial; revert with `ERC20InvalidReceiver` or `ERC20InvalidSender` if invalid. Ensure the `trusted-bridge` role is managed securely.
</Callout>
