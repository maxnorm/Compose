<DocSubtitle>
Burns ERC-20 tokens from caller or another account
</DocSubtitle>

<Callout type="info" title="Key Features">
- Exposes external functions for diamond routing.
- Enables burning tokens from caller or a specified account.
- Emits `Transfer` event on successful burns.
- Self-contained with no external dependencies.
</Callout>

## Overview

This facet implements ERC-20 token burning functionality within a diamond. It exposes `burn` and `burnFrom` functions, routing calls through the diamond proxy and accessing shared token storage. Developers add this facet to enable token destruction while maintaining upgradeability and composability.

## Functions

### burn

Burns (destroys) a specific amount of tokens from the caller's balance. Emits a Transfer event to the zero address.

Burns tokens from another account, deducting from the caller's allowance. Emits a Transfer event to the zero address.

## Best Practices

<Callout type="tip" title="Best Practice">
- Initialize the ERC20 storage struct when deploying the diamond.
- Ensure the caller has sufficient balance for `burn` or allowance for `burnFrom`.
- Verify the `Transfer` event is emitted with the zero address as the recipient.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
The `burn` function requires the caller to have sufficient token balance. The `burnFrom` function requires the caller to have sufficient allowance from the specified account. Both functions revert if these conditions are not met, preventing insufficient balance or allowance issues. Follow standard Solidity security practices for input validation and access control if additional layers are implemented.
</Callout>
