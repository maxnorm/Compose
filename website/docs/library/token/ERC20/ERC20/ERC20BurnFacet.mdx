<DocSubtitle>
Burns ERC-20 tokens from caller or other accounts
</DocSubtitle>

<Callout type="info" title="Key Features">
- External functions for burning tokens via diamond routing.
- Supports burning from caller's balance (`burn`).
- Supports burning from another account's balance with allowance (`burnFrom`).
- Emits `Transfer` events when tokens are burned.
</Callout>

## Overview

This facet implements functions to burn ERC-20 tokens within a diamond. It allows callers to destroy tokens from their own balance or from another account's balance, provided they have the necessary allowance. This facet integrates with the diamond's shared storage and routing mechanisms.

## Best Practices

<Callout type="tip" title="Best Practice">
- Initialize the diamond with this facet to enable burning functionality.
- Ensure necessary allowances are set before calling `burnFrom`.
- Validate input amounts to prevent unexpected token destruction.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
The `burn` function requires sufficient balance from the caller. The `burnFrom` function requires sufficient allowance from the token owner to the caller. Both functions emit `Transfer` events to the zero address upon successful execution. Follow standard Solidity security practices for input validation.
</Callout>
