<DocSubtitle>
Implements ERC-20 token functionality within a diamond.
</DocSubtitle>

<Callout type="info" title="Key Features">
- Exposes external functions for standard ERC-20 operations.
- Leverages the diamond storage pattern for state management.
- Self-contained, adhering to Compose facet design principles.
- Enables upgradeability of token logic without changing the diamond address.
</Callout>

## Overview

This facet implements core ERC-20 token functionality, including transfers, approvals, and balance queries, as external functions within a diamond. It routes these calls through the diamond proxy and accesses shared storage. Developers add this facet to expose standard token operations while preserving the diamond's upgradeability and modularity.

## Best Practices

<Callout type="tip" title="Best Practice">
- Initialize token name, symbol, and decimals during diamond setup.
- Enforce access control on functions that change token supply if necessary.
- Verify storage compatibility when upgrading the ERC20Facet to prevent state corruption.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
The `transfer` and `transferFrom` functions implement checks-effects-interactions pattern to mitigate reentrancy. Input validation is performed for recipient addresses and allowance checks. Errors are explicitly defined for insufficient balance, allowance, and invalid addresses. Follow standard Solidity security practices for all external calls.
</Callout>
