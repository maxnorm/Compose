<DocSubtitle>
Internal functions for ERC-20 token operations
</DocSubtitle>

<Callout type="info" title="Key Features">
- Provides `internal` functions for core ERC-20 operations.
- Manages token state using the diamond storage pattern at a defined slot.
- Emits `Transfer` and `Approval` events upon relevant state changes.
- Includes custom errors for common ERC-20 failure conditions.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

This module exposes internal functions for core ERC-20 token operations such as minting, burning, transferring, and approving allowances. Facets can import this module to implement ERC-20 functionality within a diamond, leveraging shared diamond storage for token state. Changes to token supply and balances are managed atomically.

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure the `ERC20Mod` instance correctly points to the diamond's storage slot for ERC-20 state.
- Handle specific `ERC20` errors such as `ERC20InsufficientBalance` and `ERC20InsufficientAllowance` when calling related functions.
- Implement access control within your facet to restrict sensitive operations like minting or burning if necessary.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
This module interacts with diamond storage via a designated storage position (`STORAGE_POSITION` which is `keccak256("compose.erc20")`). It reads and writes to the `ERC20Storage` struct, which contains fields like `totalSupply`, `decimals`, `name`, and `symbol`. All state changes are directly applied to this shared storage, making them immediately visible to any facet that accesses the same storage position.
</Callout>
