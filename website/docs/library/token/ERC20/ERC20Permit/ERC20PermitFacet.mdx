Implements EIP-2612 permit functionality for ERC-20 tokens
</DocSubtitle>

<Callout type="info" title="Key Features">
- Implements EIP-2612 permit functionality.
- Exposes `permit`, `DOMAIN_SEPARATOR`, and `nonces` functions.
- Facilitates gasless token approvals via signed messages.
- Designed for integration with the diamond storage pattern.
</Callout>

## Overview

This facet implements EIP-2612 permit functionality, allowing token owners to grant allowances via signed messages. It exposes the `permit` function to the diamond proxy, enabling off-chain authorization for token transfers. Developers integrate this facet to provide gasless approval mechanisms for ERC-20 tokens within a Compose diamond.

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure the `permit` function is called through the diamond proxy.
- Verify the `DOMAIN_SEPARATOR` and `nonces` are correctly used when generating signatures off-chain.
- Integrate with the ERC20PermitMod for internal logic and state management.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
Follow standard Solidity security practices. Validate all input parameters for the `permit` function. Ensure correct signature verification logic is implemented off-chain to prevent invalid permits from being processed. The `nonces` function helps mitigate replay attacks by ensuring each permit can only be used once.
</Callout>
