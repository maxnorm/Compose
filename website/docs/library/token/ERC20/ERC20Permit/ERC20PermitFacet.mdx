<DocSubtitle>
Implements EIP-2612 permit functionality for ERC-20 tokens
</DocSubtitle>

<Callout type="info" title="Key Features">
- Implements EIP-2612 permit functionality for ERC-20 tokens.
- Exposes `nonces` and `DOMAIN_SEPARATOR` for signature generation.
- Utilizes diamond storage for state management.
- Functions are callable via the diamond proxy.
</Callout>

## Overview

This facet enables EIP-2612 permit functionality within a Compose diamond, allowing token owners to grant allowances via signed messages. It exposes functions to retrieve nonces and the domain separator, facilitating the `permit` function. Developers integrate this facet to provide gasless token approvals, enhancing user experience.

## Best Practices

<Callout type="tip" title="Best Practice">
- Integrate this facet during diamond initialization to set up EIP-2612 permit capabilities.
- Ensure the `permit` function is called with valid signatures and parameters to prevent signature replay or invalid allowance setting.
- Verify that the `DOMAIN_SEPARATOR` and `nonces` are correctly managed and accessible by users for generating valid permit signatures.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
The `permit` function implements EIP-2612 and relies on the validity of the provided signature (`_v`, `_r`, `_s`). Ensure that signature validation logic is robust and that nonces are correctly managed to prevent replay attacks. Use the `ERC2612InvalidSignature` error for signature validation failures. Follow standard Solidity security practices for input validation.
</Callout>
