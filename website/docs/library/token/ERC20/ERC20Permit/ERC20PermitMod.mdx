<DocSubtitle>
ERC-2612 Permit logic and domain separator
</DocSubtitle>

<Callout type="info" title="Key Features">
- Implements ERC-2612 `permit` function for gasless approvals.
- Generates and returns the domain separator for signature EIP-712 hashing.
- Centralizes permit-related storage and logic for reusability.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

This module provides a self-contained implementation for ERC-2612 permit functionality, including the domain separator and signature validation. By abstracting this logic, facets can easily integrate EIP-2612 support without duplicating complex cryptographic operations or storage management.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

import {IERC20PermitMod} from "@compose/modules/ERC20Permit/IERC20PermitMod.sol";

contract MyERC20Facet {
    IERC20PermitMod private immutable _erc20PermitMod;

    constructor(address erc20PermitModAddress) {
        _erc20PermitMod = IERC20PermitMod(erc20PermitModAddress);
    }

    /**
     * @notice Allows a user to permit an allowance via signature.
     * @dev This function acts as a wrapper to the ERC20PermitMod.
     */
    function permit(address owner, address spender, uint256 value, uint256 deadline, uint8 v, bytes32 r, bytes32 s) external {
        // The ERC20PermitMod.permit function emits the Approval event.
        _erc20PermitMod.permit(owner, spender, value, deadline, v, r, s);
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure the `Approval` event is emitted by the calling facet when `permit` is successful, as required by the ERC-2612 standard.
- Validate the `deadline` parameter to prevent permits from expiring prematurely or remaining valid indefinitely.
- Handle `ERC2612InvalidSignature` and `ERC20InvalidSpender` errors appropriately within your facet's logic.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
This module manages its own storage for permit-related data, including the domain separator. Facets integrating this module will call its `permit` function. The `permit` function itself emits the `Approval` event. Ensure that the diamond's storage layout accommodates the storage required by this module, and that the module is correctly initialized within the diamond's deployment process. The `DOMAIN_SEPARATOR` function should be called by the facet to obtain the correct domain separator for signing.
</Callout>
