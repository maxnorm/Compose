<Callout type="info" title="Key Features">
- Internal functions for tight integration with facets.

- Compatible with the diamond storage pattern (EIP-8042).
- No external dependencies, promoting composability.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

This module provides internal functions for ERC-2612 permit validation and domain separator generation. Facets can import this module to handle permit signatures, ensuring replay protection via a unique domain separator. It leverages diamond storage for a self-contained implementation.

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure the diamond's storage layout is compatible with `ERC20PermitStorage`.
- Call `permit` within a facet that correctly handles allowance updates and event emissions.
- Verify signature validity using the returned values from `permit` before proceeding with state changes.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
This module interacts with diamond storage via the `ERC20_STORAGE_POSITION` constant, which maps to `keccak256(\"compose.erc20\")`. It utilizes the `ERC20PermitStorage` struct. Any facet importing this module can access and modify the permit-related state through the diamond's storage mechanism. Changes made are immediately visible to all facets sharing the same storage.
</Callout>
