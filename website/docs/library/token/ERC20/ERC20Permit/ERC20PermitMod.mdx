<Callout type="info" title="Key Features">
- Provides internal functions for ERC-2612 permit validation and allowance setting.
- Includes `DOMAIN_SEPARATOR` for secure off-chain signature generation.
- Emits `Approval` event upon successful permit validation, to be handled by the calling facet.
- Integrates with diamond storage for efficient data access.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

This module provides internal functions for handling ERC-2612 permit signatures and domain separators. Facets can import this module to validate permit signatures and set allowances directly within the diamond. It leverages shared diamond storage for efficient access to permit-related data and ensures signature validity against the correct domain.

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure the `ERC20PermitStorage` and `ERC20Storage` structs are properly initialized in diamond storage before calling module functions.
- Verify the returned `domainSeparator` matches the expected value before signing permits off-chain.
- Handle the `ERC2612InvalidSignature` error to gracefully manage invalid permit submissions.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
This module utilizes the diamond storage pattern, accessing shared storage via the `ERC20_STORAGE_POSITION` slot, which is set to `keccak256(\"compose.erc20\")`. The `permit` function modifies allowance values within this shared storage. All functions are `internal` and expect the calling facet to manage external interactions and event emissions. Changes made through `permit` are immediately visible to all facets interacting with the same `ERC20PermitStorage` and `ERC20Storage`.
</Callout>
