<Callout type="info" title="Key Features">

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

This module provides the necessary logic and storage for implementing ERC-2612 permits (also known as permit/transferFrom). It allows users to grant allowances via off-chain signatures, reducing gas costs for frequent token transfers. The domain separator ensures signature validity across different chains and contract deployments.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

import {ERC20PermitMod} from "./ERC20PermitMod.sol";
import {IERC20Permit} from "@openzeppelin/contracts/token/ERC20/extensions/draft-IERC20Permit.sol";

contract MyERC20Facet {
    ERC20PermitMod internal constant _erc20PermitMod = ERC20PermitMod(address(this));

    /**
     * @notice Allows a user to permit a spender via an off-chain signature.
     * @dev Requires the caller to emit the Approval event.
     */
    function permit(address owner, address spender, uint256 value, uint256 deadline, uint8 v, bytes32 r, bytes32 s) external {
        _erc20PermitMod.permit(owner, spender, value, deadline, v, r, s);
        emit Approval(owner, spender, value);
    }

    /**
     * @notice Retrieves the domain separator for ERC-20 Permit.
     */
    function DOMAIN_SEPARATOR() external view returns (bytes32) {
        return _erc20PermitMod.DOMAIN_SEPARATOR();
    }
}
`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure the calling facet emits the `Approval` event after a successful `permit` call.
- Verify that the `deadline` parameter in `permit` is in the future to prevent expired permits.
- Use custom errors (`ERC20InvalidSpender`, `ERC2612InvalidSignature`) for clear revert reasons.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
The `ERC20PermitMod` library manages its own storage for the domain separator and permit-related data. Facets integrating this module must call the `permit` function and are responsible for emitting the `Approval` event. The `DOMAIN_SEPARATOR` function can be called to retrieve the domain separator, which is crucial for off-chain signature generation.
</Callout>
