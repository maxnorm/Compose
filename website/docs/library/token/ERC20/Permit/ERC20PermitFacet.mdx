<DocSubtitle>
ERC-20 permit functionality with EIP-2612
</DocSubtitle>

<Callout type="info" title="Key Features">
- Implements EIP-2612 permit functionality.
- Provides external view functions for DOMAIN_SEPARATOR and nonces.
- Uses internal functions to access diamond storage.
- Exports its own selectors for diamond registration.
</Callout>

## Overview

This facet implements EIP-2612 permit functionality for ERC-20 tokens within a Compose diamond. It exposes functions to retrieve domain separators, nonces, and to set allowances via signed messages. Developers integrate this facet to enable gasless token approvals for users.

## Best Practices

<Callout type="tip" title="Best Practice">
- Integrate this facet to add EIP-2612 permit capabilities to your diamond.
- Ensure the diamond contract is correctly initialized with the facet's selectors.
- Verify that the signature parameters (_v, _r, _s) are valid and correctly generated for the permit function.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
The `permit` function validates the provided signature components (_v, _r, _s) against the owner's nonce and the domain separator. Replay attacks are prevented by the nonce mechanism and the unique domain separator. Calls to `permit` should ensure that the signature is valid and that the deadline has not passed. The `nonces` function increments the nonce after a successful permit, preventing reuse of the same signature.
</Callout>
