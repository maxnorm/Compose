ERC-2612 permit logic and domain separator
</DocSubtitle>

<Callout type="info" title="Key Features">
- Provides `DOMAIN_SEPARATOR` for secure ERC-2612 signature encoding.
- Internal functions for permit validation and allowance setting.
- Uses diamond storage for shared permit state.
- Emits `Approval` event upon successful permit validation.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

This module provides ERC-2612 permit functionality, enabling off-chain signing of token approvals. Facets can integrate this module to manage permit validation and storage, leveraging the diamond storage pattern for shared state. It ensures a unique domain separator for secure signature verification, preventing replay attacks across different chains or contracts.

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure signature recovery is performed correctly within the calling facet before invoking the module's permit logic.
- Verify that the `_deadline` parameter is checked to prevent expired permits from being used.
- Handle the `ERC2612InvalidSignature` error to gracefully manage invalid permit attempts.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
This module integrates with the diamond storage pattern by utilizing a dedicated storage slot for permit-related data. The `permit` function interacts with this shared storage to validate signatures and update allowances. Changes to allowances made through this module are immediately reflected in the diamond's storage and visible to all facets accessing the same storage position.
</Callout>
