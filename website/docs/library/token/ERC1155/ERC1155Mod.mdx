<DocSubtitle>
Manages ERC-1155 token balances, transfers, and URIs.
</DocSubtitle>

<Callout type="info" title="Key Features">
- Supports minting and burning of individual or batched token types.
- Implements safe transfer mechanisms compliant with ERC-1155 standards.
- Manages token URI information, including base and token-specific URIs.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

This module implements core ERC-1155 functionality, including minting, burning, and safe transfers of single or batched token types. It also provides utilities for managing token URIs. By integrating this module, diamonds can manage fungible and non-fungible token standards efficiently.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

import {IERC1155Mod} from "./interfaces/IERC1155Mod.sol";

contract MyFacet {
    IERC1155Mod internal constant ERC1155 = IERC1155Mod(address(this));

    function mintAndTransfer(address to, uint256 id, uint256 amount) external {
        // Mint tokens to the facet's address first
        ERC1155.mint(address(this), id, amount);

        // Safely transfer tokens to the recipient
        ERC1155.safeTransferFrom(address(this), to, id, amount, "");
    }

    function burnSomeTokens(uint256 id, uint256 amount) external {
        ERC1155.burn(id, amount);
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Always validate token IDs and amounts before minting or burning to prevent unexpected state changes.
- Use `safeTransferFrom` and `safeBatchTransferFrom` for contract-to-contract transfers to ensure receiver compatibility.
- Handle `ERC1155InsufficientBalance` and `ERC1155InvalidArrayLength` errors explicitly in your facet logic.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
The ERC1155Mod module interacts with a predefined storage slot within the diamond's storage. Facets calling functions like `mint`, `burn`, `safeTransferFrom`, and `safeBatchTransferFrom` will implicitly use and update this shared storage. The `getStorage` function provides direct access to the ERC1155 storage struct, allowing facets to read balances and token metadata. Ensure that no other facets modify the ERC1155 storage slot in ways that could conflict with its invariants.
</Callout>
