<DocSubtitle>
ERC-1155 token data retrieval
</DocSubtitle>

<Callout type="info" title="Key Features">
- Exposes external view functions for ERC-1155 data retrieval.
- Reads data directly from diamond storage using inline assembly.
- Provides batched querying for efficiency.
- Exports its own selectors for diamond routing.
</Callout>

## Overview

This facet provides external view functions for retrieving ERC-1155 token balances and approval status within a Compose diamond. It accesses shared diamond storage to fetch this data, enabling other facets to interact with ERC-1155 token states. Developers integrate this facet to expose core ERC-1155 data querying capabilities through the diamond proxy.

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure the ERC1155DataFacet is properly added to the diamond during initialization.
- Access token data by calling functions through the diamond proxy address.
- Verify that related ERC-1155 facets (e.g., ERC1155ApproveFacet) are also integrated for full functionality.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
This facet contains only view functions and does not modify state. Input validation is handled by the underlying ERC-1155 logic. The `balanceOfBatch` and `balanceOf` functions assume valid `_id` and `_account` parameters. The `isApprovedForAll` function checks operator status against account approvals. The `ERC1155InvalidArrayLength` error is emitted if `_accounts` and `_ids` arrays in `balanceOfBatch` have different lengths.
</Callout>
