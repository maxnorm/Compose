<DocSubtitle>
ERC-1155 token transfers and metadata within a diamond
</DocSubtitle>

<Callout type="info" title="Key Features">
- Exposes external functions for diamond routing of ERC-1155 operations.
- Implements core ERC-1155 standard functions for token transfers and balance checks.
- Manages token URIs, supporting both default and token-specific configurations.
- Self-contained, following Compose's no-inheritance, no-imports-in-facets convention.
</Callout>

## Overview

This facet implements ERC-1155 token functionality as external functions within a diamond. It manages token balances, approvals, and URIs, routing calls through the diamond proxy and accessing shared storage. Developers add this facet to expose ERC-1155 token capabilities while maintaining diamond upgradeability.

## Functions

### uri

Returns the URI for token type `_id`. If a token-specific URI is set in tokenURIs[_id], returns the concatenation of baseURI and tokenURIs[_id]. Note that baseURI is empty by default and must be set explicitly if concatenation is desired. If no token-specific URI is set, returns the default URI which applies to all token types. The default URI may contain the substring `&#123;id&#125;` which clients should replace with the actual token ID.

## Best Practices

<Callout type="tip" title="Best Practice">
- Initialize ERC1155 storage and base URI during diamond setup.
- Enforce access control on all state-changing functions like `setApprovalForAll` if required by your diamond's architecture.
- Verify storage slot compatibility and function signatures before upgrading the ERC1155 facet.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
Follow standard Solidity security practices. Input validation is crucial for parameters like `_from`, `_to`, `_ids`, and `_values` to prevent errors such as `ERC1155InsufficientBalance`, `ERC1155InvalidSender`, `ERC1155InvalidReceiver`, and `ERC1155InvalidArrayLength`. Ensure proper approval checks are handled by the caller or other facets for `safeTransferFrom` and `safeBatchTransferFrom` functions, potentially using `isApprovedForAll` or requiring `_operator` approval. The `setApprovalForAll` function emits an `ApprovalForAll` event.
</Callout>
