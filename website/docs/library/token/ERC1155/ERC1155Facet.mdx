<DocSubtitle>
Manages ERC-1155 multi-token standards.
</DocSubtitle>

<Callout type="info" title="Key Features">
- Supports both fungible and non-fungible tokens under the ERC-1155 standard.
- Provides batched operations for `balanceOf` and `transferFrom` for efficiency.
- Includes `setApprovalForAll` and `isApprovedForAll` for granular operator permissions.
</Callout>

## Overview

The ERC1155Facet implements the ERC-1155 multi-token standard, allowing a diamond to manage fungible and non-fungible tokens within a unified framework. It provides essential functions for token transfers, balance checks, and operator approvals, enabling complex token economies.

### uri

Returns the URI for token type `_id`. If a token-specific URI is set in tokenURIs[_id], returns the concatenation of baseURI and tokenURIs[_id]. Note that baseURI is empty by default and must be set explicitly if concatenation is desired. If no token-specific URI is set, returns the default URI which applies to all token types. The default URI may contain the substring `&#123;id&#125;` which clients should replace with the actual token ID.

## Best Practices

<Callout type="tip" title="Best Practice">
- Initialize the `baseURI` and `tokenURIs` using appropriate diamond upgrade mechanisms if dynamic URIs are required.
- Ensure that any external contracts interacting with this facet implement the `IERC1155Receiver` interface for safe transfers.
- Carefully manage token IDs and their associated URIs to maintain discoverability and client-side rendering.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
This facet adheres to the ERC-1155 standard. Ensure proper access control is implemented at the diamond level for functions that modify state (e.g., minting, burning, or setting approvals). Implementations of `safeTransferFrom` and `safeBatchTransferFrom` rely on the receiver contract to implement `onERC1155Received` or `onERC1155BatchReceived` for reentrancy protection; ensure receiver contracts are audited.
</Callout>
