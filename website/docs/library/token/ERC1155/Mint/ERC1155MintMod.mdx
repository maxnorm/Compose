Mint ERC-1155 tokens and manage balances
</DocSubtitle>

<Callout type="info" title="Key Features">
- Internal functions for minting and balance management.
- Emits `TransferSingle` and `TransferBatch` events upon minting.
- Performs receiver validation for contract recipients.
- Utilizes diamond storage for state management.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

This module provides internal functions to mint ERC-1155 tokens and manage their balances within the diamond storage pattern. Facets can import and utilize these functions to issue new tokens, ensuring that balance updates and event emissions are handled consistently. Receiver validation is performed for contract recipients to maintain contract security.

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure sufficient gas is available for receiver validation if the recipient is a contract.
- Verify array lengths for `_ids` and `_values` match in `mintBatch` calls to prevent `ERC1155InvalidArrayLength` errors.
- Handle the `ERC1155InvalidReceiver` error if the recipient contract does not implement `onERC1155Received` or `onERC1155BatchReceived`.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
This module interacts with diamond storage at the `STORAGE_POSITION` identified by `keccak256("erc1155")`. It reads and writes to the `ERC1155Storage` struct. All changes to balances and token states made through this module are immediately visible to any other facet that accesses the same storage position via the diamond proxy.
</Callout>
