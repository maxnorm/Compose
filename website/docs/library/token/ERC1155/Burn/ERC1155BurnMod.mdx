<DocSubtitle>
Internal ERC-1155 token burning functionality
</DocSubtitle>

<Callout type="info" title="Key Features">
- Provides `internal` functions `burn` and `burnBatch` for direct use within facets.
- Uses the diamond storage pattern (EIP-8042) for shared state management.
- Emits standard ERC-1155 `TransferSingle` and `TransferBatch` events.
- Reverts with specific errors for insufficient balance or invalid input arrays.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

This module provides internal functions for burning ERC-1155 tokens. Facets can import this module to decrease token balances and emit transfer events using shared diamond storage. It ensures that burn operations are handled consistently across different facets that interact with the same ERC-1155 storage.

## Functions

### burn

Burns a single token type from an address. Decreases the balance and emits a TransferSingle event. Reverts if the account has insufficient balance. This module does not perform approval checks. Ensure proper ownership or approval validation before calling this function.

### burnBatch

Burns multiple token types from an address in a single transaction. Decreases balances for each token type and emits a TransferBatch event. Reverts if the account has insufficient balance for any token type. This module does not perform approval checks. Ensure proper ownership or approval validation before calling this function.

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure `ERC1155BurnMod` is initialized with the diamond address.
- Verify caller permissions and token balances before calling `burn` or `burnBatch`.
- Handle potential errors like `ERC1155InsufficientBalance` and `ERC1155InvalidArrayLength`.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
This module interacts with diamond storage at the `STORAGE_POSITION` identified by `keccak256(\"erc1155\")`. All functions are internal and directly manipulate the `ERC1155Storage` struct. Changes to token balances are immediately visible to any facet that accesses this storage position via the diamond proxy.
</Callout>
