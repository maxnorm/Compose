<DocSubtitle>
Burn ERC-1155 tokens within a diamond
</DocSubtitle>

<Callout type="info" title="Key Features">
- Exposes external functions for burning single or multiple ERC-1155 token types.
- Emits `TransferSingle` and `TransferBatch` events upon successful burns.
- Utilizes diamond storage for managing token balances.
- Exports its selectors via `exportSelectors` for diamond discovery.
</Callout>

## Overview

This facet enables burning of ERC-1155 tokens through external calls to the diamond proxy. It exposes `burn` and `burnBatch` functions, handling token balance reductions and emitting necessary transfer events. Developers integrate this facet to provide token destruction capabilities while adhering to the Compose diamond pattern and maintaining upgradeability.

## Best Practices

<Callout type="tip" title="Best Practice">
- Initialize the `ERC1155BurnFacet` and its associated storage slot during diamond deployment.
- Enforce ownership or operator approval checks on `burn` and `burnBatch` functions to prevent unauthorized token destruction.
- Ensure the `ERC1155Storage` struct and its slot are compatible with other ERC-1155 facets before upgrading.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
The `burn` and `burnBatch` functions require the caller to be the token owner or an approved operator. Input validation is performed to prevent burning with insufficient balances, invalid sender addresses, or mismatched array lengths. Follow standard Solidity security practices for input validation and access control.
</Callout>
