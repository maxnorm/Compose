Manage ERC-1155 operator approvals within a diamond
</DocSubtitle>

<Callout type="info" title="Key Features">
- Exposes `setApprovalForAll` for ERC-1155 operator approvals.
- Emits `ApprovalForAll` events upon approval changes.
- Provides `exportSelectors` for function discovery.
- Integrates with the diamond storage pattern for state management.
</Callout>

## Overview

This facet provides external functions for managing ERC-1155 operator approvals within a Compose diamond. It allows external callers to grant or revoke permissions for other addresses to manage their tokens. The facet integrates with the diamond proxy for call routing and accesses shared storage for approval state.

## Best Practices

<Callout type="tip" title="Best Practice">
- Call `setApprovalForAll` through the diamond proxy to manage operator approvals.
- Ensure the `operator` address is validated before granting approval if custom logic is required.
- Use `exportSelectors` during diamond upgrade or deployment to discover facet functions.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
The `setApprovalForAll` function grants broad permissions. Ensure the `_operator` address is trusted. Reverts with `ERC1155InvalidOperator` if the operator address is zero. Follow standard Solidity security practices for input validation.
</Callout>
