<DocSubtitle>
Manages ERC-1155 metadata URIs
</DocSubtitle>

<Callout type="info" title="Key Features">
- Internal functions for metadata management, intended for facet use.
- Leverages the diamond storage pattern for persistent and shared metadata.
- Emits a `URI` event upon setting token-specific URIs.
- Supports both a default URI and token-specific URIs, with a base URI concatenation.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

This module provides internal functions for managing ERC-1155 metadata URIs, including base URIs and token-specific URIs. Facets can import this module to set and retrieve these URIs, ensuring consistent metadata handling across the diamond. Changes to metadata are stored and immediately accessible via diamond storage.

Sets the base URI prefix for token-specific URIs. The base URI is concatenated with token-specific URIs set via setTokenURI. Does not affect the default URI used when no token-specific URI is set.

### setTokenURI

Sets the token-specific URI for a given token ID. Sets tokenURIs[_tokenId] to the provided string and emits a URI event with the full computed URI. The emitted URI is the concatenation of baseURI and the token-specific URI.

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure appropriate access control is implemented in the calling facet before invoking metadata setting functions.
- Verify the diamond storage layout remains compatible when upgrading facets that interact with this module.
- Handle the `URI` event emitted by `setTokenURI` for off-chain indexing or further processing.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
This module stores metadata within the diamond's shared storage at the `STORAGE_POSITION` identified by `keccak256(\"erc1155.metadata\")`. The `ERC1155MetadataStorage` struct, containing fields like `uri` and `baseURI`, is accessed via inline assembly. Any facet that correctly accesses this storage position will see changes made by this module immediately.
</Callout>
