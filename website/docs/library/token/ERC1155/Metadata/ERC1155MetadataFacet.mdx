ERC-1155 token metadata URI retrieval
</DocSubtitle>

<Callout type="info" title="Key Features">
- Exposes the `uri` function for ERC-1155 metadata retrieval.
- Integrates with diamond storage for metadata persistence.
- Follows ERC-2535 diamond standard for facet composition.
- Provides `exportSelectors` for diamond discovery.
</Callout>

## Overview

This facet implements ERC-1155 token metadata URI retrieval within a diamond proxy. It exposes the `uri` function to query metadata URIs for specific token IDs, leveraging diamond storage. Developers integrate this facet to provide standard ERC-1155 metadata functionality while maintaining diamond's upgradeability.

## Functions

### uri

Returns the URI for token type `_id`. If a token-specific URI is set in tokenURIs[_id], returns the concatenation of baseURI and tokenURIs[_id]. Note that baseURI is empty by default and must be set explicitly if concatenation is desired. If no token-specific URI is set, returns the default URI which applies to all token types. The default URI may contain the substring `&#123;id&#125;` which clients should replace with the actual token ID.

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure the `uri` function is correctly routed through the diamond proxy.
- Set the `baseURI` and token-specific URIs during diamond initialization if concatenation is desired.
- Verify storage compatibility before upgrading to maintain metadata integrity.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
The `uri` function is read-only and does not pose reentrancy risks. Input validation for `_id` is handled by the caller. Ensure proper access control is implemented at the diamond level for any functions that modify metadata if such functionality were added in other facets.
</Callout>
