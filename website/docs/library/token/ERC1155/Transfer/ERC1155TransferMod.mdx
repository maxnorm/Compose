<DocSubtitle>
Handle ERC-1155 token transfers within a diamond
</DocSubtitle>

<Callout type="info" title="Key Features">
- Provides `internal` functions for `safeTransferFrom` and `safeBatchTransferFrom`.
- Integrates with diamond storage using a dedicated storage slot (`keccak256(\"erc1155\")`).
- Enforces EIP-1155 safe transfer validation for contract recipients.
- Emits `TransferSingle` and `TransferBatch` events upon successful transfers.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

This module provides internal functions for safe ERC-1155 token transfers, including single token and batch transfers. Facets can import this module to manage token movements using shared diamond storage. It enforces ownership, approvals, and receiver validation, ensuring compliance with EIP-1155 safe transfer requirements.

### safeBatchTransferFrom

Safely transfers multiple token types from one address to another in a single transaction. Validates ownership, approval, and receiver address before updating balances for each token type. ## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure the calling facet has the necessary approvals before invoking transfer functions.
- Handle potential errors such as `ERC1155InsufficientBalance`, `ERC1155InvalidArrayLength`, and `ERC1155MissingApprovalForAll`.
- Verify that the receiver address is valid and capable of receiving ERC-1155 tokens, especially if it's a contract.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
This module reads and writes to diamond storage at the slot identified by `keccak256(\"erc1155\")`, storing an `ERC1155Storage` struct. All transfer functions are `internal` and directly interact with this shared storage, making balance updates immediately visible to all facets that access the same storage position. The `getStorage` function provides direct access to this storage struct.
</Callout>
