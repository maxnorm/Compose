<DocSubtitle>
ERC-1155 token transfers within a diamond
</DocSubtitle>

<Callout type="info" title="Key Features">
- External functions for diamond routing of ERC-1155 transfers.
- Implements `safeTransferFrom` and `safeBatchTransferFrom`.
- Emits `TransferSingle` and `TransferBatch` events as per ERC-1155 standard.
- Exports function selectors for diamond discovery.
</Callout>

## Overview

This facet implements ERC-1155 token transfers as external functions in a diamond. It routes calls through the diamond proxy and accesses shared storage. Developers add this facet to expose token functionality while maintaining upgradeability.

## Best Practices

<Callout type="tip" title="Best Practice">
- Initialize the ERC1155Storage struct correctly within the diamond's initialization process.
- Ensure that the `ERC1155ApproveFacet` is also deployed and accessible for approval operations required by transfers.
- Verify that the `SAFE_TRANSFER_FROM_SELECTOR` and `SAFE_BATCH_TRANSFER_FROM_SELECTOR` are correctly registered with the diamond.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
Functions `safeTransferFrom` and `safeBatchTransferFrom` are `external` and intended for direct invocation via the diamond proxy. Input validation is performed to prevent invalid sender/receiver addresses and array length mismatches. The facet relies on underlying approval mechanisms (likely managed by `ERC1155ApproveFacet`) for transfers initiated by third parties. Ensure correct access control is implemented at the diamond level for sensitive operations if required.
</Callout>
