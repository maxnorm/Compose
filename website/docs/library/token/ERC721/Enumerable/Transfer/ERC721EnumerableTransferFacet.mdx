<DocSubtitle>
ERC-721 token transfers within a diamond
</DocSubtitle>

<Callout type="info" title="Key Features">
- Exposes external functions for ERC-721 token transfers.
- Utilizes diamond storage pattern for shared state management.
- Self-contained with no external dependencies beyond Compose conventions.
- Compatible with ERC-2535 diamond standard for upgradeability.
</Callout>

## Overview

This facet implements ERC-721 token transfers as external functions within a diamond. It routes calls through the diamond proxy and accesses shared storage. Developers add this facet to expose ERC-721 token functionality while maintaining upgradeability.

## Best Practices

<Callout type="tip" title="Best Practice">
- Initialize the facet with necessary configurations during diamond deployment.
- Ensure that access control mechanisms are properly set up for state-changing functions.
- Verify storage slot compatibility when upgrading to new facet versions.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
The `transferFrom` and `safeTransferFrom` functions are protected by implicit checks against custom errors: `ERC721NonexistentToken`, `ERC721IncorrectOwner`, `ERC721InvalidReceiver`, and `ERC721InsufficientApproval`. Input validation is performed before state changes. Follow standard Solidity security practices for reentrancy and access control.
</Callout>
