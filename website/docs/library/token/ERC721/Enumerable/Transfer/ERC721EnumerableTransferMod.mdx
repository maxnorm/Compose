Internal ERC721 token transfer logic

<Callout type="info" title="Key Features">
- All functions are `internal` for direct use within facets.
- Leverages the diamond storage pattern for shared state management.
- Includes specific error types for common ERC721 transfer failures.
- Supports safe transfers to receiver contracts.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

This module provides internal functions for safely transferring ERC721 tokens. Facets can import this module to manage token ownership and ensure receiver compatibility within the diamond storage pattern. Changes to token ownership are immediately reflected for all facets interacting with the shared ERC721 storage.

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure ownership and approval checks are performed before calling `transferFrom`.
- Handle `ERC721IncorrectOwner`, `ERC721InvalidReceiver`, and `ERC721NonexistentToken` errors when interacting with token transfers.
- Use `safeTransferFrom` when transferring to contracts that may implement ERC721Receiver.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
This module accesses and modifies ERC721 token state using the diamond storage pattern at the `STORAGE_POSITION` defined for `keccak256(\"erc721.enumerable\")`. The `ERC721EnumerableStorage` struct governs the shared state. Any modifications made through functions like `transferFrom` are immediately visible to all facets that access the same storage slot.
</Callout>
