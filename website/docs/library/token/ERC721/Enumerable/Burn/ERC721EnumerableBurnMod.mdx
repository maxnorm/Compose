Burns ERC-721 tokens and tracks enumeration
</DocSubtitle>

<Callout type="info" title="Key Features">
- Internal functions for burning ERC-721 tokens.
- Manages enumeration state updates upon token burn.
- Uses diamond storage pattern for shared state management.
- Functions are `external` to be called by facets, but do not perform approval checks.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

This module provides internal functions for burning ERC-721 tokens, ensuring they are removed from enumeration tracking. Facets using this module can interact with shared ERC721Storage and ERC721EnumerableStorage via diamond storage. Changes to token state are immediately visible to all facets accessing the same storage.

## Best Practices

<Callout type="tip" title="Best Practice">
- Call `burn` only after verifying token existence and ownership (handled by the facet).
- Ensure correct storage slot `STORAGE_POSITION` is used by the diamond proxy.
- Handle the `ERC721NonexistentToken` error if the facet directly calls `burn` without prior checks.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
This module utilizes the diamond storage pattern at `STORAGE_POSITION`, identified by `keccak256(\"erc721.enumerable\")`. It reads from and writes to `ERC721EnumerableStorage` and `ERC721Storage` structs within this slot. All state changes made by the `burn` function are immediately reflected across all facets that access this shared diamond storage.
</Callout>
