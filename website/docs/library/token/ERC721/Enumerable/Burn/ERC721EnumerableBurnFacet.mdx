<DocSubtitle>
Burns ERC-721 tokens and removes them from enumeration
</DocSubtitle>

<Callout type="info" title="Key Features">
- External function for burning ERC-721 tokens.
- Removes tokens from enumeration tracking.
- Utilizes diamond storage for state management.
- Exports selectors for diamond integration.
</Callout>

## Overview

This facet provides functionality to burn ERC-721 tokens within a Compose diamond. It interacts with diamond storage to remove tokens from enumeration tracking. Developers add this facet to enable token destruction while maintaining the integrity of the token supply and its ordered list.

## Best Practices

<Callout type="tip" title="Best Practice">
- Initialize the facet and its storage mappings during diamond deployment.
- Ensure the `burn` function is protected by appropriate access control mechanisms.
- Verify that the `ERC721EnumerableStorage` and `ERC721Storage` structs are correctly mapped in the diamond's storage layout.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
The `burn` function should be protected by access control to prevent unauthorized token destruction. The function handles token existence checks internally, reverting with `ERC721NonexistentToken` if the token does not exist. Input validation for `_tokenId` is crucial. Follow standard Solidity security practices for external function calls.
</Callout>
