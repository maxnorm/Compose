<DocSubtitle>
Enumerable ERC-721 token data querying
</DocSubtitle>

<Callout type="info" title="Key Features">
- Exposes external view functions for ERC-721 token enumeration.
- Uses inline assembly to access diamond storage efficiently.
- Provides `exportSelectors` for diamond discovery.
- Adheres to Compose's readability-first design principles.
</Callout>

## Overview

This facet provides external view functions for querying enumerable ERC-721 token data within a diamond. It accesses shared storage using assembly to retrieve total supply and token ownership details. Integrate this facet to expose token enumeration capabilities while maintaining diamond upgradeability.

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure this facet is added to the diamond with its correct selectors.
- Access token data using the `totalSupply`, `tokenOfOwnerByIndex`, and `tokenByIndex` functions through the diamond proxy.
- Verify that the `ERC721EnumerableStorage` and `ERC721Storage` structs are correctly initialized and managed by other facets.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
The `tokenByIndex` function reverts with `ERC721OutOfBoundsIndex` if the provided `_index` is greater than or equal to `totalSupply()`. Follow standard Solidity security practices for input validation on all external calls.
</Callout>
