<Callout type="info" title="Key Features">
- Provides `internal` functions for minting, suitable for custom facet logic.
- Manages token enumeration alongside minting operations.
- Uses the diamond storage pattern (EIP-8042) for shared state.
- Reverts with specific errors for invalid receivers or existing tokens.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

This module provides internal functions for minting ERC-721 tokens while maintaining enumeration lists. Facets can import this module to add new tokens to a diamond's ERC-721 collection using shared diamond storage. The minting process ensures that new tokens are correctly added to the internal tracking mechanisms, making them visible to all facets interacting with the ERC721EnumerableStorage.

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure the `_to` address is valid and the `_tokenId` does not already exist before calling `mint`.
- Verify the diamond's storage layout for `ERC721EnumerableStorage` compatibility during upgrades.
- Handle `ERC721InvalidReceiver` and potential reentrancy issues if custom logic is added around minting.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
This module utilizes diamond storage at the `STORAGE_POSITION` slot, identified by `keccak256(\"erc721.enumerable\")`. It uses the `ERC721EnumerableStorage` struct to manage token data and enumeration. Any facet that correctly accesses this storage slot will see changes made by this module immediately, ensuring data consistency across the diamond.
</Callout>
