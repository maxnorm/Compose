<Callout type="info" title="Key Features">
- Internal functions for ERC-721 approval logic.

- Emits `Approval` and `ApprovalForAll` events upon successful operations.

This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.

## Overview

This module provides internal functions for managing ERC-721 token approvals and operator permissions. Facets can import this module to interact with shared ERC-721 storage, ensuring consistent state management across the diamond. Changes made through this module are immediately visible to all facets accessing the same storage slot.

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure the caller has the necessary permissions before calling `approve` or `setApprovalForAll`.
- Handle `ERC721InvalidOperator` and `ERC721NonexistentToken` errors appropriately.
- Use `getStorage()` to retrieve the storage pointer for direct manipulation only when absolutely necessary and with full understanding of diamond storage.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
This module interacts with diamond storage at the slot identified by `STORAGE_POSITION`, which is `keccak256("erc721")`. The `ERC721Storage` struct is used to manage the state. All functions are `internal` and operate on this shared storage, making any modifications immediately visible to all facets that access the same storage slot.
</Callout>
