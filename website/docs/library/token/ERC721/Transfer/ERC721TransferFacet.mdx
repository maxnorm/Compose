<DocSubtitle>
ERC-721 token transfers within a diamond
</DocSubtitle>

<Callout type="info" title="Key Features">
- Exposes external functions for diamond routing (ERC-2535).
- Utilizes internal `internalTransferFrom` for core transfer logic.
- Accesses shared storage via `getStorage` and a defined `STORAGE_POSITION`.
- Supports safe transfers with optional `_data` parameter.
</Callout>

## Overview

This facet implements ERC-721 token transfers as external functions within a diamond. It routes calls through the diamond proxy and accesses shared storage via `getStorage`. Developers add this facet to expose ERC-721 token functionality while maintaining diamond upgradeability.

## Best Practices

<Callout type="tip" title="Best Practice">
- Initialize the ERC721Storage struct via an initializer function during diamond setup.
- Ensure the receiver address in `safeTransferFrom` can handle ERC-721 tokens to prevent unexpected behavior.
- Verify storage compatibility using `exportSelectors` before upgrading the diamond to include this facet.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
The `transferFrom` and `safeTransferFrom` functions perform checks for token ownership (`ERC721IncorrectOwner`) and ensure the token exists (`ERC721NonexistentToken`). `safeTransferFrom` also validates receiver capabilities (`ERC721InvalidReceiver`). Ensure appropriate approvals are managed externally before calling these functions, as `ERC721InsufficientApproval` can be triggered.
</Callout>
