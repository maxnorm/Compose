<DocSubtitle>
ERC-721 token data retrieval within a diamond
</DocSubtitle>

<Callout type="info" title="Key Features">
- Exposes external view functions for ERC-721 data retrieval.
- Accesses token data directly from diamond storage using a dedicated storage slot.
- Self-contained facet with no external dependencies or inheritance.
- Provides selectors for easy integration into diamond proxies.
</Callout>

## Overview

This facet exposes functions for querying ERC-721 token ownership and approvals. It integrates with the diamond's shared storage to provide read-only access to token data. Developers add this facet to a diamond to enable external querying of ERC-721 state.

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure the ERC721DataFacet is correctly added to the diamond's facet registry.
- Call facet functions through the diamond proxy to leverage its routing mechanism.
- Verify that `balanceOf`, `ownerOf`, `getApproved`, and `isApprovedForAll` are called with valid parameters to prevent unexpected reverts.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
The `balanceOf`, `ownerOf`, `getApproved`, and `isApprovedForAll` functions are view functions and do not modify state. They are susceptible to the ERC721NonexistentToken error if called with an invalid token ID. Follow standard Solidity security practices for input validation.
</Callout>
