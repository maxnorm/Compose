Manages ERC-721 metadata within a diamond
</DocSubtitle>

<Callout type="info" title="Key Features">
- Provides `internal` functions for metadata management.
- Utilizes the diamond storage pattern (EIP-8042) for shared state.
- No external dependencies, promoting composability.
- Compatible with ERC-2535 diamonds.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

This module provides internal functions to manage ERC-721 metadata such as name, symbol, and base URI. Facets can import this module to interact with shared diamond storage for metadata. This approach ensures consistent metadata across all facets of the diamond.

## Best Practices

<Callout type="tip" title="Best Practice">
- Call `setMetadata` only when initializing or upgrading the diamond to ensure consistent metadata.
- Use `getStorage` to access metadata values, ensuring reads reflect the latest state across all facets.
- Handle potential errors if `setMetadata` or related functions implement checks in a full implementation.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
This module reads and writes to diamond storage at the `STORAGE_POSITION` slot, identified by `keccak256(\"erc721.metadata\")`. The `ERC721MetadataStorage` struct, containing `name`, `symbol`, and `baseURI`, is managed within this slot. All facets interacting with ERC-721 metadata should use this module to ensure atomic updates and consistent access to the shared storage.
</Callout>
