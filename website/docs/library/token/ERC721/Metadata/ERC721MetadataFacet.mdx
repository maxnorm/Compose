<DocSubtitle>
ERC-721 token metadata within a diamond
</DocSubtitle>

<Callout type="info" title="Key Features">
- Exposes external functions for diamond routing of metadata queries.
- Accesses token metadata via diamond storage using EIP-2535.
- Self-contained with no external dependencies beyond standard Solidity.
- Provides specific selectors for easy integration and discovery.
</Callout>

## Overview

This facet provides external access to ERC-721 token metadata, including name, symbol, and token URIs, within a Compose diamond. It utilizes the diamond storage pattern to access shared state and routes calls through the diamond proxy. Developers add this facet to expose token metadata functionalities while maintaining upgradeability.

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure the facet is correctly initialized with the diamond's deployment process.
- Verify that `name`, `symbol`, and `baseURI` are set in diamond storage before deploying this facet.
- Grant access to the facet's functions through the diamond's access control mechanism if required.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
The `name`, `symbol`, and `tokenURI` functions are view functions and do not modify state. Input validation for `tokenURI` is handled by underlying ERC721 logic. Follow standard Solidity security practices for diamond proxy interactions.
</Callout>
