<DocSubtitle>
Mint ERC-721 tokens using diamond storage
</DocSubtitle>

<Callout type="info" title="Key Features">
- Mints ERC-721 tokens using diamond storage.
- Internal functions for facet integration.
- Utilizes EIP-8042 diamond storage pattern.
- Emits `Transfer` event on successful mint.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

This module exposes internal functions for minting ERC-721 tokens. Facets import this module to create new tokens using shared diamond storage. Changes made through this module are immediately visible to all facets using the same storage pattern.

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure the caller has the necessary permissions before calling `mintToken`.
- Verify that the `_to` address is not the zero address.
- Handle potential `ERC721InvalidReceiver` or `ERC721InvalidSender` errors.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
This module uses diamond storage at `STORAGE_POSITION` under the key `keccak256(\"erc721\")`. All functions are internal and access the shared `ERC721Storage` struct. Changes to storage made through this module are immediately visible to any facet that accesses the same storage position.
</Callout>
