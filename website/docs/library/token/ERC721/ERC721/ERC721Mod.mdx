<DocSubtitle>
Internal logic for ERC-721 token management
</DocSubtitle>

<Callout type="info" title="Key Features">
- Provides `internal` functions for ERC-721 operations, suitable for integration into custom facets.
- Leverages the diamond storage pattern (EIP-8042) using a predefined storage position.
- Supports core ERC-721 functionalities: minting, transferring, and burning tokens.
- Emits `Transfer` events upon successful token transfers and burns.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

This module provides internal functions for managing ERC-721 tokens within a diamond proxy. Facets import this module to implement token minting, transfers, and burning using shared diamond storage. This approach ensures consistent ERC-721 state across all facets interacting with the same token.

## Best Practices

<Callout type="tip" title="Best Practice">
- Call internal functions like `mint`, `transferFrom`, and `burn` only after verifying necessary permissions or conditions within your facet.
- Ensure that the `ERC721Storage` struct is correctly initialized and its fields (`name`, `symbol`, `baseURI`) are set before using token operations.
- Handle custom errors such as `ERC721NonexistentToken` and `ERC721IncorrectOwner` to provide clear feedback to users.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
This module utilizes diamond storage at the `STORAGE_POSITION` key, identified by `keccak256("compose.erc721")`. All state modifications and reads are performed against the `ERC721Storage` struct stored at this location. Facets importing this module gain access to its internal functions, which directly interact with this shared storage. Changes made via this module are immediately visible to all other facets that access the same storage position.
</Callout>
