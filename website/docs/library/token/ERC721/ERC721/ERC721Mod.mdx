Internal logic for ERC-721 token management within a diamond.
</DocSubtitle>

<Callout type="info" title="Key Features">
- Provides core ERC-721 operations: minting, burning, and transfers.
- Enforces essential ERC-721 invariants and checks for invalid operations.
- Leverages diamond storage for persistent token state management.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

The ERC721Mod provides essential internal logic for managing ERC-721 tokens using the diamond storage pattern. It enables facets to mint, burn, and transfer tokens while enforcing core ERC-721 rules. This module simplifies integration by abstracting complex state management into reusable functions.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

import {IERC721Mod } from "../mods/ERC721Mod.sol";

contract MyERC721Facet {
    // Assume ERC721Mod is deployed and its address is known
    IERC721Mod private immutable _erc721Mod;

    constructor(address _erc721ModAddress) {
        _erc721Mod = IERC721Mod(_erc721ModAddress);
    }

    /**
     * @notice Mints a new ERC721 token.
     * @param _to The address to mint the token to.
     * @param _tokenId The ID of the token to mint.
     */
    function mintToken(address _to, uint256 _tokenId) external {
        _erc721Mod.mint(_to, _tokenId);
    }

    /**
     * @notice Transfers ownership of an ERC721 token.
     * @param _from The current owner's address.
     * @param _to The new owner's address.
     * @param _tokenId The ID of the token to transfer.
     */
    function transferToken(address _from, address _to, uint256 _tokenId) external {
        _erc721Mod.transferFrom(_from, _to, _tokenId);
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure proper access control is implemented in facets calling `mint` and `burn` functions to prevent unauthorized operations.
- Handle potential revert reasons like `ERC721NonexistentToken`, `ERC721InvalidSender`, and `ERC721InvalidReceiver` in calling facets.
- When upgrading facets that interact with ERC721Mod, be mindful of storage layout compatibility and ensure the module's storage slot remains consistent.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
The ERC721Mod utilizes a predefined storage slot for its internal ERC-721 storage struct. Facets that interact with this module must be aware that operations like `mint`, `burn`, and `transferFrom` directly modify this shared diamond storage. The `getStorage` function can be used by facets to read the current state of the ERC-721 data, but direct modification of the storage struct by external facets is discouraged to maintain module integrity.
</Callout>
