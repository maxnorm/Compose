<DocSubtitle>
Manage ERC721 tokens within a Compose diamond.
</DocSubtitle>

<Callout type="info" title="Key Features">
- Supports core ERC-721 operations: mint, burn, and transfer.
- Utilizes diamond storage for persistent and upgradeable token state.
- Employs custom errors for gas-efficient and clear error reporting.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

The ERC721Mod module provides essential internal logic for ERC-721 token management, enabling custom facets to implement token minting, burning, and transfers. It leverages diamond storage to maintain a consistent and upgradeable state for all ERC-721 related data.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

import {IERC721Mod } from "@compose/modules/ERC721Mod.sol";

contract MyERC721Facet {
    IERC721Mod internal erc721Mod;

    function initialize(address _erc721ModAddress) external {
        erc721Mod = IERC721Mod(_erc721ModAddress);
    }

    function safeMint(address _to, uint256 _tokenId) external {
        // Assume access control checks are done elsewhere
        erc721Mod.mint(_to, _to, _tokenId);
    }

    function safeTransfer(address _from, address _to, uint256 _tokenId) external {
        // Assume access control checks are done elsewhere
        erc721Mod.transferFrom(_from, _to, _tokenId);
    }

    function safeBurn(uint256 _tokenId) external {
        // Assume access control checks are done elsewhere
        erc721Mod.burn(_tokenId);
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Implement robust access control within facets calling module functions to restrict minting, burning, and transfers to authorized roles.
- Handle custom errors like `ERC721IncorrectOwner` and `ERC721NonexistentToken` gracefully in facet logic to provide informative user feedback.
- Ensure the ERC721Mod facet is initialized with the correct module address during diamond upgrades to maintain state continuity.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
The ERC721Mod stores its state in a predefined slot within the diamond's storage. Facets interacting with this module should use the `getStorage` function to access the ERC721 storage struct directly when needed, or rely on the module's functions which handle storage interactions internally. Changes made through the module's functions (e.g., mint, burn, transfer) are immediately visible to all facets that have access to the diamond's storage.
</Callout>
