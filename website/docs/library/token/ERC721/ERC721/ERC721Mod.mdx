Internal logic for ERC-721 token management
</DocSubtitle>

<Callout type="info" title="Key Features">
- Exposes `internal` functions for ERC-721 operations.
- Integrates with diamond storage using a predefined slot (`keccak256(\"compose.erc721\")`).
- No external dependencies, promoting composability.
- Supports standard ERC-721 state management within a diamond.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

This module provides internal functions for minting, burning, and transferring ERC-721 tokens using diamond storage. Facets importing this module can leverage its logic to manage token state within a shared storage pattern. Changes to token ownership or metadata are immediately reflected across all facets accessing the same storage.

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure the ERC721Storage struct layout is compatible before upgrading facets.
- Call internal functions like `mint`, `burn`, and `transferFrom` with appropriate checks for token existence and ownership.
- Handle custom errors like `ERC721NonexistentToken` or `ERC721IncorrectOwner` in calling facets.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
This module utilizes diamond storage at the slot derived from `keccak256(\"compose.erc721\")`. The `getStorage()` function provides access to the `ERC721Storage` struct, which contains fields such as `name`, `symbol`, and `baseURI`. All state modifications performed by `mint`, `burn`, and `transferFrom` directly update this shared storage, making them immediately visible to all facets interacting with the same storage position.
</Callout>
