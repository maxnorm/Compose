<DocSubtitle>
Burns ERC-721 tokens within a diamond
</DocSubtitle>

<Callout type="info" title="Key Features">
- Exposes an external `burn` function for token destruction.
- Updates internal enumeration tracking upon token burning.
- Self-contained, adhering to ERC-2535 diamond standards.
- Follows Compose readability-first conventions.
</Callout>

## Overview

This facet implements the burning of ERC-721 tokens as an external function within a diamond. It provides the `burn` function to remove a token from existence and updates internal tracking. Developers integrate this facet to enable token destruction capabilities in their diamond.

## Best Practices

<Callout type="tip" title="Best Practice">
- Integrate the `ERC721BurnFacet` during diamond initialization.
- Ensure the `burn` function is called with valid `_tokenId` parameters.
- Verify that the caller has the necessary permissions to burn the token, if access control is implemented at the diamond level.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
The `burn` function removes a token permanently. Ensure that the caller has the correct authorization to burn the specified `_tokenId`. Input validation for `_tokenId` is crucial to prevent unexpected behavior. The facet relies on the diamond's dispatch mechanism for access control. Reentrancy is not a concern as the function does not make external calls after state changes.
</Callout>
