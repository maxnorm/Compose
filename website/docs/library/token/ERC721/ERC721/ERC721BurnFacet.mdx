<DocSubtitle>
Burns ERC-721 tokens within a diamond proxy
</DocSubtitle>

<Callout type="info" title="Key Features">
- Exposes an external `burn` function for token destruction.
- Emits standard ERC-721 `Transfer` events upon successful burning.
- Leverages diamond storage for state management.
- Self-contained, requiring no external dependencies beyond diamond core.
</Callout>

## Overview

This facet implements ERC-721 token burning as an external function within a diamond proxy. It interacts with shared diamond storage to remove tokens and emits `Transfer` events. Developers integrate this facet to provide token destruction capabilities while maintaining the upgradeability and modularity of the diamond pattern.

## Best Practices

<Callout type="tip" title="Best Practice">
- Integrate the `ERC721BurnFacet` during diamond initialization.
- Ensure the `ERC721Storage` struct is correctly laid out and accessible.
- Verify that the `burn` function's access control (if any) is correctly configured before deployment.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
The `burn` function should have appropriate access control mechanisms in place, typically managed by a separate access control facet, to ensure only authorized addresses can burn tokens. Input validation for `_tokenId` is critical to prevent unintended state modifications or reverts due to non-existent tokens, though specific checks are handled internally by the facet's logic. Follow standard Solidity security practices for all interactions.
</Callout>
