<DocSubtitle>
ERC-721 token management within a diamond
</DocSubtitle>

<Callout type="info" title="Key Features">
- Exposes standard ERC-721 functions including `ownerOf`, `balanceOf`, `transferFrom`, and `approve`.
- Provides `tokenURI` for accessing token metadata.
- Utilizes the diamond storage pattern for state management.
- Facilitates upgradeability by separating ERC-721 logic into a distinct facet.
</Callout>

## Overview

This facet implements ERC-721 token functionality for a diamond proxy. It provides external functions to manage token ownership, approvals, and metadata. Developers integrate this facet to expose a full ERC-721 interface while leveraging the diamond's upgradeability and storage pattern.

## Best Practices

<Callout type="tip" title="Best Practice">
- Initialize the `name`, `symbol`, and `baseURI` fields during diamond deployment.
- Ensure the `ERC721Mod` module is correctly integrated for storage access.
- Verify that all ERC-721 specific errors are handled by the calling contract or client.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
Input validation is performed within the facet functions to prevent common ERC-721 vulnerabilities. Functions such as `transferFrom` and `safeTransferFrom` include checks for ownership and approvals. Errors like `ERC721InvalidOwner`, `ERC721NonexistentToken`, and `ERC721InsufficientApproval` are emitted to indicate specific failure conditions. Follow standard Solidity security practices for external calls and state management.
</Callout>
