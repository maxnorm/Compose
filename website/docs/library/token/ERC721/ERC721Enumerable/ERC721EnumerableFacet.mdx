<DocSubtitle>
ERC-721 token management and metadata access
</DocSubtitle>

<Callout type="info" title="Key Features">
- Exposes external functions for ERC-721 token operations.
- Provides enumerable functionality to list tokens by owner index.
- Accesses shared storage via `ERC721EnumerableStorage`.
- Compatible with ERC-2535 diamond standard.
</Callout>

## Overview

This facet implements ERC-721 token functionality within a diamond proxy. It exposes standard ERC-721 functions for managing token ownership, approvals, and metadata. Developers integrate this facet to provide NFT capabilities while leveraging the diamond's upgradeability and modularity.

## Best Practices

<Callout type="tip" title="Best Practice">
- Initialize token name, symbol, and base URI during diamond deployment.
- Use the `ownerOf` and `balanceOf` functions to verify token ownership before state-changing operations.
- Ensure that the `safeTransferFrom` functions are used when transferring tokens to potentially unknown contracts.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
Follow standard Solidity security practices. Ensure that only authorized addresses can call state-changing functions like `approve`, `setApprovalForAll`, `transferFrom`, and `safeTransferFrom`. The `ownerOf`, `balanceOf`, `tokenOfOwnerByIndex`, `getApproved`, and `isApprovedForAll` functions are view functions and do not pose direct state modification risks. Input validation is handled by custom errors such as `ERC721InvalidOwner` and `ERC721NonexistentToken`.
</Callout>
