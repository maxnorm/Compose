<DocSubtitle>
Manage and burn ERC721 tokens with enumeration.
</DocSubtitle>

<Callout type="info" title="Key Features">
- Allows for the destruction of ERC721 tokens.
- Maintains enumeration order after token burns.
- Provides access to the facet's storage structure.
</Callout>

## Overview

This facet extends ERC721 functionality by enabling token burning and maintaining enumeration order. It provides the necessary functions to remove tokens from the contract while ensuring the internal tracking mechanisms remain consistent.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

import {IERC721EnumerableBurnFacet} from "@compose-protocol/diamond/contracts/facets/ERC721/IERC721EnumerableBurnFacet.sol";

contract MyDiamond {
    // ... other facets

    function burnToken(uint256 _tokenId) external {
        // Assumes the diamond has a function to delegate to the ERC721EnumerableBurnFacet
        // For example, using a DiamondLoupeFacet or a custom router
        (bool success, ) = address(this).staticcall(abi.encodeWithSelector(IERC721EnumerableBurnFacet.burn.selector, _tokenId));
        require(success, "Failed to burn token");
    }

    // ... other diamond functions
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Initialize the facet with the correct storage layout during diamond deployment.
- Ensure proper access control is implemented on the diamond level to restrict who can call the `burn` function.
- Verify that the `Transfer` event is emitted correctly upon successful token burning.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
The `burn` function should be protected by robust access control to prevent unauthorized token destruction. Ensure that the token ID provided to `burn` exists, utilizing the `ERC721NonexistentToken` error for validation. Reentrancy is not a direct concern for the `burn` function itself, but downstream effects of token transfers should be considered in the overall diamond logic.
</Callout>
