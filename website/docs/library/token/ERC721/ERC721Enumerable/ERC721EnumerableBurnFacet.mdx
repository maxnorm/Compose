<DocSubtitle>
Burns ERC-721 tokens and removes them from enumeration
</DocSubtitle>

<Callout type="info" title="Key Features">
- Provides an external function to burn ERC-721 tokens.
- Manages token removal from enumeration tracking.
- Self-contained and does not introduce external dependencies.
- Follows EIP-2535 Diamond Standard for composability.
</Callout>

## Overview

This facet provides functionality to burn ERC-721 tokens within a diamond. It interacts with the diamond's shared storage to remove tokens and maintain enumeration order. Developers integrate this facet to enable token destruction while ensuring the integrity of the token supply and tracking.

## Best Practices

<Callout type="tip" title="Best Practice">
- Initialize the facet by adding it to the diamond during deployment.
- Ensure appropriate access control is set for the `burn` function if direct calls are restricted.
- Verify that the diamond's storage layout is compatible before upgrading to include this facet.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
The `burn` function should be protected by access control mechanisms to prevent unauthorized token destruction. Input validation for `_tokenId` is handled internally by the facet, reverting with `ERC721NonexistentToken` if the token does not exist. Follow standard Solidity security practices for input validation and state management.
</Callout>
