Internal logic for enumerable ERC-721 tokens
</DocSubtitle>

<Callout type="info" title="Key Features">
- Provides internal functions for minting, burning, and transferring ERC-721 tokens.
- Manages token enumeration state using the diamond storage pattern.
- Emits `Transfer` events upon successful token operations.
- Reverts with specific custom errors for invalid operations or states.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

This module provides internal functions for managing enumerable ERC-721 tokens within a diamond. Facets can import this module to implement minting, burning, and transferring tokens, ensuring all state changes are correctly reflected in enumeration lists. It utilizes the diamond storage pattern for shared state access.

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure the `_sender` parameter accurately reflects the caller or authorized party for `burn` and `transferFrom` functions to maintain correct ownership and authorization checks.
- Handle custom errors like `ERC721NonexistentToken`, `ERC721IncorrectOwner`, and `ERC721InvalidReceiver` to gracefully manage token operations.
- Verify that the diamond's storage layout is compatible with `ERC721EnumerableStorage` before upgrading or adding new facets to prevent storage collisions.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
This module stores its state in the diamond's storage at the position defined by `keccak256(\"compose.erc721.enumerable\")`. The `getStorage()` function provides a view into this storage struct, `ERC721EnumerableStorage`. All state modifications performed by `mint`, `burn`, and `transferFrom` are directly applied to this shared diamond storage, making them immediately visible to any facet that accesses the same storage position.
</Callout>
