<Callout type="info" title="Key Features">
- Manages token enumeration state for minting, burning, and transfers.
- Provides internal logic for standard ERC-721 operations, reducing facet complexity.
- Utilizes the diamond storage pattern for efficient state management.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

This module provides the core logic for implementing enumerable ERC-721 functionality within a Compose diamond. It handles the state management for token minting, burning, and transfers, ensuring tokens are correctly tracked in enumeration lists. By integrating this module, facets can offer standard ERC-721 features while maintaining a consistent and efficient storage pattern.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

import {IERC721EnumerableMod} from "@compose-protocol/diamond/contracts/modules/erc721/ERC721EnumerableMod.sol";
import {ERC721Storage} from "@compose-protocol/diamond/contracts/modules/erc721/ERC721Storage.sol";

contract MyERC721Facet {
    IERC721EnumerableMod private immutable _erc721EnumerableMod;

    constructor(address diamondProxy) {
        _erc721EnumerableMod = IERC721EnumerableMod(diamondProxy);
    }

    function mintToken(address to, uint256 tokenId) external {
        _erc721EnumerableMod.mint(to, tokenId);
    }

    function burnToken(uint256 tokenId) external {
        _erc721EnumerableMod.burn(tokenId);
    }

    function transferToken(address from, address to, uint256 tokenId) external {
        _erc721EnumerableMod.transferFrom(from, to, tokenId);
    }

    function getEnumerableStorage() external view returns (ERC721Storage.Layout storage) {
        return _erc721EnumerableMod.getStorage();
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure proper access control is implemented in facets calling mint, burn, and transferFrom to prevent unauthorized operations.
- Always validate input parameters in facets before calling module functions to prevent reverts due to invalid data (e.g., zero addresses, non-existent tokens).
- Be mindful of storage slot collisions if customizing the ERC721Storage struct; follow Compose's storage guidelines.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
The ERC721EnumerableMod interacts with the diamond's storage using a predefined slot for its `ERC721Storage.Layout` struct. Facets integrating this module can access the entire storage layout via the `getStorage()` function. State changes made by the module's functions (`mint`, `burn`, `transferFrom`) are immediately reflected in the diamond's storage and are visible to all facets. The module enforces invariants related to token existence, ownership, and enumeration order.
</Callout>
