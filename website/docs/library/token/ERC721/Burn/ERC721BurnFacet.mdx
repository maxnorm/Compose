<DocSubtitle>
Burns ERC-721 tokens within a diamond
</DocSubtitle>

<Callout type="info" title="Key Features">
- Exposes external functions for token burning.
- Interacts with diamond storage for state management.
- Self-contained unit adhering to Compose facet principles.
</Callout>

## Overview

This facet provides functionality to burn ERC-721 tokens within a Compose diamond. It exposes external functions for token destruction, interacting with the diamond's shared storage. Developers integrate this facet to manage token lifecycles, specifically for removing tokens from circulation.

## Best Practices

<Callout type="tip" title="Best Practice">
- Initialize the facet within the diamond's deployment process.
- Ensure proper access control is implemented for functions that change state.
- Verify storage slot compatibility if upgrading this facet.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
Functions `burn` and `burnBatch` may revert with `ERC721NonexistentToken` if the token ID does not exist or `ERC721InsufficientApproval` if the caller is not approved to burn the token. Follow standard Solidity security practices for input validation and access control.
</Callout>
