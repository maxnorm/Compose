Minimal multi-token logic for ERC-6909
</DocSubtitle>

<Callout type="info" title="Key Features">
- Provides internal functions for ERC-6909 token operations.
- Utilizes the diamond storage pattern for shared state.
- No external dependencies, ensuring composability.
- Supports approvals, transfers, minting, and burning of multi-tokens.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

This module provides internal functions and storage for ERC-6909 minimal multi-token logic. Facets import this module to manage token approvals, transfers, minting, and burning using shared diamond storage. Changes made through this module are immediately visible to all facets using the same storage pattern.

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure sufficient allowances are set before calling `transfer` or `burn`.
- Verify that the sender and receiver addresses are valid before executing operations.
- Handle specific errors like `ERC6909InsufficientBalance` and `ERC6909InsufficientAllowance` returned by the module functions.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
This module uses the diamond storage pattern at the `STORAGE_POSITION` defined by `keccak256(\"compose.erc6909\")`. All functions are `internal` and access the shared `ERC6909Storage` struct. Changes to token balances and approvals made through this module are immediately visible to any facet that accesses the same storage position via the diamond proxy.
</Callout>
