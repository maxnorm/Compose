<DocSubtitle>
Minimal multi-token logic for ERC-6909
</DocSubtitle>

<Callout type="info" title="Key Features">
- Provides internal functions for core ERC-6909 operations.
- Uses the diamond storage pattern via a dedicated storage slot.
- No external dependencies, promoting composability.
- Functions are `internal` to ensure they are called only through facets.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

This module provides internal functions and storage layout for ERC-6909 minimal multi-token logic. Facets can import this module to manage token approvals, transfers, minting, and burning using shared diamond storage. Changes made through these internal functions are immediately visible to all facets interacting with the same storage slot.

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure the `ERC6909Storage` struct is correctly initialized in diamond storage.
- Call internal functions with appropriate access control checks implemented in the calling facet.
- Handle errors such as `ERC6909InsufficientBalance` and `ERC6909InsufficientAllowance` returned by module functions.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
This module utilizes the diamond storage pattern by defining a specific storage position identified by `STORAGE_POSITION` (keccak256("compose.erc6909")). All functions operate on the `ERC6909Storage` struct located at this slot. Any facet that correctly imports and accesses this storage slot will see and interact with the same token state, ensuring consistency across the diamond.
</Callout>
