<DocSubtitle>
ERC-6909 minimal multi-token logic and storage.
</DocSubtitle>

<Callout type="info" title="Key Features">
- Implements core ERC-6909 functions: `mint`, `burn`, `transfer`, `approve`, `setOperator`.
- Provides a dedicated storage layout for ERC-6909 state variables.
- Utilizes inline assembly for efficient storage access via `getStorage`.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

The ERC6909Mod provides essential internal functions and the storage layout required for implementing the ERC-6909 standard. This module enables diamonds to manage multi-token balances, approvals, and operator permissions efficiently, adhering to the ERC-6909 specification.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

import {IERC6909Mod, IERC6909Storage, STORAGE_POSITION} from "@compose-protocol/core/contracts/modules/erc6909/ERC6909Mod.sol";

contract ERC6909Facet {
    IERC6909Mod internal erc6909Mod;

    constructor(address _diamondProxy) {
        erc6909Mod = IERC6909Mod(_diamondProxy);
    }

    /**
     * @notice Approves a spender for a specific token ID.
     * @param _id The token ID to approve.
     * @param _spender The address to approve.
     * @param _amount The amount to approve.
     */
    function approve(uint256 _id, address _spender, uint256 _amount) external {
        erc6909Mod.approve(_id, _spender, _amount);
    }

    /**
     * @notice Transfers tokens from one address to another.
     * @param _id The token ID to transfer.
     * @param _from The sender's address.
     * @param _to The recipient's address.
     * @param _amount The amount to transfer.
     */
    function transfer(uint256 _id, address _from, address _to, uint256 _amount) external {
        erc6909Mod.transfer(_id, _from, _to, _amount);
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure the `ERC6909Mod` facet is correctly initialized and its storage slot is reserved.
- Handle potential `ERC6909InsufficientBalance` and `ERC6909InsufficientAllowance` errors explicitly in facet logic.
- Be mindful of operator permissions when implementing `transfer` logic, as they bypass allowance checks.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
The `ERC6909Mod` relies on a specific storage slot defined by `STORAGE_POSITION` for its internal state. Facets interacting with this module must ensure this slot is not used by other modules or facets to prevent collisions. The `getStorage` function provides a direct, albeit low-level, interface to the module's storage struct, enabling efficient state reads.
</Callout>
