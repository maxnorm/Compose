<DocSubtitle>
ERC-6909 token transfers and operator management
</DocSubtitle>

<Callout type="info" title="Key Features">
- Exposes external functions for ERC-6909 token operations.
- Manages token balances and operator approvals using diamond storage.
- Self-contained, following Compose's no-inheritance, no-import philosophy.
- Compatible with ERC-2535 diamond standard for upgradeability.
</Callout>

## Overview

This facet implements ERC-6909 token transfers and operator approvals within a diamond. It exposes external functions for managing token balances and operator status, routing calls through the diamond proxy. Developers add this facet to provide flexible, upgradeable token functionality.

## Best Practices

<Callout type="tip" title="Best Practice">
- Initialize token balances and operator approvals during diamond deployment.
- Ensure the diamond proxy is correctly configured to route calls to this facet.
- Verify storage slot compatibility when upgrading to prevent state corruption.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
Input validation is performed for receiver and sender addresses in transfer functions. Insufficient balance and allowance checks are enforced via custom errors. Operator approval logic is handled by `setOperator` and checked in `transferFrom`. Follow standard Solidity security practices for all interactions.
</Callout>
