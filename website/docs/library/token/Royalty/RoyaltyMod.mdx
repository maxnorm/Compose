<DocSubtitle>
Manage ERC-2981 royalties for tokens and defaults.
</DocSubtitle>

<Callout type="info" title="Key Features">
- Implements ERC-2981 standard for on-chain royalties.
- Supports both default royalties and token-specific overrides.
- Provides functions for setting, deleting, and querying royalty information.
- Utilizes diamond storage access for efficient state management.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

This module provides the core logic for implementing the ERC-2981 royalty standard. It allows setting and querying default royalties applicable to all tokens, as well as token-specific royalties that override the defaults. This composable approach enables diamonds to efficiently manage royalty distributions.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

import {IRoyaltyMod} from "../modules/royalty/IRoyaltyMod.sol";

contract RoyaltyFacet {
    // Assume IRoyaltyMod is correctly interfaced with the diamond proxy
    IRoyaltyMod internal royaltyMod;

    // Initialization would set royaltyMod to the diamond's RoyaltyMod facet address

    /**
     * @notice Sets a specific royalty for a token.
     * @param _tokenId The ID of the token.
     * @param _receiver The address to receive the royalty payment.
     * @param _feeBasisPoints The royalty fee in basis points (e.g., 100 for 1%).
     */
    function setTokenRoyalty(uint256 _tokenId, address _receiver, uint16 _feeBasisPoints) external {
        royaltyMod.setTokenRoyalty(_tokenId, _receiver, _feeBasisPoints);
    }

    /**
     * @notice Gets royalty information for a token.
     * @param _tokenId The ID of the token.
     * @param _salePrice The sale price of the token.
     * @return receiver The address receiving the royalty.
     * @return feeAmount The royalty amount.
     */
    function royaltyInfo(uint256 _tokenId, uint256 _salePrice) external view returns (address receiver, uint256 feeAmount) {
        return royaltyMod.royaltyInfo(_tokenId, _salePrice);
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Always validate `_receiver` and `_feeBasisPoints` before setting royalties to prevent invalid configurations.
- Use `resetTokenRoyalty` to revert to default royalties for a token, ensuring predictable state changes.
- Be aware that `royaltyInfo` will first check for token-specific royalties before falling back to default royalties.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
The RoyaltyMod stores royalty configurations in a predefined storage slot within the diamond. The `getStorage` function provides direct access to this storage struct. Facets interacting with this module should use the `IRoyaltyMod` interface. Token-specific royalties are stored using a mapping within the main royalty storage struct, keyed by `tokenId`. Default royalties are also stored within this struct. Changes to default royalties affect all tokens that do not have specific royalty configurations set.
</Callout>
