<DocSubtitle>
ERC-2981 royalty logic for tokens
</DocSubtitle>

<Callout type="info" title="Key Features">
- Implements ERC-2981 royalty standard with internal functions.
- Manages default and token-specific royalty information.
- Utilizes diamond storage for persistent state management.
- Provides clear error types for invalid royalty configurations.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

This module provides internal functions for managing ERC-2981 royalty information. Facets can import this module to set, reset, and query default and token-specific royalties using shared diamond storage. It ensures consistent royalty calculations across all facets interacting with the diamond.

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure receiver addresses are validated before setting royalties to prevent unintended recipients.
- Call `resetTokenRoyalty` to revert to default royalty settings for a token.
- Handle custom errors `ERC2981InvalidDefaultRoyalty`, `ERC2981InvalidDefaultRoyaltyReceiver`, `ERC2981InvalidTokenRoyalty`, and `ERC2981InvalidTokenRoyaltyReceiver` to manage invalid royalty configurations.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
This module stores royalty information in diamond storage at the slot identified by `keccak256(\"compose.erc2981\")`. The `RoyaltyStorage` struct, containing `defaultRoyaltyInfo`, is managed at this location. Functions like `setDefaultRoyalty` and `setTokenRoyalty` directly modify this storage. Any facet that accesses this storage slot will see the updated royalty information immediately.
</Callout>
