ERC-2981 royalty logic for tokens
</DocSubtitle>

<Callout type="info" title="Key Features">
- Implements ERC-2981 royalty information logic.
- Supports both default and token-specific royalty settings.
- Uses diamond storage for persistent royalty data.
- Provides internal functions for facet integration.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

This module provides internal functions for managing ERC-2981 royalty information, including default and token-specific settings. Facets can import this module to implement royalty queries and updates using shared diamond storage. This enables consistent royalty logic across different parts of a diamond.

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure input validation for receiver addresses and fee numerators before calling `setDefaultRoyalty` or `setTokenRoyalty`.
- Call `resetTokenRoyalty` to revert token-specific royalties to the default setting.
- Handle custom errors like `ERC2981InvalidDefaultRoyalty` and `ERC2981InvalidTokenRoyaltyReceiver` for robust error handling.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
This module utilizes diamond storage at the predefined `STORAGE_POSITION` associated with the `keccak256(\"erc2981\")` hash. It reads and writes to the `RoyaltyStorage` struct, which contains the `defaultRoyaltyInfo`. Changes made via `setDefaultRoyalty` or `setTokenRoyalty` are immediately reflected for all facets that access this storage position.
</Callout>
