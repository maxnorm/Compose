<DocSubtitle>
ERC-2981 royalty logic for tokens
</DocSubtitle>

<Callout type="info" title="Key Features">
- Implements ERC-2981 royalty standard logic internally.
- Manages royalty information using diamond storage.
- Provides functions to set, reset, and query royalties.
- All functions are `internal` for facet composition.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

This module provides internal functions and storage for ERC-2981 royalty logic. Facets can import this module to set, retrieve, and manage default and token-specific royalties using shared diamond storage. It ensures consistent royalty information across all facets interacting with token sales.

## Best Practices

<Callout type="tip" title="Best Practice">
- Use `setDefaultRoyalty` to set a fallback royalty for all tokens.
- Call `setTokenRoyalty` to override default royalties for specific tokens.
- Handle `ERC2981InvalidDefaultRoyalty`, `ERC2981InvalidDefaultRoyaltyReceiver`, `ERC2981InvalidTokenRoyalty`, and `ERC2981InvalidTokenRoyaltyReceiver` errors.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
This module interacts with diamond storage at the slot identified by `keccak256("compose.erc2981")`. It uses the `RoyaltyStorage` struct, which contains `defaultRoyaltyInfo`. Functions like `setTokenRoyalty` and `resetTokenRoyalty` directly modify this shared storage. Changes are immediately visible to any facet accessing this storage slot.
</Callout>
