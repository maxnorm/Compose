<DocSubtitle>
Manages ERC-2981 royalties with default and token-specific settings.
</DocSubtitle>

<Callout type="info" title="Key Features">
- Supports both default and token-specific royalty configurations.
- Implements the standard `royaltyInfo` function as per ERC-2981.
- Provides explicit functions for setting, deleting, and resetting royalty data.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

This module implements ERC-2981 royalty logic, allowing diamonds to define both a default royalty for all tokens and specific royalties for individual tokens. It ensures efficient storage access and provides clear functions for setting, deleting, and querying royalty information, supporting composable NFT royalty standards.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

import {IRoyaltyMod} from "path/to/IRoyaltyMod.sol";

contract RoyaltyConsumerFacet {
    // Assume IRoyaltyMod is correctly interfaced and the diamond proxy is set up.
    IRoyaltyMod internal royaltyMod;

    function _setRoyaltyMod(address _royaltyModAddress) internal {
        royaltyMod = IRoyaltyMod(_royaltyModAddress);
    }

    /**
     * @notice Sets a specific royalty for a token.
     * @param _tokenId The ID of the token.
     * @param _receiver The address to receive royalty payments.
     * @param _feeBasisPoints The royalty fee in basis points (e.g., 500 for 5%).
     */
    function setTokenSpecificRoyalty(uint256 _tokenId, address _receiver, uint256 _feeBasisPoints) external {
        royaltyMod.setTokenRoyalty(_tokenId, _receiver, _feeBasisPoints);
    }

    /**
     * @notice Queries royalty information for a token.
     * @param _tokenId The ID of the token.
     * @param _salePrice The sale price of the token.
     * @return receiver The address receiving the royalty.
     * @return feeAmount The calculated royalty amount.
     */
    function getTokenRoyaltyInfo(uint256 _tokenId, uint256 _salePrice) external view returns (address receiver, uint256 feeAmount) {
        (receiver, feeAmount) = royaltyMod.royaltyInfo(_tokenId, _salePrice);
        return (receiver, feeAmount);
    }

    /**
     * @notice Resets a token's royalty to use the default setting.
     * @param _tokenId The ID of the token.
     */
    function resetTokenSpecificRoyalty(uint256 _tokenId) external {
        royaltyMod.resetTokenRoyalty(_tokenId);
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Always validate receiver addresses and fee basis points before setting royalties to prevent errors.
- Use `resetTokenRoyalty` to revert token-specific settings, ensuring clarity and predictable fallback behavior.
- Be aware that `deleteDefaultRoyalty` will cause `royaltyInfo` to return `(address(0), 0)` for tokens without specific overrides.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
The RoyaltyMod stores royalty data in a predefined slot within the diamond's storage. The `getStorage` function provides direct access to this storage, which is crucial for internal operations. Facets interacting with this module should be aware that modifications to default or token-specific royalties are immediately reflected in subsequent calls to `royaltyInfo`. The order of operations matters: setting token-specific royalties takes precedence over default royalties, and resetting a token's royalty reverts it to using the default.
</Callout>
