ERC-2981 royalty information for tokens
</DocSubtitle>

<Callout type="info" title="Key Features">
- Implements ERC-2981 `royaltyInfo` function.
- Retrieves royalty data from diamond shared storage.
- No external dependencies beyond diamond proxy interaction.
- Facilitates on-chain royalty standard compliance.
</Callout>

## Overview

This facet provides ERC-2981 compliant royalty information retrieval for tokens within a Compose diamond. It exposes the `royaltyInfo` function, allowing external callers to query royalty details based on token ID and sale price. The facet accesses royalty data from the diamond's shared storage.

## Functions

### royaltyInfo

Returns royalty information for a given token and sale price. Returns token-specific royalty if set, otherwise falls back to default royalty. Royalty amount is calculated as a percentage of the sale price using basis points. Implements the ERC-2981 royaltyInfo function.

<ExpandableCode language="solidity" maxLines={8}>

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure the `RoyaltyFacet` is correctly initialized with default royalty information during diamond setup.
- Calls to `royaltyInfo` are performed through the diamond proxy address.
- Verify that the `STORAGE_POSITION` for royalty storage is correctly defined and unique.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
The `royaltyInfo` function is a `view` function and does not modify state. Input validation for `_tokenId` and `_salePrice` is handled internally by the function logic. Access control is not explicitly managed by this facet, relying on the diamond's overall access control mechanisms if any state-modifying actions were to be added.
</Callout>
