<DocSubtitle>
Manages ERC-2981 royalty information for tokens.
</DocSubtitle>

<Callout type="info" title="Key Features">
- Implements the ERC-2981 standard for royalty payments.
- Supports token-specific royalty configurations.
- Falls back to a default royalty configuration when token-specific settings are absent.
- Calculates royalties based on basis points of the sale price.
</Callout>

## Overview

The RoyaltyFacet implements the ERC-2981 standard, enabling tokens within a Compose diamond to return royalty information. It supports both token-specific and default royalty configurations, calculated as a percentage of the sale price.

## Functions

### royaltyInfo

Returns royalty information for a given token and sale price. Returns token-specific royalty if set, otherwise falls back to default royalty. Royalty amount is calculated as a percentage of the sale price using basis points. Implements the ERC-2981 royaltyInfo function.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

import {IRoyaltyFacet} from "@compose/diamond/facets/Royalty/IRoyaltyFacet.sol";
import {IDiamond} from "@compose/diamond/IDiamond.sol";

contract RoyaltyConsumer {
    IDiamond immutable diamond;

    constructor(address _diamondAddress) {
        diamond = IDiamond(_diamondAddress);
    }

    function getRoyaltyInfo(uint256 _tokenId, uint256 _salePrice) external view returns (address receiver, uint256 royaltyAmount) {
        // Get the RoyaltyFacet selector
        bytes4 royaltySelector = IRoyaltyFacet.royaltyInfo.selector;

        // Call the royaltyInfo function through the diamond proxy
        (bool success, bytes memory data) = address(diamond).call(abi.encodeWithSelector(royaltySelector, _tokenId, _salePrice));
        require(success, "RoyaltyFacet: call failed");

        // Decode the result
        (receiver, royaltyAmount) = abi.decode(data, (address, uint256));
        return (receiver, royaltyAmount);
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Initialize RoyaltyFacet with default royalty settings during diamond deployment.
- Use the `getStorage` function to access and modify royalty storage if needed, ensuring proper access control.
- Ensure the `RoyaltyFacet` is correctly added to the diamond's facet registry.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
Access to modify royalty settings should be restricted to authorized addresses. The `royaltyInfo` function itself is read-only and does not pose reentrancy risks. Ensure the basis points for royalties are validated to prevent excessive amounts.
</Callout>
