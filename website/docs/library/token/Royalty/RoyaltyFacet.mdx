<DocSubtitle>
Returns royalty information for NFTs
</DocSubtitle>

<Callout type="info" title="Key Features">
- Implements ERC-2981 royalty standard.
- Provides external `royaltyInfo` function for querying royalty details.
- Accesses royalty storage via internal `getStorage` function.
- Designed for composability within the Compose diamond pattern.
</Callout>

## Overview

This facet implements the ERC-2981 standard for royalty payments within a Compose diamond. It exposes the `royaltyInfo` function to query royalty details for a given token and sale price, routing calls through the diamond proxy. Developers can add this facet to enable royalty functionality for their NFTs.

## Functions

### royaltyInfo

Returns royalty information for a given token and sale price. Returns token-specific royalty if set, otherwise falls back to default royalty. Royalty amount is calculated as a percentage of the sale price using basis points. Implements the ERC-2981 royaltyInfo function.

<ExpandableCode language="solidity" maxLines={8}>
{`function royaltyInfo(uint256 _tokenId, uint256 _salePrice)
    external
    view
    returns (address receiver, uint256 royaltyAmount);`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure the `RoyaltyFacet` is correctly initialized with default royalty information if needed.
- Calls to `royaltyInfo` are read-only and do not require special access control beyond what the diamond proxy enforces.
- Verify storage slot compatibility when upgrading facets to prevent data corruption.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
The `royaltyInfo` function is a view function and does not modify state, thus posing minimal reentrancy risk. Input validation for `_tokenId` and `_salePrice` is handled internally. Follow standard Solidity security practices for diamond deployment and facet management.
</Callout>
