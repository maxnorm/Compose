<Callout type="info" title="Key Features">
- Exposes internal functions for direct use within diamond facets.
- Manages pending owner data using the diamond storage pattern.

- Designed for integration with ERC-173 ownership patterns.

This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.

## Overview

This module provides the data storage and retrieval for a two-step ownership transfer mechanism. It exposes internal functions to access and manage the pending owner address, enabling composable access control patterns within a diamond.

## Best Practices

<Callout type="tip" title="Best Practice">
- Call `pendingOwner()` to retrieve the pending owner address.
- Use `getStorage()` to access the raw storage struct for advanced interactions, understanding storage slot implications.
- Ensure any facet interacting with this data adheres to the two-step ownership transfer logic.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
This module utilizes diamond storage at `STORAGE_POSITION`, identified by `keccak256(\"erc173.owner.pending\")`. The `PendingOwnerStorage` struct holds the `pendingOwner` address. All functions are internal, allowing facets to directly interact with the shared storage, ensuring that changes to the pending owner are immediately visible across all facets utilizing this module and storage slot.
</Callout>
