<Callout type="info" title="Key Features">
- Implements ERC-173 ownership standard.
- Internal functions for seamless facet integration.
- Manages ownership directly within the diamond storage pattern.
- Supports renouncing ownership by setting the owner to `address(0)`.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

This module provides ERC-173 ownership transfer logic, allowing diamonds to manage contract ownership. Facets can import this module to transfer ownership or renounce it, ensuring a clear and auditable ownership chain. Ownership changes are managed directly in diamond storage, making them immediately visible to all facets.

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure ownership transfers are initiated by the current owner, or handle `OwnerUnauthorizedAccount` errors.
- Carefully manage the renouncement of ownership; it cannot be recovered.
- Integrate ownership transfer logic early in the diamond deployment or upgrade process.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
This module utilizes the diamond storage pattern, specifically referencing the storage slot identified by `STORAGE_POSITION` (keccak256("erc173.owner")). The `OwnerStorage` struct, containing the `owner` field, is accessed and modified through internal functions. Any facet interacting with this storage slot will see the updated owner address immediately after a successful transferOwnership call.
</Callout>
