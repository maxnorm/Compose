<DocSubtitle>
Manages contract ownership and transfers
</DocSubtitle>

<Callout type="info" title="Key Features">
- Exposes external functions for ownership management.
- Integrates with the diamond storage pattern using a defined storage slot.
- Provides `owner`, `transferOwnership`, and `renounceOwnership` functions.
- Emits `OwnershipTransferred` event upon ownership change.
</Callout>

## Overview

This facet provides ownership management for a Compose diamond. It exposes functions to retrieve the current owner, transfer ownership to a new address, and renounce ownership. Developers integrate this facet to establish a clear owner for administrative tasks and upgrades within the diamond pattern.

## Best Practices

<Callout type="tip" title="Best Practice">
- Initialize the owner address during diamond deployment.
- Use `transferOwnership` to safely transfer control to a new address.
- Call `renounceOwnership` only when the contract should become ownerless.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
The `transferOwnership` function should be called by the current owner. Input validation for `_newOwner` is handled by the function logic (setting to address(0) renounces ownership). Access to state-changing functions is implicitly controlled by the diamond's routing mechanism, ensuring only the facet's intended functions are callable externally. Follow standard Solidity security practices.
</Callout>
