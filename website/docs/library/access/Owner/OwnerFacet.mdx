<DocSubtitle>
Manages diamond ownership and transfer.
</DocSubtitle>

<Callout type="info" title="Key Features">
- Manages diamond ownership and administrative control.
- Supports ownership transfer and renouncement.
- Emits `OwnershipTransferred` event upon successful ownership changes.
</Callout>

## Overview

The OwnerFacet provides essential functions for managing the ownership of a Compose diamond. It allows for retrieving the current owner, transferring ownership to a new address, and renouncing ownership entirely. This facet is crucial for controlling administrative access and upgradeability.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

import {IOwnerFacet} from "@compose/diamond/facets/Owner/IOwnerFacet.sol";

contract OwnerFacetConsumer {
    IOwnerFacet immutable ownerFacet;

    constructor(address _diamondProxyAddress) {
        ownerFacet = IOwnerFacet(_diamondProxyAddress);
    }

    function getCurrentOwner() external view returns (address) {
        return ownerFacet.owner();
    }

    function transferDiamondOwnership(address _newOwner) external {
        ownerFacet.transferOwnership(_newOwner);
    }

    function renounceDiamondOwnership() external {
        ownerFacet.renounceOwnership();
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Only the current owner should call `transferOwnership` and `renounceOwnership`.
- Use `transferOwnership(_newOwner)` to delegate ownership before renouncing.
- Ensure the `_newOwner` address is valid and intended before execution.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
Access to `transferOwnership` and `renounceOwnership` is restricted to the current owner. Care must be taken when transferring ownership to ensure the new owner is trusted, as they will have administrative control over the diamond. Renouncing ownership makes the contract immutable and un-upgradable unless other mechanisms are in place.
</Callout>
