<DocSubtitle>
Manages contract ownership and ownership transfers
</DocSubtitle>

<Callout type="info" title="Key Features">
- Exposes external functions for ownership management.
- Uses a dedicated storage slot for owner state.
- Supports ownership transfer and renouncement.
- Compatible with ERC-2535 diamond standard.
</Callout>

## Overview

This facet provides ownership management functionality for a diamond. It exposes functions to view the current owner, transfer ownership, and renounce ownership. Developers integrate this facet to establish a clear owner for administrative tasks within the diamond.

## Best Practices

<Callout type="tip" title="Best Practice">
- Initialize the owner address during diamond deployment.
- Use `transferOwnership` to safely delegate administrative control.
- Call `renounceOwnership` only when the contract should no longer have a designated owner.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
All state-changing functions (`transferOwnership`, `renounceOwnership`) are protected by access control, reverting with `OwnerUnauthorizedAccount` if called by an unauthorized address. Use `transferOwnership` carefully to avoid unintended ownership changes. Renouncing ownership permanently removes the owner.
</Callout>
