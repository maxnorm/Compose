<Callout type="info" title="Key Features">
- Provides internal functions for owner management.
- Uses diamond storage pattern for ownership state.
- Compatible with ERC-173 contract ownership standard.
- Minimal dependencies, designed for facet composition.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

This module provides internal functions for managing ERC-173 contract ownership. Facets can import this module to check ownership and transfer ownership using shared diamond storage. Changes to ownership are immediately visible to all facets accessing the same storage.

## Best Practices

<Callout type="tip" title="Best Practice">
- Call `transferOwnership` to change the owner, setting to `address(0)` to renounce ownership.
- Use `requireOwner` to enforce ownership checks before critical operations.
- Ensure `OwnerStorage` is correctly integrated into the diamond's storage layout.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
This module utilizes diamond storage at the slot identified by `keccak2535(\"compose.owner\")`. The `OwnerStorage` struct, containing at least an `owner` field, is managed at this position. All functions operate internally, directly accessing and modifying this shared storage, making ownership changes universally visible across facets.
</Callout>
