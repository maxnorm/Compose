<DocSubtitle>
Manages ERC-173 contract ownership.
</DocSubtitle>

<Callout type="info" title="Key Features">
- Implements ERC-173 contract ownership.
- Provides `owner()`, `transferOwnership()`, and `requireOwner()` functions.
- Supports ownership renouncement by transferring to `address(0)`.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

The OwnerMod provides the fundamental storage and functions for managing contract ownership according to the ERC-173 standard. This enables secure control and administration of diamond functions, ensuring that critical operations can only be performed by the designated owner.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

import {IOwnerMod, OwnerMod} from "../modules/OwnerMod.sol";

contract MyOwnerFacet {
    // Assume OwnerMod is deployed and its storage is accessible.
    // The STORAGE_POSITION for OwnerMod is typically defined externally.
    uint256 constant OWNER_MOD_STORAGE_SLOT = 1; // Example slot

    struct OwnerModStorage {
        address _owner;
    }

    function getOwnerModStorage() internal pure returns (OwnerModStorage storage) {
        uint256 slot = OWNER_MOD_STORAGE_SLOT;
        assembly {
            slot := and(slot, 0xFFFFFFFFFFFFFFFF)
        }
        return OwnerModStorage(slot);
    }

    function owner() public view returns (address) {
        return OwnerMod(address(this)).owner();
    }

    function transferOwnership(address _newOwner) external {
        // Call the OwnerMod's transferOwnership function, which will interact with its storage.
        // This typically happens via the diamond proxy.
        OwnerMod(address(this)).transferOwnership(_newOwner);
    }

    function requireOwner() internal view {
        // This function would likely be called by other functions within this facet
        // to enforce ownership checks before proceeding.
        OwnerMod(address(this)).requireOwner();
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Call `transferOwnership` with `address(0)` to renounce ownership, making the contract immutable.
- Ensure the `OwnerMod` is initialized with the correct owner during deployment.
- Use `requireOwner()` internally within facets to protect sensitive administrative functions.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
The OwnerMod utilizes a dedicated storage slot for its `ERC173Storage` struct. Facets that need to interact with ownership logic can access this storage via the `getStorage` function or by calling the module's exported functions through the diamond proxy. The owner address is a critical invariant for the diamond; any changes must be carefully managed to maintain control.
</Callout>
