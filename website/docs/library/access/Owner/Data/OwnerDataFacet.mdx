<DocSubtitle>
Manages the owner address for a diamond
</DocSubtitle>

<Callout type="info" title="Key Features">
- Provides external `owner()` function to query the diamond's owner.
- Exposes internal `getStorage()` for facet-level storage access.
- Includes `exportSelectors()` for diamond upgradeability and discovery.
- Self-contained, adhering to Compose facet design principles.
</Callout>

## Overview

This facet provides direct access to the owner address of a Compose diamond. It exposes an external function to retrieve the owner's address and an internal function to access its storage structure. Developers add this facet to manage ownership and integrate with other access control facets.

## Best Practices

<Callout type="tip" title="Best Practice">
- Initialize the diamond with this facet to establish ownership.
- Use the `owner()` function to retrieve the current owner address.
- Ensure `exportSelectors()` is called during diamond deployment to register facet selectors.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
The `owner()` function is a view function and does not modify state, posing minimal risk. Input validation is not applicable as there are no user-supplied parameters for `owner()` or `getStorage()`. The `exportSelectors()` function is pure and has no security implications. Follow standard Solidity security practices for diamond interactions.
</Callout>
