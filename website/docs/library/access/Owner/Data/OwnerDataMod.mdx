<DocSubtitle>
Manages ERC-173 contract ownership internally
</DocSubtitle>

<Callout type="info" title="Key Features">
- Provides internal functions for ERC-173 ownership management.
- Uses diamond storage (EIP-8042) for owner state.
- `requireOwner()` enforces caller authorization based on diamond owner.
- Exposes `getStorage()` for direct access to ownership state.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

This module provides internal functions and storage for managing ERC-173 contract ownership. Facets can import this module to interact with shared ownership data, ensuring consistent owner checks and transfers across the diamond. Ownership changes are immediately reflected for all facets utilizing the diamond storage pattern.

## Best Practices

<Callout type="tip" title="Best Practice">
- Call `requireOwner()` before executing sensitive operations to enforce ownership.
- Ensure `OwnerDataMod` is correctly initialized with the intended owner address during deployment.
- Handle the `OwnerUnauthorizedAccount` and `OwnerAlreadyRenounced` errors, which may be emitted by internal `OwnerDataMod` functions if called inappropriately.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
This module utilizes the diamond storage pattern, storing ownership data at a specific slot identified by `STORAGE_POSITION` which resolves to `keccak256(\"erc173.owner\")`. The `OwnerStorage` struct contains the `owner` address. All functions in this module directly access and modify this shared storage, making any ownership changes immediately visible to all facets interacting with the diamond's storage.
</Callout>
