<DocSubtitle>
Renounces roles from an account within a diamond
</DocSubtitle>

<Callout type="info" title="Key Features">
- Exposes `renounceRole` for role revocation.
- Emits `RoleRevoked` event upon successful renouncement.
- Self-contained, adhering to ERC-2535 standards.
- Exports facet selectors using `exportSelectors`.
</Callout>

## Overview

This facet provides functionality to renounce roles from an account within a Compose diamond. It exposes the `renounceRole` function, allowing users to revoke their own roles. The facet interacts with the diamond's shared storage to manage role assignments and emits `RoleRevoked` events upon successful renouncement.

## Functions

### renounceRole

Renounces a role from the caller. Emits a RoleRevoked event. Reverts with AccessControlUnauthorizedSender If the caller is not the account to renounce the role from.

## Best Practices

<Callout type="tip" title="Best Practice">
- Call `renounceRole` through the diamond proxy to ensure correct routing.
- Verify that the caller is the account from which the role is being renounced before execution.
- Ensure the `AccessControlStorage` struct is compatible if upgrading or adding facets.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
The `renounceRole` function reverts with `AccessControlUnauthorizedSender` if the caller is not the account specified for renouncement. Input validation is handled by the function's internal checks. Follow standard Solidity security practices for diamond interactions.
</Callout>
