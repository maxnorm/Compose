<DocSubtitle>
Renounce roles for accounts in Compose diamonds
</DocSubtitle>

<Callout type="info" title="Key Features">
- Allows accounts to renounce their own roles.
- Emits a `RoleRevoked` event upon successful renouncement.
- Internal function designed for use within diamond facets.
- Utilizes diamond storage for role management.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

This module provides the `renounceRole` function for users to revoke their own assigned roles within a Compose diamond. By calling this function, an account can permanently remove its access associated with a specific role. This enhances security by allowing users to relinquish permissions when no longer needed, without requiring administrative intervention.

## Best Practices

<Callout type="tip" title="Best Practice">
- Call `renounceRole` with the correct role and your own address to revoke permissions.
- Verify the `AccessControlUnauthorizedSender` error is handled if attempting to renounce a role for another account.
- Ensure the `AccessControlStorage` struct is correctly initialized and accessible via `AccessControlRenounceMod.getStorage()`.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
This module interacts with the diamond's shared storage via the `AccessControlStorage` struct. The `renounceRole` function directly modifies this storage. Any facet that reads from `AccessControlStorage` will immediately see the updated role assignments after `renounceRole` is called. The storage is accessed through a dedicated slot identified by `STORAGE_POSITION`.
</Callout>
