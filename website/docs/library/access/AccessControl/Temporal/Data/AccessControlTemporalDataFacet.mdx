<DocSubtitle>
Manages temporal role expiry and validation
</DocSubtitle>

<Callout type="info" title="Key Features">
- Exposes functions for checking and enforcing temporal role validity.
- Utilizes custom errors `AccessControlUnauthorizedAccount` and `AccessControlRoleExpired`.
- Provides internal functions `getAccessControlStorage` and `getStorage` for internal access.
- Exports its selectors via `exportSelectors` for diamond registration.
</Callout>

## Overview

This facet provides temporal access control by managing role expiry timestamps and validating role access. It exposes functions to check if a role has expired and to enforce valid role checks, reverting with specific errors upon failure. Developers integrate this facet to implement time-bound permissions within a Compose diamond.

## Best Practices

<Callout type="tip" title="Best Practice">
- Integrate this facet into your diamond to manage time-bound role assignments.
- Use `requireValidRole` to enforce temporal access control on critical functions.
- Ensure proper initialization of role expiry timestamps via granting mechanisms.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
All external-facing functions (`getRoleExpiry`, `isRoleExpired`, `requireValidRole`) are view functions and do not modify state. The `requireValidRole` function enforces temporal access control by reverting with `AccessControlRoleExpired` if the role's expiry timestamp has passed, or `AccessControlUnauthorizedAccount` if the account does not possess the role. Follow standard Solidity security practices for input validation on any functions that interact with granting mechanisms.
</Callout>
