<DocSubtitle>
Manage role expiry and validity in diamonds
</DocSubtitle>

<Callout type="info" title="Key Features">
- Provides internal functions for temporal access control checks.
- Utilizes the diamond storage pattern for shared state management.
- Reverts with specific errors (`AccessControlRoleExpired`, `AccessControlUnauthorizedAccount`) for clarity.
- Compatible with ERC-2535 diamonds.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

This module provides internal functions for managing role expiry within a diamond's access control system. Facets can use these functions to check if a role has expired or to validate an account's role status, leveraging shared diamond storage. Changes to role expiry are immediately visible across all facets accessing the same storage.

## Best Practices

<Callout type="tip" title="Best Practice">
- Call `requireValidRole` to enforce role expiry checks before executing sensitive operations.
- Use `isRoleExpired` for conditional logic based on role expiry status.
- Ensure the `AccessControlTemporalDataMod` is correctly initialized within your facet.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
This module interacts with diamond storage via the `ACCESS_CONTROL_STORAGE_POSITION`. It accesses the `AccessControlTemporalStorage` struct. Functions are internal and operate directly on shared diamond storage, making any temporal role changes immediately visible to all facets.
</Callout>
