<DocSubtitle>
Revoke temporal roles from accounts
</DocSubtitle>

<Callout type="info" title="Key Features">
- All functions are `internal` for use within custom facets.
- Leverages the diamond storage pattern for shared state management.
- Emits a `TemporalRoleRevoked` event upon successful revocation.
- Requires callers to be the admin of the role being revoked.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

This module provides internal functions to revoke temporal roles from accounts within a diamond. It interacts with shared diamond storage, making role revocation immediately visible to all facets. Use this module to manage time-bound permissions in a composable manner.

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure that the caller has the necessary permissions to revoke roles before calling `revokeTemporalRole`.
- Verify that the diamond storage layout remains compatible after upgrades to prevent data corruption.
- Handle the `AccessControlUnauthorizedAccount` error when the caller lacks permission to revoke the specified role.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
This module accesses and modifies diamond storage at the `ACCESS_CONTROL_STORAGE_POSITION`, identified by `keccak256("compose.accesscontrol")`. The `AccessControlStorage` and `AccessControlTemporalStorage` structs are used to manage role data. Changes made to the storage via `revokeTemporalRole` are immediately visible to all facets that interact with the same storage position.
</Callout>
