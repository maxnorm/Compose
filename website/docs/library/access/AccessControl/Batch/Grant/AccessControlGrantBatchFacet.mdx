<DocSubtitle>
Grants roles to multiple accounts efficiently
</DocSubtitle>

<Callout type="info" title="Key Features">
- Grants roles to multiple accounts in a single transaction.
- Emits `RoleGranted` event for each granted account.
- Reverts with `AccessControlUnauthorizedAccount` if the caller lacks permission.
- Exports its selectors via `exportSelectors`.
</Callout>

## Overview

This facet provides batched role granting functionality for Compose diamonds. It enables efficient assignment of roles to multiple accounts in a single transaction, reducing gas costs and complexity. Developers can integrate this facet to streamline access control setup and management within their diamond.

## Best Practices

<Callout type="tip" title="Best Practice">
- Initialize roles and grant initial administrative roles using the `grantRoleBatch` function during diamond deployment.
- Ensure the caller of `grantRoleBatch` has the necessary administrative permissions for the specified role.
- Verify the `AccessControlStorage` slot is correctly configured and accessible by this facet.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
All state-changing functions must be protected by appropriate access control mechanisms. The `grantRoleBatch` function ensures that only authorized accounts can grant roles. Input validation on `_accounts` is handled internally. Follow standard Solidity security practices for handling roles and permissions.
</Callout>
