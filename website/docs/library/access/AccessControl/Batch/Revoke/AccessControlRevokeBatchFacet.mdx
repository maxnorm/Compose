<DocSubtitle>
Revokes roles from multiple accounts efficiently
</DocSubtitle>

<Callout type="info" title="Key Features">
- Efficiently revokes roles from multiple accounts in one transaction.
- Emits `RoleRevoked` event for each account processed.
- Exposes `exportSelectors` for diamond upgrade compatibility.
</Callout>

## Overview

This facet provides an efficient way to revoke a specific role from multiple accounts within a Compose diamond. It exposes the `revokeRoleBatch` function, allowing developers to manage access control by removing permissions in a single on-chain transaction. This reduces gas costs and simplifies bulk permission management.

## Functions

### revokeRoleBatch

Revokes a role from multiple accounts in a single transaction. Emits a RoleRevoked event for each account the role is revoked from. Reverts with AccessControlUnauthorizedAccount If the caller is not the admin of the role.

## Best Practices

<Callout type="tip" title="Best Practice">
- Integrate this facet into your diamond during deployment.
- Ensure the caller has the necessary administrative permissions to revoke roles.
- Use `exportSelectors` during diamond upgrade to correctly update facet mappings.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
Reverts with `AccessControlUnauthorizedAccount` if the caller does not have the required administrative role. Input validation is handled by the underlying diamond proxy and facet logic.
</Callout>
