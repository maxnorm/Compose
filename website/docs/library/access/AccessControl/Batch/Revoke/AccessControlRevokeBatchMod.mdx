<DocSubtitle>
Revoke roles from multiple accounts efficiently
</DocSubtitle>

<Callout type="info" title="Key Features">
- Provides an internal function `revokeRoleBatch` for batch role revocations.
- Utilizes the diamond storage pattern (EIP-8042) for state management.
- Emits a `RoleRevoked` event for each account a role is successfully revoked from.
- Reverts with `AccessControlUnauthorizedAccount` if the caller lacks the required permissions.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

This module provides an internal function for revoking a specified role from multiple accounts in a single transaction. It leverages the diamond storage pattern for efficient access control management. By batching revocations, it reduces gas costs and transaction complexity for common administrative tasks.

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure the caller has the necessary permissions to revoke the specified role before executing `revokeRoleBatch`.
- Handle the `AccessControlUnauthorizedAccount` error that may be returned if the caller lacks authorization.
- Verify that `AccessControlStorage` is correctly initialized and accessible by this module within the diamond's storage pattern.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">

The `AccessControlStorage` struct holds the state for role management. All functions within this module are internal and operate directly on the diamond's shared storage, ensuring that changes are immediately visible to all facets that utilize the same storage pattern.
