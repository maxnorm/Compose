<DocSubtitle>
Revokes roles from accounts within a diamond
</DocSubtitle>

<Callout type="info" title="Key Features">
- Exposes `internal` functions for role revocation.
- Uses the diamond storage pattern for role management.
- Emits `RoleRevoked` event upon successful revocation.
- Reverts with `AccessControlUnauthorizedAccount` if the caller lacks the required permissions.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

This module provides internal functions for revoking roles assigned to accounts. Facets can import this module to manage access control by removing role permissions using shared diamond storage. Changes are immediately visible to all facets interacting with the same storage.

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure the caller has the necessary administrative privileges before calling `revokeRole`.
- Handle the `AccessControlUnauthorizedAccount` custom error if the function is called by an unauthorized entity.
- Verify that the `AccessControlStorage` struct in diamond storage is compatible during upgrades.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
This module interacts with diamond storage at the `STORAGE_POSITION` dedicated to access control, identified by `keccak256(\"compose.accesscontrol\")`. The `revokeRole` function directly modifies the `AccessControlStorage` struct. Any facet reading from this storage position will immediately observe the updated role assignments.
</Callout>
