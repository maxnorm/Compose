<DocSubtitle>
Revokes roles from accounts within a diamond
</DocSubtitle>

<Callout type="info" title="Key Features">
- Exposes `revokeRole` for revoking permissions.
- Utilizes diamond storage for role management.
- Emits `RoleRevoked` event upon successful revocation.
- Integrates seamlessly with other Access Control facets.
</Callout>

## Overview

This facet provides functionality to revoke roles from accounts in a Compose diamond. It exposes the `revokeRole` function, which is callable through the diamond proxy. This facet integrates with other Access Control facets to manage permissions comprehensively within the diamond storage pattern.

## Best Practices

<Callout type="tip" title="Best Practice">
- Initialize the AccessControl facet during diamond setup to configure initial roles and administrators.
- Enforce access control on the `revokeRole` function to ensure only authorized entities can revoke permissions.
- Ensure storage compatibility with other Access Control facets when upgrading or adding new facets.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
The `revokeRole` function reverts with `AccessControlUnauthorizedAccount` if the caller lacks the required administrator role for the specified role. Input validation on `_role` and `_account` is handled by the facet implementation. Follow standard Solidity security practices for access control and reentrancy.
</Callout>
