<DocSubtitle>
Manage role assignments and checks using diamond storage
</DocSubtitle>

<Callout type="info" title="Key Features">
- Provides `internal` functions for role management and checks.
- Utilizes the diamond storage pattern for shared access control state.
- Reverts with `AccessControlUnauthorizedAccount` error on failed checks.
- No external dependencies, promoting composability.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

This module provides internal functions for managing role assignments and checking permissions within a diamond. Facets can import this module to interact with shared access control state stored in the diamond. Access checks are performed directly against the diamond's storage, ensuring consistency across all facets.

## Best Practices

<Callout type="tip" title="Best Practice">
- Call `requireRole` with specific roles before executing sensitive operations.
- Ensure the diamond's storage slot for `AccessControlStorage` is correctly initialized.
- Integrate with `AccessControlAdminMod` for managing roles and role assignments.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
This module reads and writes to the diamond's storage at the `STORAGE_POSITION` defined by `keccak256("compose.accesscontrol")`. The `AccessControlStorage` struct holds the role assignment data. All functions are internal, intended to be called by facets that compose this module, ensuring that changes to role assignments are immediately reflected across all facets accessing the same storage slot.
</Callout>
