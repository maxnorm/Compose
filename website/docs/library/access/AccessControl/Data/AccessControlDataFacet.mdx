<DocSubtitle>
Manages roles and permissions for diamond access control
</DocSubtitle>

<Callout type="info" title="Key Features">
- Exposes external functions for role checking and enforcement.
- Utilizes diamond storage for role assignments.
- Provides `getRoleAdmin` for role hierarchy inspection.
- Exports its selectors for diamond facet discovery.
</Callout>

## Overview

This facet manages roles and permissions for a diamond, providing functions to check and enforce access control. It integrates with diamond storage to store role assignments. Developers add this facet to control access to diamond functions based on assigned roles.

## Best Practices

<Callout type="tip" title="Best Practice">
- Integrate this facet into your diamond to manage role-based access control.
- Utilize `hasRole` and `requireRole` to enforce permissions on critical functions.
- Consult `AccessControlAdminFacet` and batch facets for role management operations.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
All functions are view functions and do not modify state. Input validation is handled by the caller or related facets. Reverts with `AccessControlUnauthorizedAccount` if `requireRole` fails.
</Callout>
