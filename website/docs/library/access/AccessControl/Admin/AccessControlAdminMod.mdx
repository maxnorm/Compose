<Callout type="info" title="Key Features">
- Internal functions for role administration.
- Uses the diamond storage pattern for shared state.
- Emits `RoleAdminChanged` event upon successful updates.
- Reverts with `AccessControlUnauthorizedAccount` for unauthorized calls.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

This module provides internal functions to manage the administrator role for other roles within a diamond. Facets can import this module to grant or revoke administrative privileges on roles, ensuring proper delegation of access control management. Changes made through this module are immediately reflected in the diamond's shared access control storage.

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure the caller possesses the current administrative role before calling `setRoleAdmin`.
- Handle the `AccessControlUnauthorizedAccount` error to gracefully manage unauthorized attempts.
- Use `getStorage()` to retrieve the correct storage instance when initializing the module.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
This module interacts with diamond storage at the `STORAGE_POSITION` identified by `keccak2535.keccak256("compose.accesscontrol")`. All functions are internal and operate directly on the `AccessControlStorage` struct. Changes to role administration are immediately visible to any facet accessing this storage.
</Callout>
