<DocSubtitle>
Manages role pausing and unpausing within a diamond
</DocSubtitle>

<Callout type="info" title="Key Features">
- Exposes external functions for role pausing and unpausing through the diamond proxy.
- Integrates with diamond storage pattern for persistent state management.
- Provides `isRolePaused` to query role status and `requireRoleNotPaused` for conditional checks.
- Exports its selectors for easy integration into diamond facet lists.
</Callout>

## Overview

This facet implements role-based pausing functionality for a diamond. It allows administrators to temporarily disable specific roles, preventing any account from using them. The facet integrates with the diamond's access control system and utilizes diamond storage for state management. Developers add this facet to control role availability without altering core diamond logic.

## Best Practices

<Callout type="tip" title="Best Practice">
- Initialize the AccessControlPausableFacet and its storage during diamond deployment.
- Ensure only authorized addresses (e.g., diamond owner or designated admin role) can call `pauseRole` and `unpauseRole`.
- Verify that the `AccessControlPausableFacet` selectors are correctly added to the diamond's facet registry.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
All state-changing functions (`pauseRole`, `unpauseRole`) revert with `AccessControlUnauthorizedAccount` if the caller is not the admin of the role. The `requireRoleNotPaused` function reverts with `AccessControlRolePaused` if the specified role is paused, and `AccessControlUnauthorizedAccount` if the account does not possess the role. Follow standard Solidity security practices for input validation and access control.
</Callout>
