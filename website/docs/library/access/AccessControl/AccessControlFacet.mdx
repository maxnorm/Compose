<DocSubtitle>
Manages roles and permissions within a Compose diamond.
</DocSubtitle>

<Callout type="info" title="Key Features">
- Role-based access control for granular permission management.
- Supports batch granting and revoking of roles for efficiency.
- Provides functions to query role assignments and role admin configurations.
</Callout>

## Overview

The AccessControlFacet provides a robust role-based access control system for Compose diamonds. It enables granular permission management by defining roles, assigning them to accounts, and enforcing role-based access to functions. This facet is crucial for securing administrative operations and controlling access to sensitive functionalities.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

import {DiamondCutFacet} from "@compose/diamond-proxy/facets/DiamondCutFacet.sol";
import {AccessControlFacet} from "@compose/diamond-proxy/facets/AccessControlFacet.sol";
import {IDiamondCut} from "@compose/diamond-proxy/interfaces/IDiamondCut.sol";
import {IDiamondLoupe} from "@compose/diamond-proxy/interfaces/IDiamondLoupe.sol";

contract Deployer {
    address constant DIAMOND_CUT_FACET_ADDRESS = address(0x...); // Replace with actual address
    address constant ACCESS_CONTROL_FACET_ADDRESS = address(0x...); // Replace with actual address

    function deployDiamond() public {
        // Assume diamond proxy is already deployed and accessible via IDiamondLoupe
        IDiamondLoupe diamond = IDiamondLoupe(address(0x...)); // Replace with actual diamond address
        IDiamondCut diamondCut = IDiamondCut(DIAMOND_CUT_FACET_ADDRESS);

        // Grant DEFAULT_ADMIN_ROLE to the deployer
        AccessControlFacet(ACCESS_CONTROL_FACET_ADDRESS).grantRole(diamond.DEFAULT_ADMIN_ROLE(), msg.sender);

        // Example: Grant ACCESS_CONTROL_ADMIN_ROLE to another address
        address adminAddress = address(0x123);
        bytes32 accessControlAdminRole = keccak256("ACCESS_CONTROL_ADMIN_ROLE"); // Example role
        AccessControlFacet(ACCESS_CONTROL_FACET_ADDRESS).grantRole(accessControlAdminRole, adminAddress);
    }

    function checkRole(address account, bytes32 role) public view returns (bool) {
        return AccessControlFacet(ACCESS_CONTROL_FACET_ADDRESS).hasRole(role, account);
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Initialize roles and grant initial permissions during diamond deployment. Configure `DEFAULT_ADMIN_ROLE` appropriately.
- Use the `grantRoleBatch` and `revokeRoleBatch` functions for efficient management of multiple accounts for the same role.
- Carefully manage the `ADMIN_ROLE` for each role to prevent unauthorized role management operations.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
Ensure that the caller is authorized to grant or revoke roles, especially for administrative roles. Reentrancy is not a direct concern as most functions modify state and do not call external contracts. Input validation is handled by the underlying access control logic, reverting on unauthorized operations.
</Callout>
