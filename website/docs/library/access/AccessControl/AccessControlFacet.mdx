<DocSubtitle>
Manages roles and permissions within a diamond
</DocSubtitle>

<Callout type="info" title="Key Features">
- Exposes external functions for role management and permission checks.
- Integrates seamlessly with the diamond storage pattern.
- Supports batch granting and revoking of roles for efficiency.
- Compatible with ERC-2535 diamond standard.
</Callout>

## Overview

This facet implements role-based access control for Compose diamonds. It provides functions to grant, revoke, and manage roles and their administrative roles. Developers integrate this facet to enforce permissions on diamond functions, ensuring only authorized accounts can perform sensitive operations.

## Best Practices

<Callout type="tip" title="Best Practice">
- Initialize roles and grant administrative privileges during diamond deployment.
- Use `grantRole` and `revokeRole` to manage account permissions for specific roles.
- Enforce role checks within other facets using `requireRole` before executing sensitive operations.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
All state-changing functions (`setRoleAdmin`, `grantRole`, `revokeRole`, `grantRoleBatch`, `revokeRoleBatch`, `renounceRole`) require the caller to possess the administrative role for the target role. `renounceRole` can only be called by the account itself. Use `requireRole` to validate permissions before executing critical operations.
</Callout>
