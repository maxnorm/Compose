<DocSubtitle>
Grants roles to accounts within a diamond
</DocSubtitle>

<Callout type="info" title="Key Features">
- Exposes external function `grantRole` for role assignment.
- Emits `RoleGranted` event upon successful role granting.
- Reverts with `AccessControlUnauthorizedAccount` if caller lacks admin rights.
- Utilizes diamond storage for persistent role data.
</Callout>

## Overview

This facet exposes functions to grant roles to accounts, managing access control within a Compose diamond. It integrates with diamond storage to maintain role assignments and emits events upon successful role grants. Developers add this facet to establish granular permissions for various operations.

## Best Practices

<Callout type="tip" title="Best Practice">
- Initialize roles and grant initial administrative permissions during diamond setup.
- Ensure the caller has the necessary permissions to grant roles.
- Integrate this facet with other access control facets for comprehensive permission management.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
The `grantRole` function is protected by an access control check, ensuring only authorized accounts can grant roles. Input validation is performed implicitly by the Solidity compiler for function arguments. Follow standard Solidity security practices for all interactions.
</Callout>
