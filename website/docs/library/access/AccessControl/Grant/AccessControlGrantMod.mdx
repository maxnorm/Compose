<DocSubtitle>
Grant roles to accounts within a diamond
</DocSubtitle>

<Callout type="info" title="Key Features">
- Provides `internal` functions for granting roles, suitable for facet implementation.
- Leverages the diamond storage pattern for shared state management.
- Emits `RoleGranted` event upon successful role assignment.
- Reverts with `AccessControlUnauthorizedAccount` for unauthorized grant attempts.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

This module provides internal functions to grant roles to accounts. Facets can import and use these functions to manage permissions within the diamond, leveraging shared diamond storage. Changes to role grants are immediately visible to all facets interacting with the same storage.

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure the caller has the necessary administrative privileges before calling `grantRole`.
- Verify that the `AccessControlGrantMod` is correctly initialized with the diamond's storage pointer.
- Handle the `AccessControlUnauthorizedAccount` error if role granting fails due to insufficient permissions.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
This module interacts with diamond storage using the EIP-8042 diamond storage pattern. It utilizes a specific storage slot identified by `STORAGE_POSITION` (keccak256("compose.accesscontrol")) to store role assignments. All functions within this module are `internal` and directly access this shared storage, ensuring that any modifications are immediately visible to other facets that also interact with the same storage slot.
</Callout>
