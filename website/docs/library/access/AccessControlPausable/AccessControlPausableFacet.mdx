<DocSubtitle>
Controls role pausing and unpausing within a diamond
</DocSubtitle>

<Callout type="info" title="Key Features">
- External functions for role pausing and unpausing.
- Integrates with diamond storage for role state management.
- Emits RolePaused and RoleUnpaused events upon state changes.
- Reverts with AccessControlUnauthorizedAccount or AccessControlRolePaused errors for security.
</Callout>

## Overview

This facet provides external functions to pause and unpause specific roles within a diamond. It integrates with diamond storage to manage role states, allowing temporary restrictions on role usage. Developers can add this facet to temporarily disable critical roles without altering the diamond's core logic.

### pauseRole

Temporarily disables a role, preventing all accounts from using it. Only the admin of the role can pause it. Emits a RolePaused event. Reverts with AccessControlUnauthorizedAccount If the caller is not the admin of the role.

## Best Practices

<Callout type="tip" title="Best Practice">
- Initialize role pausing/unpausing logic during diamond setup.
- Enforce role pausing on critical functions by calling `requireRoleNotPaused`.
- Ensure the caller has the necessary administrative privileges to pause or unpause roles.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
The `pauseRole` and `unpauseRole` functions revert with `AccessControlUnauthorizedAccount` if the caller is not the admin of the role. The `requireRoleNotPaused` function reverts with `AccessControlUnauthorizedAccount` if the account does not have the role, or `AccessControlRolePaused` if the role is paused. Follow standard Solidity security practices for input validation.
</Callout>
