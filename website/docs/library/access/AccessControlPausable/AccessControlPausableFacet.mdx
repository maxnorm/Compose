<DocSubtitle>
Manage roles and pausing functionality for diamond access control.
</DocSubtitle>

<Callout type="info" title="Key Features">
- Role-specific pausing and unpausing of functionality.
- Enforcement of pause status via `requireRoleNotPaused` function.
- Emits events for `RolePaused` and `RoleUnpaused` for off-chain monitoring.
</Callout>

## Overview

This facet extends diamond access control by introducing the ability to pause specific roles. It allows administrators to temporarily disable role execution, enhancing operational control and security. The facet provides functions to check role pause status, pause/unpause roles, and enforce these checks within other facets.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

import {IAccessControlPausableFacet} from "@compose/diamond/facets/AccessControlPausable/IAccessControlPausableFacet.sol";

contract MyDiamondConsumer {
    IAccessControlPausableFacet internal accessControlPausableFacet;

    constructor(address _diamondAddress) {
        accessControlPausableFacet = IAccessControlPausableFacet(_diamondAddress);
    }

    function pauseMyRole() external {
        // Assuming 'MY_ROLE' is a valid role identifier
        accessControlPausableFacet.pauseRole(bytes32(keccak256("MY_ROLE")));
    }

    function unpauseMyRole() external {
        // Assuming 'MY_ROLE' is a valid role identifier
        accessControlPausableFacet.unpauseRole(bytes32(keccak256("MY_ROLE")));
    }

    function checkRoleStatus(bytes32 _role) external view returns (bool) {
        return accessControlPausableFacet.isRolePaused(_role);
    }

    // Other facets would internally call requireRoleNotPaused
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Integrate `requireRoleNotPaused` within other facets that operate under role-based access control to enforce pausing logic.
- Ensure role administrators are properly managed through the underlying AccessControl facet before utilizing pause/unpause functionality.
- Store role identifiers consistently, preferably using `bytes32(keccak256("ROLE_NAME"))`.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
Access to `pauseRole` and `unpauseRole` is restricted to the admin of the specific role, preventing unauthorized pausing. Reentrancy is not a direct concern for the pause/unpause functions themselves, but downstream functions protected by `requireRoleNotPaused` must be audited for reentrancy. The facet relies on the underlying AccessControl facet for role management and authorization.
</Callout>
