<DocSubtitle>
Role-based access control with pausing capabilities
</DocSubtitle>

<Callout type="info" title="Key Features">
- Exposes external functions for diamond routing.
- Enables pausing and unpausing of specific roles.
- Integrates with the diamond storage pattern.
- Reverts with custom errors for unauthorized access and paused roles.
</Callout>

## Overview

This facet implements role-based access control with the ability to pause specific roles. It exposes functions to manage role pausing and check role status through the diamond proxy. Developers integrate this facet to control access to sensitive operations and temporarily disable roles during maintenance or emergencies.

## Best Practices

<Callout type="tip" title="Best Practice">
- Initialize the diamond with the AccessControlPausableFacet and grant necessary roles.
- Ensure that only authorized addresses can call `pauseRole` and `unpauseRole`.
- Use `requireRoleNotPaused` to protect sensitive functions against paused roles.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
All state-changing functions (`pauseRole`, `unpauseRole`) are protected by access control, reverting with `AccessControlUnauthorizedAccount` if the caller lacks the required administrative role. The `requireRoleNotPaused` function checks both role membership and if the role is currently paused, reverting with `AccessControlUnauthorizedAccount` or `AccessControlRolePaused` respectively. Follow standard Solidity security practices for input validation.
</Callout>
