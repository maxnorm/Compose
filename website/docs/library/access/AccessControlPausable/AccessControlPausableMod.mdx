Manages role pausing and access control checks
</DocSubtitle>

<Callout type="info" title="Key Features">
- Internal functions for role pausing and access checks.
- Uses diamond storage pattern (EIP-8042) for shared state.
- Reverts with specific errors (`AccessControlRolePaused`, `AccessControlUnauthorizedAccount`) on failure.
- No external dependencies or `using` directives.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

This module provides internal functions for pausing specific roles, preventing any account from exercising that role's permissions. Facets can import this module to enforce role-specific pausing logic within the diamond's shared storage. Changes to role pause status are immediately reflected across all facets accessing the same diamond storage.

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure `requireRoleNotPaused` is called before executing role-restricted logic.
- Call `pauseRole` and `unpauseRole` only from authorized facets.
- Verify the storage layout compatibility when upgrading facets to ensure correct state access.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
This module interacts with diamond storage at the `ACCESS_CONTROL_STORAGE_POSITION`, identified by `keccak256(\"compose.accesscontrol\")`. All state modifications and checks are performed against the `AccessControlPausableStorage` struct within this shared storage. Functions are internal, ensuring direct integration with other facets that also adhere to the diamond storage pattern.
</Callout>
