<DocSubtitle>
Manages role-based access control with pausing capabilities.
</DocSubtitle>

<Callout type="info" title="Key Features">
- Role-specific pausing: Allows granular control over role activation.
- Integrated access control: Combines role checks with pause status.
- Reverts with specific errors: `AccessControlRolePaused` and `AccessControlUnauthorizedAccount` for clear failure reasons.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

This module integrates role-based access control with the ability to pause specific roles. It ensures that actions protected by a role can be temporarily halted without affecting other operations. This is crucial for maintaining control during upgrades or emergencies.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

import {IAccessControlPausableMod} from "@compose-protocol/diamond-contracts/contracts/modules/access-control/interfaces/IAccessControlPausableMod.sol";

contract MyFacet {
    IAccessControlPausableMod public constant ACCESS_CONTROL_PAUSABLE_MODULE = IAccessControlPausableMod(address(0)); // Replace with actual deployed address

    bytes32 public constant ADMIN_ROLE = keccak256("ADMIN_ROLE");

    function grantAdminRole(address _account) external {
        // Assume role granting is handled elsewhere or by another facet
        // For demonstration, we'll simulate checking pause status
        if (ACCESS_CONTROL_PAUSABLE_MODULE.isRolePaused(ADMIN_ROLE)) {
            revert AccessControlRolePaused(ADMIN_ROLE);
        }
        // Call to grant role would go here, typically in a dedicated AccessControl facet
    }

    function pauseAdminRole() external {
        // Only an account with the ADMIN_ROLE can pause the ADMIN_ROLE itself
        ACCESS_CONTROL_PAUSABLE_MODULE.requireRoleNotPaused(ADMIN_ROLE);
        ACCESS_CONTROL_PAUSABLE_MODULE.pauseRole(ADMIN_ROLE);
    }

    function unpauseAdminRole() external {
        ACCESS_CONTROL_PAUSABLE_MODULE.unpauseRole(ADMIN_ROLE);
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Use `requireRoleNotPaused` to enforce both role membership and pause status before critical operations.
- Implement pause/unpause logic with appropriate access controls to prevent unauthorized state changes.
- Be mindful of the order of operations when integrating with other facets; pausing a role might affect dependent functionalities.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
This module interacts with the diamond's storage to manage role pausing states. Facets can query the pause status of roles using `isRolePaused` and enforce pause conditions with `requireRoleNotPaused`. The module's storage is accessible via `getAccessControlStorage` and `getStorage`. Ensure that the AccessControlPausableMod facet is added to the diamond and its functions are correctly routed.
</Callout>
