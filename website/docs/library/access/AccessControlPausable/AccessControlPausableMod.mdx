<DocSubtitle>
Manages role-based access control with pausing capabilities.
</DocSubtitle>

<Callout type="info" title="Key Features">
- Role-based access control checks.
- Ability to pause and unpause specific roles.
- Reverts with specific errors (`AccessControlUnauthorizedAccount`, `AccessControlRolePaused`) for clear failure reasons.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

This module provides a robust system for managing roles and permissions within a Compose diamond. It integrates pausing functionality for specific roles, allowing for controlled suspension of operations without disabling the entire diamond. This ensures flexibility and safety during maintenance or emergencies.

contract MyFacet {
    IAccessControlPausableMod public constant accessControlPausableMod = IAccessControlPausableMod(address(this)); // Example address

    address public constant PAUSER_ROLE = keccak256("PAUSER_ROLE");

    function grantPauserRole() external {
        // Assuming a function to grant roles exists and is implemented elsewhere
        // For demonstration, we'll directly call pause/unpause
    }

    function pauseMyFeature() external {
        accessControlPausableMod.requireRoleNotPaused(PAUSER_ROLE);
        // Proceed with pausing logic
    }

    function unpauseMyFeature() external {
        accessControlPausableMod.requireRoleNotPaused(PAUSER_ROLE); // Ensure caller has the role to unpause
        // Proceed with unpausing logic
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Use `requireRoleNotPaused` in functions to enforce role-based access and ensure operations are not paused.
- Implement role granting and revocation logic in separate facets or through an ownership mechanism.
- Carefully manage the assignment of the `PAUSER_ROLE` to prevent unauthorized pausing or unpausing of critical functionalities.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
The AccessControlPausableMod interacts with the diamond's storage to manage role pausing states. Facets that utilize this module should ensure they have access to the diamond's storage layout and correctly reference the module's functions. The `requireRoleNotPaused` function checks both role ownership and the paused status of that role before allowing execution to proceed. The module's storage is accessible via `getAccessControlStorage` and `getStorage`.
</Callout>
