<DocSubtitle>
Manages time-bound role assignments for access control
</DocSubtitle>

<Callout type="info" title="Key Features">
- Grants roles with specific expiry timestamps.
- Provides functions to check role expiry and validity.
- Revokes temporal roles explicitly.
- Compatible with ERC-2535 diamond standard.
</Callout>

## Overview

This facet implements temporal access control, allowing roles to be granted with specific expiry timestamps. It integrates with the diamond's access control system to enforce time-sensitive permissions. Developers add this facet to manage roles that should automatically revoke after a certain period, enhancing dynamic permission management within a diamond.

## Best Practices

<Callout type="tip" title="Best Practice">
- Initialize temporal roles during diamond deployment or upgrade.
- Enforce role validity using `requireValidRole` before critical operations.
- Utilize `grantRoleWithExpiry` and `revokeTemporalRole` for dynamic permission management.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
All state-changing functions (`grantRoleWithExpiry`, `revokeTemporalRole`) are protected by access control, reverting with `AccessControlUnauthorizedAccount` if the caller is not the role's admin. The `requireValidRole` function validates role existence and expiry, reverting with `AccessControlUnauthorizedAccount` or `AccessControlRoleExpired` respectively. Follow standard Solidity security practices for input validation.
</Callout>
