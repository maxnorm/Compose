<DocSubtitle>
Manages time-bound role assignments and checks for expired access.
</DocSubtitle>

<Callout type="info" title="Key Features">
- Grants roles with a specified expiry timestamp.
- Automatically revokes roles past their expiry time via `requireValidRole` checks.
- Provides granular control over time-limited access permissions.
</Callout>

## Overview

The AccessControlTemporalFacet extends the core access control mechanism by introducing time-limited role assignments. It allows administrators to grant roles that automatically expire and provides functions to check the validity of these temporal roles. This enables sophisticated access control policies based on time constraints.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

import {DiamondProxy, DiamondCutFacet} from "@compose-protocol/diamond-proxy/contracts/DiamondProxy.sol";
import {AccessControlFacet} from "@compose-protocol/access-control/contracts/AccessControlFacet.sol";
import {AccessControlTemporalFacet} from "@compose-protocol/access-control/contracts/AccessControlTemporalFacet.sol";

contract DeployDiamond {
    function deploy() external {
        // Assume diamond proxy and cut facet are already deployed and initialized
        DiamondProxy diamondProxy = DiamondProxy(address);
        DiamondCutFacet diamondCutFacet = DiamondCutFacet(address);

        // Add AccessControl and AccessControlTemporal facets
        address[] memory facetAddresses = new address[](2);
        facetAddresses[0] = address(new AccessControlFacet());
        facetAddresses[1] = address(new AccessControlTemporalFacet());

        bytes[] memory facetCalldata = new bytes[](2);
        // Assuming default initializers for facets
        facetCalldata[0] = AccessControlFacet.initialize.selector;
        facetCalldata[1] = AccessControlTemporalFacet.initialize.selector;

        diamondCutFacet.diamondCut(new DiamondCutFacet.FacetCut[](0), facetAddresses, facetCalldata, address(0));

        // Grant admin role to deployer for AccessControlFacet
        AccessControlFacet(address).grantRole(AccessControlFacet.DEFAULT_ADMIN_ROLE, msg.sender);

        // Grant a role with expiry
        uint64 expiryTimestamp = uint64(block.timestamp) + 3600; // Expires in 1 hour
        AccessControlTemporalFacet(address).grantRoleWithExpiry(AccessControlFacet.DEFAULT_ADMIN_ROLE, msg.sender, expiryTimestamp);

        // Check if role is expired
        bool expired = AccessControlTemporalFacet(address).isRoleExpired(AccessControlFacet.DEFAULT_ADMIN_ROLE, msg.sender);
        require(!expired, "Role is expired!");

        // Require valid role
        AccessControlTemporalFacet(address).requireValidRole(AccessControlFacet.DEFAULT_ADMIN_ROLE, msg.sender);
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure the `AccessControlFacet` is deployed and initialized before the `AccessControlTemporalFacet` to establish the base role management structure.
- Always check `isRoleExpired` or use `requireValidRole` before performing sensitive operations that rely on time-bound access.
- Store expiry timestamps as `uint64` to minimize gas costs for storage.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
The `grantRoleWithExpiry` and `revokeTemporalRole` functions are restricted to the admin of the respective role, preventing unauthorized modifications. Input validation on the expiry timestamp is crucial; ensure it is a future timestamp to prevent immediate expiration or unintended behavior. Be mindful of potential reentrancy if other facets interact with role checks during grant or revoke operations.
</Callout>
