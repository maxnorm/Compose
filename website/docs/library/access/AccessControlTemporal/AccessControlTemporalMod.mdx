<DocSubtitle>
Manages time-bound role assignments for accounts
</DocSubtitle>

<Callout type="info" title="Key Features">
- Grants roles with explicit expiry timestamps.
- Provides functions to check role expiry and revoke temporal roles.
- Uses diamond storage pattern for shared state management.
- Functions are `internal` for facet composition.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

This module manages time-bound role assignments, allowing roles to automatically expire. Facets can grant roles with specific expiry timestamps and check their validity. This enhances security by ensuring temporary access eventually ceases without manual revocation.

## Best Practices

<Callout type="tip" title="Best Practice">
- Call `requireValidRole` before executing sensitive operations to ensure roles are active and not expired.
- Use `grantRoleWithExpiry` for temporary permissions, reducing the need for manual revocation.
- Handle `AccessControlRoleExpired` and `AccessControlUnauthorizedAccount` errors gracefully in calling facets.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
This module interacts with diamond storage using the `ACCESS_CONTROL_STORAGE_POSITION` key, referencing the `AccessControlTemporalStorage` struct. All state changes are made directly to this shared storage, making them immediately visible to any facet that accesses the same storage position.
</Callout>
