Manages role grants with expiration dates
</DocSubtitle>

<Callout type="info" title="Key Features">
- Internal functions for facet integration.
- Manages roles with expiration timestamps.
- Uses diamond storage pattern for shared state.
- Reverts with explicit errors for expired or unauthorized roles.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

This module provides temporal role management for Compose diamonds. Facets can grant roles to accounts with specific expiration timestamps using shared diamond storage. The `requireValidRole` function ensures that only active, non-expired roles are considered, reverting with specific errors for unauthorized or expired access.

## Best Practices

<Callout type="tip" title="Best Practice">
- Call `requireValidRole` before executing sensitive operations to enforce temporal access control.
- Use `grantRoleWithExpiry` to assign roles with a defined active period.
- Handle `AccessControlRoleExpired` and `AccessControlUnauthorizedAccount` errors returned by validation functions.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
This module integrates with diamond storage at the `ACCESS_CONTROL_STORAGE_POSITION` using the `AccessControlTemporalStorage` struct. All state changes made via `grantRoleWithExpiry` and `revokeTemporalRole` are immediately reflected across all facets accessing the same storage position. The `requireValidRole` function reads directly from this shared storage.
</Callout>
