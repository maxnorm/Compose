<DocSubtitle>
Temporally manage role assignments with expiry.
</DocSubtitle>

<Callout type="info" title="Key Features">
- Grants roles with a specific expiry timestamp.
- Automatically revokes roles upon expiry.
- Provides functions to check role expiry and validate current role status.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

This module extends role-based access control by introducing time-bound role assignments. It allows granting roles that automatically expire after a specified timestamp, enhancing security and operational flexibility by ensuring temporary privileges are automatically revoked. This is crucial for managing short-term access needs within a diamond.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

import {IAccessControlTemporalMod} from "@compose/diamond/facets/IAccessControlTemporalMod.sol";

contract MyFacet {
    IAccessControlTemporalMod internal accessControlTemporal;

    function setup(address _accessControlTemporalFacet) external {
        accessControlTemporal = IAccessControlTemporalMod(_accessControlTemporalFacet);
    }

    function grantTemporaryAdmin(address _account, uint64 _expiry) external {
        accessControlTemporal.grantRoleWithExpiry(keccak256("ADMIN_ROLE"), _account, _expiry);
    }

    function checkAdminStatus(address _account) external view {
        accessControlTemporal.requireValidRole(keccak256("ADMIN_ROLE"), _account);
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Use `requireValidRole` to enforce time-bound access control checks, preventing unauthorized actions by expired roles.
- Store expiry timestamps using `uint64` for gas efficiency and compatibility with common timestamp formats.
- Consider the order of operations when integrating with other access control mechanisms to avoid unintended privilege escalation or denial.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
This module interacts with the diamond's storage to manage temporal role assignments. Facets calling `grantRoleWithExpiry`, `isRoleExpired`, or `requireValidRole` will read and potentially write to the AccessControlTemporal storage. The `requireValidRole` function enforces both the presence of the role and its non-expired status, making it a critical check for secure operations. Ensure the AccessControlTemporal module is initialized and accessible by facets requiring its functionality.
</Callout>
