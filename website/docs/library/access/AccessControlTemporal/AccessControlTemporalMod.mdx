<DocSubtitle>
Grants and manages time-bound roles for access control.
</DocSubtitle>

<Callout type="info" title="Key Features">
- Roles can be granted with a specific expiry timestamp.
- Automatically enforces role expiry, revoking access without manual intervention.
- Provides functions to check role validity and expiry status.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

This module extends standard access control by allowing roles to be granted with an expiry timestamp. It ensures that access is automatically revoked once the role's validity period concludes, enhancing security and automating permission management within Compose diamonds.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

import {IAccessControlTemporalMod} from "@compose/modules/access-control-temporal/IAccessControlTemporalMod.sol";

contract MyFacet {
    IAccessControlTemporalMod internal accessControlTemporalMod;

    constructor(address _accessControlTemporalMod) {
        accessControlTemporalMod = IAccessControlTemporalMod(_accessControlTemporalMod);
    }

    function grantUserRole(address _user, bytes32 _role, uint64 _expiry) external {
        accessControlTemporalMod.grantRoleWithExpiry(_role, _user, _expiry);
    }

    function checkUserRole(address _user, bytes32 _role) external view {
        accessControlTemporalMod.requireValidRole(_role, _user);
        // Role is valid and not expired
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Use `grantRoleWithExpiry` with a future timestamp to enforce time-limited permissions.
- Call `requireValidRole` before executing sensitive operations to ensure the caller's role is still active.
- Ensure the `AccessControlTemporalMod` facet is correctly initialized with the diamond proxy.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
The `AccessControlTemporalMod` module manages its own storage, distinct from the core Access Control module. When using `grantRoleWithExpiry`, ensure that the `_expiry` timestamp is set correctly. The `requireValidRole` function checks both the existence of the role assignment and its validity against the current block timestamp. Facets interacting with this module should be aware of potential `AccessControlRoleExpired` reverts.
</Callout>
