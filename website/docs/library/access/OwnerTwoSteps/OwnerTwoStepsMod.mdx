<DocSubtitle>
Two-step ownership transfer logic for diamonds
</DocSubtitle>

<Callout type="info" title="Key Features">
- Implements ERC-173 two-step ownership transfer.
- All functions are `internal` or `external` for direct facet interaction.
- Uses diamond storage for owner and pending owner state.
- Provides `owner()`, `pendingOwner()`, `transferOwnership()`, `acceptOwnership()`, and `renounceOwnership()` functions.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

This module provides a secure, two-step process for transferring contract ownership. By requiring both initiation and acceptance, it prevents accidental or malicious ownership changes. Facets can integrate this module to manage critical administrative functions within the diamond storage pattern.

## Best Practices

<Callout type="tip" title="Best Practice">
- Call `transferOwnership` only from the current owner to initiate a transfer.
- Ensure the pending owner calls `acceptOwnership` to finalize the transfer.
- Use `renounceOwnership` to set the owner to `address(0)` when no owner is desired.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
This module manages ownership state within dedicated storage slots in the diamond. `OWNER_STORAGE_POSITION` holds the `OwnerStorage` struct, and `PENDING_OWNER_STORAGE_POSITION` holds the `PendingOwnerStorage` struct. Functions like `transferOwnership` and `acceptOwnership` directly modify this state. Changes are immediately reflected for any facet that reads from these storage positions via the diamond proxy.
</Callout>
