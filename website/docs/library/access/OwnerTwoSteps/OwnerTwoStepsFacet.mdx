<DocSubtitle>
Manages contract ownership with two-step transfers
</DocSubtitle>

<Callout type="info" title="Key Features">
- Provides `transferOwnership`, `acceptOwnership`, and `renounceOwnership` functions.
- Uses distinct storage slots for owner and pending owner state.
- Emits `OwnershipTransferStarted` and `OwnershipTransferred` events.
</Callout>

## Overview

This facet implements a two-step ownership transfer mechanism for a diamond. It exposes external functions to initiate and accept ownership changes, enhancing security by requiring explicit confirmation from both the current and new owner. Calls are routed through the diamond proxy, integrating seamlessly with its upgradeability.

## Best Practices

<Callout type="tip" title="Best Practice">
- Initialize ownership using `transferOwnership` during diamond deployment.
- Ensure the new owner explicitly calls `acceptOwnership` to finalize the transfer.
- Use `renounceOwnership` only when relinquishing control permanently.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
The `transferOwnership` function reverts if the caller is not the current owner. The `acceptOwnership` function reverts if the caller is not the pending owner. The `renounceOwnership` function reverts if the caller is not the current owner. Follow standard Solidity security practices.
</Callout>
