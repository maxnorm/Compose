<DocSubtitle>
Manages contract ownership with a two-step transfer process.
</DocSubtitle>

<Callout type="info" title="Key Features">
- Two-step ownership transfer for enhanced security.
- `transferOwnership` initiates a transfer, `acceptOwnership` finalizes it.
- `renounceOwnership` allows relinquishing ownership.
</Callout>

## Overview

This facet provides a robust ownership management system for Compose diamonds, enforcing a two-step ownership transfer to prevent accidental ownership loss. It allows the current owner to initiate a transfer and a new owner to accept it, ensuring secure control transitions.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

import {IOwnerTwoStepsFacet} from "@compose/contracts/src/facets/OwnerTwoSteps/IOwnerTwoStepsFacet.sol";

contract OwnerManager {
    IOwnerTwoStepsFacet public ownerFacet;

    constructor(address _diamondAddress) {
        ownerFacet = IOwnerTwoStepsFacet(_diamondAddress);
    }

    function initiateOwnershipTransfer(address _newOwner) external {
        ownerFacet.transferOwnership(_newOwner);
    }

    function acceptNewOwnership() external {
        ownerFacet.acceptOwnership();
    }

    function getCurrentOwner() external view returns (address) {
        return ownerFacet.owner();
    }

    function getPendingOwner() external view returns (address) {
        return ownerFacet.pendingOwner();
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Initialize the facet with the diamond proxy address to interact with its storage.
- Ensure the caller is the current owner when calling `transferOwnership`.
- The pending owner must call `acceptOwnership` to finalize the transfer.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
Only the current owner can initiate a transfer. The pending owner must explicitly accept the ownership. Accidental calls to `transferOwnership` without subsequent acceptance by the intended recipient will not result in a permanent ownership change. Ensure the initial owner address is correctly set during deployment.
</Callout>
