## Overview

This library provides internal functions to prevent reentrant calls within a diamond. Facets can import and use these functions to safeguard against reentrancy attacks. By calling `enter` before executing sensitive logic and `exit` afterwards, facets ensure that a function cannot be re-entered before its initial execution completes.

## Best Practices

<Callout type="tip" title="Best Practice">
- Call `NonReentrancyMod.enter()` at the beginning of any function that should not be reentered.
- Call `NonReentrancyMod.exit()` at the end of the function, after all critical operations are complete.
- Ensure custom errors or revert messages clearly indicate reentrancy prevention if `NonReentrancyMod.enter()` reverts.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
This library does not interact with diamond storage directly. It maintains its own reentrancy state internally within the context of a single function execution. The `enter` and `exit` functions manage a simple boolean flag or equivalent mechanism to track whether a function is currently active, preventing nested calls.
</Callout>
