<Callout type="info" title="Key Features">
- Provides `enter` and `exit` functions to manage reentrancy.
- Emits a `Reentrancy` error on reentrant calls.
- Designed for use with the diamond storage pattern.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

This library provides functions to prevent reentrant calls within diamond functions. Facets can import this library to ensure that sensitive operations are not vulnerable to reentrancy attacks. By using `enter` and `exit`, facets can enforce a single execution context for critical code paths, maintaining data integrity.

## Best Practices

<Callout type="tip" title="Best Practice">
- Call `enter` at the beginning of any function that should not be reentrant.
- Call `exit` at the end of the function after all critical state changes are complete.
- Handle the `Reentrancy` error to gracefully manage reentrant call attempts.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
This library does not directly interact with diamond storage. It provides internal functions that can be used by any facet to manage execution flow and prevent reentrancy. The state for the reentrancy guard is managed locally within the calling facet's context.
</Callout>
