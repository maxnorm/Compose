## Overview

This library provides internal functions to prevent reentrant function calls. Facets can import and use these functions to ensure that a function's execution is not interrupted by another call to itself or another function within the same execution context. This is crucial for maintaining state integrity and security in composable diamond architectures.

## Best Practices

<Callout type="tip" title="Best Practice">
- Call `NonReentrancyMod.enter()` at the beginning of functions that should not be reentrant.
- Call `NonReentrancyMod.exit()` at the end of these functions to reset the reentrancy guard.
- Ensure custom errors are handled if reentrancy is detected.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
This library does not interact with diamond storage. Its functionality is managed internally within the library's scope for each function call. The `enter` and `exit` functions manage a local reentrancy guard specific to the execution context of the calling function.
</Callout>
